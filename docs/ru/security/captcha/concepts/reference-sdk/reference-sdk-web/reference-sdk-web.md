## {heading(Отобразить капчу)[id=reference-sdk-web-show]}

Метод `captchaWidget.show()` отображает виджет капчи пользователю в веб-приложении. Виджет капчи отображается во всплывающем окне (`iframe`).

Параметры метода:

[cols="2,2,2,4", options="header", width=100%]
|===
|Параметр
|Обязательный
|Тип
|Описание

|`container`
|![](../../../../../assets/check.svg "inline")
|`HTMLElement`
|HTML-элемент, в котором нужно отобразить виджет с капчей

|`view`
|![](../../../../../assets/check.svg "inline")
|`string`
|Вид капчи. Возможные значения:

- `popup` — всплывающее окно;
- `block` — блок

|`iframeSrc`
|![](../../../../../assets/check.svg "inline")
|`string`
|Ссылка для запуска виджета во фронтенде вашего приложения, полученная в параметре `link` в ответе {linkto(../../../../../tools-for-using-services/api/api-spec/captcha-api#api-spec-captcha)[text=метода]} `GET /captchaNotRobot.createSession`. Содержит сессионный токен и метаданные:

- `domain` — домен, на котором будет использована капча;
- `session_token` — сессионный токен;
- `variant` — вид отображения капчи (`block` или `popup`)

|`captchaType`
|![](../../../../../assets/check.svg "inline")
|`string`
|Тип капчи. Возможное значение — `type_1`

|`autofocus`
|![](../../../../../assets/no.svg "inline")
|`boolean`
|Признак автофокусировки капчи. Используется только для значения `block` в аргументе `view`. Возможные значения:

- `true` — автофокус включен (по умолчанию), фокус на странице будет перемещен на веб-элемент с капчей;
- `false` — автофокус выключен, фокус на странице останется на том же месте, где и был до появления веб-элемента с капчей

|`scheme`
|![](../../../../../assets/no.svg "inline")
|`string`
|Цветовая схема виджета с капчей. Возможные значения:

- `light` — светлая;
- `dark` — темная.

По умолчанию соответствует цветовой схеме страницы или приложения, куда интегрирована капча

|`lang`
|![](../../../../../assets/no.svg "inline")
|`string`
|Язык локализации. Если не указан, значение будет определено на сервере. Возможные значения:

- `ru` — русский;
- `be` — белорусский;
- `de` — немецкий;
- `en` — английский;
- `es` — испанский;
- `fr` — французский;
- `kk` — казахский;
- `pl` — польский;
- `tr` — турецкий;
- `uk` — украинский;
- `uz` — узбекский

|`onClose`
|![](../../../../../assets/no.svg "inline")
|`() => void`
|Callback-уведомление о закрытии капчи
|===

Метод возвращает promise-объект с результатом прохождения капчи:

```console
promise resolve <ТОКЕН_ПРОХОЖДЕНИЯ_КАПЧИ> 
promise reject <ОШИБКА>
```

Здесь:

- `<ТОКЕН_ПРОХОЖДЕНИЯ_КАПЧИ>` — токен успешного прохождения капчи, если пользователь прошел капчу;
- `<ОШИБКА>` — ошибка `close`, если пользователь закрыл окно капчи.

## {heading(Закрыть капчу)[id=reference-sdk-web-close]}

Метод `captchaWidget.close()` закрывает виджет капчи. Метод ничего не возвращает в ответе.

Пример вызова метода:

```typescript
captchaWidget.close();
```
