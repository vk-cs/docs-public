# {heading(Что нового)[id=whats-new]}

## {heading(Конфигурации {var(cloud)})[id=whats-new-configurations]}

Обновлен набор конфигураций {var(cloud)}.

Добавлена конфигурация «Легкое облако», которая позволяет сократить минимальное количество серверов для развертывания {var(cloud)} и уменьшить потребление ресурсов слоем управления.

Данная конфигурация рекомендуется:

- Для применения в периферийных вычислениях (edge computing).
- В качестве начальной для миграции с классической или виртуализированной инфраструктуры в частное облако для малого и среднего бизнеса.

## {heading(Отказоустойчивость)[id=whats-new-reliability]}

Появилась возможность размещать компоненты {var(cloud)} в нескольких зонах доступности.

Зоны доступности предназначены для разделения узлов рабочих нагрузок с целью защиты от различных негативных сценариев: отказ электропитания, отказ сети и прочее.

## {heading(Подсистема виртуальных серверов Cloud Servers)[id=whats-new-cloud-servers]}

Появилась возможность создавать ВМ с использованием графических процессоров. ВМ может использовать как весь графический процессор целиком (GPU), так и его часть (vGPU).

## {heading(Подсистема виртуальных рабочих мест Cloud Desktop)[id=whats-new-cloud-desktop]}

Добавлена Подсистема виртуальных рабочих мест Cloud Desktop. Cloud Desktop основана на технологии Virtual Desktop Infrastructure (VDI). Она предоставляет пользователям доступ к виртуальным рабочим местам, созданным на удаленном сервере и функционирующим в виртуализированном окружении.

Область применения виртуальных рабочих мест:

- Создание или модернизация коммуникационной инфраструктуры масштаба предприятия.
- В образовательных или иных организациях, в которых предусмотрено использование одного рабочего места множеством лиц, с возможностью очистки рабочего места по завершению сеанса работы.
- В предприятиях с разветвленной филиальной сетью.

## {heading(Безопасность)[id=whats-new-security]}

Теперь необходимо явно указывать список подсетей, из которых нужен доступ для входа на Портал администратора или интеграции со внешними системами. Из других подсетей доступ будет запрещен.
