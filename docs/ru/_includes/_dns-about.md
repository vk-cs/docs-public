{ifdef(public)}Сервис{/ifdef}DNS {var(cloud)} предоставляет инструменты для управления доменными зонами и записями в {ifdef(public)}публичных и {/ifdef}приватных сетях. Он интегрирован с экосистемой облачных сервисов {var(cloud)} и позволяет настраивать маршрутизацию трафика, обеспечивать доступность ресурсов и безопасное взаимодействие между компонентами инфраструктуры.

{ifdef(public)}
## {heading(Приватный DNS)[id=dns-about-private]}
{/ifdef}

*Приватный DNS* — функциональность DNS-сервера, работающего в проектных сетях {var(cloud)}. Позволяет обращаться к инстансам по DNS-именам.

{ifdef(public)}Сервис{/ifdef}{ifndef(public)}{var(cloud)}{/ifndef} поддерживает настройку приватной зоны и имен портов виртуальных машин. DNS-сервер отвечает по тем же адресам, что и порты DHCP в сети. Для работы приватного DNS в сети должен быть включен DHCP-сервер.

{ifndef(public)}
Использование DHCP-сервера и приватного DNS-сервера всегда включено для виртуальных сетей, но может быть отключено для их отдельных подсетей.
{/ifndef}

{note:warn}
Cерверы пересылки запросов приватного DNS — `8.8.8.8`, `8.8.4.4`. Изменение этих адресов не поддерживается.
{/note}
