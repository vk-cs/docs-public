# {heading(Работа с сертификатами)[id=certificates_manage]}

Замена TLS-сертификата доступна для сервисов: Cloud Trino, Cloud Airflow, Cloud ClickHouse, Cloud PostgreSQL, Redis, OpenSearch.

## {heading(Просмотр списка сертификатов)[id=certificates_view]}

1. Перейдите в раздел **Data Platform** → **Сертификаты**.
1. Ознакомьтесь со списком сертификатов в таблице. Для каждого сертификата отображается: название, текущий статус, описание и срок действия.

## {heading(Создание сертификата)[id=certificates_create]}

1. Перейдите в раздел **Data Platform** → **Сертификаты**.
1. Нажмите кнопку **Добавить**.
1. Задайте название сертификата.
1. (Опционально) Задайте описание сертификата.
1. Загрузите файлы сертификата. Для каждого файла выберите способ добавления:

   - **Загрузка сертификата** — перетащите файл в область загрузки или выберите файл на устройстве.
   - **Вставить как текст** — вставьте содержимое сертификата в текстовое поле.

   Доступные типы файлов:

   - **Сертификат** — публичный сертификат сервера в формате PEM. Сервер предъявляет его клиенту при установке TLS-соединения.
   - **Цепочка сертификатов** — промежуточные сертификаты в формате PEM. Связывают серверный сертификат с доверенным корневым удостоверяющим центром. Без цепочки клиент не сможет проверить подлинность сертификата.
   - **Приватный ключ** — приватный ключ в формате PEM, соответствующий сертификату. Используется сервером для завершения TLS-рукопожатия.

   Чтобы заменить или удалить загруженный файл, нажмите **•••** рядом с файлом и выберите нужное действие.

1. Нажмите кнопку **Создать**.

## {heading(Просмотр информации о сертификате)[id=certificates_info]}

1. Перейдите в раздел **Data Platform** → **Сертификаты**.
1. Нажмите на название нужного сертификата.
1. Ознакомьтесь с информацией:

   - **Название** — имя сертификата.
   - **Описание** — описание сертификата.
   - **ID сертификата** — уникальный идентификатор сертификата.
   - **Кем выдан** — центр сертификации, выдавший сертификат.
   - **Статус** — текущий статус сертификата.
   - **Срок действия** — период, в течение которого сертификат действителен.
   - **Дата добавления** — дата и время создания сертификата.
   - **Добавил** — пользователь, создавший сертификат.
   - **Подключенные экземпляры** — экземпляры сервисов, к которым подключен сертификат.

## {heading(Удаление сертификата)[id=certificates_delete]}

1. Перейдите в раздел **Data Platform** → **Сертификаты**.
1. Нажмите **•••** для нужного сертификата и выберите пункт **Удалить**.
1. Подтвердите удаление.
