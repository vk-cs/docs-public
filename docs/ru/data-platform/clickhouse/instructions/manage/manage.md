## {heading(Просмотр и редактирование настроек экземпляра)[id=clickhouse_settings]}

{tabs}

{tab(Личный кабинет)}

{ifdef(public)}
1. [Перейдите](https://msk.cloud.vk.com/app/) в личный кабинет {var(cloud)}.
{/ifdef}
   
1. Перейдите в раздел **Data Platform** → **Экземпляры сервисов**.
1. Нажмите на имя нужного экземпляра.
1. На странице экземпляра перейдите на вкладку **Настройки**.
1. Найдите нужный параметр в таблице настроек экземпляра, при необходимости воспользуйтесь строкой поиска.
1. Просмотрите историю изменения настроек, для этого нажмите ![ ](../../assets/clock-icon.svg "inline").
1. Отредактируйте нужные настройки:
    1. Нажмите кнопку **Редактировать**.
    1. Измените значения нужных параметров в таблице.
    1. Если необходимо вернуться к значению по умолчанию, нажмите **•••** для нужного параметра и выберите пункт **Сбросить**.
    1. Нажмите кнопку **Сохранить изменения**.

{/tab}

{/tabs}

## {heading(Добавление подключения)[id=clickhouse_add_connect]}

{tabs}

{tab(Личный кабинет)}

{include(../../../_includes/_clickhouse.md)[tags=open]}

1. На странице экземпляра перейдите на вкладку **Подключения**.
1. Нажмите кнопку **Добавить подключение**.
1. Выберите источник данных:

   - `S3 VK Cloud` — подключение к хранилищу {var(s3)} в том же проекте.
   - `S3 внешний` — подключение к внешнему объектному хранилищу S3.
   - `Iceberg Metastore с S3 VK Cloud` — подключение к экземпляру Cloud Iceberg Metastore в том же проекте {var(cloud)}.
   - `Iceberg Metastore с внешним S3` — подключение к внешнему каталогу Cloud Iceberg Metastore.
   - `ClickHouse` — подключение к базе данных Cloud ClickHouse.
   - `PostgreSQL` — подключение к базе данных PostgreSQL.
   - `MySQL` — подключение к базе данных MySQL.

1. Настройте параметры подключения к источнику:

   {tabs}
   
   {tab(S3 VK Cloud)}

   - **Название**: задайте имя подключения. Имя должно начинаться со строчной латинской буквы и содержать только заглавные и строчные латинские буквы, цифры и знаки `_`. Имя будет использоваться при работе с Cloud ClickHouse.
   - **Бакет**: имя бакета {var(s3)}, к которому будет подключаться экземпляр Cloud ClickHouse.

   {/tab}
   
   {tab(S3 внешний)}
   
   - **Название**: задайте имя подключения. Имя должно начинаться со строчной латинской буквы и содержать только заглавные и строчные латинские буквы, цифры и знаки `_`. Имя будет использоваться при работе с Cloud ClickHouse.
   - **Регион**: регион, в котором расположено ваше хранилище S3.
   - **Access Key**: идентификатор ключа доступа к хранилищу.
   - **Secret Key**: секретный ключ доступа к хранилищу.
   - **S3 URL**: URL-адрес вашего хранилища S3.
   - **Бакет**: имя бакета в хранилище S3, к которому будет подключаться экземпляр Cloud ClickHouse.

   {/tab}
   
   {tab(Iceberg Metastore с S3 VK Cloud)}
   
   - **Название**: задайте имя подключения. Имя должно начинаться со строчной латинской буквы и содержать только заглавные и строчные латинские буквы, цифры и знаки `_`. Имя будет использоваться при работе с Cloud ClickHouse.
   - **Бакет**: имя бакета {var(s3)}, к которому будет подключаться экземпляр Cloud ClickHouse.
   - **Путь к файлам в бакете**: имя директории в бакете, которая будет доступна экземпляру Cloud ClickHouse.

   {/tab}
   
   {tab(Iceberg Metastore с внешним S3)}
   
   - **Название**: задайте имя подключения. Имя должно начинаться со строчной латинской буквы и содержать только заглавные и строчные латинские буквы, цифры и знаки `_`. Имя будет использоваться при работе с Cloud ClickHouse.
   - **Регион**: регион, в котором расположен ваш внешний каталог Cloud Iceberg Metastore.
   - **Access Key**: уникальный идентификатор, используемый для аутентификации при доступе к каталогу.
   - **Secret Key**: приватный ключ для доступа к каталогу.
   - **S3 URL**: URL-адрес вашего каталога Cloud Iceberg Metastore.
   - **Бакет**: имя бакета, к которому будет подключаться экземпляр Cloud ClickHouse.
   - **Путь к файлам в бакете**: имя директории в бакете, которая будет доступна экземпляру Cloud ClickHouse.

   {/tab}
   
   {tab(ClickHouse, PostgreSQL, MySQL)}
   
   - **Название**: задайте имя подключения. Имя должно начинаться со строчной латинской буквы и содержать только заглавные и строчные латинские буквы, цифры и знаки `_`. Имя будет использоваться при работе с Cloud ClickHouse.
   - **Имя хоста**: адрес сервера для подключения.
   - **Имя пользователя**: имя учетной записи пользователя базы данных.
   - **Имя базы данных**: имя базы данных, к которой будет подключаться экземпляр Cloud ClickHouse.
   - **Пароль пользователя**: пароль учетной записи пользователя базы данных.

   {/tab}
   
   {/tabs}

1. Нажмите кнопку **Сохранить изменения**.

{/tab}

{/tabs}

## {heading(Восстановление экземпляра из резервной копии)[id=clickhouse_restore]}

{tabs}

{tab(Личный кабинет)}

{note:warn}
Восстановление из резервной копии перезапишет все данные в экземпляре. Все изменения, сделанные после создания копии, будут безвозвратно утеряны.
{/note}

{include(../../../_includes/_clickhouse.md)[tags=open]}

1. На странице экземпляра перейдите на вкладку с резервными копиями.
1. Нажмите **•••** для нужной резервной копии и выберите пункт **Восстановить**.
1. (Опционально) Измените значения параметров экземпляра. Параметры описаны в разделе о {linkto(../create#clickhouse_create)[text=создании экземпляра]}.
1. Нажмите кнопку **Создать**.

   Будет создан новый экземпляр сервиса Cloud ClickHouse.

{/tab}
