# {heading(Создание экземпляра сервиса)[id=nifi_create]}

{include(../../../_includes/_standalone.md)[tags=difference]}

1. Перейдите в раздел **Data Platform** → **Экземпляры сервисов**.
1. Нажмите кнопку **Создать экземпляр**.
1. Выполните шаги:

   1. {linkto(#nifi_create_configuration)[text=Конфигурация]}.
   1. {linkto(#nifi_create_parameters)[text=Параметры]}.
   1. {linkto(#nifi_create_service)[text=Обслуживание]}.
   1. {linkto(#nifi_create_initialization)[text=Учетные данные]}.

## {heading(Шаг 1. Конфигурация)[id=nifi_create_configuration]}

1. Выберите:

   1. Тип сервиса `NiFi`.
   1. Версию сервиса.
   1. Конфигурацию сервиса:
      
      - `Single`: конфигурация из одного узла с возможностью расширения до нескольких дополнительных узлов. Такое решение снижает расходы на развертывание тестовой инфраструктуры, но не обеспечивает отказоустойчивость: сбой узла приведет к недоступности всего экземпляра. Конфигурация из одного узла подходит для разработки и тестирования.
      - `Кластер`: конфигурация из нескольких узлов. Такое решение подходит для обработки больших объемов данных и обеспечивает высокую производительность и отказоустойчивость: сбой одного узла не приведет к остановке работы экземпляра.

1. Нажмите кнопку **Следующий шаг**.

## {heading(Шаг 2. Параметры)[id=nifi_create_parameters]}

1. Задайте параметры:

   1. **Название**: название экземпляра сервиса.
   1. (Опционально) **Описание**: описание экземпляра сервиса.
   1. **Назначить внешний IP** (**Автоматическое назначение IP** для Standalone): автоматическое назначение внешнего IP-адреса экземпляру сервиса или выбор существующего внешнего IP-адреса.
   1. (Только для Standalone в составе {var(cloud)}, опционально) Добавьте группу безопасности. Если нужной группы безопасности нет в списке, создайте ее по инструкции из **Руководства администратора {var(cloud)}** → **Сети** → [Настройка firewall](https://cloud.vk.com/docs/on-premises/private-cloud/ru/4_3/admin-guide/service_management/network/network_firewall_setup).
   1. (Пропустить для Standalone) Выберите кластер Kubernetes из выпадающего списка или создайте новый. В этот кластер будет добавлен worker-узел для размещения экземпляра NiFi.
   1. Выберите опцию `Базовая` или `Продвинутая`:
      
      - Для опции **Базовая** в конфигурации сервиса `Single` доступен шаблон `Small`, который содержит 1 узел NiFi (4 CPU, 8 ГБ RAM) и 1 узел Zookeeper (1,5 CPU, 2 ГБ RAM).
      - Для опции **Базовая** в конфигурации сервиса `Кластер` доступен шаблон `Small`, который содержит 3 узла NiFi (4 CPU, 8 ГБ RAM) и 3 узла Zookeeper (2 CPU, 4 ГБ RAM).
      - Для опции **Продвинутая**:

        1. В блоке **Nifi** задайте параметры:

           - **CPU**: количество процессоров.
           - **RAM**: количество оперативной памяти.
           - **Тип диска**: тип диска. Выберите тип, либо он будет назначен автоматически.
           - **Размер диска**: размер диска в ГБ.
           - **Количество Nifi**: общее количество узлов NiFi.
            
        1. В блоке **Zookeeper** задайте параметры:

           - **CPU**: количество процессоров.
           - **RAM**: количество оперативной памяти.
           - **Тип диска**: тип диска. Выберите тип, либо он будет назначен автоматически.
           - **Размер диска**: размер диска в ГБ.
           - **Количество Zookeeper**: общее количество узлов ZooKeeper.

      {include(../../../_includes/_standalone.md)[tags=total_quotas]}

1. Нажмите кнопку **Следующий шаг**.

## {heading(Шаг 3. Обслуживание)[id=nifi_create_service]}

1. В блоке **Техническое окно** выберите один или несколько дней и укажите время проведения технических работ с учетом вашей временной зоны. Продолжительность технических работ: 4 часа.
1. Нажмите кнопку **Следующий шаг**.

## {heading(Шаг 4. Учетные данные)[id=nifi_create_initialization]}

1. В блоке **Учетная запись** задайте параметры:

   1. **Логин администратора для доступа к NiFi**: логин администратора. Требования к логину:

      - Допустимы только цифры, латинские буквы и символ `_`.
      - Первый символ: латинская буква любого регистра или `_`.
      - Недопустимы имена: `admin`, `root`.

   1. **Пароль администратора для доступа к NiFi**: пароль администратора. Чтобы задать пароль, нажмите кнопку **Сгенерировать** или укажите свой. Требования к паролю:

      - Не менее 16 и не более 50 символов.
      - Минимум одна заглавная и одна строчная буквы латинского алфавита.
      - Минимум одна цифра.
      - Минимум один из символов: `!`, `?`, `%`, `#`, `/`, `(`, `)`, `-`, `+`, `*`.

      {note:err}
      Сохраните пароль. Восстановить забытый пароль нельзя.
      {/note}

1. Нажмите кнопку **Создать**.
