|===
|Термин
|Определение

|ACID-транзакции
|Atomicity, Consistency, Isolation, Durability, стандарт в котором транзакция (последовательность операций) выполняется полностью, либо не выполняется вовсе. Обеспечивает надежную и предсказуемую работу БД и сохраняет целостность данных в случае сбоев

|Airflow
|Apache Airflow, открытое ПО для создания, выполнения, мониторинга и оркестрации потоков операций по обработке данных

|API
|Application Programming Interface, интерфейс программирования приложений

|Bouncer
|Менеджер соединений, который управляет и оптимизирует подключения между клиентами и сервером БД

|Ceph
|Программно-определяемая распределенная файловая система с открытым исходным кодом

|CI/CD-инструменты
|Программные средства, автоматизирующие процессы непрерывной интеграции (Continuous Integration, CI) и непрерывной поставки/развертывания (Continuous Delivery/Deployment, CD) в разработке ПО

|ClickHouse
|Столбцовая аналитическая СУБД для хранения и обработки исторических данных

|ClickHouse Keeper
|Cервис, который координирует репликацию данных и распределяет DDL-запросы в кластерах ClickHouse

|CPU
|Central Processing Unit, центральный процессор

|Cron-выражение
|Строка, состоящая из шести или семи полей. Определяет расписание выполнения задач в планировщике задач Cron

|DAG
|Directed Acyclic Graph, направленный ациклический граф

|DDL-запрос
|Data Definition Language, команда для определения и управления структурой БД

|Docker-контейнер
|Легковесная, изолированная среда, которая содержит все необходимое для запуска приложения, включая код, библиотеки и зависимости

|Docker-образ
|Шаблон, содержащий все необходимое для создания и запуска Docker-контейнера

|Docker Registry
|Серверное приложение для хранения и управления Docker-образами

|ETL-процессы
|Процессы управления данными: извлечение, преобразование, загрузка данных (Extract, Transform, Load)

|Flink
|Apache Flink, сервис для потоковой обработки и передачи любых объемов данных из источников в целевые системы

|GraphX
|Библиотека для обработки и анализа графов в Apache Spark, которая позволяет выполнять параллельные вычисления на графах и предоставляет API для работы с графовыми структурами данных

|Greenplum
|Аналитическая, массивно-параллельная (MPP) СУБД на основе PostgreSQL, предназначенная для хранения и обработки больших объемов данных

|Hadoop-кластер
|Набор узлов, которые работают как единое целое для хранения и обработки больших объемов данных с использованием фреймворка Apache Hadoop

|High IOPS SSD
|Диск, обеспечивающий высокую производительность, которая характеризуется большим количеством операций ввода-вывода в секунду (Input/Output Operations Per Second)

|IP
|Internet Protocol, интернет-протокол

|Iceberg
|Apache Iceberg, открытый формат таблиц для хранения и организации структур данных в Data LakeHouse

|Jatoba
|Защищенная СУБД, основанная на модернизированном ядре СУБД PostgreSQL

|Java Vector API
|Интерфейс Java для работы с векторными вычислениями, которые используют возможности SIMD-процессоров (Single Instruction, Multiple Data). Он позволяет писать код, который может выполнять одну и ту же операцию над множеством данных одновременно 

|JDBC
|API для языка программирования Java, который позволяет приложениям взаимодействовать с реляционными БД, обеспечивая универсальный интерфейс для выполнения SQL-запросов и управления данными

|Job Manager
|Компонент Apache Flink, который управляет выполнением задач, распределяет ресурсы и координирует выполнение потоков данных в кластере

|JupyterHub
|Платформа для запуска и управления множеством сессий Jupyter Notebook для разных пользователей через веб-интерфейс

|JupyterHub Notebook
|Интерактивная веб-среда для создания и обмена документами, содержащими код, уравнения, визуализации и текст

|Kafka
|Apache Kafka, распределенный программный брокер сообщений с открытым исходным кодом

|Kafka Connect
|Компонент для масштабируемой и надежной потоковой передачи данных между Apache Kafka и другими системами

|Kubernetes
|Открытое ПО для автоматизации развертывания, масштабирования и управления контейнеризированными приложениями

|MLflow
|Платформа с открытым исходным кодом для управления полным жизненным циклом моделей машинного обучения (ML)

|MLlib
|Библиотека машинного обучения для Apache Spark, предоставляющая инструменты для выполнения различных задач машинного обучения, таких как классификация, регрессия, кластеризация и обработка данных

|MPP-архитектура
|Massively Parallel Processing, архитектура, в которой множество процессоров работают паралелльно, обрабатывая разные части задачи одновременно для повышения производительности и масштабируемости

|NiFi
|Apache NiFi, платформа для автоматизации и управления потоками данных

|OLAP-запрос
|Запрос для анализа многомерных данных в системах аналитической онлайн-обработки (Online Analytical Processing)

|OpenSearch
|Программный комплекс для создания поисковых систем и аналитических решений

|Pandas API
|Интерфейс библиотеки Pandas для Python, предоставляющий инструменты для удобной и эффективной работы с данными, включая их обработку, анализ и визуализацию

|Parquet
|Столбцовый формат хранения данных с открытым исходным кодом, оптимизированный для эффективного хранения и обработки больших объемов данных

|PostgreSQL
|Свободная объектно-реляционная СУБД

|PyTorch
|Библиотека с открытым исходным кодом для машинного обучения, которая поддерживает автоматическое дифференцирование и динамическое построение нейронных сетей

|RAM
|Random Access Memory, оперативная память

|Redis
|Высокоэффективная и высокоскоростная СУБД NoSQL-типа, которая хранит данные в оперативной памяти в виде пар ключ/значение

|S3 (S3 внешний)
|Simple Storage Service, сервис, где хранятся цифровые данные большого объема. Работает по одноименному протоколу

|S3 VK Cloud
|Облачное хранилище объектов {var(s3)}, предоставляемое компанией VK Tech. Позволяет хранить большие объемы данных с малыми затратами

|SDN
|Software Defined Network, программно-определяемая сеть

|Single
|Конфигурация экземпляра сервиса из одного узла, не отказоустойчивая. Подходит для задач разработки и тестирования

|Scikit-learn
|Библиотека Python для машинного обучения, предоставляющая простые и эффективные инструменты для анализа данных и моделирования

|Spark
|Apache Spark, платформа для распределенной обработки больших данных, обеспечивающая высокую скорость вычислений и поддержку различных аналитических задач

|Spark History Server
|Компонент Apache Spark, который позволяет просматривать и анализировать историю выполненных задач Spark. Он предоставляет доступ к логам, метрикам и визуализациям выполнения задач. Используется для диагностики проблем и оптимизации производительности

|Spark SQL
|Модуль Apache Spark, который предоставляет возможность работать с данными с помощью SQL-запросов, а также интегрировать SQL с программами на языках Scala, Java, Python и R

|Structured Streaming
|Компонент Apache Spark, который позволяет обрабатывать потоковые данные в реальном времени с использованием высокоуровневого API, обеспечивая поддержку непрерывных вычислений и отказоустойчивость

|SQL
|Structured Query Language, язык структурированных запросов

|SQL Firewall
|Система безопасности для защиты БД от несанкционированного доступа и вредоносных SQL-запросов

|Task Manager
|Компонент Apache Flink, который выполняет задачи, обрабатывает данные и управляет ресурсами, выделенными для выполнения потоков данных в кластере

|TensorFlow
|Библиотека с открытым исходным кодом для машинного обучения и нейронных сетей. Поддерживает как обучение, так и развертывание моделей на различных платформах

|Time travel
|Функция Apache Iceberg, которая позволяет запрашивать и анализировать данные из предыдущих состояний таблицы, используя сохраненные снимки (snapshots)

|TLS-шифрование
|Протокол, который обеспечивает безопасную передачу данных по сети и предотвращает несанкционированный доступ и перехват информации

|Trino
|Высокопроизводительный распределенный SQL-движок, предназначенный для выполнения быстрых аналитических запросов к данным из разных источников, таких как базы данных, файловые системы и объектные хранилища, без необходимости их копирования

|URL
|Uniform Resource Locator, адрес сайта или сервиса

|WAL
|Write-Ahead Logging, метод логирования, который обеспечивает целостность данных в СУБД

|ZooKeeper
|Сервис для координации распределенных систем, который управляет их конфигурацией, синхронизацией и поддержанием состояния

|Бакет
|Контейнер для хранения данных (объектов) в S3-хранилищах

|БД
|База данных

|ВМ
|Виртуальная машина

|Дифференциальная резервная копия
|Резервная копия, которая содержит в себе только различия между предыдущей полной копией и текущим моментом времени. Если между дифференциальными копиями не выполнялось полное копирование, размер каждой следующей копии будет больше предыдущей

|Зона доступности
|Один или несколько центров обработки данных (ЦОД), в которых могут быть размещены компоненты {var(data-p)}

|Инкрементальная резервная копия
|Резервная копия, которая содержит только изменения данных после любого предыдущего резервного копирования: полного или дифференциального. Размер этой копии зависит только от количества изменившихся данных за период времени между созданием резервных копий

|Кибер Бэкап
|Система резервного копирования и восстановления данных

|Кластер
|Отказоустойчивая конфигурация экземпляра сервиса из нескольких узлов. Подходит для любых задач, в том числе для повседневной работы с данными в production-среде

|Логи
|Файлы с записями о событиях и действиях пользователя или программы в хронологическом порядке

|ПО
|Программное обеспечение

|Под
|Наименьшая единица развертывания в Kubernetes, содержащая один или несколько контейнеров, которые совместно используют ресурсы сети и хранилища

|Полная резервная копия
|Резервная копия, которая включает в себя файлы, базы данных, настройки и конфигурации, необходимые для восстановления системы в случае ее сбоя, потери данных или других проблем

|Поток данных
|Упорядоченное движение информации от источника к месту назначения. Может включать в себя процессы преобразования, обработки и передачи данных

|Репликация
|Процесс копирования и поддержания синхронности данных между несколькими системами или узлами

|Сервис
|Система, интегрированная с {var(data-p)}

|СУБД
|Система управления базами данных

|Топик
|Логическая категория или канал для группировки сообщений в Apache Kafka, аналог таблицы в БД или папке в файловой системе

|Шард
|Часть данных, которая хранится на отдельном узле кластера или сервере БД
