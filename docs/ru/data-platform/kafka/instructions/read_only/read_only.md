# {heading(Вывод экземпляра из режима только для чтения)[id=kafka_read_only]}

Если на диске хотя бы одного брокера Cloud Kafka остается меньше 10% свободного места, {var(data-p)} автоматически переводит экземпляр в режим только для чтения (read-only). Это предотвращает переполнение диска и помогает сохранить работоспособность кластера. При попытке опубликовать сообщения возникнет ошибка и запись в топики не выполнится.

В режиме только для чтения:

- Запись сообщений в топики недоступна;
- Чтение сообщений доступно.

Режим применяется ко всему экземпляру, даже если место закончилось только на одном брокере.

## {heading(Отображение режима только для чтения)[id=kafka_read_only_view]}

При переходе экземпляра в режим только для чтения в разделе **Data Platform** → **Экземпляры сервисов** для экземпляра сервиса в столбце **Состояние сервиса** отобразится статус `Только для чтения`.

Узнать на диске какого из брокеров закончилось свободное место можно в разделе **Data Platform** → **Экземпляры сервисов** на вкладке **Статус компонентов**.

## {heading(Вывод экземпляра из только для чтения)[id=kafka_read_only]}

Чтобы вывести экземпляр из режима только для чтения, увеличьте размеры дисков для брокеров:
1. Перейдите в раздел **Data Platform** → **Экземпляры сервисов** на вкладку **Статус компонентов**.
1. Нажмите **Масштабирование дисков**.
1. Выберите **Вертикальное масштабирование**.
1. Увеличьте размер дисков.
1. Нажмите кнопку **Сохранить изменения**.
