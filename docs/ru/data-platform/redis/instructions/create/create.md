# {heading(Создание экземпляра сервиса)[id=redis_create]}

{include(../../../_includes/_standalone.md)[tags=difference]}

1. Перейдите в раздел **Data Platform** → **Экземпляры сервисов**.
1. Нажмите кнопку **Создать экземпляр**.
1. Выполните шаги:

   1. {linkto(#redis_create_configuration)[text=Конфигурация]}.
   1. {linkto(#redis_create_parameters)[text=Параметры]}.
   1. {linkto(#redis_create_service)[text=Обслуживание]}.
   1. {linkto(#redis_create_user)[text=Учетные данные]}.

## {heading(Шаг 1. Конфигурация)[id=redis_create_configuration]}

1. Выберите:

   1. Тип сервиса `Redis`.
   1. Версию сервиса.
   1. Конфигурацию сервиса:
      
      - `Single`: конфигурация из одного узла с возможностью расширения до нескольких дополнительных узлов. Решение снижает расходы на развертывание тестовой инфраструктуры, но не обеспечивает отказоустойчивость: сбой узла приведет к недоступности всего экземпляра. Конфигурация из одного узла подходит для разработки и тестирования.
      - `Кластер`: конфигурация из нескольких узлов. Решение подходит для обработки больших объемов данных и обеспечивает высокую производительность и отказоустойчивость: сбой одного узла не приведет к остановке работы экземпляра.

1. Нажмите кнопку **Следующий шаг**.

## {heading(Шаг 2. Параметры)[id=redis_create_parameters]}

1. Задайте параметры:

   1. **Название**: название экземпляра сервиса.
   1. (Опционально) **Описание**: описание экземпляра сервиса.
   1. **Название базы данных**: название базы данных.
   1. **Назначить внешний IP** (**Автоматическое назначение IP** для Standalone):  автоматическое назначение внешнего IP-адреса экземпляру сервиса или выбор существующего внешнего IP-адреса.
   1. (Только для Standalone в составе {var(cloud)}, опционально) Добавьте группу безопасности. Если нужной группы безопасности нет в списке, создайте ее по инструкции из **Руководства администратора {var(cloud)}** → **Сети** → [Настройка firewall](https://cloud.vk.com/docs/on-premises/private-cloud/ru/4_3/admin-guide/service_management/network/network_firewall_setup).
   1. (Пропустить для Standalone) Выберите кластер Kubernetes из выпадающего списка или создайте новый. В этот кластер будет добавлен worker-узел для размещения экземпляра Redis.
   1. Выберите опцию `Базовая` или `Продвинутая` и настройте параметры узлов кластера:
      
      - Для опции **Базовая** в конфигурации сервиса `Single` выберите один из готовых шаблонов:

        - `Малый`: содержит 1 узел Redis (2 CPU, 8 RAM, Ceph 10 ГБ).
        - `Побольше`: содержит 1 узел Redis (4 CPU, 32 RAM, Ceph 10 ГБ).
        - `Еще больше`: содержит 1 узел Redis (16 CPU, 128 RAM, Ceph 10 ГБ).

      - Для опции **Базовая** в конфигурации сервиса `Кластер` выберите один из готовых шаблонов:

        - `Малый с репликацией`: содержит 3 узла Redis (2 CPU, 8 RAM, Ceph 10 ГБ).
        - `Побольше с репликацией`: содержит 3 узла Redis (4 CPU, 32 RAM, Ceph 10 ГБ).
        - `Еще больше с репликацией`: содержит 3 узла Redis (16 CPU, 128 RAM, Ceph 10 ГБ).
        - `Малый 3-шардовый с репликацией`: содержит 2 узла Redis (2 CPU, 8 RAM, Ceph 10 ГБ).
        - `Побольше 3-шардовый с репликацией`: содержит 2 узла Redis (4 CPU, 32 RAM, Ceph 10 ГБ).
        - `Еще больше 3-шардовый с репликацией`: содержит 2 узла Redis (16 CPU, 128 RAM, Ceph 10 ГБ).

      - Для опции **Продвинутая** в блоке **Redis** задайте параметры контейнера, который запустит сервер Redis:

        - **CPU**: количество процессоров.
        - **RAM**: количество оперативной памяти.
        - **Тип диска**: тип диска. Выберите тип самостоятельно или он будет назначен автоматически.
        - **Размер диска**: размер диска в ГБ.
        - **Количество Redis**: общее количество контейнеров.

      {include(../../../_includes/_standalone.md)[tags=total_quotas]}

1. Нажмите кнопку **Следующий шаг**.

## {heading(Шаг 3. Обслуживание)[id=redis_create_service]}

1. В блоке **Техническое окно** выберите один или несколько дней и время проведения технических работ с учетом вашей временной зоны. Продолжительность технических работ: 4 часа.
1. В блоке **Полная резервная копия** укажите максимальное количество полных копий для хранения. При превышении максимального количества старые копии будут перезаписаны.
1. Нажмите кнопку **Следующий шаг**.

## {heading(Шаг 4. Учетные данные)[id=redis_create_user]}

1. Задайте параметры:

   1. **Логин администратора для доступа к Redis**: логин администратора. Требования к логину:

      - Допустимы только цифры, латинские буквы и символ `_`.
      - Первый символ — латинская буква любого регистра или `_`.
      - Недопустимые имена: `os_admin`, `root`, `dataplatform_moth`, `default`.
      - Не более 50 символов.

   1. **Пароль администратора для доступа к Redis**: пароль администратора. Чтобы задать пароль, нажмите кнопку **Сгенерировать** или укажите свой. Требования к паролю:

      - Не менее 8 символов.
      - Минимум одна заглавная и одна строчная буквы латинского алфавита.
      - Минимум одна цифра.
      - Минимум один из символов: `!`, `?`, `%`, `#`, `/`, `(`, `)`, `-`, `+`, `*`.

      {note:err}
      Сохраните пароль. Восстановление забытого пароля недоступно.
      {/note}

   Чтобы добавить еще одну учетную запись, нажмите кнопку **Добавить учетную запись**.

1. Нажмите кнопку **Создать**.
