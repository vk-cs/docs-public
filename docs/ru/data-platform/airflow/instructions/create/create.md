# {heading(Создание экземпляра сервиса)[id=airflow_create]}

{include(../../../_includes/_standalone.md)[tags=difference]}

1. Перейдите в раздел **Data Platform** → **Экземпляры сервисов**.
1. Нажмите кнопку **Создать экземпляр**.
1. На шаге **Конфигурация**:
   1. Выберите тип сервиса `Cloud Airflow`.
   1. Выберите версию сервиса.
   1. Нажмите кнопку **Следующий шаг**.
1. На шаге **Параметры**:
   1. Задайте название и описание для экземпляра.
   1. (Пропустить для Standalone) Включите параметр **Назначить внешний IP**, если требуется доступ к экземпляру из внешней сети.
   1. (Только для Standalone, опционально) Задайте IP-адрес для экземпляра сервиса из подсети. Для этого отключите опцию **Автоматическое назначение IP**. По умолчанию IP-адрес назначается экземпляру автоматически.
   1. (Только для Standalone в составе {var(cloud)}, опционально) Добавьте группу безопасности. Если нужной группы безопасности нет в списке, создайте ее по инструкции из **Руководства администратора {var(cloud)}** → **Сети** → [Настройка firewall](https://cloud.vk.com/docs/on-premises/private-cloud/ru/4_3/admin-guide/service_management/network/network_firewall_setup).
   1. (Пропустить для Standalone) Выберите кластер Kubernetes из выпадающего списка или создайте новый. В этот кластер будет добавлен worker-узел для размещения экземпляра Cloud Airflow.

   1. Выберите опцию `Базовая` или `Продвинутая` и настройте параметры узлов кластера:

      - Для опции **Базовая** выберите один из готовых шаблонов кластера.
      - Для опции **Продвинутая** задайте нужное количество CPU, объем RAM и количество узлов для компонентов Cloud Airflow:

        - **Api server**: веб‑сервер Cloud Airflow, который предоставляет пользовательский интерфейс и REST API для мониторинга и управления рабочими процессами.
        - **DAG Processor**: компонент, который извлекает из хранилища файлы DAG, анализирует их и подготавливает к выполнению планировщиком.
        - **Triggerer**: компонент, который ожидает и обрабатывает отложенные события, чтобы возобновлять задачи в состоянии ожидания.
        - **Scheduler**: компонент, который планирует и запускает задачи DAG с учетом расписания и зависимостей между ними.
        - **Celery Worker**: рабочие ноды Cloud Airflow. Дополнительно задайте:
          
          - тип и размер диска;
          - срок хранения логов Cloud Airflow в днях.
            
            {note:warn}
            Установите количество дней в соответствии с размером диска Celery Worker, чтобы избежать переполнения дискового пространства. Изменить размер диска в процессе эксплуатации невозможно.
            {/note}

   1. Нажмите кнопку **Следующий шаг**.
1. На шаге **Обслуживание**:

   1. Выберите дни недели и время начала проведения технических работ. В это время на сервере могут выполняться технические работы, в том числе создание резервных копий. Сервис может быть временно недоступен.

      {note:info}
      Длительность технических работ — 4 часа. Временная зона — GMT+03:00.
      {/note}

   1. Нажмите кнопку **Следующий шаг**.
1. На шаге **Подключения**:
   1. Нажмите **Добавить подключение**. 
   1. Выберите тип подключения:
      - `Подключение к хранилищу VK Cloud S3` — подключение к {var(s3)} в том же проекте.
      - `Подключение к внешнему S3 хранилищу` — подключение к внешнему хранилищу S3.
   1. Выберите готовое подключение или создайте новое.
      
      Чтобы создать новое подключение в блоке **Подключение** выберите `Новое подключение` и задайте для него параметры:

      {include(../../../_includes/_airflow.md)[tags=connections_airflow]}

   1. Нажмите кнопку **Следующий шаг**.
1. На шаге **Учетные данные**:
   1. Задайте логин учетной записи для доступа к Cloud Airflow. Требования к логину:

      - Допустимы только цифры, латинские буквы и символ `_`.
      - Первый символ — латинская буква любого регистра или `_`.
      - Недопустимые имена: `os_admin`, `root`, `dataplatform_moth`.

   1. Выберите роль для учетной записи:

      {include(../../../_includes/_airflow.md)[tags=roles_airflow]}
      
   1. Придумайте или сгенерируйте пароль. Требования к паролю:

      - Не менее 16 символов.
      - Минимум одна заглавная и одна строчная буквы латинского алфавита.
      - Минимум одна цифра.
      - Минимум один из символов: `!`, `"`, `#`, `$`, `%`, `&`, `(`, `)`, `*`, `+`, `,`, `-`, `.`, `:`, `;`, `<`, `=`, `>`, `?`, `@`, `[`, `]`, `^`, `_`, `{`, `}`, `~`.

      {note:err}
      Сохраните пароль. Восстановление пароля недоступно.
      {/note}

   1. Чтобы добавить еще одну учетную запись, нажмите кнопку **Добавить учетную запись**.
   1. Нажмите кнопку **Создать**.
