<!-- Конфигурации экземпляра -->

{includetag(configuration)}

- `Single`: в этой конфигурации экземпляр сервиса будет расположен на одном сервере.

    Решение подходит для разработки и тестирования, снижает расходы на развертывание тестовой инфраструктуры. Не обеспечивает отказоустойчивость: сбой узла приводит к остановке всей работы.

- `Кластер`: в этой конфигурации будет создан отказоустойчивый кластер с максимальной доступностью, автоматическим восстановлением и переключением после сбоев.

    Решение подходит для хранения баз данных большинства проектов, в том числе высоконагруженных, с высокими требованиями к сохранности информации.

{/includetag}

<!-- Новая конфигурация экземпляра на Atom -->

{includetag(configuration_atom)}

- `Одиночный узел`: конфигурация из одного узла, не отказоустойчивая. Экземпляр в такой конфигурации подходит для задач разработки и тестирования.
- `Кластер`: отказоустойчивая конфигурация из нескольких узлов. Экземпляр в такой конфигурации подходит для любых задач, в том числе для повседневной работы с данными в production-среде.

{/includetag}

<!-- Роли для баз данных -->

{includetag(roles_db)}

- `Владелец базы данных` — пользователь с максимально широким набором прав. При удалении пользователей права на созданные ими объекты переходят к владельцу БД. У экземпляра сервиса может быть только один владелец, его нельзя удалить.

- `Стандартный пользователь` — пользователь, который может создавать, модифицировать и просматривать объекты: таблицы, схемы, функции и др. Ему принадлежат права на созданные объекты. Количество пользователей с данной ролью не ограничено.

- `Наблюдатель` — пользователь, который может просматривать таблицы БД, но не может создавать или модифицировать их.
  Количество пользователей с данной ролью не ограничено.

- `Аудитор безопасности`  — пользователь, который может просматривать существующих пользователей, назначенные роли и системные таблицы. Аудитор не может создавать новые объекты, изменять или просматривать существующие пользовательские объекты. Количество пользователей с данной ролью не ограничено.

{/includetag}
