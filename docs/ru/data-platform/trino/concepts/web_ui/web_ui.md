# {heading(Веб-интерфейс Trino)[id=trino-web-ui]}

Cloud Trino предоставляет веб-интерфейс, с помощью которого можно просматривать активные и завершенные запросы, отслеживать их статусы и прогресс, анализировать производительность. Подробнее — в [официальной документации Trino](https://trino.io/docs/current/admin/web-interface.html).

## {heading(Доступ к интерфейсу)[id=trino-ui-access]}

Ссылка на веб-интерфейс Trino отображается на вкладке **Общая информация** экземпляра сервиса в поле **Trino Admin UI**. Для аутентификации используйте логин и пароль пользователя Cloud Trino. Подробнее о подключении через веб-интерфейс — в инструкции {linkto(../../instructions/connect#trino_connect-ui)[text=Подключение к экземпляру сервиса]}.

## {heading(Основные разделы интерфейса)[id=trino-ui-sections]}

### {heading(Главная страница)[id=trino-ui-overview]}

Главная страница веб-интерфейса — **Cluster overview**. На странице отображается следующая информация:

- Сведения о версии Trino, окружении и времени работы координатора.
- Панель мониторинга на которой отображаются основные метрики кластера:

   - **Running queries**: количество выполняющихся запросов.
   - **Queued queries**: количество запросов в очереди.
   - **Blocked queries**: количество заблокированных запросов.
   - **Active workers**: количество активных worker-узлов Trino.
   - **Runnable drivers**: количество работоспособных драйверов на текущий момент.
   - **Reserved memory**: объем зарезервированной памяти для выполняющихся запросов.
   - **Rows/sec**: количество строк таблиц, обрабатываемых в секунду на текущий момент.
   - **Bytes/sec**: скорость обработки данных (строк в секунду, байт в секунду).
   - **Worker parallelism**: эффективность использования worker-узлов.

- Список запросов. Активные запросы находятся в начале списка, за ними следуют завершенные. Для каждого запроса отображаются: ID, текст SQL-запроса, статус, процент выполнения, пользователь запустивший запрос и источник запроса.

Для навигации по запросам используйте следующие инструменты:

- поиск по тексту запроса, ID, пользователю, источнику;
- фильтрация по статусу:

   - **Running** — выполняющиеся;
   - **Queued** — в очереди;
   - **Finished** — завершенные;
   - **Failed** — с ошибкой.

- сортировка по различным параметрам;
- настройка отображения: количество запросов на странице, частота обновления.

### {heading(Страница запроса)[id=trino-ui-query-detail]}

Чтобы перейти на страницу с подробной информацией о запросе, нажмите на его ID. Страница содержит вкладки:

- **Overview**: сводка с параметрами запроса, временная шкала, статистика использования ресурсов, полный текст SQL-запроса, список стадий выполнения.
- **Live plan**: схема выполнения запроса по стадиям. Схему можно приближать и перемещать. При нажатии на стадию открывается вкладка **Stage performance**.
- **Stage performance**: детализация выбранной стадии выполнения: использование ресурсов, обработанные данные.
- **Splits**: график временных шкал для каждой стадии.
- **JSON**: полное JSON-представление запроса со всей статистикой.
- **References**: таблицы, на которые сослался запрос, и вызванные функции.
