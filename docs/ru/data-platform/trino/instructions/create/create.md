{tab(Личный кабинет)}

{ifdef(public)}
1. [Перейдите](https://msk.cloud.vk.com/app/) в личный кабинет {var(cloud)}.
1. Выберите проект, где нужно создать экземпляр сервиса.
{/ifdef}

1. Перейдите в раздел **Data Platform → Экземпляры сервисов**.
1. Нажмите кнопку **Создать экземпляр**.
1. На шаге **Конфигурация**:

   1. Выберите тип сервиса `Cloud Trino`.
   1. Выберите нужную версию сервиса.
   1. Выберите конфигурацию экземпляра сервиса:

      - `Одиночный узел`: конфигурация из одного узла, не отказоустойчивая. Экземпляр в такой конфигурации подходит для задач разработки и тестирования.
      - `Кластер`: отказоустойчивая конфигурация из нескольких узлов. Экземпляр в такой конфигурации подходит для работы в production-среде.

   1. Нажмите кнопку **Следующий шаг**.

1. На шаге **Параметры**:

   1. Задайте для экземпляра название и описание.

   1. {ifndef(public)} (Пропустить для Standalone) {/ifndef} Включите опцию **Назначить внешний IP**, если необходим доступ к экземпляру из внешней сети.

   {ifndef(public)}
   1. (Только для Standalone, опционально) Задайте IP-адрес для экземпляра сервиса из подсети. Для этого отключите опцию **Автоматическое назначение IP**. По умолчанию IP-адрес назначается экземпляру автоматически.
   1. (Только для Standalone в составе {var(cloud)}, опционально) Добавьте группу безопасности. Если нужной группы безопасности нет в списке, создайте ее по инструкции из **Руководства администратора {var(cloud)}** → **Сети** → [Настройка firewall](https://cloud.vk.com/docs/on-premises/private-cloud/ru/4_3/admin-guide/service_management/network/network_firewall_setup).
   1. (Пропустить для Standalone) Выберите кластер Kubernetes из выпадающего списка или создайте новый. В этот кластер будет добавлен worker-узел для размещения экземпляра Cloud Trino.
   {/ifndef}
   {ifdef(public)}
   1. Выберите кластер Kubernetes. В этот кластер [Cloud Containers](/ru/kubernetes/k8s) будет добавлен [worker-узел](/ru/kubernetes/k8s/concepts/architecture#k8s-architecture-topology) для размещения экземпляра Cloud Trino.
      Если нужного кластера нет в списке:

      1. Выберите пункт **Создать новый кластер**.
      1. Выберите сеть и подсеть для кластера из выпадающего списка **Сеть**.

         Если нужной сети нет в списке, выберите пункт **Создать новую сеть** и укажите [SDN](/ru/networks/vnet/concepts/sdn) и адрес подсети.

      1. Выберите [зону доступности](/ru/start/concepts/architecture#architecture-az), где будет размещен узел кластера Cloud Containers.
   {/ifdef}
   1. Настройте параметры узлов, на которых будет развернут экземпляр сервиса:
      
      {tabs}

      {tab(Одиночный узел)}

      В конфигурации **Одиночный узел** экземпляр будет развернут на одном узле. 
   
      Выберите вариант настройки:

      - **Базовая**: будет использован готовый шаблон узла с 4 CPU, 6 RAM.
      - **Продвинутая**: настройте параметры узла самостоятельно — в блоке **Сoordinator** задайте количество CPU и RAM.

      {/tab}

      {tab(Кластер)}

      В конфигурации **Кластер** настройте количество узлов и их параметры:
      
      Выберите вариант настройки:

      - **Базовая**: выберите один из готовых шаблонов кластера. Шаблоны отличаются количеством worker-узлов, количеством CPU и RAM на каждом узле.
      - **Продвинутая**: задайте количество CPU и RAM для master-узла кластера **Coordinator** и для worker-узла **Worker**. Также вы можете задать количество узлов **Worker**.

      {/tab}

      {/tabs}

      {ifndef(public)}
      {include(../../../_includes/_standalone.md)[tags=total_quotas]}
      {/ifndef}
      
   1. Нажмите кнопку **Следующий шаг**.
1. На шаге **Обслуживание**:

   1. Выберите дни недели и время начала проведения технических работ. В это время на сервере могут выполняться технические работы, в том числе создание резервных копий — сервис будет недоступен.

      {note:info}
      Длительность технических работ — 4 часа. Временная зона — GMT+03:00. 
      {/note}

   1. Если хотите изменить задачи, которые будут выполняться при обслуживании экземпляра, в блоке **Задачи по расписанию** нажмите **Обслуживание iceberg подключений** и настройте параметры:
      
         - **Частота запуска** — время начала и периодичность обслуживания подключений к Iceberg. Чтобы изменить значение параметра:
           
            1. Выберите в списке **Создать индивидуальное расписание**.
            2. Задайте время и частоту запуска в формате Cron-выражения.

         - (Опционально) Задайте дополнительные параметры:
           
            - **Продолжительность** — максимальное время выполнения в секундах. Значение по умолчанию — `3600`.
            - **Возраст снимков** — максимальный возраст снимков файлов метаданных и данных. Снимки старше указанного значения будут удалены. Значение по умолчанию — `7d`.
            - **Возраст orphan файлов** — максимальный возраст orphan-файлов данных. Файлы старше указанного значения будут удалены. Значение по умолчанию — `7d`.
            - **Минимальный размер файла манифеста** — минимальный размер файла манифеста. Файлы манифестов с размером ниже указанного значения будут оптимизированы. Значение по умолчанию — `128MB`.
            - **Удалять файлы метаданных** — опция для удаления устаревших файлов метаданных после выполнения коммита. Включена по умолчанию.
            - **Количество версий файлов метаданных** — максимальное количество версий файлов метаданных. Устаревшие версии будут удалены, если включена опция **Удалять файлы метаданных**. Значение по умолчанию — `10`.

   1. Нажмите кнопку **Следующий шаг**.

1. На шаге **Подключения** добавьте к экземпляру сервиса Trino источники данных:

   {note:info}
   Источники данных можно добавить и после создания экземпляра сервиса.
   {/note}

   1. Нажмите кнопку **Добавить подключение**.

   {include(../../../_includes/_trino.md)[tags=connect]}
   
   1. (Опционально) Чтобы добавить еще подключение, нажмите кнопку **Добавить подключение** и укажите его параметры.
   1. Нажмите кнопку **Следующий шаг**.

1. На шаге **Учетные данные**:

   1. Задайте логин администратора для доступа к Trino.
   1. Придумайте или сгенерируйте пароль администратора.

      {include(../../../_includes/_trino.md)[tags=password]}

      {note:info}
      Утерянный пароль невозможно восстановить, но можно {linkto(../management#trino_change_password)[text=поменять]} на новый.
      {/note}

   1. (Опционально) Чтобы добавить еще одну учетную запись администратора, нажмите кнопку **Добавить учетную запись** и укажите логин и пароль.
   1. Нажмите кнопку **Создать**.

      Дождитесь завершения операции. Создание экземпляра Cloud Trino может занять длительное время.

{/tab}

{/tabs}
