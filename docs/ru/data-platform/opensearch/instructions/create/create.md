# {heading(Создание экземпляра сервиса)[id=opensearch_create]}

{include(../../../_includes/_standalone.md)[tags=difference]}

1. Перейдите в раздел **Data Platform** → **Экземпляры сервисов**.
1. Нажмите кнопку **Создать экземпляр**.
1. Выполните шаги:
   1. {linkto(#opensearch_create_configuration)[text=Конфигурация]}.
   1. {linkto(#opensearch_create_parameters)[text=Параметры]}.
   1. {linkto(#opensearch_create_service)[text=Обслуживание]}.
   1. {linkto(#opensearch_create_user)[text=Учетные данные]}.

## {heading(Шаг 1. Конфигурация)[id=opensearch_create_configuration]}

1. Выберите тип сервиса `OpenSearch`.
1. Выберите версию сервиса.
1. Нажмите кнопку **Следующий шаг**.

## {heading(Шаг 2. Параметры)[id=opensearch_create_parameters]}

1. В поле **Название** введите название экземпляра сервиса.
1. (Опционально) В поле **Описание** введите описание экземпляра сервиса.
1. (Только для Standalone, опционально) Задайте IP-адрес для экземпляра сервиса из подсети. Для этого отключите опцию **Автоматическое назначение IP**. По умолчанию IP-адрес назначается экземпляру автоматически.
1. (Только для Standalone в составе {var(cloud)}, опционально) Добавьте группу безопасности. Если нужной группы безопасности нет в списке, создайте ее по инструкции из **Руководства администратора {var(cloud)}** → **Сети** → [Настройка firewall](https://cloud.vk.com/docs/on-premises/private-cloud/ru/4_3/admin-guide/service_management/network/network_firewall_setup).
1. (Пропустить для Standalone) Выберите кластер Kubernetes из выпадающего списка или создайте новый. В этот кластер будет добавлен worker-узел для размещения экземпляра OpenSearch.
1. В блоке **Узлы** задайте параметры группы подов для узлов OpenSearch, которые управляют кластером:

   - **CPU**: количество ядер процессора.
   - **RAM**: объем оперативной памяти.
   - **Тип диска**: выберите тип диска, либо он будет назначен автоматически.
   - **Размер диска**: размер диска в ГБ.
   - **Количество Masters**: общее количество узлов.

1. В блоке **Дашборд** задайте параметры группы подов для OpenSearch Dashboards:

   - **CPU**: количество ядер процессора.
   - **RAM**: объем оперативной памяти.
   - **Количество Dashboards**: общее количество дашбордов.

1. Нажмите кнопку **Следующий шаг**.

## {heading(Шаг 3. Обслуживание)[id=opensearch_create_service]}

1. В блоке **Техническое окно** выберите один или несколько дней и время проведения технических работ с учетом вашей временной зоны. Продолжительность технических работ: 4 часа.
1. В блоке **Полная резервная копия** укажите максимальное время хранения резервной копии в днях. 
1. Нажмите кнопку **Следующий шаг**.

## {heading(Шаг 4. Учетные данные)[id=opensearch_create_user]}

1. Задайте **Логин для доступа к OpenSearch**. Требования к логину:

   - Допустимы только цифры, латинские буквы и символ `_`.
   - Первый символ: латинская буква любого регистра или `_`.
   - Недопустимые имена: `os_admin`, `root`, `dataplatform_moth`, `default`.
   - Не более 50 символов.

1. **Роль**: роль пользователя. В настоящий момент для пользователя доступна только роль `Администратор`.
1. Задайте **Пароль для доступа к OpenSearch**. Для этого нажмите кнопку **Сгенерировать** или укажите свой. Требования к паролю:

   - Не менее 8 символов.
   - Минимум одна заглавная и одна строчная буквы латинского алфавита.
   - Минимум одна цифра.
   - Минимум один из символов: `!`, `?`, `%`, `#`, `/`, `(`, `)`, `-`, `+`, `*`.

   {note:err}
   Сохраните пароль. Восстановление забытого пароля недоступно.
   {/note}

1. (Опционально) Чтобы добавить еще одну учетную запись, нажмите кнопку **Добавить учетную запись**.
1. Нажмите кнопку **Создать**.
