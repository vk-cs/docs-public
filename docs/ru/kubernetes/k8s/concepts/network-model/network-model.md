# {heading(Сетевая модель Kubernetes)[id=k8s-network-model]}

Kubernetes предназначен для совместного использования машин между приложениями. Как правило, для совместного использования компьютеров требуется, чтобы два приложения не пытались использовать одни и те же порты.

Каждый под получает свой IP-адрес. Это создает чистую, обратно совместимую модель, при которой с подами можно работать как с ВМ или физическими хостами с точки зрения распределения портов, именования, обнаружения служб, балансировки нагрузки, конфигурации приложений и миграции.

Kubernetes предъявляет следующие фундаментальные требования к любой сетевой реализации (запрещая любые политики преднамеренной сегментации сети):

* Модули на узле могут связываться со всеми модулями на всех узлах без использования NAT.
* Агенты на узле, например системные демоны, могут связываться со всеми модулями на этом узле.
* Для платформ, поддерживающих работу с подами в сети хоста, например Linux, модули в хост-сети узла могут связываться со всеми модулями на всех узлах без NAT.

IP-адреса Kubernetes существуют в области видимости пода — контейнеры в рамках общего доступа к сетевым пространствам имен подов, включая их IP-адреса. Все контейнеры внутри пода могут достигать порты друг друга через `localhost`. Контейнеры внутри пода должны координировать использование порта.
