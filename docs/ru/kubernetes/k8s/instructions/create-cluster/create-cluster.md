# {heading(Создание кластера)[id=k8s-create-cluster]}

{ifdef(public)}
Различные способы создания кластера Kubernetes.
{/ifdef}

{ifndef(public)}
{note:info}
К ВМ кластера Kubernetes после создания применяется политика `soft-anti-affinity`: ВМ по возможности размещаются на разных вычислительных узлах.
{/note}

Чтобы создать кластер через личный кабинет:

1. {linkto(../../../../tools-for-using-services/account/instructions/lk-entry#tools-account-lk-entry)[text=Перейдите]} в личный кабинет {var(cloud)}.
1. {linkto(../../../../tools-for-using-services/account/instructions/project-settings/manage#project-view-quotas)[text=Убедитесь]}, что квот достаточно для создания кластера.
1. Перейдите в раздел **Кластеры Kubernetes** → **Кластеры Kubernetes**.
1. Нажмите кнопку **Добавить**.

   Если в {var(cloud)} нет созданных кластеров, название кнопки — **Создать кластер**.

1. {linkto(#k8s-create-cluster-create)[text=Настройте параметры инстанса кластера]}.
1. {linkto(#k8s-create-cluster-nodes-group)[text=Настройте группы узлов]}.

Кластер будет создан в течение 5-20 минут.

## {heading(1. Создание кластера)[id=k8s-create-cluster-create]}

1. Задайте параметры:

   * **Имя кластера** — название создаваемого кластера.
   * **Версия Kubernetes** — версия кластера.
   * **Зона доступности** — зона доступности ВМ из списка зон.
   * **Тип виртуальной машины Master** — {linkto(../../concepts/flavors#k8s-flavors-vm-flavor)[text=шаблон конфигурации ВМ]} из списка.
   * **Тип Master-диска** — {linkto(../../../../computing/iaas/concepts/data-storage/disk-types#iaas-disk-types)[text=тип диска]} из списка. Выберите тип, либо он будет назначен автоматически.
   * **Количество узлов Master** — количество создаваемых master-узлов.
   * **Размер диска на Master-узле** — объем диска на каждом master-узле в ГБ.
   * **Сеть** — сеть и подсеть кластера, в которой будут размещаться master- и worker-узлы кластера. Если нужной сети нет в списке, {linkto(../../../../networks/vnet/instructions/net#vnet-net-add)[text=создайте]} их.
   * **Использовать сеть Load Balancer** — настроить использование сети балансировщика нагрузки. 

     Включите эту опцию, чтобы для создаваемых кластером балансировщиков нагрузки использовалась отдельная подсеть в выбранной сети. При активации переключателя появляется поле **Сеть Load Balancer** с выбором подсети, в которой будут создаваться балансировщики нагрузки. Если нужной подсети нет в списке, {linkto(../../../../networks/vnet/instructions/net#vnet-net-subnet-add)[text=создайте ее]}. 

     По умолчанию опция отключена, и балансировщики нагрузки используют ту же подсеть, что и узлы кластера.

   * **Использовать подсеть пода** — настроить использование подсети, которую будут использовать поды для коммуникации между собой. 

     При активации переключателя появляется поле **Подсеть пода** с выбором подсети, в которой будут создаваться поды. Не должна пересекаться с подсетью нод кластера. По умолчанию используется значение `10.100.0.0/16`.

   * **Назначить внешний IP** — включите эту опцию, чтобы API-эндпоинту кластера и предустановленному Ingress-контроллеру (если он выбран на предыдущем шаге) были назначены внешние IP-адреса, иначе они будут назначены из подсети кластера. 

     По умолчанию опция включена, что позволяет получить доступ к кластеру и Ingress-контроллеру из интернета.

   * **Доверенные Docker Registry** — доверенные репозитории Docker Registry, у которых не проверяется TLS при загрузке образов. Чтобы добавить адрес репозитория, нажмите кнопку **Добавить адрес репозитория** и введите значение в появившееся поле. Примеры значений: `172.24.4.3`, `172.24.4.3:5000`, `example.com`.
   * **Включить мониторинг** — включите эту опцию, чтобы установить в кластер агент сбора метрик. {linkto(../../../../monitoring-services/monitoring/instructions/manage-dashboards#monitoring-manage-dashboards)[text=Визуализация метрик]} будет доступна в разделе **Мониторинг**.

1. Нажмите кнопку **Следующий шаг**.

## {heading(2. Группы узлов)[id=k8s-create-cluster-nodes-group]}

1. Задайте параметры:

   * **Название группы** — название группы узлов кластера.
   * **Тип Node-узлов** — шаблон конфигурации ВМ из списка шаблонов, доступных в проекте.
   * **Зона доступности** — зона доступности ВМ.
   * **Тип диска** — {linkto(../../../../computing/iaas/concepts/data-storage/disk-types#iaas-disk-types)[text=тип диска]} из списка. Выберите тип, либо он будет назначен автоматически.
   * **Размер диска** — объем диска узла в ГБ.
   * **Количество узлов Node** — количество развертываемых узлов.
   * **Включить автомасштабирование** — при выборе пункта размер кластера увеличится за счет добавления нового узла в кластер. Расширение кластера выполняется, если не могут запуститься новые поды из-за нехватки ресурсов. После падения нагрузки узел удалится. При включении появляются поля:

     * **Минимальное количество узлов** — минимально возможное количество узлов при автомасштабировании.
     * **Максимальное количество узлов** — максимально возможное количество узлов при автомасштабировании.

   * **Процент недоступных нод при обновлении кластера** — процентное количество узлов, которые можно вывести из группы узлов при обновлении кластера.
   * **Kubernetes labels** — пары **ключ-значение**, которые добавляются к объектам как поды. Чтобы добавить метку, нажмите **Добавить label** и заполните соответствующие поля.
   * **Node taints** — параметр узла, который не допускает запуска пода на определенном узле. Чтобы добавить параметр, нажмите **Добавить taint** и заполните соответствующие поля.

1. (Опционально) Добавьте дополнительную группу узлов, нажав кнопку **Добавить группу узлов** и заполнив поля, описанные выше.
1. Нажмите кнопку **Создать кластер**.
{/ifndef}
