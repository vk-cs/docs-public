# {heading(Fluent Bit)[id=k8s-install-private-logaas-integration]}

Аддон Fluent Bit позволяет организовать доставку логов из кластера Kubernetes в сервис логирования.

Источниками логов выступают службы kubelet и поды (pods), расположенные на узлах кластера.

Вне зависимости от выбранного варианта установки аддон будет установлен в виде DaemonSet на все узлы кластера, включая master-узлы.

## {heading(Установка аддона)[id=k8s-install-advanced-logaas-integration-install]}

Чтобы установить аддон Fluent Bit:

1. {linkto(../../../../../../tools-for-using-services/account/instructions/lk-entry#tools-account-lk-entry)[text=Перейдите]} в личный кабинет {var(cloud)}.
1. Перейдите в раздел **Кластеры Kubernetes** → **Кластеры Kubernetes** и нажмите на имя нужного кластера.
1. Перейдите на вкладку **Аддоны**.
1. Нажмите кнопку **Установить** на карточке аддона **logaas-integration**.
1. В поле **Версия** выберите необходимую версию аддона и нажмите **Установить аддон**.
1. При необходимости измените:

   * Название приложения.
   * Название пространства имен, куда будет установлен аддон.
   * Код аддона.

     Подробное описание параметров кода доступно [в репозитории Fluent Bit](https://github.com/fluent/helm-charts/blob/main/charts/fluent-bit/values.yaml).

1. Нажмите **Установить аддон**. Установка аддона может занять продолжительное время.

При необходимости выполните тонкую настройку аддона используя:

* Пользовательские фильтры.

  Задайте в коде аддона одно или несколько правил для пользовательских фильтров в поле `customFilter`, чтобы в сервис логирования попадали только логи с заданным минимальным уровнем критичности. Эти правила можно настроить на уровне определенного пространства имен и на уровне определенных подов в пространстве имен. Пример кода:

  ```console
  customFilter:
    - namespace: kube-system
      rules: # Одно правило для пространства имен
      - min_level: WARN
      - min_level: ERROR
        podprefix: kube-internal
    - namespace: default
      rules: # Несколько правил для подов в пространстве имен
        - podprefix: test-pod
          min_level: WARN
        - podprefix: someapi
          min_level: DEBUG
  ```

  Здесь:

  * `namespace` — имя пространства имен.
  * `min_level` — буквенное обозначение минимального уровня критичности. Примеры: WARN, ERROR, DEBUG.
  * `podprefix` — префикс имени пода.

* Пользовательские шаблоны

  Задайте в коде аддона одно или несколько правил для пользовательских шаблонов customRegexp, чтобы выставлять желаемый уровень критичности для логов в обход стандартных механизмов аддона. Если часть записи в логе совпадает с заданным шаблоном Lua ([pattern](https://www.lua.org/manual/5.4/manual.html#6.4.1)), то этой записи присваивается заданный в правиле уровень критичности. Если совпадений нет, то записи присваивается уровень критичности, определенный с помощью парсеров Fluent Bit. Это может быть полезно, если механизм автоматического определения уровня, который применяется в аддоне, некорректно определяет уровень логов конкретного приложения.

  Эти правила действуют на уровне определенного пространства имен и на уровне определенных подов в пространстве имен. Пример кода:

  ```console
  - namespace: kube-system
    rules: # Одно правило для пространства имен: настройка нескольких уровней
    - levels:
      - level: WARN
        reg_exp: "text message"
  - namespace: default
    rules: # Несколько правил для подов в пространстве имен: настройка нескольких
    - podprefix: test-pod
      levels:
      - level: ERROR
        reg_exp: "text message"
      - level: WARN
        reg_exp: "json message"
    - podprefix: another-another-test-pod
      levels:
      - level: WARN
        reg_exp: "text message"
      - level: FATAL
        reg_exp: "some"
      - levels:
      - level: ERROR
        reg_exp: "text message"
      - level: INFO
        reg_exp: "some"
  ```

  Здесь:

  * `namespace` — имя пространства имен.
  * `level` — буквенное обозначение минимального уровня критичности. Примеры: WARN, ERROR, DEBUG, INFO, FATAL.
  * `podprefix` — префикс имени пода.
  * `reg_exp` — шаблон Lua.

Подробнее о работе с аддоном — в официальной документации [Fluent Bit](https://docs.fluentbit.io/manual/installation/getting-started-with-fluent-bit).
