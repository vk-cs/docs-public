# {heading(Изменение адресов DNS-серверов в сети кластера)[id=k8s-change-dns]}

В статье показан пример изменения IP-адресов DNS-серверов, которые используются для разрешения доменных имен хостов в кластере.

## {heading(Подготовительные шаги)[id=k8s-change-dns-prepare]}

1. {linkto(/ru/tools-for-using-services/cli/openstack-cli#tools-cli-openstack)[text=Установите OpenStack CLI и пройдите аутентификацию]}, если это еще не сделано.
1. [Перейдите](https://msk.cloud.vk.com/app/) в личный кабинет {var(cloud)}.
1. Перейдите в раздел **Кластеры Kubernetes** → **Кластеры Kubernetes**.
1. Нажмите на имя кластера, для которого нужно изменить DNS-настройки.
1. На вкладке **Общие данные** узнайте имя сети и подсети, в которой расположен кластер.
1. Узнайте у своего DNS-провайдера IP-адреса DNS-серверов, которые нужно использовать вместо назначенных кластеру.

## {heading(1. Измените адреса DNS-серверов в настройках сети)[id=k8s-change-dns-network-settings]}

1. Получите ID подсети, которую нужно отредактировать:

   ```console
   openstack subnet list
   ```

1. Сбросьте все назначенные IP-адреса:

   ```console
   openstack subnet set --no-dns-nameservers <ID_ПОДСЕТИ>
   ```

   Здесь `<ID_ПОДСЕТИ>` — идентификатор подсети, которую вы хотите отредактировать.

1. Укажите новый IP-адрес:

   ```console
   openstack subnet set --dns-nameserver <АДРЕС_DNS> <ID_ПОДСЕТИ>
   ```
   
   Вы можете указать более одного IP-адреса, в таком случае перед каждым адресом повторите параметр `--dns-nameserver`. Пример команды:

   ```console
   openstack subnet set --dns-nameserver <АДРЕС_DNS_1> --dns-nameserver <АДРЕС_DNS_2> --dns-nameserver <АДРЕС_DNS_3> <ID_ПОДСЕТИ>
   ```

## {heading(2. Перезагрузите кластер)[id=k8s-change-dns-restart-cluster]}

Чтобы изменения вступили в силу:

1. {linkto(../../../instructions/manage-cluster#k8s-manage-cluster-stop)[text=Остановите]} ваш кластер и дождитесь полного завершения работы всех узлов.
1. {linkto(../../../instructions/manage-cluster#k8s-manage-cluster-start)[text=Запустите]} кластер заново и дождитесь инициализации всех узлов.

{note:warn}
Перезагрузка приводит к временной недоступности сервисов на кластере. Выполняйте остановку и запуск кластера в часы наименьшей активности приложения.
{/note}

При запуске кластера все его узлы получат обновленные DNS-настройки.
