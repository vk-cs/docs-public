|===
|Термин
|Определение

{ifdef(private-cert)}
|ACL
|Access Control List, список разрешений на действия для секрета или контейнера
{/ifdef}

|API
|Application Programming Interface, программный интерфейс приложения — набор классов, процедур, функций, структур или констант, которыми одна компьютерная программа может взаимодействовать с другой программой

|BIOS
|Basic Input/Output System, базовая система ввода-вывода

|Ceph
|Программная объектная отказоустойчивая сеть хранения данных

{ifndef(private-cert)}
|CIFS
|Common Internet File System, так же называют SMB (Server Message Block) — сетевой протокол прикладного уровня для удаленного доступа к файлам, принтерам и другим сетевым ресурсам, а также для межпроцессного взаимодействия
{/ifndef}

|CLI
|Command Line Interface, интерфейс командной строки

|CPU
|Central Processing Unit, центральный процессор

{ifndef(private-cert)}
|DBaaS
|Database as a Service, база данных как сервис — облачная модель, в которой пользователи могут эксплуатировать среду базы данных, выбирая из преднастроенного каталога сервисов
{/ifndef}

|DHCP
|Dynamic Host Configuration Protocol, протокол динамической настройки узла

|DNS
|Domain Name System, система доменных имен

{ifndef(private-cert)}
|GPU
|Graphics Processing Unit, графический процессор
{/ifndef}

|HDD
|Hard Disk Drive, жесткий диск

|HTTP
|HyperText Transfer Protocol, протокол передачи гипертекста

|HTTPS
|HyperText Transfer Protocol Secure, защищенный протокол передачи гипертекста

{ifndef(private-cert)}
|Helm
|Менеджер пакетов для Kubernetes, который используется для быстрой установки сложных приложений, таких как CRM, e-commerce, БД и прочих
{/ifndef}

|IaC
|Infrastructure as Code, инфраструктура как код — автоматизация инфраструктуры с использованием принципов и практик разработки программного обеспечения

|IaaS
|Infrastructure as a Service, инфраструктура как сервис — модель обслуживания в облачных вычислениях, по которой потребителям предоставляются информационно-технологические ресурсы — виртуальные серверы с заданной вычислительной мощностью, операционной системой и доступом к сети

{ifndef(private-cert)}
|Image-based приложение
|Продукт, который развертывается на основе образов виртуальных машин в облачном проекте пользователя
{/ifndef}

|IP
|Internet Protocol, интернет-протокол

|JSON
|JavaScript Object Notation, текстовый формат обмена данными, основанный на JavaScript

{ifndef(private-cert)}
|kubectl
|Инструмент командной строки для управления кластерами Kubernetes. Используется для развертывания приложений, проверки ресурсов кластера и просмотра логов
{/ifndef}

|LLD
|Low Latency Drive, тип накопителя, оптимизированный для работы с минимальными задержками и высокой производительностью. Представлен типом диска Low Latency NVMe (ef-nvme)

|Low Latency NVMe
|LLD-диск, реализованный на основе интерфейса NVMe. Располагается на одном гипервизоре с ВМ

{ifndef(private-cert)}
|Master-узел
|Главный узел, который отвечает за управление кластером Kubernetes и является точкой входа
{/ifndef}

|Multiqueue
|Множественные очереди, позволяют обрабатывать сетевые пакеты, распределяя их между несколькими виртуальными процессорами

|NAT
|Network Address Translation, перевод веб-адресов

{ifndef(private-cert)}
|NFS
|Network File System, протокол сетевого доступа к файловым системам

|PaaS
|Platform as a Service, платформа как сервис — модель предоставления облачных вычислений, при которой потребитель получает доступ к использованию информационно-технологических платформ: операционных систем, систем управления базами данных, связующему программному обеспечению, средствам разработки и тестирования, размещенным у провайдера. В этой модели вся информационно-технологическая инфраструктура, включая вычислительные сети, серверы, системы хранения, целиком управляются {var(cloud)} же определяется набор доступных для потребителей видов платформ и набор управляемых параметров платформ, а потребителю предоставляется возможность использовать платформы, создавать их виртуальные экземпляры, устанавливать, разрабатывать, тестировать, эксплуатировать на них прикладное программное обеспечение, при этом динамически изменяя количество потребляемых вычислительных ресурсов
{/ifndef}

|PCI
|Peripheral Component Interconnect, шина ввода-вывода для подключения периферийных устройств к материнской плате компьютера

|RAM
|Random Access Memory, оперативная память

|Root-диск, загрузочный диск
|Основной диск, с которого запускается операционная система при старте ВМ

{ifndef(private-cert)}
|SaaS
|Software as a Service, программное обеспечение как сервис — модель предоставления облачных вычислений, при которой потребитель получает доступ к использованию независимых сервисов, например кластеры Kubernetes

|SaaS-приложение
|Централизованно установленный мультитенантный продукт
{/ifndef}

|SDS Ceph
|Software-Defined Storage Ceph — программная объектная отказоустойчивая сеть хранения данных

|SSD
|Solid-State Drive, твердотельный накопитель — энергонезависимое немеханическое запоминающее устройство на основе микросхем памяти, альтернатива HDD

|SSH
|Secure Shell — защищенный протокол удаленного соединения между компьютером-клиентом и компьютером-сервером, позволяющий выполнять удаленное управление и передачу данных между ними

|TCP
|Transmission Control Protocol, протокол контроля передачи данных

|Terraform
|Open-source программное обеспечение Infrastructure as Code (IaC) или «инфраструктура как код», созданное HashiCorp. Оно позволяет пользователям определять и предоставлять инфраструктуру центра обработки данных, используя декларативный язык конфигурации, известный как язык конфигурации HashiCorp, или, опционально, JSON.

|URL
|Uniform Resource Locator, унифицированный указатель ресурса — система унифицированных адресов электронных ресурсов, или единообразный определитель местонахождения ресурса

{ifndef(private-cert)}
|VDI
|Virtual Desktop Infrastructure, технология виртуализации рабочих столов, которая позволяет запускать и управлять множеством виртуальных компьютеров на централизованных серверах

|vGPU
|Virtual GPU, технология виртуализации графического процессора, которая позволяет одному физическому GPU обслуживать сразу несколько виртуальных машин (ВМ)
{/ifndef}

|VLAN
|Virtual Local Area Network, виртуальная локальная компьютерная сеть

|VNC
|Virtual Network Computing, система удаленного доступа к рабочему столу компьютера

|VPN
|Virtual Private Network, виртуальная частная сеть

{ifndef(private-cert)}
|XaaS
|Everything as a Service, все как сервис — модель предоставления различных услуг в облачной среде (программного обеспечения, платформ, инфраструктуры)
{/ifndef}

|YAML
|Язык сериализации данных, обычно используемый в файлах конфигурации, таких как Kubernetes, Docker, Ansible и других

|Балансировщик нагрузки (load balancer)
|Балансировщики нагрузки позволяют распределять поток запросов (или сетевой трафик) между несколькими устройствами, например серверами, с целью:

* Горизонтального масштабирования.
* Обеспечения отказоустойчивости (резервирования).
* Оптимизации использования ресурсов (за счет динамического добавления или удаления устройств в зависимости от нагрузки)

|Виртуальная сеть (virtual network)
|Сеть, созданная с помощью виртуализации (виртуального маршрутизатора). Работает аналогично традиционной сети. Обеспечивает связь между виртуальными машинами (ВМ) и другими устройствами

|Виртуальный маршрутизатор
|Программная реализация сетевого маршрутизатора, которая работает аналогично традиционному маршрутизатору, но запускается с помощью средств виртуализации

{ifndef(private-cert)}
|ВРМ
|Виртуальные рабочие места
{/ifndef}

|Гипервизор (hypervisor)
|Выделенный сервер с настроенным программным гипервизором, обслуживающий запущенные экземпляры ВМ.

Каждый гипервизор имеет ОС, установленную на жесткие диски, настроенный гипервизор KVM, настроенные модули поддержки SDS Ceph и/или СХД, а также агенты для работы с сетевыми настройками

|Группа безопасности (security group)
|Наборы настраиваемых разрешающих правил, которые регулируют права сетевого доступа (вход по определенному протоколу, через конкретный порт, для определенных IP-адресов или групп безопасности)

|Группа узлов (node group)
|Множество узлов с общим шаблоном конфигурации ВМ (flavor)

|Деплой-нода, узел развертывания (deploy node)
|Вспомогательный узел для развертывания и конфигурирования {var(cloud)}. Обычно включает в себя репозиторий, в котором хранятся все артефакты, необходимые для процессов установки и обновления {var(cloud)}

|Диск (volume)
|Сетевое блочное устройство, которое обеспечивает хранилище данных для ВМ. Все диски в {var(cloud)} являются сетевыми и защищены репликацией данных, обеспечивающей надежность хранения и отказоустойчивость

|Журнал серийной консоли
|Способ получить удаленный доступ к серверу вне зависимости от состояния сети или операционной системы. Доступ осуществляется через систему удаленного доступа VNC (Virtual Network Computing)

|Зона доступности (availability zone)
|Логическое деление и объединение гипервизоров для физической их изоляции и обеспечения отказоустойчивости

|Инстанс (instance), ВМ
|Виртуальная машина, которая запускается и работает в облаке

{ifndef(private-cert)}
|Инстанс сервиса (service instance)
|Экземпляр сервиса, развернутый у пользователя

|Кластер Kubernetes
|Кластер Kubernetes состоит из набора машин, так называемых узлов, которые запускают контейнеризированные приложения. Кластер должен иметь как минимум один рабочий узел
{/ifndef}

|Ключевая пара (key pair)
|Взаимосвязанная пара, состоящая из открытого и закрытого асимметричных ключей.

Открытый ключ используется для шифрования данных, может быть широко распространен.

Закрытый ключ используется для расшифровки данных, хранится только у владельца ключевой пары

{ifndef(private-cert)}
|Метод балансировки
|Метод распределения заданий между несколькими сетевыми устройствами, например серверами, с целью оптимизации использования ресурсов
{/ifndef}

|Образ, образ диска (image)
|Файл, несущий в себе полную копию содержимого диска: структуру файловой системы и данные, находящиеся на диске. Образы используются для создания не пустых дисков, в том числе загрузочных.

Обычно образ, используемый в {var(cloud)} при создании загрузочного диска ВМ, содержит установленную операционную систему, а иногда и дополнительно ПО. Имя образа, чаще всего, отражает тип и версию операционной системы и другие особенности содержания образа

|ОС
|Операционная система

|ПО
|Программное обеспечение

|Под (pod)
|Группа из одного или нескольких контейнеров приложений, включающая общие используемые хранилища (тома), IP-адрес и информацию по их запуску

|Пользователь
|Арендатор облачных ресурсов

|Правило группы безопасности
|Набор параметров, определяющих условия прохождения трафика. Правила объединяются в группы, которые в свою очередь применяются для портов инстансов

|Резервная копия
|Копия (ВМ или диска) в том состоянии, когда была создана точка восстановления

{ifndef(private-cert)}
|СУБД
|Система управления базами данных
{/ifndef}

|Снимок диска, снэпшот (snapshot)
|Состояние файловой системы на определенный момент времени. Создание снимка диска останавливает запись гостевой ОС на короткое время, сохраняет состояние файлов системы и все последующие изменения записывает в отдельный файл

|Теги инстанса (tag)
|Произвольные сочетания символов, позволяющие индексировать инстансы или фильтровать их

|Узел (node)
|Отдельная физическая или виртуальная машина, на которой развернуты и выполняются компоненты {var(cloud)}

|Управляющий узел (controller node)
|Тип узла, обеспечивающий управление {var(cloud)} и автоматизацию множества процессов ее работы, но не предоставляющий ресурсы для конечных пользователей. Управляющие узлы предоставляют API и другие интерфейсы для работы с {var(cloud)}, обеспечивают работу внутренних сервисов {var(cloud)} и планировщиков, а также некоторые инструменты для администрирования {var(cloud)}

{ifndef(private-cert)}
|Файловое хранилище (file storage)
|Виртуальная файловая система, которую можно подключать к ВМ
{/ifndef}

|ЦОД
|Центр обработки данных — специализированное здание или помещение для размещения серверного и сетевого оборудования, оснащенное всей необходимой инфраструктурой. Примеры: электропитание, охлаждение, сети связи и прочие

|Шаблон конфигурации ВМ (flavor)
|Конфигурация инстанса, определяющая характеристики его ресурсов: количество и тип CPU, объем RAM и некоторые другие параметры
