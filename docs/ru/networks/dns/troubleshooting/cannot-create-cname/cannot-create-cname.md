# {heading(Не удалось создать CNAME-запись для корневого домена)[id=dns-cannot-create-cname]}

При попытке создать CNAME-запись для корневого домена (домена второго уровня) возникает ошибка. Согласно стандартам DNS (RFC), CNAME-запись нельзя создавать для корневого домена (APEX, например, `example.com`), только для поддоменов (`www.example.com`).

### {heading(Решение)[id=dns-cannot-create-cname-decision]}

Для корневого домена используйте запись типа ALIAS (или ANAME), если ваш DNS-провайдер ее поддерживает. DNS от {var(cloud)} поддерживает создание ALIAS-записей для таких сценариев.
