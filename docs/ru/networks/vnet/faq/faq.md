{cut(Для чего используются подсети в моем облаке и к каким зонам доступности они относятся?)}

Для анализа подсетей обратитесь в [техническую поддержку](/ru/contacts).
В обращении сообщите:

- {linkto(../../../tools-for-using-services/account/instructions/project-settings/manage#project-pid-view)[text=PID проекта]};
- {linkto(../../../networks/vnet/instructions/net#vnet-net-view)[text=ID подсетей]}.

{/cut}

{cut(Как управлять PTR-записями для IP-адреса?)}

Чтобы настроить обратный DNS-запрос с помощью PTR-записи, обратитесь в [техническую поддержку](/ru/contacts).
В обращении сообщите IP-адрес и соответствующее ему доменное имя.

{/cut}

{cut(Будет ли IP-адрес шлюза доступен из другого проекта?)}

Для организации сетевой связности между разными проектами {linkto(../../../networks/vnet/instructions/shared-nets#vnet-shared-nets)[text=используйте]} общие сети. Вы можете передать сеть из одного проекта в другой, после чего настроить маршрутизацию в обоих проектах.

{/cut}

{cut(Наблюдается периодический рост ошибок 5XX на магистральном канале. Что делать?)}

[Обратитесь в техническую поддержку](/ru/contacts) для диагностики сетевых проблем. Обязательно приложите результаты трассировки (MTR) в момент возникновения проблемы (минимум 1000 пакетов) и точные временные метки инцидентов.

{/cut}

{cut(Ограничивается ли трафик до внешних SMTP-серверов)}

Для защиты от спама на новых проектах исходящий трафик через порты SMTP (25, 465, 587) по умолчанию ограничен.

Если вы не можете отправить почту, обратитесь в [техническую поддержку](/ru/contacts), чтобы снять ограничение.

{/cut}

{cut(Есть ли ограничения на доступ к публичным ресурсам с IP-адресов {var(cloud)}?)}

{var(cloud)} предоставляет стандартный доступ в интернет без блокировки публичных ресурсов.

Вы можете проверить это, {linkto(../../../networks/vnet/quick-start#vnet-quick-start)[text=создав]} ВМ с доступом в интернет.

{/cut}

{cut(Какие есть особенности настройки VPN на Windows Server?)}

Для работы VPN (IPsec/IKEv2) на Windows Server настройте разрешающие правила:

1. В группах безопасности {var(cloud)}.
   
   Откройте входящий трафик по протоколу UDP на порты 500 и 4500. В качестве источника укажите IP-адрес или диапазон адресов VPN-клиентов.

1. В Брандмауэре Windows на виртуальной машине.
   
   Создайте правила, разрешающие входящие подключения по UDP к портам 500 и 4500.

{/cut}

{cut(Как сменить Floating IP-адрес?)}

Чтобы назначить ВМ новый Floating IP-адрес:

1. {linkto(../../../networks/vnet/instructions/ip/floating-ip#vnet-floating-ip-disassociate)[text=Отвяжите]} текущий Floating IP-адреса от порта.
1. {linkto(../../../networks/vnet/instructions/ip/floating-ip#vnet-floating-ip-add)[text=Получите]} новый Floating IP-адрес.

{note:warn}
Существующие Floating IP-адреса тарифицируются, даже если они не назначены какому-либо порту. Если отвязанный Floating IP-адрес вам больше не нужен, [удалите](/ru/networks/vnet/instructions/ip/floating-ip#vnet-floating-ip-delete) его.
{/note}

{/cut}

{cut(Как перенести порт с одной ВМ на другую?)}

Чтобы перенести порт с одной ВМ на другую:

1. {linkto(../../../networks/vnet/instructions/ports#vnet-ports-off)[text=Отключите]} порт от первой виртуальной машины.
1. {linkto(../../../networks/vnet/instructions/ports#vnet-ports-on)[text=Подключите]} порт ко второй виртуальной машине.

{/cut}

{cut(Есть ли возможность выбрать конкретный диапазон IP-адресов, если нет возможности разрешать сразу несколько IP-зон?)}

Выбор конкретного диапазона или подсети для публичного IP (Floating IP) в интерфейсе не предусмотрен. Адрес назначается случайным образом из доступного пула. Для реализации белых списков на вашей стороне рекомендуется использовать VPN-туннель или выделенное подключение, чтобы взаимодействовать через стабильный внутренний IP

{/cut}

{cut(Можно ли управлять продвинутым маршрутизатором через OpenStack CLI?)}

Нет, управление {linkto(../concepts/router#vnet-router-advanced)[text=продвинутым маршрутизатором]} через команды OpenStack CLI не поддерживается, так как это специализированный сервис {var(cloud)}. Для управления продвинутым маршрутизатором используйте {linkto(../instructions/advanced-router#vnet-advanced-router)[text=личный кабинет]} или {linkto(../../../tools-for-using-services/api/api-spec/network-api/neutron-sprut-api#api-spec-neutron-sprut)[text=API-запросы]}.

{/cut}
