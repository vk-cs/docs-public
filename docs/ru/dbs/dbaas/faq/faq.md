# {heading(Вопросы и ответы)[id=dbaas-faq]}

{cut(Как создать базу данных?)}

Вы можете создать базу данных с помощью {linkto(../instructions/create#dbaas-create)[text=пошаговой инструкции]} или {linkto(../quick-start#dbaas-quick-start)[text=быстрого старта]}.

{/cut}

{cut(Какие СУБД доступны в облаке?)}

Полный {linkto(../concepts/types#dbaas-types)[text=список]} поддерживаемых СУБД и их {linkto(../concepts/work-configs#dbaas-work-configs)[text=конфигурации]}.

{/cut}

{cut(Как выбрать нужную конфигурацию инстанса БД?)}

Вы можете выбрать нужный тип конфигурации при {linkto(../instructions/create#dbaas-create)[text=создании инстанса БД]}. После создания инстанса тип конфигурации изменить нельзя, но вы всегда можете добавить реплику. Подробнее о типах конфигураций читайте в разделе {linkto(../concepts/work-configs#dbaas-work-configs)[text=Типы конфигураций инстансов БД]}.

{/cut}

{cut(Могу ли я самостоятельно изменить конфигурационный файл инстанса БД?)}

Прямой доступ к конфигурационному файлу не предусмотрен, но некоторые параметры можно {linkto(../instructions/db-config#dbaas-db-config)[text=изменить]} через личный кабинет.

Если нужного параметра нет, обратитесь в [техническую поддержку](/ru/contacts) для согласования правок в конфигурации. Часть параметров критичны для работы инстанса баз данных и не могут быть изменены.

{/cut}

{cut(От чего зависит стоимость сервиса?)}

Стоимость сервиса зависит от выбранной {linkto(../concepts/work-configs#dbaas-work-configs)[text=конфигурации]}. Оплата рассчитывается только за использованные ресурсы, подробнее в статье {linkto(../tariffication#dbaas-tariffication)[text=Тарификация]}.

{/cut}

{cut(Как добавить пользователя базы данных?)}

Воспользуйтесь инструкцией {linkto(../instructions/users#dbaas-users)[text=Управление пользователями]}.

{/cut}

{cut(Резервное копирование включено по умолчанию?)}

По умолчанию резервное копирование не включено. Вы можете его включить при {linkto(../instructions/create#dbaas-create)[text=создании инстанса]} или настроить в разделе [Резервное копирование](https://msk.cloud.vk.com/app/services/databases/backups/).

{/cut}

{cut(Включено ли резервное копирование для моей БД?)}

Чтобы проверить, включено ли резервное копирование:

1. [Перейдите](https://msk.cloud.vk.com/app/) в личный кабинет {var(cloud)}.
1. Выберите {linkto(../../../tools-for-using-services/account/concepts/projects#tools-account-concepts-projects)[text=проект]}, где находится нужный инстанс БД.
1. Перейдите в раздел **Базы данных** → **Резервное копирование**.
1. Перейдите на вкладку **Автоматическое** или **Point-in-time recovery**.

   Если рядом с именем нужного плана зеленый индикатор, резервное копирование работает по плану. Если индикатор красный, резервное копирование приостановлено.

Резервное копирование можно включить при {linkto(../instructions/create#dbaas-create)[text=создании инстанса БД]} или настроить отдельно, {linkto(../../../storage/backups/instructions/create-backup-plan#backup-plan-create-db)[text=создав план резервного копирования]}.

{/cut}

{cut(Можно ли изменить версию БД при восстановлении из резервной копии?)}

Да, но только для СУБД PostgreSQL и на более новую версию.

Для всех остальных СУБД доступна только версия, соответствующая версии резервной копии.

{/cut}

{cut(Можно ли выбрать новую версию PostgreSQL при восстановлении из резервной копии?)}

Да, можно.

При {linkto(../../../storage/backups/instructions/restore-from-backup#backup-restore)[text=восстановлении из резервной копии]} — автоматической, ручной или {linkto(../../../storage/backups/instructions/point-in-time-recovery-pitr#backup-pitr)[text=созданной в определенный момент времени (PITR)]} — выбор новой версии доступен в интерфейсе, если для текущей версии инстанса есть обновление.

{/cut}

{cut(Как происходит автомасштабирование диска БД?)}

Если автомасштабирование включено, по достижении порогового значения свободного места диск расширяется на 10 ГБ.

{/cut}

{cut(Как происходит масштабирование баз данных?)}

Вы можете изменить тип виртуальной машины, на которой размещены базы данных, или увеличить размер диска. При смене типа ВМ изменения вступают в силу после перезагрузки ВМ.

Также вы можете подключить автомасштабирование размера диска базы данных. Тогда при увеличении объема данных размер диска увеличится автоматически.

Инструкции по масштабированию инстанса БД в разделе {linkto(../instructions/manage-instance#dbaas-manage-instance)[text=Управление инстансом БД]}.

{/cut}

{cut(Как распределяются по ЦОД мастер, синхронная и асинхронная реплики?)}

Мастер и реплики находятся в одном ЦОД, но по запросу инженеры [технической поддержки](/ru/contacts) могут разнести их в разные ЦОД.

{/cut}

{cut(В конфигурации базы данных Master-Replica репликация асинхронная или синхронная?)}

Для **Master-Replica** конфигураций работает асинхронная репликация. Для кластера используется синхронная и асинхронная репликация.

{/cut}

{cut(Как создать реплику баз данных?)}

Воспользуйтесь инструкцией статьи {linkto(../instructions/replication#dbaas-replication)[text=Репликации]}.

{/cut}

{cut(Как установить расширения мониторинга для БД?)}

Установка расширений подробно описана в разделе {linkto(../instructions/managing-extensions#dbaas-managing-extensions)[text=Управление расширениями]}.

Для PostgreSQL мониторинг включен по умолчанию.

{/cut}

{cut(Каким образом настраивается автопереключение между узлами базы данных?)}

При использовании кластера баз данных при недоступности мастера переключение на другие узлы будет настроено автоматически. При использовании {linkto(../concepts/work-configs#dbaas-work-configs)[text=конфигурации]} **Master-Replica** переключение потребуется делать вручную:

- {linkto(../instructions/manage-instance/mysql#dbaas-mysql-switch-master)[text=MySQL]};
- {linkto(../instructions/manage-instance/postgresql#dbaas-postgresql-switch-master)[text=PostgreSQL]}.

{/cut}

{cut(Какой максимальный объем данных можно хранить в СУБД?)}

Объем ограничен возможностями диска — 2 ТБ для High-IOPS SSD, 5 Тб для SSD (для одного шарда), либо объемом RAM (Redis, Tarantool).

{/cut}

{cut(Можно ли создать пользователя с правами только на чтение на БД через личный кабинет?)}

Нет, это возможно только через прямой SQL-запрос.

{/cut}

{cut(Предоставляется ли root-доступ к ВМ, на которой размещен инстанс БД?)}

Нет, прямой доступ с правами суперпользователя (например, по SSH) к ВМ, на которой работает инстанс БД, не предоставляется. Все операции по управлению инстансом выполняются через личный кабинет или API.

{/cut}

{cut(Каким образом организован доступ пользователей к PostgreSQL?)}

Доступ пользователей к базам данных организован в соответствии с ролевой моделью самой PostgreSQL. Пользователь, имеющий максимальный доступ к одной базе данных, будет иметь доступ к другим базам по схеме `public`.

{/cut}

{cut(Можно ли использовать теги в Terraform?)}

{var(cloud)} не поддерживает теги в Terraform.

{/cut}

{cut(Можно ли перенести базу данных в другой проект?)}

Перенос объектов PaaS-сервисов между проектами не поддерживается. Виртуальная машина, на которой была развернута база данных, может быть перенесена в другой проект только как обычная виртуальная машина. Перенести такую виртуальную машину как инстанс базы данных или создать инстанс базы данных с диском, перенесенным из другого проекта, невозможно.

{/cut}

{cut(Как настроить интеграцию с внешними облачными сервисами?)}

Платформа не блокирует исходящий трафик к API других облачных провайдеров. Настройка интеграции между сервисами в {var(cloud)} и внешними сервисами выполняется стандартными средствами этих приложений.

{/cut}

{cut(Как настроить облачную версию PostgreSQL?)}

Сервис Cloud Databases предоставляет управляемые базы данных. Базовые параметры (CPU, RAM, диски) настраиваются при создании кластера. Дополнительные параметры конфигурации PostgreSQL можно изменять в личном кабинете.

{/cut}

{cut(Какой протокол используется при создании экземпляра Kafka?)}

По умолчанию используются защищенные протоколы SASL и SSL.

{/cut}

{cut(Возможно ли как-то изменить или расширить набор доступных макросов?)}

Доступные макросы для настройки репликации в ClickHouse заданы платформой {var(cloud)}. Их список нельзя изменить или расширить пользовательскими макросами.

{/cut}

{cut(Как обновить версию инстанса БД без прав администратора?)}

Cloud Databases — управляемый сервис, поэтому прямой доступ с правами `root/admin` к операционной системе и административным командам СУБД ограничен. Обновлять версию СУБД, изменять ее конфигурацию и управлять пользователями можно только через интерфейс [личного кабинета](https://msk.cloud.vk.com/app/) {var(cloud)} или API, а не через консоль внутри ВМ.

{/cut}
