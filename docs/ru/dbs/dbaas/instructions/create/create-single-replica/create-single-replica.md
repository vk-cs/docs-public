# {heading(Конфигурации Single и Master-Replica)[id=dbaas-create-single-replica]}

{tab(Личный кабинет)}

1. {ifdef(public)}[Перейдите](https://msk.cloud.vk.com/app/){/ifdef}{ifndef(public)}{linkto(../../../../../tools-for-using-services/account/instructions/lk-entry#tools-account-lk-entry)[text=Перейдите]}{/ifndef} в личный кабинет {var(cloud)}.
{ifndef(public)}
1. Убедитесь, что {linkto(../../../../../tools-for-using-services/account/instructions/project-settings/manage#project-view-quotas)[text=квот]} достаточно для создания инстанса БД.
{/ifndef}
1. Выберите проект, где нужно создать инстанс БД.
1. Перейдите в раздел **Базы данных → Инстансы баз данных**.
1. Нажмите кнопку **Создать базу данных** или **Добавить**.
1. На шаге **Конфигурация**:

   1. Выберите нужный тип базы данных.
   1. Выберите нужную версию.
   1. Выберите конфигурацию **Single** или **Master-Replica**. Конфигурация **Master-Replica** доступна не для всех {linkto(../../../concepts/work-configs#dbaas-work-configs-available-configs)[text=типов СУБД]}.
   1. Нажмите кнопку **Следующий шаг**.

1. На шаге **Создание инстанса**:

   1. Задайте общие параметры инстанса:

      - **Имя инстанса базы данных** или **Имя Master-инстанса базы данных**: может содержать только латинские буквы, цифры и спецсимволы `.`, `-`, `_`.

        Имена хостов инстанса будут состоять из указанного имени и суффикса, соответствующего СУБД.

        {note:info}
        В конфигурации **Master-Replica** к имени инстанса добавится суффикс `-1`, не влияющий на имена хостов. Например, если задано имя инстанса `my-instance`, то итоговое имя инстанса примет вид `my-instance-1`, а хосты получат имена вида `my-instance-...`, но не `my-instance-1-...`.
        {/note}

      {ifdef(public)}
      - **Категория виртуальной машины**: выберите {linkto(../../../../../computing/iaas/concepts/vm/flavor#iaas-flavor)[text=категорию]} предустановленных конфигураций ВМ.
      {/ifdef}

      - **Тип виртуальной машины**: шаблон конфигурации для инстанса БД.

        {ifdef(public)}
        Шаблоны с высокопроизводительными CPU доступны [по запросу в службу поддержки](/ru/contacts).
        {/ifdef}
        {ifndef(public)}
        При создании кластера PostgreSQL рекомендуется использовать конфигурацию не ниже, чем 4 ядра CPU и 8 ГБ оперативной памяти.
        {/ifndef}

      {ifdef(public)}
      - **Зона доступности**: {linkto(../../../../../start/concepts/architecture#architecture-az)[text=зона доступности]} для инстанса БД.
      {/ifdef}
      {ifndef(public)}
      - **Зона доступности**: зона доступности для инстанса.
      {/ifndef}

      - **Тип диска**: {linkto(../../../../../computing/iaas/concepts/data-storage/disk-types#iaas-disk-types)[text=тип диска]} для инстанса БД.

      - **Размер диска**: размер диска (в гигабайтах).

        Чем больше размер диска, тем выше его производительность в некоторых дисковых операциях.

        Для инстансов PostgreSQL вместо параметра **Размер диска** указываются два значения: **Размер основного диска** и **WAL-диск**. Размер WAL-диска рассчитывается автоматически как `Размер основного диска` × `0,2`.

        {note:warn}
        Для инстансов с PostgreSQL версии 14 и выше автоматически включается режим только для чтения, если остаток свободного места на основном или WAL-диске достигает меньшего из двух значений: 5% от объема диска или 4 ГБ.
        {/note}

      - **Включить автомасштабирование диска**: выберите эту опцию, чтобы размер диска автоматически увеличивался при заполнении диска данными.
        {ifdef(public)}
        Поддержка функциональности зависит от {linkto(../../../concepts/disks-autoscaling#dbaas-disks-autoscaling-autoscaling)[text=типа и версии СУБД]}.
        {/ifdef}
        При выборе этой опции также укажите **Максимальный размер диска**.

        Для инстансов PostgreSQL вместо параметра **Максимальный размер диска** указываются два значения: **Максимальный размер диска** и **Максимальный размер WAL-диска**. Максимальный размер WAL-диска рассчитывается автоматически как `Максимальный размер диска` × `0,2`.

      - **Сеть**: сеть, в которой будет размещаться инстанс БД. Если нужной сети нет в списке, {linkto(../../../../../networks/vnet/instructions/net#vnet-net-add)[text=создайте ее]}.

      - **Назначить внешний IP**: выберите эту опцию, чтобы назначить Floating IP-адрес инстансу БД.

        {ifdef(public)}
        Такой инстанс БД будет доступен из интернета.

        {note:warn}
        Использование Floating IP-адреса {linkto(../../../../../networks/vnet/tariffication#vnet-tariffication-charged)[text=тарифицируется]}.
        {/note}
        {/ifdef}

      - **Настройки Firewall**: список групп безопасности для инстанса БД.
      - **Создать реплику**: эта опция влияет на конфигурацию инстанса и определяет, будет ли добавлена реплика.

        {ifdef(public)}
        Опция доступна для следующих типов СУБД:

        - MySQL,
        - PostgreSQL.
        {/ifdef}
        {ifndef(public)}
        Опция доступна для PostgreSQL.
        {/ifndef}

        Опция действует так:

        - Если эта опция не выбрана, будет создан инстанс БД в конфигурации **Single**: с одним хостом с ролью `Master`.
        - Если эта опция выбрана, будет создан инстанс БД в конфигурации **Master-Repica**: с одним хостом в роли `Master` и другим хостом в роли `Replica`.

        {note:info}
        Имя каждой реплики состоит из имени инстанса и суффикса с порядковым номером хоста (например, `my-instance-3`). Нумерация хостов инстанса начинается с `1`.
        {/note}

   1. Настройте резервное копирование. При необходимости параметры резервного копирования можно {linkto(../../../../../storage/backups/instructions/create-backup-plan#backup-plan-create-db)[text=задать]} или {linkto(../../../../../storage/backups/instructions/manage-backup-plan#backup-plan-edit)[text=изменить]} после создания инстанса БД.

      {ifdef(public)}
      Для Tarantool резервное копирование недоступно.\
      {/ifdef}

      {tabs}

      {tab(Отключено)}

      Выберите эту опцию, чтобы не использовать резервное копирование для инстанса БД.

      {/tab}

      {ifdef(public)}
      {tab(Point-in-time recovery)}

      Опция доступна только для PostgreSQL. Эта опция позволяет выполнять резервное копирование по схеме Point-In-Time-Recovery (PITR).

      Если опция выбрана, задайте параметры резервного копирования:

      - **Время начала**: время начала резервного копирования в формате `ЧЧ:ММ`. Указанный в подсказке часовой пояс соответствует часовому поясу вашего устройства. Формат времени — 24 часа.

      - **Хранить, кол-во копий**: количество копий, которое необходимо хранить.

      - **Интервал резервного копирования**: периодичность создания резервных копий.

      {/tab}
      {/ifdef}

      {tab(Полное)}

      Задайте параметры резервного копирования:

      - Укажите **Периодичность резервного копирования**.
      - При необходимости включите резервное копирование по {linkto(../../../../../storage/backups/concepts/gfs-backup#backup-gfs)[text=стратегии GFS]} и настройте параметры хранения.
      - Если резервное копирование по стратегии GFS не включено, задайте максимальное количество полных резервных копий.

      {/tab}

      {/tabs}

   1. Нажмите кнопку **Следующий шаг**.

1. На шаге **Инициализация**:

   1. Выберите **Тип создания** и укажите параметры инициализации баз данных.

      {tabs}
      
      {tab(Новая база данных)}

      Будет создана новая пустая база данных.

      Для Redis параметров инициализации не предусмотрено.

      {ifdef(public)}
      Для Tarantool: задайте имя пользователя и пароль.
      {/ifdef}

      Для других типов СУБД:

      - Задайте имя базы данных.
      - Задайте имя пользователя и пароль.

      {/tab}

      {tab(Восстановить из копии)}
      
      Эта опция неактивна, если:

      - нет резервных копий, соответствующих выбранному типу и версии СУБД.
      {ifdef(public)}
      - резервное копирование не поддерживается (для Tarantool).
      {/ifdef}

      Базы данных будут восстановлены из резервной копии.

      Для PostgreSQL выберите **Тип бэкапа**:

      {ifdef(public)}
      - **Point-in-time recovery**:

        - Из выпадающего списка **Бэкап** выберите резервную копию, начиная с которой должно выполняться восстановление.
        - При необходимости выберите **Дату и время**. Базы данных будут восстановлены в состояние на указанный момент времени.

          Если не выбирать дату и время, то восстановление будет выполнено на последний доступный момент времени.
      {/ifdef}

      - **Полный**: из выпадающего списка **Бэкап** выберите резервную копию, из которой необходимо выполнить восстановление.

      Для других типов СУБД из выпадающего списка **Бэкап** выберите резервную копию, из которой необходимо выполнить восстановление.

      {/tab}
      
      {/tabs}

   1. Нажмите кнопку **Создать базу данных**.

      Дождитесь завершения операции. Создание инстанса БД может занять длительное время.

      {ifndef(public)}
      На инстанс БД после создания автоматически устанавливается агент, отправляющий данные в {linkto(../../../monitoring#dbaas-monitoring)[text=сервис мониторинга]} {var(cloud)}.
      {/ifndef}
