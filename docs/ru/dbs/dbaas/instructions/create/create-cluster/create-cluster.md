# {heading(Конфигурация Кластер)[id=dbaas-create-cluster]}

{ifndef(public)}
{note:warn}
К ВМ инстансов БД в конфигурации `Кластер` после создания применяется политика `soft-anti-affinity`: ВМ по возможности размещаются на разных вычислительных узлах.
{/note}
{/ifndef}

{ifdef(public)}
## {heading(OpenSearch)[id=dbaas-create-cluster-opensearch]}

{tabs}

{tab(Личный кабинет)}

1. [Перейдите](https://msk.cloud.vk.com/app/) в личный кабинет {var(cloud)}.
1. Выберите проект, где нужно создать кластер.
1. Перейдите в раздел **Базы данных → Инстансы баз данных**.
1. Нажмите кнопку **Создать базу данных** или **Добавить**.
1. На шаге **Конфигурация**:

   1. Выберите тип базы данных `OpenSearch`.
   1. Выберите нужную версию OpenSearch, например, `2`.
   1. Нажмите кнопку **Следующий шаг**.

1. На шаге **Параметры**:

   1. Задайте общие параметры кластера:

      - **Имя кластера базы данных**: может содержать только латинские буквы, цифры и спецсимволы `-`, `_`.

        Имена узлов кластера будут состоять из указанного имени и суффикса. Суффикс будет различаться для разных СУБД.
      
      - **Зона доступности**: {linkto(../../../../../start/concepts/architecture#architecture-az)[text=зона доступности]} для кластера.
      - **Сеть**: сеть, в которой будет размещаться кластер. Если нужной сети нет в списке, {linkto(../../../../../networks/vnet/instructions/net#vnet-net-add)[text=создайте ее]}.

   1. Настройте резервное копирование. При необходимости параметры резервного копирования можно {linkto(../../../../../storage/backups/instructions/create-backup-plan#backup-plan-create-db)[text=задать]} или {linkto(../../../../../storage/backups/instructions/manage-backup-plan#backup-plan-edit)[text=изменить]} после создания кластера.

      {tabs}
      
      {tab(Отключено)}
            
      Выберите эту опцию, чтобы не использовать резервное копирование для кластера.

      {/tab}
      
      {tab(Полное)}
      
      Задайте параметры резервного копирования:

      - Укажите **Периодичность резервного копирования**.
      - При необходимости включите резервное копирование по {linkto(../../../../../storage/backups/concepts/gfs-backup#backup-gfs)[text=стратегии GFS]} и настройте параметры хранения.
      - Если резервное копирование по стратегии GFS не включено, задайте максимальное количество полных резервных копий.

      {/tab}

      {/tabs}

   1. Нажмите кнопку **Следующий шаг**.

1. На шаге **Ноды**:

   1. В блоке **Dashboards** выберите опцию **Включить Dashboards**, если необходим доступ к веб-интерфейсу [OpenSearch Dashboards](https://opensearch.org/docs/latest/dashboards/quickstart-dashboards/).

      Затем задайте параметры:

      - **Категория виртуальной машины**: выберите {linkto(../../../../../computing/iaas/concepts/vm/flavor#iaas-flavor)[text=категорию]} предустановленных конфигураций ВМ.

      - **Тип виртуальной машины**: шаблон конфигурации для кластера.

        Шаблоны с высокопроизводительными CPU доступны [по запросу в службу поддержки](/ru/contacts).

      - **Зона доступности**: {linkto(../../../../../start/concepts/architecture#architecture-az)[text=зона доступности]} для кластера.

      - **Тип диска**: {linkto(../../../../../computing/iaas/concepts/data-storage/disk-types#iaas-disk-types)[text=тип диска]} для кластера.

      - **Размер диска, GB**: размер диска (в гигабайтах).

        Чем больше размер диска, тем выше его производительность в некоторых дисковых операциях.

      - **Назначить внешний IP**: выберите эту опцию, чтобы иметь доступ к веб-интерфейсу из интернета.

        Узлу с ролью `Dashboards` будет назначен Floating IP-адрес.

        {note:warn}
        Использование Floating IP-адреса {linkto(../../../../../networks/vnet/tariffication#vnet-tariffication-charged)[text=тарифицируется]}.
        {/note}

   1. В блоке **Master Node** выберите опцию **Использовать выделенные master ноды**, если необходимо назначить роль `master` на выделенный узел кластера.

      {note:info}
      Если эта опция не выбрана, роль `master` будет назначена узлам с ролью `data`. {linkto(../../../concepts/architecture#dbaas-architecture-opensearch)[text=Подробнее о структуре кластера и ролях]}.
      {/note}

      Затем задайте параметры:

      - **Категория виртуальной машины**: выберите {linkto(../../../../../computing/iaas/concepts/vm/flavor#iaas-flavor)[text=категорию]} предустановленных конфигураций ВМ.

      - **Тип виртуальной машины**: шаблон конфигурации для кластера.

        Шаблоны с высокопроизводительными CPU доступны [по запросу в службу поддержки](/ru/contacts).

      - **Зона доступности**: {linkto(../../../../../start/concepts/architecture#architecture-az)[text=зона доступности]} для кластера.

      - **Тип диска**: {linkto(../../../../../computing/iaas/concepts/data-storage/disk-types#iaas-disk-types)[text=тип диска]} для кластера.

      - **Размер диска, GB**: размер диска (в гигабайтах).

        Чем больше размер диска, тем выше его производительность в некоторых дисковых операциях.

   1. В блоке **Data Node** выполните настройки узлов с ролью `data`.

      Задайте параметры:

      - **Категория виртуальной машины**: выберите {linkto(../../../../../computing/iaas/concepts/vm/flavor#iaas-flavor)[text=категорию]} предустановленных конфигураций ВМ.

      - **Тип виртуальной машины**: шаблон конфигурации для кластера.

        Шаблоны с высокопроизводительными CPU доступны [по запросу в службу поддержки](/ru/contacts).

      - **Зона доступности**: {linkto(../../../../../start/concepts/architecture#architecture-az)[text=зона доступности]} для кластера.

      - **Тип диска**: {linkto(../../../../../computing/iaas/concepts/data-storage/disk-types#iaas-disk-types)[text=тип диска]} для кластера.

      - **Размер диска, GB**: размер диска (в гигабайтах).

        Чем больше размер диска, тем выше его производительность в некоторых дисковых операциях.

      - **Количество узлов**.

   1. Нажмите кнопку **Следующий шаг**.

1. На шаге **Инициализация**:

   1. Выберите **Тип создания** и укажите параметры инициализации баз данных.

      {tabs}
      
      {tab(Новая база данных)}
            
      Будет создана новая пустая база данных.

      Задайте:

      - Имя базы данных.
      - Имя пользователя и пароль.

      {/tab}
      
      {tab(Восстановить из копии)}
      
      Эта опция неактивна, если нет резервных копий, соответствующих выбранному типу и версии СУБД.

      Базы данных будут восстановлены из резервной копии.

      {note:warn}
      Перед восстановлением убедитесь, что:

      - Количество шардов в новом инстансе БД такое же, как в резервной копии инстанса.
      - Размер дисков в новом инстансе БД не меньше, чем в резервной копии.
      - Названия шардов нового инстанса БД такие же, как в резервной копии.
      {/note}

      Из выпадающего списка **Бэкап** выберите резервную копию, из которой необходимо выполнить восстановление.

      {/tab}

      {/tabs}

   1. Нажмите кнопку **Создать базу данных**.

      Дождитесь завершения операции. Создание кластера может занять длительное время.

{/tab}
