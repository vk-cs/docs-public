# {heading(Резервное копирование данных в объектное хранилище с помощью rclone)[id=rclone-backup-scenarios]}

С помощью сервиса {var(s3)} можно организовать резервное копирование данных, выбрав подходящий инструмент под конкретную задачу. Для работы из командной строки подходит rclone — утилита, поддерживающая S3-совместимые хранилища, шифрование на лету и гибкие политики хранения.

Для примера далее используются следующие данные:

- Имя бакета: `backup-bucket`
- Регион: `ru-msk`
- Эндпоинт {var(s3)}: `https://hb.ru-msk.vkcloud-storage.ru`

## {heading(Подготовительные шаги)[id=rclone-backup-scenarios-prepare]}

1. Создайте аккаунт для сервиса {var(s3)}, если это еще не сделано:
   1. [Перейдите](https://msk.cloud.vk.com/app) в личный кабинет {var(cloud)}.
   1. Выберите проект.
   1. Перейдите в раздел **Object Storage** → **Аккаунты**.
   1. Нажмите кнопку **Добавить аккаунт** или **Добавить**.
   1. Задайте имя аккаунта.
   1. Нажмите кнопку **Создать**.
   1. В открывшемся окне скопируйте и сохраните идентификатор ключа доступа (**Access Key ID**) и секретный ключ (**Secret Key**).

      {note:warn}
      После закрытия окна восстановить секретный ключ будет невозможно. Если ключ утерян, создайте новый.
      {/note}

1. Убедитесь, что [AWS CLI](../../../../tools-for-using-services/cli/aws-cli) установлен и настроен для работы с созданным аккаунтом {var(s3)}. Это понадобится, если планируете управлять жизненным циклом объектов через CLI.
1. {linkto(../../instructions/buckets/create-bucket#s3-instructions-create-bucket)[text=Создайте]} бакет с именем `backup-bucket`.
1. Установите утилиту rclone, следуя [официальной документации](https://rclone.org/install/).

## {heading(1. Настройте rclone для работы с {var(s3)})[id=rclone-backup-scenarios-step1]}

1. Запустите интерактивную настройку rclone:

   ```console
   rclone config
   ```

1. Выберите опцию `n` (new remote) для создания нового подключения и задайте параметры:

    * Укажите имя подключения, например `vkcloud`.
    * Выберите тип хранилища `s3` (Storage).
    * В качестве провайдера выберите `Other` (любой S3-совместимый провайдер).
    * Задайте параметры подключения.

    Итоговая конфигурация в файле `~/.config/rclone/rclone.conf` должна выглядеть так:

    ```ini
    [vkcloud]
    type = s3
    provider = Other
    env_auth = false
    access_key_id = <ACCESS_KEY>
    secret_access_key = <SECRET_KEY>
    region = ru-msk
    endpoint = https://hb.ru-msk.vkcloud-storage.ru
    ```

    Здесь:

    - `<ACCESS_KEY>` — значение **Access Key ID**, полученное при создании аккаунта {var(s3)}.
    - `<SECRET_KEY>` — значение **Secret Key**, полученное при создании аккаунта {var(s3)}.
    - `ru-msk` — код региона аккаунта. Для региона Казахстан используйте `kz-ast`.
    - `https://hb.ru-msk.vkcloud-storage.ru` — эндпоинт сервиса {var(s3)} региона Москва. Для региона Казахстан используйте `https://hb.kz-ast.vkcloud-storage.ru`.

1. Проверьте подключение, выполнив команду:

   ```console
   rclone lsd vkcloud:
   ```

   В выводе должен появиться созданный бакет `backup-bucket`.

## {heading((Опционально) 2. Настройте фильтры для оптимизации трафика)[id=rclone-backup-scenarios-step2]}

При резервном копировании можно исключить временные и ненужные файлы, чтобы сократить объем передаваемых данных и время выполнения операции. Для этого создайте файл-фильтр `filter.txt` и укажите в нем файлы, которые нужно исключить:

```text
# filter.txt
- *.tmp
- node_modules/**
+ *
```

Имя этого файла нужно будет указать с флагом `--filter-from` при вызове команды синхронизации локального и удаленного хранилища.

Знак `+` перед именем файла означает, что файл должен быть включен в резервное копирование, знак `-` — проигнорирован. В примере файла выше фильтры будут работать следующим образом:

* исключить все файлы, заканчивающиеся на `.tmp` (на любом уровне вложенности);
* исключить папку `node_modules` и все ее содержимое рекурсивно;
* включить все остальное, что не совпало с предыдущими правилами.

{note:info}
Порядок фильтров в `--filter-from` важен: rclone обрабатывает правила сверху вниз. Первое совпавшее правило определяет, будет файл включен или исключен.
{/note}

Если случай сложный или файлов для фильтрации немного, можно будет указать их напрямую в команде синхронизации `sync` через флаги `--exclude` или `--include`.

## {heading((Опционально) 3. Настройте шифрование данных на лету)[id=rclone-backup-scenarios-step3]}

Утилита rclone поддерживает шифрование данных перед отправкой в хранилище с помощью подключения типа `crypt`. Это позволяет хранить данные в зашифрованном виде, даже если бакет не имеет серверного шифрования.

Чтобы настроить шифрование при синхронизации локального и удаленного хранилища:

1. Запустите интерактивную настройку rclone:

   ```console
   rclone config
   ```

1. Выберите опцию `n` (new remote) для создания нового подключения и задайте параметры:

    * Укажите имя подключения с шифрованием, например `vkcloud-encrypted`.
    * Выберите тип `crypt`.
    * В качестве источника подключения укажите `vkcloud:backup-bucket`.
    * Задайте основной и дополнительный пароли для шифрования.

   Итоговая конфигурация в файле `~/.config/rclone/rclone.conf` должна выглядеть так:

   ```ini
   [vkcloud-encrypted]
   type = crypt
   remote = vkcloud:backup-bucket
   password = <ПАРОЛЬ>
   password2 = <ПАРОЛЬ_СОЛЬ>
   ```

   Здесь:

   * `vkcloud:backup-bucket` — полный путь к удаленному хранилищу, который состоит из имени удаленного хранилища, настроенного ранее через `rclone config`, и названия бакета (хранилища) в сервисе VK Object Storage.
   * `<ПАРОЛЬ>` — зашифрованный пароль.
   * `<ПАРОЛЬ_СОЛЬ>` — дополнительный зашифрованный пароль, который алгоритм утилиты rclone объединяет с основным паролем для вычисления итогового криптографического ключа.

   {note:warn}
   Основной и дополнительный пароли невозможно восстановить. Сохраните их в безопасном месте — без них расшифровка данных будет невозможна.
   {/note}

## {heading((Опционально) 4. Настройте политики хранения резервных копий)[id=rclone-backup-scenarios-step4]}

Чтобы автоматически удалять устаревшие резервные копии, используйте встроенные возможности rclone:

1. Удалите файлы старше заданного количества дней:

   ```console
   rclone delete vkcloud:backup-bucket --min-age 30d
   ```

Здесь:

   * `vkcloud:backup-bucket` — полный путь к удаленному хранилищу, который состоит из имени удаленного хранилища, настроенного ранее через `rclone config`, и названия бакета (хранилища) в сервисе VK Object Storage.
   * `--min-age` — опция для фильтрации файлов по возрасту, которая позволяет копировать или обрабатывать файлы не старше заданного времени, например не старше 30 дней.

Для инкрементальных бэкапов с сохранением предыдущих версий используйте флаг `--backup-dir` напрямую в команде синхронизации `sync`.

## {heading(5. Проверьте работоспособность резервного копирования)[id=rclone-backup-scenarios-check]}

1. Создайте тестовый файл:

   ```console
   mkdir -p /data
   echo "test backup data" > /data/test-file.txt
   ```

1. Выполните резервное копирование с шифрованием, заданными фильтрами и настроенными политиками хранения резервных копий:

   ```console
   rclone sync /data vkcloud-encrypted:/current \
     --filter-from filter.txt \
     --verbose \
     --backup-dir vkcloud:backup-bucket/archive/$(date +%Y-%m-%d)
   ```

   Здесь:

      * `/data` — источник, откуда нужно перенести данные.
      * `vkcloud-encrypted:/current` — подсключение к папке `/current` удаленного хранилища (VK Object Storage) с шифрованием данных.
      * `--filter-from filter.txt` — файл-фильтр с правилами фильтрации данных при синхронизации.
      * `--backup-dir` — опция для переопределения поведения команды `sync`: при каждом запуске текущая версия файлов из папки `current` будет перемещена в папку `archive/<ДАТА>`, а новые файлы загружены в `current`.
      * `--verbose` — опция включения подробного вывода: имена файлов, статус передачи, пропуски, ошибки, статистика и прогресс по потокам.

   {cut(Пример вывода команды)}

   ```console
   Transferred:        16 B / 16 B, 100%, 0 B/s, ETA -
   Transferred:            1 / 1, 100%
   Elapsed time:         0.5s
   ```

   {/cut}

1. Проверьте, что файл появился в бакете:

   ```console
   rclone ls vkcloud-encrypted:
   ```

   В выводе должен присутствовать файл `test-file.txt`.

1. Убедитесь, что данные в бакете зашифрованы — выведите список объектов в бакете VK Object Storage напрямую:

   ```console
   rclone ls vkcloud:backup-bucket
   ```

   Имена файлов и их содержимое должны отличаться от оригинальных, так как `crypt` шифрует и имена, и данные.

1. Восстановите файл из резервной копии:

   ```console
   mkdir -p /restored-data
   rclone copy vkcloud-encrypted:test-file.txt /restored-data/
   ```

1. Проверьте содержимое восстановленного файла:

   ```console
   cat /restored-data/test-file.txt
   ```

   Вывод должен содержать: `test backup data`.
