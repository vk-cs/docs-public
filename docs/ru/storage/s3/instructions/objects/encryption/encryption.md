# {heading(Шифрование объектов)[id=s3-instructions-encryption]}

В {var(s3)} есть возможность шифровать объекты в двух режимах:

- Шифрование на стороне сервера (Server-side encryption, SSE), без возможности управления ключами (SSE-E).
- Шифрование с помощью управляемых клиентом ключей (SSE-C).

{note:warn}
Независимо от параметров шифрования объекты размером меньше 21 байта хранятся в незашифрованном виде.
{/note}

## {heading(Шифрование на стороне сервера)[id=s3-instructions-encryption-server]}

SSE-E выполняется по схеме Envelope Encryption. Каждый объект шифруется уникальным ключом с помощью службы управления ключами шифрования (Key Management Service, KMS). В качестве KMS в {var(s3)} используется Hashicorp Vault. В виде дополнительной защиты SSE шифрует ключ с помощью корневого ключа, который он регулярно меняет. Для шифрования данных используется 256-битный Advanced Encryption Standard (AES-256).

Объекты шифруются перед сохранением в бакет и расшифровываются при скачивании из него. Шифрование на стороне сервера предполагает два типа шифрования:

- Шифрование бакета по умолчанию. Загрузка и скачивание объекта производится без указания заголовков. Шифрование применяется ко всем новым объектам, при этом загруженные ранее объекты остаются без изменений. При отключении шифрования бакета соответственно: все новые объекты будут загружены без шифрования.
- Шифрование объекта, загружаемого в нешифрованный бакет. При загрузке объекта укажите заголовок `x-amz-server-side-encryption`: AES-256. При скачивании объекта данный заголовок не указывается.

Чтобы установить шифрование бакета по умолчанию, выполните команду:

```console
aws s3api put-bucket-encryption \
   --bucket <ИМЯ_БАКЕТА> \
   --endpoint-url=<URL_АДРЕС_СЕРВИСА> \
   --server-side-encryption-configuration '{
      "Rules": [
         {
         "ApplyServerSideEncryptionByDefault": {
  	        "SSEAlgorithm": "AES256"}}]}'
```

Чтобы удалить шифрование бакета, выполните команду:

```console
aws s3api delete-bucket-encryption \
   --bucket <ИМЯ_БАКЕТА> \
   --endpoint-url=<URL_АДРЕС_СЕРВИСА>
```

Чтобы получить конфигурацию шифрования бакета, выполните команду:

```console
aws s3api get-bucket-encryption \
   --bucket <ИМЯ_БАКЕТА> \
   --endpoint-url=<URL_АДРЕС_СЕРВИСА>
```

Пример вывода:

```console
{
   "ServerSideEncryptionConfiguration": {
      "Rules": [
         {
            "ApplyServerSideEncryptionByDefault": {
               "SSEAlgorithm": "AES256"
            }
         }
      ]
   }
}
```

## {heading(Шифрование с помощью управляемых пользователем ключей)[id=s3-instructions-encryption-key]}

Шифрование SSE-C производится ключом, предоставленным пользователем. Ключ используется только для шифрования и расшифровки объекта. После запроса ключ удаляется и не хранится в метаданных объекта. Ключ шифрования указывается при запросе через API.

Чтобы использовать шифрование пользовательскими ключами:

1. Сгенерируйте ключ шифрования алгоритмом AES-256. Посчитайте для него хеш MD5.
1. Загрузите объект и передайте ключ и хеш, закодированными в Base64, в заголовках:

   - `x-amz-server-side-encryption-customer-key`.
   - `x-amz-server-side-encryption-customer-key-MD5`.
   - `x-amz-server-side-encryption-customer-algorithm: AES256`.
