# {heading(Мониторинг)[id=s3-instructions-monitoring]}

## {heading(Просмотр журнала событий)[id=s3-instructions-monitoring-view]}

IAM Only с помощью сервиса Cloud Logging ведет журнал событий.

Чтобы посмотреть журнал событий в личном кабинете, перейдите в раздел **Мониторинг** → **Журнал событий**.

Чтобы посмотреть подробную информацию по событию, нажмите на запись в таблице журнала.

Чтобы обновить журнал событий, нажмите кнопку **Обновить**.

Чтобы скачать журнал событий одним файлом, нажмите кнопку **Скачать**.

{note:info}
Файл может содержать до 1000 записей.
{/note}

## {heading(Фильтрация событий)[id=s3-instructions-monitoring-filter]}

1. Перейдите в раздел **Мониторинг** → **Журнал событий**.
1. Нажмите кнопку **Поиск и фильтры**.
1. В области фильтрации укажите одно или несколько значений фильтра:

   - **Статус** — фильтр по статусу логов.
   - **Продукт** — фильтр по продукту.
   - Фиксированный временной период, за который нужно отобразить события в журнале. Может принимать значения: `5 минут`, `15 минут`, `Час`, `День` или `3 дня`.
   - Заданный временной период, за который нужно отобразить события в журнале. Установите вручную или с помощью календаря.

1. Нажмите кнопку **Найти**.

## {heading(Поиск событий)[id=s3-instructions-monitoring-search]}

1. Перейдите в раздел **Мониторинг** → **Журнал событий**.
1. Нажмите кнопку **Поиск и фильтры**.
1. В поле поиска укажите запрос (подробнее — в разделе {linkto(#s3-instructions-monitoring-language)[text=%text]}).
1. Нажмите кнопку **Найти**.

## {heading(Язык поисковых запросов)[id=s3-instructions-monitoring-language]}

Язык поисковых запросов позволяет:

- искать по одному или нескольким фильтрующим выражениям;
- применять логические операции и операции сравнения.

Для фильтрации журналов событий используйте параметры фильтров, перечисленные в {ifndef(s3-pdf)}таблице ниже.{/ifndef}{ifdef(s3-pdf)}{linkto(#tab_search_filter_options)[text=таблице %number]}.

{/ifdef}

|===
|Параметр
|Формат
|Описание

|`message`
|`message: "<ЗНАЧЕНИЕ>"`
|Поиск записей, в сообщениях которых есть заданные значения. Является параметром по умолчанию, в запросе можно не указывать

|`timestamp`
|`timestamp <ОПЕРАТОР_СРАВНЕНИЯ> "<ЗНАЧЕНИЕ>"`

|`stream_id`
|`stream_id: "<ИДЕНТИФИКАТОР_РЕСУРСА>"`

{note:info}

{/note}

### {heading(Требования)[id=s3-instructions-monitoring-requirements]}

1. Символы `"`, `'` и `\` в выражении экранируются с помощью `\`.
1. Значение можно не брать в кавычки, если оно:

   - Начинается с буквы латинского алфавита и содержит только буквы латинского алфавита, цифры и знаки подчеркивания.
   - Является целым числом без знака.

1. Значения для `timestamp` представлены в формате `RFC3339` (подробнее — в [официальной документации](https://www.ietf.org/rfc/rfc3339.txt)).

### {heading(Фильтры с несколькими условиями)[id=s3-instructions-monitoring-multiple-filters]}

Объединить несколько условий в одном фильтре можно с помощью логических операторов:

- `AND`,
- `OR`,
- `NOT`.

Используйте скобки для задания определенного порядка вычислений:

Пример:

```console
<ПАРАМЕТР_1>: "<ЗНАЧЕНИЕ_1>" AND (<ПАРАМЕТР_2> = "<ЗНАЧЕНИЕ_2>" OR <ПАРАМЕТР_3> "<ЗНАЧЕНИЕ_3>")
```

В этом примере будут найдены записи, в которых параметр `<ПАРАМЕТР_1>` содержит значение `"<ЗНАЧЕНИЕ_1>"` и выполнено хотя бы одно из двух условий:

1. Значение параметра `<ПАРАМЕТР_2>` равно `"<ЗНАЧЕНИЕ_2>"`.
1. Значение параметра `<ПАРАМЕТР_3>` меньше, чем `"<ЗНАЧЕНИЕ_3>"` в лексикографическом смысле.
