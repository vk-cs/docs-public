# {heading(Управление IAM Only)[id=s3-instructions-iamo-manage]}

## {heading(Управление аккаунтом)[id=s3-instructions-iamo-manage-account]}

1. Перейдите в личный кабинет.
1. Нажмите на имя пользователя в шапке страницы и выберите пункт **Настройки аккаунта**.
1. На вкладке **Профиль пользователя** ознакомьтесь с информацией.

{note:warn}
Для изменения учетных данных или пароля обратитесь к администратору {var(s3)}.
{/note}

## {heading(Управление проектами)[id=s3-instructions-iamo-manage-project]}

Проекты — организационные единицы виртуальной инфраструктуры в IAM Only. Пользователи могут быть участниками одного или нескольких проектов. Роли определяют, какие действия могут выполнять пользователи. Проекты используются для разделения ресурсов, ролей или других архитектурных особенностей реализуемых задач. 

Создание проектов и пользователей, назначение пользователям ролей и внесение изменений для существующих проектов и пользователей выполняет администратор {var(s3)}.

### {heading(Выбор проекта)[id=s3-instructions-iamo-manage-project-selection]}

1. Перейдите в личный кабинет.
1. В шапке страницы раскройте список проектов.

   Если больше пяти проектов, для более удобного выбора проекта используйте:

   - поиск проекта по идентификатору или названию;
   - фильтр по ролям пользователя;

1. Выберите проект.

### {heading(Настройки проекта)[id=s3-instructions-iamo-manage-project-settings]}

1. Перейдите в личный кабинет.
1. Нажмите на имя пользователя в шапке страницы и выберите пункт **Настройки проекта**.
1. Выберите нужную вкладку:

   - **Общая информация** — общая информация о проекте.
   - **Доступ по API** — значения параметров для использования API в текущем проекте.
   - **API Endpoints** — перечень точек доступа для сервисов текущего проекта.
   - **Terraform** — инструкция по подключению ПО Terraform к проекту, в том числе конфигурационные файлы Terraform.
