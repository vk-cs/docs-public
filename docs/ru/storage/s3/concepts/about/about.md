# {heading(Введение)[id=s3-concepts-about]}

{var(s3)} — это {ifdef(public)}{linkto(../../../../computing/iaas/concepts/data-storage/storage-types#iaas-concepts-storage-types)[text=объектное хранилище данных]}{/ifdef}{ifdef(s3,s3-pdf)}объектное хранилище данных{/ifdef}, основанное на протоколе S3. Оно подходит для хранения данных большого объема, к которым нечасто обращаются. Работать с {var(s3)} можно в личном кабинете {ifdef(public)}{var(cloud)}{/ifdef}{ifdef(s3,s3-pdf)}{linkto(../../instructions/iamo/iamo-auth#s3-instructions-iamo-auth)[text=IAM Only]}{/ifdef}, с помощью интерфейса командной строки AWS CLI, а также ряда {linkto(#s3-concepts-about-tools)[text=других инструментов]}.

{var(s3)} отличает:

{ifdef(public)}
- низкая стоимость хранения данных в сравнении с другими {linkto(../../../../computing/iaas/concepts/data-storage/storage-types#iaas-concepts-storage-types)[text=типами хранилищ]} в {var(cloud)};
{/ifdef}
- высокая масштабируемость и доступность;
- простота интеграции с приложениями.

Данные в {var(s3)} хранятся в {linkto(#s3-concepts-about-bucket)[text=бакетах]} в виде независимых {linkto(#s3-concepts-about-object)[text=объектов]} и их {linkto(#s3-concepts-about-metadata)[text=метаданных]}.

## {heading(Бакет)[id=s3-concepts-about-bucket]}

*Бакет* — это контейнер для хранения {linkto(#s3-concepts-about-object)[text=объектов]}. В бакет может быть загружено любое количество объектов. Общий объем объектов в бакете также неограничен. Пользователь, создавший бакет, получает полные права доступа к бакету и объектам в нем и может делиться этими правами с другими.

У каждого бакета есть имя, которое используется для доступа к объектам внутри него. Имя должно удовлетворять определенным {linkto(#s3-concepts-about-bucket-naming)[text=требованиям]}. После создания бакета изменить его имя нельзя.

В проекте можно создать {ifdef(public)}{linkto(../../../../tools-for-using-services/account/concepts/quotasandlimits#quotasandlimits-s3)[text=несколько бакетов]}{/ifdef}{ifdef(s3,s3-pdf)}несколько бакетов{/ifdef}, но нельзя вложить один бакет в другой. Внутри каждого бакета можно построить иерархию директорий. Для этого к именам объектов нужно добавить {linkto(#s3-concepts-about-object-key)[text=префиксы]}, которые {var(s3)} интерпретирует как имена директорий.

Бакет можно использовать для хостинга статических сайтов, {linkto(../../instructions/buckets/manage-bucket#s3-instructions-manage-bucket-domain)[text=привязав к нему свой домен]}{ifdef(public)}, а также в {linkto(../../instructions/buckets/manage-bucket#s3-instructions-manage-bucket-cdn)[text=сервисе CDN в качестве источника]}{/ifdef}.

В {var(s3)} поддерживаются:

- {linkto(../../concepts/lifecycle#s3-concepts-lifecycle)[text=автоматическое удаление]} объектов из бакета по заданным правилам;
- {linkto(../../concepts/features#s3-concepts-features-webhooks)[text=настройка уведомлений]} о событиях в бакете;
- {linkto(../../instructions/buckets/manage-bucket#s3-instructions-manage-bucket-sync)[text=синхронизация]} содержимого бакета и локальной директории;
- кросс-доменные запросы ресурсов в бакете по технологии {linkto(../../instructions/access-management/cors#s3-instructions-cors)[text=CORS]}.

### {heading(Правила именования бакетов)[id=s3-concepts-about-bucket-naming]}

Имя бакета должно:

- быть уникальным для всего сервиса {var(s3)}.
- содержать от 4 до 63 символов.
- начинаться и заканчиваться буквой или цифрой.
- состоять только из строчных латинских букв, цифр и символов: `.`, `-`.

  Для лучшей совместимости рекомендуется избегать использования точек (`.`) в именах бакетов. Если включить точки в имя сегмента, вы не сможете использовать адресацию в стиле виртуального хоста через HTTPS, если не выполните собственную проверку сертификата. Это связано с тем, что сертификаты безопасности, используемые для виртуального хостинга бакетов, не работают для бакетов с точками в их именах.

  Это ограничение не влияет на сегменты, используемые для статического хостинга веб-сайтов, поскольку статический хостинг веб-сайтов доступен только через HTTP.

Имя бакета не должно:

- совпадать с именем других бакетов сервиса {var(s3)}.
- начинаться с префикса `xn--`.
- заканчиваться суффиксом `-ol-s3` или `s3alias`.
- начинаться или заканчиваться символами `-` и `.`.
- содержать подряд две точки `..`, дефис и точку `-.`, точку и дефис `.-`.

  Примеры: `my..bucket`, `my-.bucket`, `my.-bucket`.

- иметь формат IP-адреса или схожий с ним, например `192.168.5.4`.

Имя бакета содержится в URL-адресах находящихся в нем объектов, поэтому не используйте в имени конфиденциальную информацию. Она будет отображаться в ссылке на объект, если вы предоставите публичный доступ к нему.

{cut(Примеры имен бакетов, соответствующих рекомендуемым правилам)}

- `docexamplebucket1`
- `log-delivery-march-2020`
- `my-hosted-content`

{/cut}

{cut(Примеры недопустимых имен бакетов)}

- `doc_example_bucket` (содержит символы подчеркивания)
- `DocExampleBucket` (содержит прописные буквы)
- `doc-example-bucket-` (заканчивается дефисом)

{/cut}

## {heading(Объект)[id=s3-concepts-about-object]}

*Объект* — это файл, загруженный в хранилище, и его {linkto(#s3-concepts-about-metadata)[text=метаданные]}. Файл может быть любого формата, например: медиафайл, резервная копия, массив данных BigData.

Объекты в {var(s3)} хранятся в {linkto(#s3-concepts-about-bucket)[text=бакетах]}. В зависимости от размера файла используется {linkto(../../concepts/features#s3-concepts-features-object-uploading)[text=стандартная или составная загрузка]} объекта в бакет.

При загрузке объекта ему назначаются:

- {linkto(#s3-concepts-about-object-key)[text=ключ объекта]} — специальный идентификатор, по которому осуществляется доступ к объекту;
- {linkto(#s3-concepts-about-metadata)[text=метаданные]}, описывающие свойства объекта;
- {linkto(../../instructions/objects/manage-object#s3-instructions-manage-object-access)[text=уровень доступа]};
- {linkto(#s3-concepts-about-storage-class)[text=класс хранения]}.

Наряду с операциями загрузки, скачивания, копирования и удаления объектов в {var(s3)} поддерживаются:

- {linkto(../../instructions/objects/manage-object#s3-instructions-manage-object-access)[text=изменение уровня доступа]} для объекта;
- предоставление доступа к объекту по {linkto(../../instructions/access-management/signed-url#s3-instructions-signed-url)[text=подписанной ссылке]};
- временная и постоянная {linkto(../../concepts/objects-lock#s3-concepts-object-lock)[text=блокировка]} объекта от удаления или перезаписи.

## {heading(Ключ объекта)[id=s3-concepts-about-object-key]}

*Ключ объекта* — это комбинация имени объекта и префиксов к имени. Префиксы отделяются друг от друга и от имени объекта разделителем `/`. Префиксы необязательны, но позволяют организовывать структуру хранения объектов для более удобной их идентификации.

Модель данных {var(s3)} представляет собой плоскую структуру: вы создаете бакет, в котором хранятся объекты. Нет иерархии вложенных бакетов. Однако если в ключе указаны префиксы к имени объекта, {var(s3)} будет использовать их как имена директорий в бакете. Например, для объекта с ключом `photos/myphoto1.jpg` префиксом будет `photos/`. Если будет загружен новый объект с ключом `photos/myphoto2.jpg`, то они будут объединены общим префиксом `photos/`. Для удобства пользователя в {linkto(../../connect/s3-file-managers#s3-connect-file-managers)[text=файловых менеджерах]} и личном кабинете {ifdef(public)}{var(cloud)}{/ifdef}{ifdef(s3,s3-pdf)}IAM Only{/ifdef} объекты `photos/myphoto1.jpg` и `photos/myphoto2.jpg` будут отображаться как файлы `myphoto1.jpg` и `myphoto2.jpg` в директории `photos/`.

При выборе имени объекта и префиксов рекомендуется следовать приведенным ниже правилам.

### {heading(Рекомендации по созданию ключа объекта)[id=s3-concepts-about-object-key-rules]}

Ключ объекта состоит из символов Юникод. Длина ключа в кодировке UTF-8 — не более 1024 байт.

В ключе объекта вы можете использовать любой символ Юникод. Однако некоторые символы могут вызвать проблемы с рядом приложений и веб-протоколов. Чтобы избежать проблем, следуйте рекомендациям:  

- В ключе объекта рекомендуется использовать только латинские буквы, цифры и спецсимволы `/`, `!`, `-`, `_`, `.`, `*`, `'`, `(`, `)`. Символ `/` отделяет друг от друга префиксы и имя объекта.
- Ключ объекта не должен:

  - заканчиваться косой чертой `/`;
  - содержать две точки подряд: `..`;
  - состоять только из одной точки: `.`;
  - состоять только из пробела или содержать ведущие или завершающие пробелы.

- В ключе объекта не рекомендуется использовать приведенные ниже символы. Некоторые из них нуждаются в кодировании при их использовании в URL-адресах, а некоторые являются непечатаемыми и могут не обрабатываться вашим браузером. Чтобы обеспечить корректную адресацию объекта с такими символами в ключе, требуется дополнительная обработка ключа объекта в коде.

  Нежелательные символы:

  - спецсимволы `$`, `&`, `@`, `=`, `;`, `:`, `+`, `\`, `%`, `,`, `?`, `#`;
  - символы с ASCII-кодами `00h`–`1Fh` (десятичные коды `0`–`31`);
  - символ `DEL` с ASCII-кодом `7Fh` (десятичный код `127`);
  - пробел, так как в некоторых случаях могут быть потеряны значимые последовательности пробелов, особенно при наличии нескольких пробелов;
  - все невидимые символы Юникод.

{cut(Примеры ключей объектов, соответствующих рекомендациям)}

- `Development/Projects.xls`
- `Finance/statement1.pdf`
- `Finance/taxdocument.pdf`
- `s3-dg.pdf`

На основе префиксов `Development/` и `Finance/` {var(s3)} создаст директории в бакете. Открыв директорию `Development`, вы увидите в ней объект `Projects.xlsx`, в директории `Finance` найдете объекты `statement1.pdf` и `taxdocument.pdf`. Имя `s3-dg.pdf` не имеет префикса, поэтому этот объект будет размещаться на корневом уровне бакета.

{/cut}

## {heading(Метаданные объекта)[id=s3-concepts-about-metadata]}

У каждого объекта {var(s3)} есть *метаданные*. Это пары ключ/значение, описывающие свойства объекта. Различают метаданные, присваиваемые сервисом (например, класс хранения или дата создания), и пользовательские метаданные. Некоторые сервисные метаданные доступны пользователю для управления, в частности класс хранения объекта.

Вы можете установить метаданные объекта во время его загрузки. После загрузки напрямую изменить метаданные объекта нельзя. Единственный способ сделать это — {linkto(../../instructions/objects/manage-object#s3-instructions-manage-object-copy)[text=скопировать]} объект. При копировании нужно указать ключ объекта одновременно в качестве источника и цели и задать для копии новые метаданные. Таким способом вы можете изменить:

- пользовательские метаданные,
- системные метаданные, управляемые пользователем, например класс хранения.

Метаданные, управляемые сервисом, при этом будут сброшены. Например, будет сброшена дата создания объекта.

## {heading(Классы хранения)[id=s3-concepts-about-storage-class]}

Классы хранения отличаются назначением {ifdef(public)}и [стоимостью хранения](https://cloud.vk.com/pricelist) {/ifdef}данных и доступны для бакетов, объектов и составных загрузок. В {var(cloud)} поддерживается несколько классов хранения:

- `Hotbox` — для хранения и быстрой раздачи большого количества файлов для высоконагруженных сайтов, приложений, медиасервисов, онлайн-СМИ, сайтов с многопользовательским контентом и мобильных приложений. В API обозначается названием `STANDARD`.
- `Icebox` — для редко используемых данных: {ifdef(public)}бэкапов, {/ifdef}логов, медиаконтента, научных, статистических данных, а также рабочих архивов. В API обозначается названием `STANDARD_IA`.

{ifdef(public)}
- `BackupBucket` (только для бакетов) — для {linkto(../../../../storage/backups#cloud-backup)[text=резервных копий инстансов]}, созданных как автоматически, так и вручную. Бакеты такого класса нельзя самостоятельно создать или удалить. После удаления резервных копий бакет BackupBucket может остаться пустым. Он необходим объектному хранилищу для дальнейшей работы с планами резервного копирования в проекте. В API обозначается названием `BACKUP`.
{/ifdef}
