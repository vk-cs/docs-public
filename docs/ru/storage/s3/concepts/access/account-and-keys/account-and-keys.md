В сервисе VK Object Storage для доступа к бакетам и объектам используются два независимых друг от друга вида ключей:

- [Ключи доступа, привязанные к аккаунту VK Object Storage](#account_access_keys). Дают доступ ко всем бакетам аккаунта.
- [Ключи доступа, привязанные к бакету](#bucket_access_keys). Дают доступ к одному бакету или части объектов в этом бакете.

## {heading(Аккаунт VK Object Storage и его ключи доступа)[id=account_access_keys]}

В личном кабинете VK Cloud для работы с сервисом VK Object Storage ключи доступа не используются. Но если вы хотите работать с бакетами и объектами через [AWS CLI](/ru/tools-for-using-services/cli/aws-cli), [API](/ru/tools-for-using-services/api/api-spec/s3-rest-api) или [файловые менеджеры](../../../connect/s3-file-managers), в проекте нужно [создать](../../../instructions/access-management/access-keys#create_account_and_access_key) аккаунт VK Object Storage и ключи доступа, которые будут к нему привязаны.

Вы можете создать несколько аккаунтов VK Object Storage. Их количество в одном проекте ограничено [квотами](/ru/tools-for-using-services/account/concepts/quotasandlimits#object_storage_limits). Одновременно с аккаунтом автоматически создаются ключи доступа к нему. Ключи доступа к аккаунту — это пара из публичного и секретного ключей. Секретный ключ доступен только при создании пары, восстановить его позднее невозможно, но вы можете [создать](../../../instructions/access-management/access-keys#add_new_access_key) новые ключи доступа. К одному аккаунту можно привязать до двух пар ключей.

Управление аккаунтами VK Object Storage и привязанными к ним ключами доступно только в личном кабинете VK Cloud.

## {heading(Ключи доступа к бакету)[id=bucket_access_keys]}

Ключи доступа к бакету привязываются к конкретному бакету, а не к аккаунту. Они наделяют пользователя правом работать с объектами только в этом бакете или в одной его директории, но не дают доступа к другим бакетам и объектам в проекте.

Ключи доступа к бакету — это пара из публичного и секретного ключей. Секретный ключ доступен только при создании пары, восстановить его позднее невозможно, но вы можете [создать](../../../instructions/access-management/bucket-keys#add-keys) новые ключи доступа.

При добавлении ключей доступа в бакет в качестве параметра можно указать [префикс к имени объекта](../../about#object_key), который VK Object Storage интерпретирует как имя директории в бакете. Такие ключи называются префиксными (prefix access keys). Префиксные ключи дают доступ только к объектам в директории бакета, заданной префиксом. Ключи, созданные без указания префикса, дают доступ ко всем объектам в бакете.

Как и ключи, привязанные к аккаунту, ключи доступа к бакету дают возможность работать с объектами в нем через AWS CLI, API или файловые менеджеры. Управлять ключами доступа к бакету можно [через личный кабинет](../../../instructions/access-management/bucket-keys) или [с помощью API](/ru/tools-for-using-services/api/api-spec/s3-rest-api/pak-api).
