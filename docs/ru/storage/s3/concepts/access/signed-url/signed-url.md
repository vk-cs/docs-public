По умолчанию все объекты и бакеты в проекте являются приватными. Однако вы можете разрешить другим пользователям скачивать из бакета или загружать в него объекты, создав предварительно подписанный URL-адрес. В нем указываются:

- Публичный ключ доступа. Например, публичный [ключ доступа, привязанный к аккаунту VK Object Storage](../account-and-keys#account_access_keys), или публичный [ключ доступа бакета](../account-and-keys#bucket_access_keys).
- Имя бакета.
- Ключ объекта.
- Срок действия URL-адреса.
- Подпись, зашифрованная секретным ключом доступа.

{note:info}
По умолчанию предварительно подписанный URL-адрес позволяет скачивать указанный в нем объект. Вы также можете создавать URL-адреса для загрузки объектов в бакет. Подробнее — в [официальной документации Amazon AWS](https://docs.amazonaws.cn/en_us/AmazonS3/latest/userguide/PresignedUrlUploadObject.html).
{/note}

Любой, у кого есть подписанный URL-адрес, получает доступ к указанному в нем объекту. Авторизации в VK Cloud для этого не требуется. Подписанный URL-адрес можно использовать несколько раз, но только в течение указанного в нем срока действия.

Возможности подписанного URL-адреса ограничены правами того, кто его подписал. Например, URL-адрес перестанет работать, если удален аккаунт Object Cloud, ключом которого подписан этот URL-адрес.

{cut(Общий вид подписанного URL)}

```console
https://hb.ru-msk.vkcloud-storage.ru/<ИМЯ_БАКЕТА>/<КЛЮЧ_ОБЪЕКТА>?
     X-Amz-Algorithm=<АЛГОРИТМ>
    &X-Amz-Expires=<ВРЕМЯ_ДЕЙСТВИЯ>
    &X-Amz-SignedHeaders=<СПИСОК_ЗАГОЛОВКОВ>
    &X-Amz-Date=<ДАТА_И_ВРЕМЯ>
    &X-Amz-Credential=<КЛЮЧ_ДОСТУПА>/<YYYYMMDD>/ru-msk/s3/aws4_request
    &X-Amz-Signature=<ПОДПИСЬ>
```

Здесь:

- `<КЛЮЧ_ОБЪЕКТА>` — ключ объекта, к которому предоставляется доступ.
- `<ИМЯ_БАКЕТА>` — имя бакета, в котором находится объект.
- `<АЛГОРИТМ>` — версия подписи и алгоритм ее вычисления, например `AWS4-HMAC-SHA256`.
- `<ВРЕМЯ_ДЕЙСТВИЯ>` — время действия URL-адреса в секундах. Отсчитывается от момента, указанного в заголовке `X-Amz-Date`. Максимальное значение — 604800 секунд (7 дней).
- `<СПИСОК_ЗАГОЛОВКОВ>` — список подписываемых заголовков запроса. Заголовки разделяются символом `;`. Обязательно должны быть подписаны заголовок `Host` и все заголовки `X-Amz-*`, которые используются в запросе.
- `<ДАТА_И_ВРЕМЯ>` — дата и время истечения срока действия URL-адреса в формате ISO8601, например `20180719T000000Z`. Дата должна совпадать по значению с датой в заголовке `X-Amz-Credential`.
- `<КЛЮЧ_ДОСТУПА>` — Access Key ID, публичный ключ доступа.
- `<YYYYMMDD>` — дата в формате `YYYYMMDD` для подстановки в заголовок `X-Amz-Credential`. Должна совпадать по значению с датой, установленной в заголовке `X-Amz-Date`.  
- `<ПОДПИСЬ>` — подпись запроса. Запрос подписывается с помощью секретного ключа доступа.

{/cut}

[Составить](../../../instructions/access-management/signed-url) подписанный URL-адрес можно с помощью AWS CLI или SDK.
