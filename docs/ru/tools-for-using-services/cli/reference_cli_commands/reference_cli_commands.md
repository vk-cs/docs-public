# {heading(Справочник Openstack CLI)[id=reference_cli_commands]}

Для управления {var(cloud)} с помощью OpenStack CLI выполните подготовительные операции из раздела {linkto(../openstack-cli#tools-cli-openstack)[text=%text]}, и выполните команды, приведенные ниже.

## {heading(Встроенная справка по командам)[id=cli_help]}

```console
openstack help <КОМАНДА>
```

Здесь `<КОМАНДА>` — команда, по которой необходимо получить справку.

## {heading(Управление шаблонами (типами) ВМ)[id=cli_flavor]}

### {heading(Получение списка шаблонов ВМ)[id=cli_openstack_flavor_list]}

```console
openstack flavor list
      [--public | --private | --all]
      [--long]
```

Здесь:

- `--public` — показать только публичные шаблоны ВМ.
- `--private` — показать только приватные шаблоны ВМ.
- `--all` — показать все шаблоны ВМ: публичные и приватные (по умолчанию).
- `--long` — выводить дополнительные поля.

### {heading(Просмотр информации о шаблоне ВМ)[id=cli_flavor_info]}

```console
openstack flavor show <ШАБЛОН_ВМ>
```

Здесь `<ШАБЛОН_ВМ>` — название или идентификатор шаблона ВМ.

## {heading(Управление ВМ)[id=cli_vm]}

### {heading(Просмотр списка ВМ)[id=cli_openstack_server_list]}

```console
openstack server list
      [--reservation-id <РЕЗЕРВИРОВАНИЕ>]
      [--ip <IP-АДРЕС>]
      [--name <ИМЯ>]
      [--instance-name <ИНСТАНС>]
      [--status <СТАТУС>]
      [--flavor <ШАБЛОН_ВМ>]
      [--image <ОБРАЗ>]
      [--host <УЗЕЛ>]
      [--all-projects]
      [--project <ПРОЕКТ>]
      [--project-domain <ДОМЕН_ПРОЕКТА>]
      [--user <ПОЛЬЗОВАТЕЛЬ>]
      [--user-domain <ДОМЕН_ПОЛЬЗОВАТЕЛЯ>]
      [--long]
      [--marker <МАРКЕР>]
      [--limit <ЛИМИТ>]
      [--deleted]
      [--changes-since <ВРЕМЯ>]
```

Здесь:

- `--reservation-id <РЕЗЕРВИРОВАНИЕ>` — отображает экземпляры ВМ, соответствующие резервированию.
- `--ip <IP-АДРЕС>` — регулярное выражение для отбора ВМ по соответствующим адресам.
- `--name <ИМЯ>` — регулярное выражение для отбора ВМ по имени.
- `--instance-name <ИНСТАНС>` — регулярное выражение для отбора ВМ по имени инстанса.
- `--status <СТАТУС>` — фильтровать ВМ по указанному статусу.
- `--flavor <ШАБЛОН_ВМ>` — фильтровать ВМ по указанному шаблону ВМ (по названию или идентификатору).
- `--image <ОБРАЗ>` — фильтровать ВМ по указанному образу (по названию или идентификатору).
- `--host <УЗЕЛ>` — фильтровать ВМ по гипервизору размещения.
- `--all-projects` — включать в выборку все проекты.
- `--project <ПРОЕКТ>` — фильтровать ВМ в указанном проекте (по названию или идентификатору).
- `--project-domain <ДОМЕН_ПРОЕКТА>` — фильтровать ВМ по домену проекта по названию или идентификатору. Опция используется в случае конфликтов между названиями проектов.
- `--user <ПОЛЬЗОВАТЕЛЬ>` — фильтровать ВМ указанного пользователя (по имени или идентификатору).
- `--user-domain <ДОМЕН_ПОЛЬЗОВАТЕЛЯ>` — фильтровать ВМ по домену пользователя (по имени или идентификатору). Опция используется в случае конфликтов между именами пользователей.
- `--long` — выводить дополнительные поля.
- `--marker <МАРКЕР>` — последняя ВМ предыдущей страницы. Выводит весь список ВМ после `<МАРКЕР>`, если не указано иное. Если используется с опцией `--deleted`, маркер `<МАРКЕР>` должен быть идентификатором (ID), иначе допускается использование названия ВМ или ID.
- `--limit <ЛИМИТ>` — максимальное количество ВМ в выводимом списке. Если указано значение `-1` — выводятся все ВМ. Если указанное значение `<ЛИМИТ>` превышает значение конфигурационного параметра `osapi_max_limit` — выводится `osapi_max_limit` ВМ.
- `--deleted` — выводить только удаленные ВМ.
- `--changes-since <ВРЕМЯ>` — выводит список ВМ, измененных после указанного момента времени. Указываемое время должно быть в формате ISO 8061. Пример: `2016-03-05T06: 27: 59Z`.

### {heading(Добавление ВМ)[id=cli_openstack_server_create]}

```console
openstack server create
    (--image <ОБРАЗ> | --volume <ДИСК>)
    --flavor <ШАБЛОН_ВМ>
    [--security-group <ГРУППА_БЕЗОПАСНОСТИ>]
    [--key-name <КЛЮЧЕВАЯ_ПАРА>]
    [--property <КЛЮЧ=ЗНАЧЕНИЕ>]
    [--file <ФАЙЛ_НАЗНАЧЕНИЯ=ИСХОДНЫЙ_ФАЙЛ>]
    [--user-data <ДАННЫЕ>]
    [--availability-zone <ЗОНА_ДОСТУПНОСТИ>]
    [--block-device-mapping <УСТРОЙСТВО=МАППИНГ>]
    [--nic <net-id=UUID_СЕТИ,v4-fixed-ip=IP_АДРЕС,v6-fixed-ip=IP_АДРЕС,port-id=UUID_ПОРТА,auto,none>]
    [--hint <КЛЮЧ=ЗНАЧЕНИЕ>]
    [--config-drive <КОНФИГУРАЦИОННЫЙ_ДИСК>]
    [--min <КОЛИЧЕСТВО>]
    [--max <КОЛИЧЕСТВО>]
    [--wait]
    <ВМ>
```

Здесь:

- `--image <ОБРАЗ>` — создание ВМ с использованием существующего образа (название или идентификатор).
- `--volume <ДИСК>` — создание ВМ с использованием указанного диска (название или идентификатор). Флаг автоматически создает отображение блочного устройства с индексом загрузки `0`.
- `--flavor <ШАБЛОН_ВМ>` — создание ВМ с использованием указанного шаблона конфигурации ВМ (название или идентификатор). Подробная информация приведена в разделе {linkto(#cli_openstack_flavor_list)[text=%text]}.
- `--security-group <ГРУППА_БЕЗОПАСНОСТИ>` — создание ВМ с назначением группы безопасности. Может использоваться несколько раз для назначения нескольких групп безопасности.
- `--key-name <КЛЮЧЕВАЯ_ПАРА>` — используемая ключевая пара. Необязательный флаг.
- `--property <КЛЮЧ=ЗНАЧЕНИЕ>` — создание ВМ с дополнительными свойствами. Может использоваться несколько раз для установки нескольких свойств.
- `--file <ФАЙЛ_НАЗНАЧЕНИЯ=ИСХОДНЫЙ_ФАЙЛ>` — имя файла, который добавляется в образ ОС. Пример: SSH-ключи.
- `--user-data <ДАННЫЕ>` — пользовательские данные для сервера метаданных.
- `--availability-zone <ЗОНА_ДОСТУПНОСТИ>` — создание ВМ в заданной зоне доступности.
- `--block-device-mapping <УСТРОЙСТВО=МАППИНГ>` — маппинг блочных устройств.
- `--nic <net-id=UUID_СЕТИ,v4-fixed-ip=IP_АДРЕС,v6-fixed-ip=IP_АДРЕС,port-id=UUID_ПОРТА,auto,none>` — создать сетевой адаптер. Можно использовать несколько раз для создания нескольких адаптеров. Необходимо указывать идентификатор сети или идентификатор порта, но не оба сразу:

  - `net-id` — подключить сетевой адаптер к сети с UUID `<UUID_СЕТИ>`.
  - `v4-fixed-ip` — фиксированный IPv4-адрес сетевого адаптера. Необязательный флаг.

   <!--- // ** `v6-fixed-ip` — фиксированный IPv6-адрес сетевого адаптера. Необязательный флаг. -->

- `port-id` — подключить сетевой адаптер к порту с UUID `<UUID_ПОРТА>`.
- `none` — сеть не подключена.
- `auto` — автоматическое выделение сети.

  - `--hint <КЛЮЧ=ЗНАЧЕНИЕ>` — дополнительные параметры для планировщика (scheduler).
  - `--config-drive <КОНФИГУРАЦИОННЫЙ_ДИСК>` — использовать указанный диск в качестве конфигурационного. Установите значение `True` для использования эфемерного диска.
  - `--min <КОЛИЧЕСТВО>` — минимальное количество ВМ для запуска. Значение по умолчанию — `1`.
  - `--max <КОЛИЧЕСТВО>` — максимальное количество ВМ для запуска. Значение по умолчанию — `1`.
  - `--wait` — дождаться завершения создания ВМ.
  - `<ВМ>` — имя ВМ.

### {heading(Изменение параметров ВМ)[id=cli_openstack_server_set]}

```console
openstack server set
      [--name <НАЗВАНИЕ>]
      [--root-password <ПАРОЛЬ>]
      [--property <КЛЮЧ=ЗНАЧЕНИЕ>]
      [--state <СТАТУС>]
      <ВМ>
```

Здесь:

- `--name <НАЗВАНИЕ>` — новое название ВМ.
- `--root-password` — установить пароль для доступа к ВМ пользователю по умолчанию. Для каждого шаблона ВМ может быть свое имя пользователя. Пример: для встроенного образа с ОС CentOS 7 имя пользователя по умолчанию — `centos`.
- `--property <КЛЮЧ=ЗНАЧЕНИЕ>` — дополнительные свойства ВМ. Может использоваться несколько раз для установки нескольких свойств.
- `--state <СТАТУС>` — установить статус для ВМ.
- `<ВМ>` — название или идентификатор ВМ.

### {heading(Изменение размера ВМ)[id=cli_openstack_server_resize]}

```console
openstack server resize
    [--flavor <ШАБЛОН_ВМ> | --confirm | --revert]
    [--wait]
    <ВМ>
```

Здесь:

- `--flavor <ШАБЛОН_ВМ>` — название или идентификатор шаблона конфигурации ВМ.
- `--confirm` — подтвердить изменение размера ВМ.
- `--revert` — восстановить состояние ВМ до изменения размера.
- `--wait` — дождаться завершения изменения размера ВМ.
- `<ВМ>` — название или идентификатор ВМ.

### {heading(Миграция ВМ)[id=cli_openstack_server_migrate]}

```console
openstack server migrate
    [--live <УЗЕЛ>]
    [--shared-migration | --block-migration]
    [--disk-overcommit | --no-disk-overcommit]
    [--wait]
    <ВМ>
```

Здесь:

- `--live` — выполнить live-миграцию ВМ на вычислительный узел `<УЗЕЛ>`.
- `--shared-migration` — выполнить общую live-миграцию (по умолчанию).
- `--block-migration` — выполнить блочную live-миграцию.
- `--disk-overcommit` — разрешить избыточную фиксацию диска на целевом узле.
- `--no-disk-overcommit` — не выполнять избыточную фиксацию диска на целевом узле (по умолчанию).
- `--wait` — дождаться завершения миграции.
- `<ВМ>` — название или идентификатор мигрируемой ВМ.

- `--volume <ДИСК>` — название или идентификатор диска, из которого создать образ.

- `--unprotected` — защитить образ от удаления.

- `--private` — фильтр по приватным дискам.

### {heading(Создание диска)[id=cli_openstack_volume_create]}

```console
openstack volume create
      [--size <РАЗМЕР_ДИСКА>]
      [--type <ТИП_ДИСКА>]
      [--image <ОБРАЗ> | --snapshot <СНИМОК> | --source <ДИСК> | --source-replicated <РЕПЛИЦИРУЕМЫЙ_ДИСК>]
      [--description <ОПИСАНИЕ>]
      [--project <ПРОЕКТ>]
      [--availability-zone <ЗОНА_ДОСТУПНОСТИ>]
      [--consistency-group <ГРУППА_КОНСИСТЕНТНОСТИ>]
      [--property <КЛЮЧ=ЗНАЧЕНИЕ>]
      [--hint <КЛЮЧ=ЗНАЧЕНИЕ>]
      [--bootable | --non-bootable]
      [--read-only | --read-write]
      <НАЗВАНИЕ>
```

Здесь:

- `--size <РАЗМЕР_ДИСКА>` — размер диска в ГБ.
- `--type <ТИП_ДИСКА>` — название или идентификатор типа диска.
- `--image <ОБРАЗ>` — название или идентификатор образа для создания диска.
- `--snapshot <СНИМОК>` — название или идентификатор снимка для создания диска.
- `--source <ДИСК>` — диск, который необходимо использовать в качестве источника (клонирование).
- `--source-replicated <РЕПЛИЦИРУЕМЫЙ_ДИСК>` — название или идентификатор реплицируемого диска для клонирования.
- `--description <ОПИСАНИЕ>` — описание диска.
- `--project <ПРОЕКТ>` — название или идентификатор проекта, за которым необходимо закрепить диск.
- `--availability-zone <ЗОНА_ДОСТУПНОСТИ>` — название или идентификатор зоны доступности, в которой необходимо создать диск.
- `--consistency-group <ГРУППА_КОНСИСТЕНТНОСТИ>` — группа консистентности для диска.
- `--property <КЛЮЧ=ЗНАЧЕНИЕ>` — дополнительные свойства диска. Может использоваться несколько раз для установки нескольких свойств.
- `--hint <КЛЮЧ=ЗНАЧЕНИЕ>` — дополнительные параметры для планировщика (scheduler).
- `--bootable` — ключ, который используется, чтобы пометить диск как загрузочный. Из загрузочных дисков можно создавать ВМ.
- `--non-bootable` — ключ, который используется, чтобы пометить диск как незагрузочный (используется по умолчанию).
- `--read-only` — ключ, который используется, чтобы установить режим «только для чтения».
- `--read-write` — ключ, который используется, чтобы установить режим «чтение-запись» (используется по умолчанию).
- `<НАЗВАНИЕ>` — название диска.

### {heading(Изменение параметров диска)[id=cli_openstack_volume_set]}

```console
openstack volume set
      [--name <НАЗВАНИЕ>]
      [--size <РАЗМЕР_ДИСКА>]
      [--description <ОПИСАНИЕ>]
      [--type <ТИП_ДИСКА>]
      [--retype-policy <ПОЛИТИКА_МИГРАЦИИ>]
      [--bootable | --non-bootable]
      [--read-only | --read-write]
      <ДИСК>
```

<!--- // специально убран параметр --state, т.к. он доступен только админам -->

Здесь:

- `--name <НАЗВАНИЕ>` — новое название диска.
- `--retype-policy <ПОЛИТИКА_МИГРАЦИИ>` — политика миграции диска при его переписывании. Возможные значения: `never` или `on-demand`. Значение по умолчанию `never`. Необходимо указывать вместе с флагом `--type`.

Описание остальных параметров приведено в разделе {linkto(#cli_openstack_volume_create)[text=%text]}.

## {heading(Управление сетями)[id=cli_network]}

### {heading(Просмотр списка сетей)[id=cli_openstack_network_list]}

```console
openstack network list
      [--external | --internal]
      [--long]
      [--name <НАЗВАНИЕ>]
      [--enable | --disable]
      [--project <ПРОЕКТ>]
      [--project-domain <ДОМЕН_ПРОЕКТА>]
      [--share | --no-share]
      [--status <СТАТУС>]
      [--provider-network-type <ТИП_СЕТИ>]
      [--provider-physical-network <ФИЗИЧЕСКАЯ_СЕТЬ>]
      [--provider-segment <СЕГМЕНТ_СЕТИ>]
```

Здесь:

* `--external` — отобразить только внешние сети.
* `--internal` — отобразить только внутренние сети.
* `--long` — вывести дополнительные свойства.
* `--name <НАЗВАНИЕ>` — отобразить сеть с названием `<НАЗВАНИЕ>`.
* `--enable` — отобразить только сети со статусом `enabled`.
* `--disable` — отобразить только сети со статусом `enabled`.
* `--project <ПРОЕКТ>` — фильтровать сети в указанном проекте (по названию или идентификатору).
* `--project-domain <ДОМЕН_ПРОЕКТА>` — фильтровать сети по домену проекта по названию или идентификатору. Опция используется в случае конфликтов между названиями проектов.
* `--share` — показать общие сети для нескольких проектов.
* `--no-share` — показать общие сети только для одного проекта.
* `--status <СТАТУС>` — отобразить сети в определенных статусах. Возможные значения:

   * `ACTIVE` — активна.
   * `BUILD` — в процессе создания.
   * `DOWN` — отключена.
   * `ERROR` — ошибка работы.

* `--provider-network-type <ТИП_СЕТИ>` — отобразить сети определенного типа. Возможные значения: `flat`, `geneve`, `gre`, `local`, `vlan`, `vxlan`.
* `--provider-physical-network <ФИЗИЧЕСКАЯ_СЕТЬ>` — отобразить сеть с именем физической сети `<ФИЗИЧЕСКАЯ_СЕТЬ>`.
* `--provider-segment <СЕГМЕНТ_СЕТИ>` — отобразить список сетей в соответствии с идентификатором VLAN для сетей VLAN или идентификатором Tunnel ID для GENEVE/GRE/VXLAN сетей.

### {heading(Просмотр информации о сети)[id=cli_network_info]}

```console
openstack network show <СЕТЬ>
```
Здесь `<СЕТЬ>` — название или идентификатор сети.

### {heading(Создание сети)[id=cli_openstack_network_create]}

```console
openstack network create
      [--share | --no-share]
      [--enable | --disable]
      [--project <ПРОЕКТ>]
      [--project-domain <ДОМЕН_ПРОЕКТА>]
      [--description <ОПИСАНИЕ>]
      [--availability-zone-hint <ЗОНА_ДОСТУПНОСТИ>]
      [--external | --internal]
      [--default | --no-default]
      [--qos-policy <ПРАВИЛО_QOS>]
      [--provider-network-type <ТИП_СЕТИ>]
      [--provider-physical-network <ФИЗИЧЕСКАЯ_СЕТЬ>]
      <СЕТЬ>
```

Здесь:

- `--share` — установить сеть как публичную (доступной на других проектах).
- `--no-share` — установить сеть как приватную (по умолчанию).
- `--enable` — признак доступности сети (по умолчанию).
- `--disable` — признак недоступности сети.
- `--project <ПРОЕКТ>` — название или идентификатор проекта, в которую добавить сеть.
- `--project-domain <ДОМЕН_ПРОЕКТА>` — присвоить сеть домену проекта по названию или идентификатору. Флаг используется в случае конфликтов между названиями проектов.
- `--description <ОПИСАНИЕ>` — описание сети.
- `--availability-zone-hint <ЗОНА_ДОСТУПНОСТИ>` — зона доступности для сети (требуется расширение `Network Availability Zone`).
- `--external` — установить сеть как внешнюю.
- `--internal` — установить сеть как внутреннюю (по умолчанию).
- `--default` — использовать в виде внешней сети по умолчанию.
- `--no-default` — не использовать в виде внешней сети по умолчанию (по умолчанию).
- `--qos-policy <ПРАВИЛО_QOS>` — название или идентификатор правила QoS для сети.
- `--provider-network-type <ТИП_СЕТИ>` — физический механизм, с помощью которого реализуется виртуальная сеть. Возможные значения: `flat`, `geneve`, `gre`, `local`, `vlan`, `vxlan`.
- `--provider-physical-network <ФИЗИЧЕСКАЯ_СЕТЬ>` — название физической сети (только для внешней сети).
- `<СЕТЬ>` — имя сети.


### {heading(Создание порта)[id=cli_openstack_port_create]}

```console
openstack port create
      --network <СЕТЬ>
      [--description <ОПИСАНИЕ>]
      [--device <ID_УСТРОЙСТВА>]
      [--device-owner <ВЛАДЕЛЕЦ_ПОРТА>]
      [--vnic-type <ТИП_VNIC>]
      [--host <ID_УЗЛА>]
      [--dns-name <DNS_ИМЯ>]
      [--fixed-ip subnet=<ПОДСЕТЬ>,ip-address=<IP-АДРЕС>]
      [--enable | --disable]
      [--mac-address <MAC_АДРЕС>]
      [--project <ПРОЕКТ>]
      [--project-domain <ДОМЕН_ПРОЕКТА>]
      [--security-group <ГРУППА_БЕЗОПАСНОСТИ> | --no-security-group]
      [--enable-port-security | --disable-port-security]
      [--allowed-address ip-address=<IP-АДРЕС>[,mac-address=<MAC_АДРЕС>]]
      <ПОРТ>
```

Здесь:

- `--network <СЕТЬ>` — название или идентификатор сети, для которой создается порт.
- `--description <ОПИСАНИЕ>` — описание для порта.
- `--device <ID_УСТРОЙСТВА>` — идентификатор устройства порта.
- `--device-owner <ВЛАДЕЛЕЦ_ПОРТА>` — имя владельца порта.
- `--vnic-type <ТИП_VNIC>` — тип VNIC для порта. Возможные значения: `direct`, `direct-physical`, `macvtap`, `normal` (по умолчанию), `baremetal`.
- `--host <ID_УЗЛА>` — идентификатор узла, к которому будет привязан порт.
- `--dns-name <DNS_ИМЯ>` — назначить DNS-имя для порта.
- `--fixed-ip subnet=<ПОДСЕТЬ>,ip-address=<IP-АДРЕС>` — IP-адрес в указанной подсети (название или идентификатор) для порта.
- `--enable` — сделать порт активным (значение по умолчанию).
- `--disable` — сделать порт неактивным.
- `--mac-address <MAC_АДРЕС>` — MAC-адрес порта.
- `--project <ПРОЕКТ>` — закрепить порт за проектом по названию или идентификатору.
- `--project-domain <ДОМЕН_ПРОЕКТА>` — присвоить порт домену проекта по названию или идентификатору. Флаг используется в случае конфликтов между названиями проектов.
- `--security-group <ГРУППА_БЕЗОПАСНОСТИ>` — группа безопасности для порта. Опция `--no-security-group` не добавляет группы безопасности порту (по умолчанию).
- `--enable-port-security` — включить защиту для порта (по умолчанию).
- `--disable-port-security` — отключить защиту для порта.
- `--allowed-address ip-address=<IP-АДРЕС>[,mac-address=<MAC_АДРЕС>]` — пара разрешенных адресов, связанных с портом.
- `<ПОРТ>` — название порта.

### {heading(Изменение порта)[id=cli_openstack_port_set]}

```console
openstack port set
      [--description <ОПИСАНИЕ>]
      [--device <ID_УСТРОЙСТВА>]
      [--device-owner <ВЛАДЕЛЕЦ_ПОРТА>]
      [--vnic-type <ТИП_VNIC>]
      [--host <ID_УЗЛА>]
      [--dns-name <DNS_ИМЯ>]
      [--enable | --disable]
      [--name <НАЗВАНИЕ>]
      [--fixed-ip subnet=<ПОДСЕТЬ>,ip-address=<IP-АДРЕС>]
      [--no-fixed-ip]
      [--security-group <ГРУППА_БЕЗОПАСНОСТИ>]
      [--no-security-group]
      [--enable-port-security]
      [--disable-port-security]
      [--allowed-address ip-address=<IP-АДРЕС>[,mac-address=<MAC_АДРЕС>]]
      [--no-allowed-address]
      <ПОРТ>
```

Здесь:

- `--name <НАЗВАНИЕ>` — новое название порта.
- `--no-fixed-ip` — убрать все IP-адреса для порта.
- `--no-allowed-address` — убрать все разрешенные адреса для порта.

Описание остальных параметров приведено в разделе {linkto(#cli_openstack_port_create)[text=%text]}.

### {heading(Удаление порта)[id=cli_openstack_port_delete]}

Здесь `<ПОРТ> [<ПОРТ_2> ...]` — название или идентификатор порта. Несколько портов необходимо указывать через пробел.

### {heading(Правила группы безопасности)[id=cli_sec_rule]}

#### {heading(Просмотр списка правил групп безопасности)[id=cli_sec_rule_list]}

```console

      [--protocol <ПРОТОКОЛ>]

      [--long]
      [<ГРУППА_БЕЗОПАСНОСТИ>]
```

Здесь:

- `--protocol <ПРОТОКОЛ>` — фильтрация правил по протоколу IP. Возможные значения: `icmp`, `tcp`, `udp`.
- `--ingress` — фильтрация по правилам входящего трафика.
- `--egress` — фильтрация по правилам исходящего трафика.
- `--long` — вывести дополнительные свойства.
- `<ГРУППА_БЕЗОПАСНОСТИ>` — фильтрация правил по названию или идентификатору группы безопасности.

#### {heading(Просмотр информации о правиле группы безопасности)[id=cli_sec_rule_info]}

```console
openstack security group rule show <ПРАВИЛО>
```

Здесь `<ПРАВИЛО>` — идентификатор правила группы безопасности.

#### {heading(Создание правил групп безопасности)[id=cli_sec_rule_create]}

```console
openstack security group rule create
      [--remote-ip <IP-АДРЕС> | --remote-group <ГРУППА_БЕЗОПАСНОСТИ>]
      [--description <ОПИСАНИЕ>]
      [--dst-port <ПОРТ>]
      [--icmp-type <ТИП_ICMP>]
      [--icmp-code <КОД_ICMP>]
      [--protocol <ПРОТОКОЛ>]
      [--ingress | --egress]
      [--ethertype <ТИП_ETHERNET_КАДРА>]
      [--project <ПРОЕКТ>]
      [--project-domain <ДОМЕН_ПРОЕКТА>]
      <ПРАВИЛО>
```

Здесь:

- `--remote-ip <IP-АДРЕС>` — блок адреса подключения в нотации CIDR, на который будет применена фильтрация. Значение по умолчанию — `0.0.0.0/0`.
- `--remote-group <ГРУППА_БЕЗОПАСНОСТИ>` — название или идентификатор удаленной группы безопасности.
- `--description <ОПИСАНИЕ>` — описание правила.
- `--dst-port <ПОРТ>`— порт назначения. Может быть одиночным портом или диапазоном портов (`137:139`). Требуется для протоколов TCP и UDP. Игнорируется для ICMP протокола.
- `--icmp-type <ТИП_ICMP>` — тип ICMP для протокола ICMP IP.
- `--icmp-code <КОД_ICMP>` — код ICMP для протокола ICMP IP.
- `--protocol <ПРОТОКОЛ>` — IP протокол. Возможные значения: `icmp`, `tcp`, `udp`. Значение по умолчанию `tcp`.
- `--ingress` — применить правило для входящего трафика (используется по умолчанию).
- `--egress` — применить правило для исходящего трафика.
- `--ethertype <ТИП_ETHERNET_КАДРА>` — тип Ethernet-кадра (двухбайтовое шестнадцатеричное число).
- `--project <ПРОЕКТ>` — название или идентификатор проекта, в котором создается правило.
- `--project-domain <ДОМЕН_ПРОЕКТА>` — присвоить правило домену проекта по названию или идентификатору. Флаг используется в случае конфликтов между названиями проектов.
- `<ПРАВИЛО>` — название правила группы безопасности.

#### {heading(Удаление правил группы безопасности)[id=cli_sec_rule_delete]}

```console

```
Здесь `<ПРАВИЛО>` — идентификатор правила группы безопасности.

- `--vip-port-id <VIP_ПОРТ>` — фильтр по названию или идентификатору порта.
- `--provisioning-status` — вывести балансировщики в определенном статусе подготовки. Возможные значения: `ACTIVE`, `DELETED`, `ERROR`, `PENDING_CREATE`, `PENDING_UPDATE`, `PENDING_DELETE`.
- `--operating-status` — вывести балансировщики в определенном статусе работы. Возможные значения: `ONLINE`, `DRAINING`, `OFFLINE`, `DEGRADED`, `ERROR`, `NO_MONITOR`.
- `--provider <ПРОВАЙДЕР>` — фильтр по провайдеру.
- `--availability-zone <ЗОНА_ДОСТУПНОСТИ>` — фильтр по зоне доступности.

```console
openstack loadbalancer listener create
      [--name <НАЗВАНИЕ>]
      [--description <ОПИСАНИЕ>]
      --protocol <ПРОТОКОЛ>
      [--connection-limit <ЛИМИТ>]
      [--insert-headers <ЗАГОЛОВОК=ЗНАЧЕНИЕ> [, <ЗАГОЛОВОК_2=ЗНАЧЕНИЕ_2>, ...]]
      --protocol-port <ПОРТ>
      [--timeout-client-data <ВРЕМЯ_ОЖИДАНИЯ>]
```

Здесь:

- `--description <ОПИСАНИЕ>` — описание правила балансировки.
- `--protocol <ПРОТОКОЛ>` — протокол балансировки. Возможные значения: `TCP`, `HTTP`, `HTTPS`.
- `--connection-limit <ЛИМИТ>` — максимальное количество подключений, разрешенных для правила балансировки.
- `--insert-headers <ЗАГОЛОВОК=ЗНАЧЕНИЕ> [, <ЗАГОЛОВОК_2=ЗНАЧЕНИЕ_2>, …]` — заголовки, которые будут включены в запрос к серверу.
- `--protocol-port <ПОРТ>` — порт протокола балансировки.
- `--timeout-client-data <ВРЕМЯ_ОЖИДАНИЯ>` — тайм-аут бездействия фронтенда в миллисекундах. Значение по умолчанию: `50000`.
- `--timeout-member-connect <ВРЕМЯ_ОЖИДАНИЯ>` — тайм-аут подключения к серверу в миллисекундах. Значение по умолчанию: `5000`.
- `--timeout-member-data <ВРЕМЯ_ОЖИДАНИЯ>` — тайм-аут бездействия сервера в миллисекундах. Значение по умолчанию: `50000`.
- `--timeout-tcp-inspect <ВРЕМЯ_ОЖИДАНИЯ>` — ожидание дополнительных TCP-пакетов для проверки содержимого в миллисекундах. Значение по умолчанию: `0`.
- `--enable` — включить правило балансировки (по умолчанию).
- `--disable` — отключить правило балансировки.
- `--allowed-cidr <РАЗРЕШЕННЫЙ_CIDR>` — разрешенный IP-адрес или маска сети в нотации CIDR. Несколько значений необходимо указывать с отдельным ключом `--allowed-cidr`. Пример: `--allowed-cidr <РАЗРЕШЕННЫЙ_CIDR_1> --allowed-cidr <РАЗРЕШЕННЫЙ_CIDR_2>`.
- `<БАЛАНСИРОВЩИК>` — название или идентификатор балансировщика нагрузки.

```console
      [--session-persistence <СВОЙСТВА>]
      --lb-algorithm <МЕТОД_БАЛАНСИРОВКИ>
```

- `--listener <ПРАВИЛО>` — название или идентификатор правила балансировки, которое привязывается к пулу серверов.
- `--loadbalancer <БАЛАНСИРОВЩИК>` — название или идентификатор балансировщика нагрузки, который привязывается к пулу серверов.
- `--session-persistence <СВОЙСТВА>` — свойства непрерывности сети для обработчика (в формате **ключ=значение**).
- `--lb-algorithm <МЕТОД_БАЛАНСИРОВКИ>` — метод балансировки. Возможные значения: `SOURCE_IP`, `ROUND_ROBIN`, `LEAST_CONNECTIONS`.
- `--enable` — включить пул серверов (по умолчанию).
- `--disable` — отключить пул серверов.
- `--tls-container-ref <ССЫЛКА_НА_КОНТЕЙНЕР>` — ссылка на TLS-контейнер.
- `--ca-tls-container-ref <ССЫЛКА_НА_КОНТЕЙНЕР>` — ссылка на TLS-контейнер с CA-сертификатом.
- `--crl-container-ref <ССЫЛКА_НА_КОНТЕЙНЕР>` — ссылка на контейнер со списком отозванных CA-сертификатов.
- `--enable-tls` — включить TLS-шифрование.
- `--disable-tls` — отключить TLS-шифрование.


```console

```
Здесь `<МОНИТОР>` — идентификатор монитора.

#### {heading(Создание монитора)[id=cli_openstack_loadbalancer_healthmonitor_create]}

```console
openstack loadbalancer healthmonitor create
      [--name <МОНИТОР>]
      --delay <ЗАДЕРЖКА>
      [--domain-name <ДОМЕН>]
      [--expected-codes <СТАТУСЫ>]
      [--http-method {GET,POST,DELETE,PUT,HEAD,OPTIONS,PATCH,CONNECT,TRACE}]
      [--http-version <ВЕРСИЯ_HTTP>]
      --timeout <ВРЕМЯ_ОЖИДАНИЯ>
      --max-retries <КОЛИЧЕСТВО_ПОПЫТОК>
      [--url-path <URL>]
      --type <ТИП>
      [--max-retries-down <КОЛИЧЕСТВО_ПОПЫТОК>]
      [--enable | --disable]
      <ПУЛ_СЕРВЕРОВ>
```

Здесь:

- `--name <МОНИТОР>` — название монитора.
- `--delay <ЗАДЕРЖКА>` — время в секундах между отправкой запросов участникам.
- `--domain-name <ДОМЕН>` — доменное имя, которое будет добавлено в HTTP-заголовок для проверки работоспособности HTTP.
- `--expected-codes <СТАТУСЫ>` — перечень HTTP-статусов, которые ожидается получить в случае работоспособности.
- `--http-method` — HTTP-методы, которые может использовать монитор. Возможные значения: `GET`, `POST`, `DELETE`, `PUT`, `HEAD`, `OPTIONS`, `PATCH`, `CONNECT`, `TRACE`.
- `--http-version <ВЕРСИЯ_HTTP>` — используемая версия HTTP.
- `--timeout <ВРЕМЯ_ОЖИДАНИЯ>` — максимальный тайм-аут в секундах.
- `--max-retries <КОЛИЧЕСТВО_ПОПЫТОК>` — число успешных попыток переподключения перед изменением рабочего статуса участника на `ONLINE`.
- `--url-path <URL>` — URL-адрес, на который монитор будет отправлять запросы по проверке работоспособности.
- `--type <ТИП>` — тип монитора. Обязательный параметр. Возможные значения: `PING`, `HTTP`, `HTTPS`, `TCP`.
- `--max-retries-down <КОЛИЧЕСТВО_ПОПЫТОК>` — максимально количество попыток подключения, после которых монитор переходит в состояние `ERROR`.
- `--enable` — включить монитор (по умолчанию).
- `--disable` — отключить монитор.
- `<ПУЛ_СЕРВЕРОВ>` — название или идентификатор пула для монитора.

## {heading(Управление маршрутизаторами)[id=cli_openstack_router_manage]}

### {heading(Просмотр списка маршрутизаторов)[id=cli_router_list]}

```console
openstack router list
      [--name <НАЗВАНИЕ>]
      [--enable | --disable]
      [--long]
      [--project <ПРОЕКТ>]
      [--project-domain <ДОМЕН_ПРОЕКТА>]
```

Здесь:

- `--name <НАЗВАНИЕ>` — фильтрация маршрутизаторов по названию.
- `--enable` — вывести список активных маршрутизаторов.
- `--disable` — вывести список неактивных маршрутизаторов.
- `--long` — вывести дополнительные свойства.
- `--project <ПРОЕКТ>` — фильтрация маршрутизаторов по названию или идентификатору проекта.
- `--project-domain <ДОМЕН_ПРОЕКТА>` — фильтровать маршрутизатор по домену проекта по названию или идентификатору. Опция используется в случае конфликтов между названиями проектов.

### {heading(Просмотр информации о маршрутизаторе)[id=cli_router_info]}

```console
openstack router show <МАРШРУТИЗАТОР>
```

Здесь `<МАРШРУТИЗАТОР>` — название или идентификатор маршрутизатора.

### {heading(Создание маршрутизатора)[id=cli_openstack_router_create]}

```console
openstack router create
    [--project <ПРОЕКТ> [--project-domain <ДОМЕН_ПРОЕКТА>]]
    [--enable | --disable]
    [--distributed]
    [--ha]
    [--description <ОПИСАНИЕ>]
    [--availability-zone-hint <ЗОНА_ДОСТУПНОСТИ>]
    <МАРШРУТИЗАТОР>
```

Здесь:

- `--project <ПРОЕКТ> [--project-domain <ДОМЕН_ПРОЕКТА>]` — название проекта и, при необходимости, домена, к которому будет относиться маршрутизатор.
- `--enable` — сделать маршрутизатор активным (значение по умолчанию).
- `--disable` — сделать маршрутизатор неактивным.
- `--distributed` — признак распределенности. Маршрутизатор создается только на вычислительных узлах.
- `--ha` — признак высокой доступности (High availability). Маршрутизатор создается на всех узлах.

  {note:warn}
  Для использования параметров `--distributed` и `--ha` требуется поддержка со стороны Neutron и соответствующие настройки.
  {/note}
  
- `--description <ОПИСАНИЕ>` — описание для маршрутизатора.
- `--availability-zone-hint <ЗОНА_ДОСТУПНОСТИ>` — зона доступности `<ЗОНА_ДОСТУПНОСТИ>`.
- `<МАРШРУТИЗАТОР>` — название маршрутизатора.

### {heading(Изменение параметров маршрутизатора)[id=cli_openstack_router_set]}

```console
openstack router set
      [--name <НАЗВАНИЕ>]
      [--description <ОПИСАНИЕ>]
      [--enable | --disable]
      [--route destination=<ПОДСЕТЬ>,gateway=<IP-АДРЕС>]
      [--no-route]
      [--ha | --no-ha]
      [--external-gateway <СЕТЬ>]
      <МАРШРУТИЗАТОР>
```

Здесь:

- `--name <НАЗВАНИЕ>` — новое название маршрутизатора.
- `--route destination=<ПОДСЕТЬ>,gateway=<IP-АДРЕС>` — указание статического маршрута. Здесь:

  - `<ПОДСЕТЬ>` — целевая подсеть в нотации CIDR.
  - `<IP-АДРЕС>` — IP-адрес ВМ.

- `--no-route` — очистить статические маршруты маршрутизатора. Чтобы перезаписать имеющиеся статические маршруты, используйте оба флага: `--route` и `--no-route`.
- `--external-gateway <СЕТЬ>` — внешняя сеть (с доступом к сетям вне {var(cloud)}. Обычно используется сеть `ext-net`.

Описание остальных параметров приведено в разделе {linkto(#cli_openstack_router_create)[text=%text]}.

Здесь:

- `--tag <ТЕГ>` — название тега, который необходимо удалить для маршрутизатора.
- `--all-tag` — удалить все теги для маршрутизатора.

Описание остальных параметров приведено в разделе {linkto(#cli_openstack_router_set)[text=%text]}.

```console
openstack floating ip list
      [--network <СЕТЬ>]
      [--port <ПОРТ>]
      [--long]
      [--status <СТАТУС>]
      [--project <ПРОЕКТ>]
```

- `--port <ПОРТ>` — фильтрация Floating IP-адресов по названию или идентификатору порта.

- `--description <ОПИСАНИЕ>` — описание Floating IP-адреса.
- `--project <ПРОЕКТ>` — название или идентификатор проекта, за которым закрепить IP-адрес.
- `--project-domain <ДОМЕН_ПРОЕКТА>` — присвоить IP-адрес домену проекта по названию или идентификатору. Флаг используется в случае конфликтов между названиями проектов.
