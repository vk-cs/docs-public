# {heading(Сеанс доступа и блокировка пользователя)[id=tools-account-user-session]}

{ifdef(private,private-pg,private-pdf,private-pg-pdf)}
- По умолчанию ограничений на количество одновременных сеансов доступа нет. Установить ограничение в один активный сеанс доступа для каждой учетной записи может только администратор.
{/ifdef}

{ifdef(private-cert)}
- Для каждой учетной записи единовременно может быть запущен только один сеанс доступа.
{/ifdef}

- После {ifndef(private-cert)}1 часа{/ifndef}{ifdef(private-cert)}5 минут{/ifdef} бездействия сеанс доступа будет завершен. Пройдите аутентификацию заново.

- Учетная запись может быть заблокирована, если не использовалась более 45 дней.

  {ifdef(private,private-pg,private-pdf,private-pg-pdf)}
  {note:warn}
  Администратор {var(cloud)} может изменить период неактивности пользователя, заданный по умолчанию.
  {/note}
  {/ifdef}

  При входе в личный кабинет с помощью заблокированной учетной записи на экране появится ошибка входа `403`.

  Блокировка учетной записи владельца проекта не ограничивает работу других пользователей в проекте. Для разблокировки учетной записи обратитесь к администратору {var(cloud)}.
