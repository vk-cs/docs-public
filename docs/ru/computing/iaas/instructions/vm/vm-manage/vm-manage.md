# {heading(Управление ВМ)[id=iaas-vm-manage]}

Изменить параметры и состояние виртуальной машины в {var(cloud)} можно в {ifdef(public)}[личном кабинете](https://msk.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}личном кабинете{/ifdef} или с помощью интерфейса командной строки OpenStack CLI.

## {heading(Получение ID виртуальной машины)[id=iaas-vm-manage-id-get]}

Для управления ВМ с помощью OpenStack CLI нужен идентификатор виртуальной машины.

{tabs}

{tab(Личный кабинет)}

1. {ifdef(public)}[Перейдите](https://msk.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}{linkto(../../../../../tools-for-using-services/account/instructions/lk-entry#tools-account-lk-entry)[text=Перейдите]}{/ifdef} в личный кабинет {var(cloud)}.
1. Перейдите в раздел **Облачные вычисления** → **Виртуальные машины**.
1. В списке ВМ нажмите на имя виртуальной машины, идентификатор которой нужно узнать.
1. На вкладке **Общая информация** найдите в таблице строку **ID виртуальной машины**.

{/tab}

{tab(OpenStack CLI)}

1. {linkto(../../../../../tools-for-using-services/cli/openstack-cli#tools-cli-openstack)[text=Установите]} клиент OpenStack CLI и пройдите аутентификацию в проекте, если этого еще не сделано.
1. Выполните одно из действий.

   - Если известно имя ВМ, получите информацию о виртуальной машине и найдите строку с `id`:

     ```console
     openstack server show <ИМЯ_ВМ>
     ```

   - Если точное имя ВМ неизвестно, выведите список виртуальных машин и найдите в нем нужную:

     ```console
     openstack server list
     ```

{/tab}

{/tabs}

## {heading(Запуск, остановка, перезагрузка ВМ)[id=iaas-vm-manage-start-stop-restart]}

{tabs}

{tab(Личный кабинет)}

1. {ifdef(public)}[Перейдите](https://msk.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}{linkto(../../../../../tools-for-using-services/account/instructions/lk-entry#tools-account-lk-entry)[text=Перейдите]}{/ifdef} в личный кабинет {var(cloud)}.
1. Перейдите в раздел **Облачные вычисления** → **Виртуальные машины**.
1. Измените состояние ВМ одним из способов.

   - С помощью групповых операций — для нескольких ВМ:

     1. В списке виртуальных машин выберите с помощью флажков ВМ, состояние которых необходимо изменить.
     1. Над списком ВМ нажмите кнопку с нужным действием.
     1. Подтвердите действие.

   - Через контекстное меню — для одной ВМ:

     1. В списке виртуальных машин нажмите ![ ](../../../../../assets/more-icon.svg "inline") для ВМ, состояние которой необходимо изменить.
     1. Выберите и подтвердите действие.

   - На странице виртуальной машины:

     1. В списке виртуальных машин нажмите на имя ВМ, состояние которой необходимо изменить.
     1. На странице ВМ перейдите на вкладку **Общая информация**.
     1. Справа над таблицей с параметрами ВМ нажмите на значок нужного действия.
     1. Подтвердите действие.

{/tab}

{tab(OpenStack CLI)}

1. {linkto(../../../../../tools-for-using-services/cli/openstack-cli#tools-cli-openstack)[text=Установите]} клиент OpenStack и пройдите аутентификацию в проекте, если этого еще не сделано.
1. Выполните нужную команду.

   - Запустить ВМ:

     ```console
     openstack server start <ID_ВМ>
     ```

   - Остановить ВМ:

     ```console
     openstack server stop <ID_ВМ>
     ```

   - Перезагрузить ВМ:

     ```console
     openstack server reboot <ID_ВМ>
     ```

{/tab}

{/tabs}

## {heading(Принудительный перезапуск ВМ)[id=iaas-vm-manage-restart]}

Если виртуальная машина не отвечает, используйте принудительный перезапуск.

{note:info}
Принудительный перезапуск ВМ соответствует выключению и включению питания (power cycling).<br/>Перезагрузка предполагает корректное завершение работы операционной системы ВМ (graceful shutdown).
{/note}

{tabs}

{tab(Личный кабинет)}

1. {ifdef(public)}[Перейдите](https://msk.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}{linkto(../../../../../tools-for-using-services/account/instructions/lk-entry#tools-account-lk-entry)[text=Перейдите]}{/ifdef} в личный кабинет {var(cloud)}.
1. Перейдите в раздел **Облачные вычисления** → **Виртуальные машины**.
1. Вызовите принудительный перезапуск ВМ одним из способов.

   - Через контекстное меню:

     1. В списке виртуальных машин нажмите ![ ](../../../../../assets/more-icon.svg "inline") для нужной ВМ.
     1. Выберите пункт **Перезапустить принудительно** и подтвердите действие.

   - На странице виртуальной машины:

     1. В списке виртуальных машин нажмите на имя ВМ, состояние которой необходимо изменить.
     1. На странице ВМ перейдите на вкладку **Общая информация**.
     1. Над таблицей с параметрами ВМ нажмите кнопку **Еще**.
     1. Нажмите **Перезапустить принудительно** и подтвердите действие.

{/tab}

{tab(OpenStack CLI)}

1. {linkto(../../../../../tools-for-using-services/cli/openstack-cli#tools-cli-openstack)[text=Установите]} клиент OpenStack и пройдите аутентификацию в проекте, если этого еще не сделано.

1. Вызовите принудительный перезапуск ВМ:

   ```console
   openstack server reboot --hard <ID_ВМ>
   ```

{/tab}

{/tabs}

   - Через контекстное меню:

     1. В списке виртуальных машин нажмите ![ ](../../../../../assets/more-icon.svg "inline") для нужной ВМ.

   - На странице виртуальной машины:

## {heading(Удаление ВМ)[id=iaas-vm-manage-delete]}

В {var(cloud)} виртуальная машина по умолчанию удаляется вместе с root-диском. Удаление дополнительных дисков зависит от настроек соответствующей политики.

{note:err}
Операция удаления необратима! Сохраните нужные данные перед удалением ВМ.
{/note}

1. Подготовьте виртуальную машину к удалению.

   - Если нужно сохранить root-диск, клонируйте его или {linkto(../../../../../computing/iaas/instructions/vm/vm-root-replace#iaas-vm-root-replace)[text=замените на ненужный]}.
   - {linkto(../../../../../computing/iaas/instructions/volumes/volumes-connect#iaas-volumes-connect-dismount-disk)[text=Отключите от ВМ]} дополнительные диски, которые нужно сохранить.

     Дополнительные диски, для которых в политике удаления установлено свойство `delete_on_termination`: `False`, можно не отключать от ВМ.

     {cut(Проверить политику удаления дисков — OpenStack CLI)}

     1. Убедитесь, что клиент OpenStack {linkto(../../../../../tools-for-using-services/cli/openstack-cli#openstack-install)[text=установлен]}, и {linkto(../../../../../tools-for-using-services/cli/openstack-cli#openstack-authorize)[text=пройдите аутентификацию]} в проекте.
     1. Выполните команду:

        ```console
        openstack server show <ID_ВМ> --os-compute-api-version 2.42
        ```

        Найдите строку `volumes_attached`. В ней перечислены идентификаторы дисков и указана политика их удаления — свойство `delete_on_termination` (`True` или `False`).

     {/cut}

1. После сохранения необходимых данных удалите виртуальную машину.

   {tabs}

   {tab(Личный кабинет)}
      
   1. {ifdef(public)}[Перейдите](https://msk.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}{linkto(../../../../../tools-for-using-services/account/instructions/lk-entry#tools-account-lk-entry)[text=Перейдите]}{/ifdef} в личный кабинет {var(cloud)}.
   1. Перейдите в раздел **Облачные вычисления** → **Виртуальные машины**.
   1. Удалите ВМ одним из способов.

      - С помощью групповых операций — для нескольких ВМ:

        1. В списке виртуальных машин выберите ВМ, которые необходимо удалить.
        1. В меню над списком ВМ нажмите кнопку **Удалить**.
        1. Подтвердите действие.

      - Через контекстное меню — для одной ВМ:

        1. В списке виртуальных машин нажмите ![ ](../../../../../assets/more-icon.svg "inline") для ВМ, которую необходимо удалить.
        1. Выберите пункт **Удалить** и подтвердите действие.

      - На странице виртуальной машины:

        1. В списке виртуальных машин нажмите на имя ВМ, которую необходимо удалить.
        1. На странице ВМ перейдите на вкладку **Общая информация**.
        1. Справа над таблицей с параметрами ВМ нажмите на значок корзины.
        1. Подтвердите действие.

   {/tab}

   {tab(OpenStack CLI)}
   
   1. {linkto(../../../../../tools-for-using-services/cli/openstack-cli#tools-cli-openstack)[text=Установите]} клиент OpenStack и пройдите аутентификацию в проекте, если этого еще не сделано.
   1. Удалите ВМ:

      ```console
      openstack server delete <ID_ВМ>
      ```

   {/tab}
   
   {/tabs}

## {heading(Переименование и изменение типа ВМ)[id=iaas-vm-manage-rename]}

В {var(cloud)} можно переименовать виртуальную машину, а также изменить тип ВМ — количество процессоров (vCPU) и объем оперативной памяти (RAM).

{note:info}
При изменении типа ВМ виртуальная машина будет перезагружена.
{/note}

{tabs}

{tab(Личный кабинет)}

1. {ifdef(public)}[Перейдите](https://msk.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}{linkto(../../../../../tools-for-using-services/account/instructions/lk-entry#tools-account-lk-entry)[text=Перейдите]}{/ifdef} в личный кабинет {var(cloud)}.
1. Перейдите в раздел **Облачные вычисления** → **Виртуальные машины**.
1. Измените название или тип ВМ одним из способов.

   - Через контекстное меню:

     1. В списке виртуальных машин нажмите ![ ](../../../../../assets/more-icon.svg "inline") для нужной ВМ.
     1. Выберите действие.
     1. Заполните поле ввода и нажмите кнопку **Сохранить**.

   - На странице виртуальной машины:

     1. В списке виртуальных машин нажмите на имя ВМ, состояние которой необходимо изменить.
     1. На странице ВМ перейдите на вкладку **Общая информация**.
     1. Над таблицей с параметрами ВМ нажмите кнопку **Еще**.
     1. Выберите действие.
     1. Заполните поле ввода и нажмите кнопку **Сохранить**.

{/tab}

{tab(OpenStack CLI)}

1. {linkto(../../../../../tools-for-using-services/cli/openstack-cli#tools-cli-openstack)[text=Установите]} клиент OpenStack и пройдите аутентификацию в проекте, если этого еще не сделано.
1. Выполните нужное действие.

   - Переименовать ВМ:

     ```console
     openstack server set --name <НОВОЕ_ИМЯ> <ID_ВМ>
     ```

   - Изменить тип ВМ:

     1. Получите список доступных шаблонов конфигурации и скопируйте ID шаблона:

        ```console
        openstack flavor list --all
        ```

     1. Запустите изменение типа ВМ и дождитесь окончания процесса:

        ```console
        openstack server resize --flavor <ID_ШАБЛОНА> <ID_ВМ>
        ```

{/tab}

{/tabs}

## {heading(Присвоение тегов)[id=iaas-vm-manage-tags]}

Теги позволяют отфильтровать список виртуальных машин и быстрее найти нужную ВМ. Используйте существующие теги или создайте собственный тег и выберите для него цвет.

{tabs}

{tab(Личный кабинет)}

1. {ifdef(public)}[Перейдите](https://msk.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}{linkto(../../../../../tools-for-using-services/account/instructions/lk-entry#tools-account-lk-entry)[text=Перейдите]}{/ifdef} в личный кабинет {var(cloud)}.
1. Перейдите в раздел **Облачные вычисления** → **Виртуальные машины**.
1. Нажмите ![ ](../../../../../assets/more-icon.svg "inline") для нужной ВМ.
1. Отметьте флажками нужные теги.
1. Для добавления нового тега нажмите **Создать новый тег**, введите его имя, выберите цвет и нажмите кнопку **Добавить тег**.

{/tab}

{tab(OpenStack CLI)}

1. {linkto(../../../../../tools-for-using-services/cli/openstack-cli#tools-cli-openstack)[text=Установите]} клиент OpenStack и пройдите аутентификацию в проекте, если этого еще не сделано.
1. Выполните нужную команду.

   - Вывести список тегов ВМ:

     ```console
     nova server-tag-list <ID_ВМ>
     ```

   - Присвоить тег ВМ:

     ```console
     nova server-tag-add <ID_ВМ> <ТЕГ>
     ```

   - Удалить тег у ВМ:

     ```console
     nova server-tag-delete <ID_ВМ> <ТЕГ>
     ```

   - Удалить все теги у ВМ:

     ```console
     nova server-tag-delete-all <ID_ВМ>
     ```

{/tab}

{/tabs}

## {heading(Установка и изменение пароля)[id=iaas-vm-manage-password]}

Пароль {linkto(../../../../../computing/iaas/concepts/oper-system#iaas-oper-system-default-account)[text=учетной записи по умолчанию]} нужно установить самостоятельно при первой авторизации в OC. Установленный пароль можно будет изменить.

Условия для установки пароля:

- виртуальная машина запущена и процесс загрузки ОС завершился;
- установлен и запущен гостевой агент QEMU (состояние — `running`).

{note:info}
В виртуальных машинах, созданных из образа {var(cloud)}, гостевой агент QEMU есть по умолчанию.
{/note}

Команда для проверки состояния гостевого агента QEMU:

{ifdef(public)}
{tabs}

{tab(Windows)}

```console
sc query qemu-ga
```

{/tab}

{tab(Linux)}
{/ifdef}

```console
systemctl status qemu-guest-agent
```
{ifdef(public)}
{/tab}

{/tabs}
{/ifdef}

Чтобы установить пароль:

{tabs}

{tab(Личный кабинет)}

1. {ifdef(public)}[Перейдите](https://msk.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}{linkto(../../../../../tools-for-using-services/account/instructions/lk-entry#tools-account-lk-entry)[text=Перейдите]}{/ifdef} в личный кабинет {var(cloud)}.
1. Перейдите в раздел **Облачные вычисления** → **Виртуальные машины**.
1. Воспользуйтесь одним из способов:

   - Через контекстное меню:

     1. В списке виртуальных машин нажмите ![ ](../../../../../assets/more-icon.svg "inline") для нужной ВМ.
     1. Выберите пункт **Установить пароль**.
     1. Введите новый пароль или нажмите кнопку **Сгенерировать**.
     1. Нажмите кнопку **Задать пароль**.

   - На странице виртуальной машины:

     1. В списке виртуальных машин нажмите на имя ВМ, состояние которой необходимо изменить.
     1. На странице ВМ перейдите на вкладку **Общая информация** или на вкладку **Консоль**.
     1. Над таблицей с параметрами ВМ или над окном с консолью нажмите кнопку **Установить пароль**.
     1. Введите новый пароль или нажмите кнопку **Сгенерировать**.

        {ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}
        {note:err}
        Сохраните пароль. В системе не предусмотрена возможность восстановить пароль.
        {/note}
        {/ifdef}

     1. Нажмите кнопку **Задать пароль**.

{/tab}

{tab(OpenStack CLI)}

1. {linkto(../../../../../tools-for-using-services/cli/openstack-cli#tools-cli-openstack)[text=Установите]} клиент OpenStack и пройдите аутентификацию в проекте, если этого еще не сделано.
1. Выполните команду:

   ```console
   openstack server set --root-password <ID_ВМ>
   ```

{/tab}

{/tabs}

{note:info}
Пароль должен быть длиной от 8 символов и содержать буквы латинского алфавита, цифры и спецсимволы `!`, `"`, `#`, `$`, `%`, `&`, `(`, `)`, `*`, `+`, `,`, `-`, `.`, `:`, `;`, `<`, `=`, `>`, `?`, `@`, `[`, `]`, `^`, `_`, `` ` ``, `{`, `}`, `~`.
{/note}

## {heading(Восстановление пароля)[id=iaas-vm-manage-password-recovery]}

Восстановление пароля, сгенерированного для пользователя по умолчанию, доступно только для виртуальных машин Windows, созданных в приватной сети. Чтобы получить пароль, необходим приватный ключ ключевой пары, выбранной при создании ВМ.

{note:warn}
Если после создания ВМ пароль был изменен или ВМ была создана во внешней сети, {linkto(#iaas-vm-manage-password)[text=установите новый пароль]}.
{/note}

{tabs}

{tab(Личный кабинет)}

1. [Перейдите](https://msk.cloud.vk.com/app/) в личный кабинет {var(cloud)}.
1. Перейдите в раздел **Облачные вычисления** → **Виртуальные машины**.
1. Откройте окно получения пароля одним из способов.

   - Через контекстное меню:

     1. В списке виртуальных машин нажмите ![ ](../../../../../assets/more-icon.svg "inline") для нужной ВМ.
     1. Выберите пункт **Получить пароль**.

   - На странице виртуальной машины:

     1. В списке виртуальных машин нажмите на имя ВМ, состояние которой необходимо изменить.
     1. На странице ВМ перейдите на вкладку **Общая информация**.
     1. Над таблицей с параметрами ВМ нажмите кнопку **Еще**.
     1. Выберите **Получить пароль**.

1. В открывшемся окне нажмите **Файл приватного ключа** или вставьте содержимое ключа в поле **Приватный ключ**.
1. Нажмите кнопку **Расшифровать пароль**.
1. В окне появится поле **Пароль**. Скопируйте пароль, нажав значок в поле.

{/tab}

{/tabs}

## {heading(Восстановление доступа к ВМ по ключу)[id=iaas-vm-manage-recovery-access-key]}

Для восстановления доступа к виртуальной машине Linux по SSH с использованием ключевой пары необходимо знать пароль пользователя ОС.

1. Получите публичный ключ.

   - Если утрачен файл с публичным ключом, перейдите в директорию с файлом приватного ключа и выполните команду:

     ```console
     ssh-keygen -y -f <ПРИВАТНЫЙ_КЛЮЧ> > <ПУБЛИЧНЫЙ_КЛЮЧ>
     ```
     Здесь:

     - `<ПРИВАТНЫЙ_КЛЮЧ>` — имя файла с приватным ключом.
     - `<ПУБЛИЧНЫЙ_КЛЮЧ>` — имя файла с публичным ключом.

   - Если утрачен файл с приватным ключом, {ifdef(public)}{linkto(../../../../../tools-for-using-services/vk-cloud-account/instructions/account-manage/keypairs#vk-cloud-account-manage-keypairs-create)[text=создайте новую ключевую пару]}{/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}{linkto(../../../../../tools-for-using-services/account/instructions/account-manage-private/keypairs-private#vk-cloud-account-manage-keypairs-create)[text=создайте новую ключевую пару]}{/ifdef}.

   {ifdef(public)}
1. Создайте ссылку на файл с публичным ключом.

   1. [Перейдите](https://msk.cloud.vk.com/app/) в личный кабинет {var(cloud)}.
   1. Перейдите в раздел **Объектное хранилище** → **Бакеты**.
   1. Используйте существующий бакет или {linkto(../../../../../storage/s3/instructions/buckets/create-bucket#s3-instructions-create-bucket)[text=создайте новый]}.
   1. {linkto(../../../../../storage/s3/instructions/objects/upload-object#s3-instructions-upload-object)[text=Добавьте файл]} с публичным ключом в бакет.
   1. {linkto(../../../../../storage/s3/instructions/access-management/signed-url#s3-instructions-signed-url)[text=Включите доступ]} по ссылке к этому файлу.
   1. Скопируйте ссылку на файл с публичным ключом.
   {/ifdef}

1. Загрузите публичный ключ на виртуальную машину.

   1. Перейдите в раздел **Облачные вычисления** → **Виртуальные машины**.
   1. В списке виртуальных машин нажмите на имя нужной ВМ.
   1. На странице ВМ перейдите на вкладку **Консоль**.
   1. Используйте имя пользователя по умолчанию и пароль для входа в ОС ВМ.
   1. Скачайте файл с публичным ключом:

      ```console
      wget <ССЫЛКА>
      ```
      Здесь `<ССЫЛКА>` — ссылка на файл с публичным ключом.

   1. Скопируйте публичный ключ в файл `authorized_keys`:

      ```console
      cat <ФАЙЛ> >> ~/.ssh/authorized_keys
      ```
      Здесь `<ФАЙЛ>` — файл с публичным ключом.

1. Проверьте возможность доступа по ключу.

   1. Перейдите в терминал компьютера, на котором хранится приватный ключ.
   1. Подключитесь к ВМ по SSH:

      ```console
      ssh -i <ФАЙЛ> <ЛОГИН>@<IP-АДРЕС>
      ```
      Здесь:

      - `<ФАЙЛ>` — путь к файлу с приватным ключом.
      - `<ЛОГИН>` — логин.
      - `<IP-АДРЕС>` — внешний IP-адрес ВМ.

## {heading(Просмотр журнала событий)[id=iaas-vm-manage-view-logs]}

Журнал событий содержит информацию об изменении состояния ВМ и о действиях пользователя.

{tabs}

{tab(OpenStack CLI)}

1. {linkto(../../../../../tools-for-using-services/cli/openstack-cli#tools-cli-openstack)[text=Установите]} клиент OpenStack и пройдите аутентификацию в проекте, если этого еще не сделано.
1. Выполните нужную команду.

   - Просмотреть журнал событий (расширенный формат):

     ```console
     openstack server event list --long <ID_ВМ>
     ```

   - Просмотреть журнал событий (краткий формат):

     ```console
     openstack server event list <ID_ВМ>
     ```

   - Используйте идентификатор события из колонки **Request ID**, чтобы вывести подробную информацию о событии:

     ```console
     openstack server event show <ID_ВМ> <ID_СОБЫТИЯ>
     ```

Примеры событий:

- `start` / `stop` — запуск / остановка ВМ;
- `resize` / `confirmResize` — запуск изменения типа ВМ / подтверждение;
- `extend_volume` — увеличение размера диска;
- `create` — создание ВМ;
- `live-migration` — миграция ВМ.

{/tab}

{/tabs}
