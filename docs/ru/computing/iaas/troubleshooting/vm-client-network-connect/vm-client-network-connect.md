# {heading(ВМ не видит внешнюю сеть)[id=iaas-vm-client-network-connect]}

Проблемы подключения виртуальной машины к внешней сети могут быть связаны с отсутствием сетевой связанности (VPN) между виртуальной сетью {var(cloud)} и внешней сетью.

### {heading(Решение)[id=iaas-vm-client-network-connect-decision]}

Создайте VPN-туннель между виртуальной сетью {var(cloud)} и внешней сетью. 

Для организации VPN-туннеля между облаком и небольшой удаленной сетью воспользуйтесь {linkto(../../../../networks/vnet/how-to-guides/onpremise-connect/vpn-tunnel#vnet-vpn-tunnel)[text=инструкцией]}. Либо посмотрите другие {linkto(../../../../networks/vnet/concepts/onpremise-connect#vnet-onpremise-connect)[text=варианты подключения]} к удаленной инфраструктуре. 

Выбор варианта подключения зависит от SDN проекта, доступа к интернету из удаленной инфраструктуры и требований к отказоустойчивости соединения.
