Ошибка `Недействительный пароль` при подключении сетевого диска может означать неверные учетные данные или настройки доступа.

### Решение

Проверьте правила доступа для файлового хранилища:

1. Перейдите в раздел **Облачные вычисления** → **Файловое хранилище**.
1. Нажмите на имя нужного файлового хранилища.
1. Перейдите на вкладку **Правила доступа**.

Для протоколов CIFS и SMB должен быть разрешен доступ для пользователя или IP-адреса и задан корректный пароль.

Если проблема сохраняется, [обратитесь в техническую поддержку](/ru/contacts).
