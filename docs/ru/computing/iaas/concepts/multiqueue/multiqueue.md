# {heading(Множественные очереди (Multiqueue))[id=iaas-multiqueue]}

{var(cloud)} поддерживает включение множественных очередей (Multiqueue) у образа ВМ.

Множественные очереди позволяют обрабатывать сетевые пакеты, распределяя их между несколькими виртуальными процессорами.

## {heading(Условия для включения множественных очередей)[id=iaas-multiqueue-conditions-for-enabling]}

Чтобы включить функцию множественных очередей, должны быть выполнены следующие условия:

* Образ ВМ загружен пользователем самостоятельно.
* Образ ВМ отличается от образов по умолчанию (предустановленных в личном кабинете).

## {heading(Сценарии использования)[id=iaas-multiqueue-usage-scenarios]}

Множественные очереди обеспечивают повышение производительности в следующих случаях:

* Пакеты трафика относительно большие.
* ВМ активна во многих сетевых подключениях одновременно, при этом трафик проходит между:

    * Разными ВМ.
    * ВМ с хостом.
    * ВМ с внешней системой.

* Количество очередей равно количеству виртуальных процессоров (vCPU). Это связано с тем, что поддержка нескольких очередей оптимизирует сходство прерываний RX и выбор очереди TX, чтобы сделать конкретную очередь частной для определенного vCPU.

## {heading(Ограничения)[id=iaas-multiqueue-restrictions]}

Функция множественных очередей virtio-net обеспечивает повышение производительности, но имеет некоторые ограничения:

* ОС ВМ ограничена ~ 200 векторами MSI. Для каждой очереди сетевого адаптера требуется вектор MSI, а также любое устройство virtio или назначенное устройство PCI. Определение экземпляра с несколькими сетевыми адаптерами virtio и виртуальными CPU может привести к превышению лимита гостевого MSI.
* Множественные очереди хорошо работают для входящего трафика, но иногда могут вызвать снижение производительности для исходящего трафика.
* Включение множественных очередей увеличивает общую пропускную способность сети, но одновременно увеличивают потребление ресурсов CPU.
* Включение функции множественных очередей virtio-net в конфигурации хоста QEMU не включает функциональность в ОС ВМ. Администратору ОС ВМ необходимо вручную включить его для каждой сетевой карты, для которой требуется эта функция, с помощью `ethtool`.
* Если множественные очереди были включены на хосте, но не были включены администратором в ОС ВМ, векторы MSI будут использоваться впустую.
* Если количество виртуальных сетевых адаптеров в экземпляре ВМ пропорционально количеству виртуальных CPU, включение множественных очередей менее важно.
