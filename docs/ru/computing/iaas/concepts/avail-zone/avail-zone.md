# {heading(Зоны доступности)[id=iaas-avail-zone]}

Зона доступности — это один или несколько центров обработки данных (ЦОД), в которых могут быть размещены компоненты облачной инфраструктуры.

Каждая зона изолирована от сбоев в других зонах доступности. Размещение виртуальных ресурсов в нескольких зонах обеспечивает отказоустойчивость и снижает вероятность потери данных.

{var(cloud)} распределяет по {ifdef(public)}{linkto(../../../../start/concepts/architecture#architecture-az)[text=поддерживаемым зонам доступности]}{/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cert)}поддерживаемым зонам доступности{/ifdef} следующие виртуальные ресурсы:

- виртуальные машины;
- диски: размещаются в сетевом блочном хранилище и автоматически реплицируются внутри своей зоны доступности{ifdef(private,private-pg,private-pdf,private-pg-pdf)};
- файловые хранилища{/ifdef},
- балансировщики нагрузки.

Существуют ресурсы, для которых нет зон доступности, например, {ifdef(public)}бакет VK Object Storage или {/ifdef}виртуальная сеть. {ifdef(public)}Для проектов региона Москва эти ресурсы являются глобальными (не привязаны к зоне).

{note:info}
Сеть, созданную в зонах доступности QAZ и KTP, невозможно объединить стандартными способами с сетями из других зон доступности, так как они находятся в другом {linkto(../../../../tools-for-using-services/account/concepts/regions#tools-account-concepts-regions)[text=регионе]}. В этом случае сетевую связность можно обеспечить средствами {linkto(../../../../networks/vnet/how-to-guides/onpremise-connect/vpn-tunnel#vnet-vpn-tunnel)[text=VPN]}.
{/note}
{/ifdef}
