Сіз {var(cloud)} платформасында image-based қолданбаларыңыздың инстанстарын орналастыру кезінде SSL-сертификаттарды автоматты түрде орната аласыз, сондай-ақ орнатылған SSL-сертификаттарды дер кезінде жаңарта аласыз. Бұл қолданбаларыңызға пайдаланушылармен деректер алмасу үшін қорғалған HTTPS протоколын пайдалануға мүмкіндік береді.

Төменде мысалда [SSL-сертификаттарды басқару жүйесін](/kz/tools-for-using-services/vendor-account/concepts/ssl-cert-concept) image-based қолданбаға қалай біріктіруге болатыны көрсетілген.

## {heading(Дайындық қадамдары)[id=preparatory_steps]}

Егер мұны әлі жасамаған болсаңыз, image-based қолданба инстанстары орналасатын доменді тіркеңіз.

## {heading(1. Қолданба инстансының ВМ бейнесін дайындаңыз)[id=adjust_vm_image]}

image-based қолданба орнатылған ВМ [бейнесін дайындау](/kz/tools-for-using-services/vendor-account/manage-apps/ibservice_add/ib_image_create) кезінде:

1. SSL-сертификат файлы мен оның жабық кілті сақталатын директорияны жасаңыз.

   Егер ВМ-ға бірнеше қолданба және оларға арналған SSL-сертификаттар орнатылса, қауіпсіздікті арттыру үшін әр қолданбаға бөлек директория жасау ұсынылады.

1. (Опционалды) SSL-сертификат файлы мен оның жабық кілтіне қолжетімділікті баптаңыз:

   1. Осы файлдардың иесі болып тағайындалатын пайдаланушыны жасаңыз.
   1. SSL-сертификат файлы мен оның жабық кілті үшін пайдаланушылар тобын жасаңыз.
   1. Пайдаланушы мен топқа SSL-сертификат файлы мен оның жабық кілтіне, сондай-ақ олар орналастырылатын директорияға қол жеткізу құқықтарын беріңіз.

      Қауіпсіздікті арттыру үшін қажет болғаннан артық құқықтар берілмегені жөн. Бұл жағдайда келесі қатынау құқықтарын орнату жеткілікті:

      - директория үшін — `755` (иесінде шектеусіз қолжетімділік бар, ал топ пен қалған пайдаланушыларда тек оқу және орындау құқықтары бар);
      - файлдар үшін — `644` (иесінде оқу және жазу құқықтары бар, ал топ пен қалған пайдаланушыларда тек оқу құқығы бар).

1. (Опционалды) SSL-сертификат жаңартылған сайын инстанс ВМ-да қандай әрекеттер орындалуы керектігін көрсетіңіз:

   1. Қажетті әрекеттері бар bash-скриптті дайындап, скрипт файлын ВМ бейнесіне орналастырыңыз.
   1. Егер SSL-сертификаттың иесі-пайдаланушы жасалған болса, оған скрипт орналасқан директорияға қол жеткізу және скриптті орындау құқығын беріңіз.
   1. Скрипттің дұрыс орындалатынын тексеріңіз.

## {heading(2. Қолданбаның Terraform манифесін дайындаңыз)[id=adjust_manifest]}

Қолданба инстансын орналастыру инфрақұрылымының сипаттамасы бар [Terraform манифесіне](/kz/tools-for-using-services/vendor-account/manage-apps/ibservice_add/tf_manifest/tf_manifest_steps) [ivkcs_cert](/kz/tools-for-using-services/vendor-account/manage-apps/ivkcs/ivkcs_resources/ivkcs_cert) ресурсын қосыңыз.

{cut(Базалық баптаулары бар ресурс мысалы)}

Мысалда келтірілген ресурс конфигурациясы:

- `example.com` домені үшін SSL-сертификат жасайды.
- SSL-сертификат пен оның жабық кілтін `/etc/ssl/certs/` директориясына орналастырады.
- SSL-сертификат әр жаңартылған сайын `/opt/scripts/reload_nginx.sh` скриптін орындайды.

```hcl
resource "ivkcs_cert" "web_cert" {
  uuid              = var.instance_uuid
  domain            = "example.com"
  path_cert         = "/etc/ssl/certs/"
  renewal_hook_path = "/opt/scripts/reload_nginx.sh"
}
```

Мұнда:

- `uuid` — image-based қолданба инстансының идентификаторы.
- `domain` — image-based қолданбаңыздың инстанстары орналасатын және оған SSL-сертификат шығарылатын доменнің FQDN-ы.
- `path_cert` — SSL-сертификат файлы мен оның жабық кілті сақталатын директорияға жол.
- `renewal_hook_path` — SSL-сертификат жаңартылғаннан кейін орындалатын скрипт бар директорияға жол.

{/cut}

{cut(Кеңейтілген баптаулары бар ресурс мысалы)}

Мысалда келтірілген ресурс конфигурациясы:

- `secure.example.com` домені үшін SSL-сертификат жасайды.
- SSL-сертификат пен оның жабық кілтін `/opt/app/ssl/` директориясына орналастырады.
- SSL-сертификат әр жаңартылған сайын `/opt/app/scripts/restart_service.sh` скриптін орындайды.
- `nginx` пайдаланушысын SSL-сертификат файлы мен оның жабық кілтінің иесі етіп тағайындайды.
- `ssl-cert` тобын SSL-сертификат файлы мен оның жабық кілті үшін тағайындайды.
- SSL-сертификатпен операциялар үшін тайм-ауттарды орнатады.

```hcl
resource "ivkcs_cert" "secure_cert" {
  uuid              = var.instance_uuid
  domain            = "secure.example.com"
  path_cert         = "/opt/app/ssl/"
  renewal_hook_path = "/opt/app/scripts/restart_service.sh"
  cert_files_owner  = "nginx"
  cert_files_group  = "ssl-cert"

  timeouts {
    create = "30m"
    update = "15m"
    delete = "5m"
  }
}
```

Мұнда:

- `uuid` — image-based қолданба инстансының идентификаторы.
- `domain` — image-based қолданбаңыздың инстанстары орналасатын және оған SSL-сертификат шығарылатын доменнің FQDN-ы.
- `path_cert` — SSL-сертификат файлы мен оның жабық кілті сақталатын директорияға жол.
- `renewal_hook_path` — SSL-сертификат жаңартылғаннан кейін орындалатын скрипт бар директорияға жол.
- `cert_files_owner` — SSL-сертификат файлы мен оның жабық кілтінің иесі-пайдаланушы.
- `cert_files_group` — SSL-сертификат файлы мен оның жабық кілті үшін пайдаланушылар тобы.
- `timeouts.create` — SSL-сертификатты жасау тайм-ауты.
- `timeouts.update` — SSL-сертификатты жаңарту тайм-ауты.
- `timeouts.delete` — SSL-сертификатты жою тайм-ауты.

{/cut}
