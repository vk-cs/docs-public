# {heading(Image-based қосымшаны жаңарту)[id=ibservice_update]}

Image-based қосымшаның конфигурациясын жаңарту мыналарды қамтуы мүмкін:

* Тарифтік жоспарлар мен опциялардың параметрлерін өзгерту.
* Тарифтік жоспарлар мен опцияларды қосу.
* Тарифтік жоспарлар мен опцияларды жою.
* Terraform манифестерінде сипатталған ресурстар параметрлерін өзгерту.
* Terraform манифестеріне ресурстар қосу.

{note:err}

Дүкеннің ағымдағы нұсқасы Terraform манифестері конфигурациясынан ресурстарды жоюды қолдамайды. Бұл функционал келесі нұсқада іске асырылады.

Манифестердегі барлық өзгерістер кері үйлесімділікті қолдауы тиіс.

{/note}

Дүкендегі image-based қосымшаны жаңарту үшін:

1. Сервистік пакеттің файлдарын өңдеңіз. Файл параметрлері [Дүкенге image-based қосымшаны қосу](../ibservice_add) бөлімінде берілген.
1. `service.yaml` файлында сервистің жаңа ревизиясын көрсетіңіз.

   {note:warn}

   Егер көрсетілген сервис ревизиясы дүкенге әлдеқашан жүктелген болса, сервис жаңартылмайды.

   {/note}
1. Егер тарифтік опциялар файлдары (`parameters` директориясы) өзгертілсе, кемінде бір өзгертілген опцияны пайдаланатын әрбір тарифтік жоспар үшін жаңа ревизияны көрсетіңіз.

   Егер тарифтік жоспарды сипаттайтын файлдар өзгертілсе, `plans/<PLAN_NAME>/plan.yaml` файлында жаңа ревизияны көрсетіңіз.

   {note:warn}

   Егер көрсетілген жоспар ревизиялары дүкенге әлдеқашан жүктелген болса, тарифтік жоспарлар мен опциялар жаңартылмайды.

   {/note}
1. Жаңартылған сервистік пакетті жүктеп, сервистің жаңа ревизиясын дүкеннің ашық атаулар кеңістігінде жариялаңыз (толығырақ — [Дүкенге image-based қосымшаны жүктеу](../ibservice_add/ibservice_upload) бөлімінде).

   Алдыңғы жарияланған ревизия дүкеннің тестілік атаулар кеңістігіне ауысады. Тестілік атаулар кеңістіктерінде сервистің барлық жарияланбаған ревизиялары сақталады.

   Жаңа ревизияны жариялау сервис инстанстарының бұрыннан бар даналарына әсер етпейді. Бұл сервис инстанстарының ревизиясы өзгермейді.
