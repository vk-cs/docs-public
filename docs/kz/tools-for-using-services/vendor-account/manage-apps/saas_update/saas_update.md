# {heading(SaaS-қолданбаны жаңарту)[id=saas_update]}

{include(/kz/_includes/_translated_by_ai.md)}

SaaS-қолданбаның конфигурациясын жаңарту мыналарды қамтуы мүмкін:

* Тарифтік жоспарлар мен опциялардың параметрлерін өзгерту.
* Тарифтік жоспарлар мен опцияларды қосу.
* Тарифтік жоспарлар мен опцияларды жою.

Marketplace дүкенінде SaaS-қолданбаны жаңарту үшін:

1. Сервис конфигурациясы бар файлды өңдеңіз.

   1. Сервистің ағымдағы жарияланған ревизиясын дүкеннің тестілік атау кеңістіктерінде қолжетімді күйде сақтау үшін JSON-файлға сервистің жаңа конфигурациясын сипаттайтын жаңа секцияны кірістіріңіз:

      {caption(null)[align=left;position=above]}
      ```json
      {
        "services": [
          {
          <SERVICE_PARAMETERS>,
          "preview": {
              "parameters": [
              ]
            },
          "plans": [
            ]
          },
          {
          <SERVICE_PARAMETERS>,
          "preview": {
              "parameters": [
              ]
            },
          "plans": [
            ]
          }
        ]
      }
      ```
      {/caption}

      Егер сервистің ағымдағы ревизиясын дүкеннің тестілік атау кеңістігінде қолжетімді күйде сақтау талап етілмесе, бұл қадамды өткізіп жіберіңіз.
   1. Сервистің жаңа конфигурациясын сипаттаңыз (толығырақ — [SaaS-қолданбаның конфигурациясы](../saas_add/saas_configure) бөлімінде).
   1. Сервистің жаңа конфигурациясында сервистің жаңа ревизиясын көрсетіңіз.

      {note:warn}

      Егер көрсетілген сервис ревизиясы дүкенге әлдеқашан жүктелген болса, сервис жаңартылмайды.

      {/note}
   1. Егер тарифтік жоспарлардың параметрлері өзгертілген болса, онда сервистің жаңа конфигурациясында жоспарлардың жаңа ревизияларын көрсетіңіз.

      {note:warn}

      Егер көрсетілген жоспар ревизиялары дүкенге әлдеқашан жүктелген болса, тарифтік жоспарлар жаңартылмайды.

      {/note}

1. Егер тарифтік жоспарлар қосылған немесе жойылған болса, брокер директориясындағы `resources/plan_mapping.json` файлын өңдеңіз (толығырақ — {linkto(#saas_broker_add_configuration)[text=%text]} бөлімінде).
1. Дүкен брокерден сервистің ағымдағы конфигурациясын мерзімді түрде сұратады, сондықтан жаңа конфигурация автоматты түрде дүкеннің тестілік атау кеңістігіне түседі.

   {note:err}

   Дүкен сервистің ағымдағы конфигурациясын ала алуы үшін брокер тәулік бойы қолжетімді болуы тиіс.

   {/note}
1. Сервистің жаңа ревизиясын дүкеннің тестілік атау кеңістігінде тестілеңіз (толығырақ — {linkto(#saas_upload_testmarketplace)[text=%text]} бөлімінде).
1. Сервистің жаңа ревизиясын дүкеннің ашық атау кеңістігінде жариялаңыз (толығырақ — {linkto(#saas_upload_publish)[text=%text]} бөлімінде).

   Егер сервистің алдыңғы жарияланған ревизиясы сервис конфигурациясы бар файлда сақталған болса, ол дүкеннің тестілік атау кеңістігіне ауысады. Тестілік атау кеңістіктерінде сервис конфигурациясында сипатталған жарияланбаған барлық ревизиялар сақталады.

   Сервис конфигурациясынан жойылған сервис ревизиялары дүкеннің тестілік те, ашық та атау кеңістіктерінде қолжетімді болмайды.

   Жаңа ревизияны жариялау сервистің бұрыннан бар инстанстарына (тенанттарына/аккаунттарына) әсер етпейді. Бұл сервис инстанстарының ревизиясы, тіпті ол сервис конфигурациясынан жойылып, дүкенде қолжетімсіз болса да, өзгермейді.
