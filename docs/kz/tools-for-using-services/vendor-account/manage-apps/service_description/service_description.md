# {heading(Дүкен үшін сервис сипаттамасын толтыру)[id=service_description]}

Сервис сипаттамасы мыналарды қамтиды:

* Белгішені.
* Қысқаша сипаттаманы.
* Толық сипаттаманы.

Сервис сипаттамасы оның конфигурациясында беріледі.

## {heading(Белгіше)[id=service_description_icon]}

Сервис белгішесі сервис картасында көрсетіледі:

* Дүкен каталогында.
* Сервис бетінің **Сервис сипаттамасы** қойындысында.

Image-based қолданба үшін белгіше [сервис пакетінің](/kz/tools-for-using-services/vendor-account/manage-apps/ibservice_add/ib_structure) құрамында жүктеледі.

SaaS-қолданба үшін [marketplace@cloud.vk.com](mailto:marketplace@cloud.vk.com) мекенжайына SVG форматындағы, өлшемі кемінде 62 × 62 пиксель болатын тіркелген белгішесі бар хат жіберіңіз. Жауап хатта [JSON-файлда](/kz/tools-for-using-services/vendor-account/manage-apps/saas_add/saas_configure/saas_json) көрсетілетін белгішеге сілтеме жіберіледі.

## {heading(Сервистің қысқаша сипаттамасы)[id=service_description_short]}

Сервистің қысқаша сипаттамасы сервис картасында көрсетіледі:

* Дүкен каталогында.
* Сервис бетінің **Сервис сипаттамасы** қойындысында.

Мәтін сервис конфигурациясындағы `short_description` параметрінде көрсетіледі:

* Image-based қолданба үшін — сервис пакетінің `service.yaml` файлында.
* SaaS-қолданба үшін — JSON-файлда.

## {heading(Сервистің толық сипаттамасы)[id=service_description_full]}

Сервистің толық сипаттамасы сервис бетінің **Сервис сипаттамасы** қойындысында көрсетіледі.

Мәтін сервис конфигурациясында көрсетіледі:

* Image-based қолданба үшін — сервис пакетінің `full_description.md` файлында.
* SaaS-қолданба үшін — JSON-файлдағы `full_description` параметрінде.

Жалпы жағдайда сервистің толық сипаттамасы келесі бөлімдерді қамтуы тиіс:

* Сервис туралы ақпарат.
* Тарифтеу ережелері.
* Пайдалы сілтемелер.
* Техникалық қолдау.
* Лицензиялық келісім.

### {heading(Сервис туралы ақпарат)[id=service_description_info]}

Бөлімде келесі ақпаратты келтіріңіз:

* Сервистің қысқаша шолуы.

   Сервистің не екенін, қандай міндеттерді шешуге көмектесетінін сипаттаңыз (негізгі пайдалану сценарийлері).
* Жеткізуші кім екенін:

   * Вендор.
   * Дистрибьютор.
   * VK.

   {note:info}

   Opensource-шешімдер үшін жеткізуші VK болып табылады.

   {/note}
* Жеткізуші туралы ақпаратты (міндетті емес).

   {note:info}

   Opensource-шешімдер үшін жеткізуші туралы ақпарат көрсетілмейді.

   {/note}
* Сервис сипаттамаларын. Кесте түрінде келесі деректерді көрсетіңіз:

   * Сервис түрін:

      * Image-based.
      * SaaS.

   * Сервис нұсқасын (болған жағдайда). Мысалы, Usergate 7.01.

      {note:info}

      Сервис нұсқасы оның конфигурациясында көрсетілген сервис ревизиясы болып табылмайды.

      {/note}
   * Image-based қолданба үшін — ОЖ-ның нақты нұсқасын. Мысалы, Ubuntu 22.04.
   * Сервис санатын. Санат мысалдары:

      * Аналитика.
      * Дерекқорлар.
      * Қауіпсіздік.
      * Әзірлеу құралдары.
      * VPN.

Image-based қолданба үшін бөлімде қосымша ақпаратты көрсетіңіз:

* Сервисті орналастыру нұсқаларын:

   * Бір ВМ.
   * Кластер.

* Қандай бұлттық платформа инфрақұрылымы қолданылатынын. Мысалы, ВМ, виртуалды желілер, DNS.
* Желіге қойылатын талаптарды:

   * ВМ сыртқы желіге қол жеткізе алуы тиіс.
   * Ішкіжелі баптауларында private DNS өшірілуі тиіс.

Пайдаланушыда сервис туралы толық түсінік қалыптастыру үшін бөлімге сервис схемалары мен интерфейс скриншоттарын қосыңыз (міндетті емес).

{caption(Сервис туралы ақпарат бөлігін толтыру мысалы, Markdown пішімі)[align=left;position=above]}
```markdown
UserGate NGFW содержит все функции решений класса UTM (Unified Threat Management), обеспечивая безопасность от всевозможных интернет-угроз и управление интернет-доступом. UserGate NGFW осуществляет защиту от атак, управление трафиком, аутентификацию интернет-пользователей, а также обеспечивает безопасность посещения сотрудниками интернет-ресурсов, ограждает их от загрузки опасного и вредоносного содержимого.

Поставщик сервиса: компания UserGate.

## Решаемые задачи

### Как межсетевой экран

* Разграничение внешнего интернет-пространства от внутренней или корпоративной сети на физической границе.
* VPN-шлюз для организации доступа и аутентификации конкретных удаленных пользователей к конкретным информационным системам.
* Организация гибридной среды типа локальная инфраструктура-облако посредством Site-to-Site VPN L2TP IPSec и объединения филиалов с центром и облаком VK Cloud.

## Характеристики сервиса:

|----------------------|------------------|

<br>Осы өнімнің жұмыс істеуі үшін мыналар пайдаланылады:
> * Виртуальная машина.
> * Виртуальные сети.
> * Floating IP-адреса.

**Минимальные требования к инфраструктуре**
Виртуальные машины:
* vCPU: 2.
* RAM: 8 ГБ.
* Размер дисков: не менее 100 ГБ.
* Сеть:
    * ВМ должна иметь доступ во внешнюю сеть.
    * Ішкі желі баптауларында private DNS **өшірілген** болуы керек.
```

### {heading(Тарифтеу ережелері)[id=service_description_tariffication]}

Бөлімде келесі ақпаратты келтіріңіз:

* Тарифтеу түрін (толығырақ — {linkto(/kz/tools-for-using-services/vendor-account/manage-apps/concepts/about#xaas_billing)[text=%text]} бөлімінде).
* Егер сервисте бірнеше тарифтік жоспар болса, олармен сервис бетіндегі тарифтік жоспарлар матрицасында танысуға болатынын көрсетіңіз.
* Егер сервисте ақылы тарифтік опциялар болса, мұндай опциялар тарифтік жоспар құнына қосымша тарифтелетінін көрсетіңіз.
* Егер сервисте келісімшарттық бағасы бар тарифтік жоспарлар болса, осы туралы ақпаратты көрсетіңіз.
* Image-based қолданба үшін бұлттық платформа инфрақұрылымының құны бөлек тарифтелетінін көрсетіңіз.

{note:err}

Егер image-based қолданба сервисті қосу сәтінде көлемі мен құнын анықтау мүмкін емес бұлттық платформа инфрақұрылымын пайдаланса (мысалы, жұмыс барысында жиналатын ақпаратты сақтау — бэкаптар), бұл туралы тарифтеу ережелерінің сипаттамасында хабарлаңыз.

{/note}

{caption(Тарифтеу ережелері бөлімін толтыру мысалы, Markdown пішімі)[align=left;position=above]}
```markdown
# Правила тарификации
Сервис Bring Your Own License моделі бойынша ұсынылады. Тарификация тек пайдаланылатын VK Cloud инфрақұрылымы үшін жүргізіледі. Оның құнын тарифтік жоспарды конфигурациялау шеберіндегі **Детализация** блогында бағалауға болады. Қорытынды құны тарифтік жоспарды конфигурациялау шеберінің соңғы бетінде көрсетіледі.
```
{/caption}
