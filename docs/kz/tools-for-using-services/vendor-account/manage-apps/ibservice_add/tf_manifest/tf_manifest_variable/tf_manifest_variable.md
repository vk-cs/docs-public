# {heading(Ресурстар конфигурациясына арналған кіріс айнымалылары)[id=tf_manifest_variable]}

{include(/kz/_includes/_translated_by_ai.md)}

Ресурстар конфигурациясына арналған кіріс айнымалыларын сипаттаңыз. Айнымалыларды шартты түрде {linkto(#tab_tf_variables)[text=%number кестеде]} келтірілген топтарға бөлуге болады.

{caption(Кесте {counter(table)[id=numb_tab_tf_variables]} — Terraform айнымалылары)[align=right;position=above;id=tab_tf_variables;number={const(numb_tab_tf_variables)}]}
[cols="2,5,2", options="header"]
|===
|Топ
|Сипаттама
|Қолдау көрсетілетін түрлер

|Арнайы
|
Брокерден алынған деректерді пайдалануға мүмкіндік беретін айнымалылар.

Келесі айнымалыларға қолдау көрсетіледі:

* `instance_uuid` — сервисті жайылту идентификаторы. iVK CS провайдерінің көптеген ресурстарын сипаттау үшін пайдаланылады.
* `email` — сервисті қосқан пайдаланушының email-ы.

Манифестте мұндай айнымалыларды жариялаған кезде, олардың мәнін Terraform автоматты түрде алады
|string

|Сыртқы
|
Тарифтік жоспардың конфигурациялау шеберінде бапталатын тарифтік опциялар мәндерін орнатуға мүмкіндік беретін айнымалылар.

Айнымалы атауы тарифтік опцияның YAML файлының атауына сәйкес келуі тиіс.

Мұндай айнымалыны манифестте жариялаудан бөлек, ол:

* Өзінің YAML файлында тарифтік опция ретінде сипатталуы тиіс.
* Тарифтік жоспарда көрсетілуі тиіс (`plans/<PLAN_NAME>/plan.yaml` файлы).
* Тарифтік жоспардың конфигурациялау шеберінде көрсетілуі тиіс (`plans/<PLAN_NAME>/display.yaml` файлы).

Сыртқы айнымалы үшін әдепкі мәнді орнатуға болады. Егер сыртқы мәнді алу мүмкін болмаса, сол мән қолданылады
|number, string, boolean

|Жергілікті
|
Тек манифестте қолданылатын айнымалылар. Айнымалыны жариялау кезінде `default` аргументінде мәнін көрсетіңіз
|Terraform қолдайтын түрлер
|===
{/caption}

{note:warn}

Айнымалыны жариялау кезінде `type` параметрінде түрін көрсетіңіз.

{/note}

{caption(Арнайы айнымалыларды жариялау мысалы)[align=left;position=above]}
```hcl
# Идентификатор развертывания сервиса
variable "instance_uuid" {
  type    = string
}

# Email пользователя
variable "email" {
  type    = string
}
```
{/caption}

{caption(Сыртқы және жергілікті айнымалыларды жариялау мысалы)[align=left;position=above]}
```hcl
# Имя виртуальной сети, где будет создана ВМ
variable "sub_network" {
  type    = string
  default = "test-net"
}

# Количество CPU для ВМ
variable "flavor_vcpus" {
  type    = number
  default = 2
}
```
{/caption}

Кіріс айнымалыларын сипаттау кезінде `locals` блоктарын пайдалануға болады. Олардың көмегімен есептеулерді, соның ішінде басқа жарияланған айнымалылар негізіндегі есептеулерді пайдаланып айнымалыларды орнатуға болады.
