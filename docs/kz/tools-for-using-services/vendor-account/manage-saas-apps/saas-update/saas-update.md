SaaS-қосымшаны оның тарифтік жоспарларына өзгерістер енгізілген жағдайда жаңарту қажет:

* тарифтік жоспарлардың немесе опциялардың параметрлері өзгерсе;
* тарифтік жоспарлар немесе опциялар қосылса;
* тарифтік жоспарлар немесе опциялар жойылса.

Осындай өзгерістер болған жағдайда Marketplace-тегі SaaS-қосымшаның конфигурациясын жаңартыңыз:

1. Сервис конфигурациясының сипаттамасы бар `catalog_<ИМЯ_СЕРВИСА>.json` JSON-файлын өңдеңіз.

   1. Marketplace-тің тестілік namespace-терінде сервистің ағымдағы жарияланған ревизиясына қолжетімділікті сақтау үшін JSON-файлға сервистің жаңа конфигурациясын сипаттайтын қосымша секцияны кірістіріңіз.

      {cut(Қосымша секцияның құрылымы)}

      {include(/kz/_includes/_saas_add_update.md)[tags=service_config]}

      {/cut}

   1. JSON-файлда сервистің жарияланған ревизияларының сипаттамасы бар барлық секциялар сақталғанына көз жеткізіңіз.

      {note:err}

      Мұндай секцияларды жою осы ревизиялардың жойылуына әкеледі. Жоюға тек сервистің жарияланбаған ревизиялары үшін ғана рұқсат етіледі: тестілік немесе жарияланымнан алынған.

      {/note}

   1. Сервистің жаңа конфигурациясында жаңа [сервис ревизиясы](../../../vendor-account/reference/saas-apps-reference/saas-param) көрсетілгеніне көз жеткізіңіз. Егер көрсетілген сервис ревизиясы Marketplace-ке әлдеқашан жүктелген болса, сервис жаңартылмайды.

   1. Егер ағымдағы тарифтік жоспарлардың параметрлерін өзгерткен болсаңыз, сервистің жаңа конфигурациясында жаңа [жоспар ревизиялары](../../../vendor-account/reference/saas-apps-reference/saas-plan) көрсетілгеніне көз жеткізіңіз. Егер көрсетілген жоспар ревизиялары Marketplace-ке әлдеқашан жүктелген болса, тарифтік жоспарлар жаңартылмайды.

1. Егер тарифтік жоспарларды қосқан немесе жойған болсаңыз, брокер директориясындағы `resources/plan_mapping.json` файлын өңдеңіз.

   {cut(resources/plan_mapping.json файлының құрылымы)}

   ```json
   {
      "<ИМЯ_СЕРВИСА>": {
         "<ID_ПЛАНА>": <SAAS_ID_ПЛАНА>,
         "<ID_ПЛАНА>": <SAAS_ID_ПЛАНА>
      }
   }
   ```

   Мұнда:

   * `<ИМЯ_СЕРВИСА>` — сервис атауы.
   * `<ID_ПЛАНА>` — `catalog_<ИМЯ_СЕРВИСА>.json` файлында көрсетілген тарифтік жоспарлардың ID-лері.
   * `<SAAS_ID_ПЛАНА>` (`string` пішімі) — сервис жағындағы `<ID_ПЛАНА>` тарифтік жоспарларының ID-леріне сәйкес келетін тарифтік жоспарлардың ID-лері.

   `plan_mapping.json` файлы мазмұнының мысалы:

   ```json
   {
     "VKT": {
       "0dc54b75-XXXX-89d5a47eef71": 1,
       "312b628e-XXXX-080ab4ca3acb": 2,
       "e2ae1588-XXXX-3f5081a799a9": 3,
       "b54e7247-XXXX-7a9dccc22e7b": 4
     }
   }
   ```

   {/cut}

   Брокер Marketplace-ке сервис конфигурацияларының тізімін мерзімді түрде беріп отыратындықтан, жаңа конфигурация Marketplace-тің тестілік namespace-іне автоматты түрде түседі.

   {note:err}

   Marketplace сервистің ағымдағы конфигурациясын уақтылы ала алуы үшін брокер тәулік бойы қолжетімді болуы тиіс.

   {/note}

1. Marketplace-тің тестілік namespace-інде сервистің жаңа ревизиясын тестілеңіз.

   {include(/kz/_includes/_saas_add_update.md)[tags=test_service]}

1. Қажет болса, сервистің жаңа ревизиясының конфигурациясына өзгерістер енгізіңіз:

   1. Брокер директориясының `resources` ішкі директориясына өтіңіз.
   1. `catalog_<ИМЯ_СЕРВИСА>.json` және `plan_mapping.json` файлдарын өңдеңіз.
   1. Тестілеуді тағы бір рет қайталаңыз.

1. Егер тестілеу және жөндеу уақытына жаңа ревизияда құнның тестілік мәндерін көрсеткен болсаңыз, `catalog_<ИМЯ_СЕРВИСА>.json` файлын өңдеп, онда нақты мәндерді көрсетіңіз.

1. Сервистің жаңа ревизиясын Marketplace-тің ашық namespace-інде жариялаңыз. Ол үшін [marketplace@cloud.vk.com](mailto:marketplace@cloud.vk.com) мекенжайына хат жіберіңіз. Хатта сервис ID-ін және оның жаңа ревизиясын көрсетіңіз.

   Сервис конфигурациясына модерация жүргізіледі. Осыдан кейін сервистің жаңа ревизиясы Marketplace-тің тестілік namespace-інен сервистің конфигурациясының [JSON-файлындағы](../../reference/saas-apps-reference/saas-param) `metadata` параметрінде көрсетілген ашық namespace-ке ауыстырылады. Сервистің жаңа ревизиясы осы ашық namespace-ке қолжетімділігі бар барлық пайдаланушыларға қолжетімді болады.

   {note:warn}

   Ашық namespace-терде тек соңғы жарияланған сервис ревизиясы ғана қолжетімді.

   {/note}

   Алдыңғы жарияланған ревизия Marketplace-тің тестілік namespace-іне ауысады. Тестілік namespace-терде сервис конфигурациясының JSON-файлында сипатталған барлық жарияланбаған ревизиялар сақталады.

   Сервис конфигурациясынан жойылған сервис ревизиялары Marketplace-тің тестілік те, ашық та namespace-терінде қолжетімді болмайды.

   Жаңа ревизияны жариялау сервистің бұрыннан бар инстанстарына (тенанттарына/аккаунттарына) әсер етпейді. Бұл сервис инстанстарының ревизиясы, тіпті ол сервис конфигурациясынан жойылып, Marketplace-те енді қолжетімсіз болса да, өзгермейді.
