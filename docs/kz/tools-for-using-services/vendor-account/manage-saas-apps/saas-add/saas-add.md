Marketplace сервисі мен SaaS-қосымша [SaaS-брокер](/kz/tools-for-using-services/vendor-account/manage-apps/concepts/about#xaas_vendor_index_type_services) арқылы VK OSB протоколы бойынша өзара әрекеттеседі. Протоколды {var(cloud)} платформасы ұсынады.

SaaS-брокерге қойылатын талаптар:

* Брокер сервис инстанстарының өмірлік циклін сипаттайтын әдістерді іске асыруы тиіс. Әдістер сызбада көрсетілген.

    {cut(Брокердің Marketplace және SaaS-қосымшамен өзара әрекеттесу сызбасы)}

    ![](../assets/SaaS_broker.png){params[noBorder=true]}

    {/cut}

* Брокерде сенімді сертификаттау орталығы берген сертификаты бар қауіпсіз HTTPS протоколы бойынша аутентификация бапталуы тиіс.

    {note:err}

    Өздігінен қол қойылған сертификаттарды пайдалануға тыйым салынады.

    {/note}

SaaS-қосымшаның жеткізушісі SaaS-брокерді толықтай өздігінен немесе {var(cloud)} платформасы ұсынатын қалып негізінде әзірлейді. Қалыпта дүкен мен брокер арасындағы өзара әрекеттесу әлдеқашан іске асырылған. Оны брокердің нақты SaaS-қосымшамен өзара әрекеттесу сипаттамасымен толықтыру қажет. Қалып Python тілінде [FastAPI](https://fastapi.tiangolo.com/tutorial/first-steps) фреймворкі негізінде әзірленген.

Бұл нұсқаулық {var(cloud)} ұсынған қалып негізінде SaaS-брокерді әзірлеу жағдайына арналған.

## {heading(3. Сервис конфигурациясын брокерге жүктеңіз)[id=saas_load_configuration_to_broker]}

1. Брокер бар директорияға өтіңіз.
1. Сервис конфигурациясы бар `catalog_<ИМЯ_СЕРВИСА>.json` файлын `resources` ішкі директориясына орналастырыңыз.
1. `resources/plan_mapping.json` файлында мыналарды көрсетіңіз:

   * `<ИМЯ_СЕРВИСА>` — сервис атауы.
   * `<ID_ПЛАНА>` — `catalog_<ИМЯ_СЕРВИСА>.json` файлында көрсетілген тарифтік жоспарлардың ID-лері.
   * `<SAAS_ID_ПЛАНА>` (`string` пішімі) — сервис жағындағы `<ID_ПЛАНА>` тарифтік жоспарларының ID-леріне сәйкес келетін тарифтік жоспарлардың ID-лері.

      ```json
      {
        "<ИМЯ_СЕРВИСА>": {
          "<ID_ПЛАНА>": <SAAS_ID_ПЛАНА>,
          "<ID_ПЛАНА>": <SAAS_ID_ПЛАНА>
        }
      }
      ```

   `plan_mapping.json` файлы мазмұнының мысалы:

   ```json
   {
     "VKT": {
       "0dc54b75-XXXX-89d5a47eef71": 1,
       "312b628e-XXXX-080ab4ca3acb": 2,
       "e2ae1588-XXXX-3f5081a799a9": 3,
       "b54e7247-XXXX-7a9dccc22e7b": 4
     }
   }
   ```

1. Брокер бар директорияда `.env.example` файлын `.env` деп қайта атаңыз.
1. `.env` файлында `BROKER_MODE` ішінде сервис атауын көрсетіңіз:

   ```json
   BROKER_MODE=<ИМЯ_СЕРВИСА>
   ```

## 4. (Опционалды) Брокерді жергілікті түрде тестілеңіз

1. Брокер директориясындағы `.env` файлында орта айнымалыларын сипаттаңыз:

   * `BROKER_PROVIDER_CLIENT_ID` — SaaS-қосымшаға қол жеткізуге арналған брокер атауы.
   * `BROKER_PROVIDER_SECRET` — брокердің SaaS-қосымшаға қол жеткізу паролі.
   * `BROKER_PROVIDER_URL` — SaaS-қосымша URL-і.
   * `BROKER_USERNAME` — брокермен сервистераралық өзара әрекеттесу үшін дүкен атауы.
   * `BROKER_PASSWORD` — брокермен сервистераралық өзара әрекеттесу үшін дүкен паролі.

1. Брокер директориясындағы `.env` файлында брокердің ДБ-сына қол жеткізуге арналған айнымалылардың мәндерін көрсетіңіз.
1. Python кітапханаларын келесі пәрменді орындап орнатыңыз:

   {tabs}

   {tab(Linux)}

   ```console
   pip install -r requirements/prod.txt
   ```

   {/tab}

   {/tabs}

1. Келесі пәрменді орындап брокерді іске қосыңыз:

   {tabs}

   {tab(Linux)}

   ```console
   gunicorn app.main:app --log-file - --workers ${UVICORN_WORKERS:-1} --worker-class uvicorn.workers.UvicornWorker --bind 0.0.0.0:${BROKER_PORT:-8000} --timeout ${WORKER_TIMEOUT:-90}
   ```

   {/tab}

   {/tabs}

   Брокерді іске қосу пәрменіне жауап мысалы:

   ```console
   INFO:     Started server process [34934]
   INFO:     Waiting for application startup.
   INFO:     Application startup complete.
   INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
   ```

1. Брокер орнатылғаннан кейін браузерде [http://0.0.0.0:8000/docs](http://0.0.0.0:8000/docs) бетін ашыңыз.

   Бұл бетте Swagger редакторындағы брокер API-інің сипаттамасы бар.

1. API сұрауларын келесі параметрлерді пайдаланып орындаңыз:

   * VK OSB протоколының `x-broker-api-version` нұсқасының мәні (мысалы, `"0.1"`).
   * `.env` файлындағы `BROKER_USERNAME` мәні.
   * `.env` файлындағы `BROKER_PASSWORD` мәні.

## {heading(7. Дүкенде сервисті тестілеңіз)[id=saas_test_marketplace]}

Сервистің {var(cloud)} ішінде қалай жұмыс істейтінін тексеру үшін оны дүкеннің тестілік namespace-інде тестілеңіз:

1. Сервисті тестілеу үшін берілетін бонустарды алу мақсатында [marketplace@cloud.vk.com](mailto:marketplace@cloud.vk.com) мекенжайына хат жіберіңіз. Қаражат {var(cloud)} ішіндегі жобаңыздың бонустық шотына есептеледі.
1. Егер сервис конфигурациясында көрсетілген тестілік namespace-терге қолжетімділігіңіз болмаса, [marketplace@cloud.vk.com](mailto:marketplace@cloud.vk.com) мекенжайына хат жіберіңіз. Хатта мыналарды көрсетіңіз:

   * Сервис конфигурациясының [JSON-файлындағы](../../reference/saas-apps-reference/saas-param) `metadata` параметрінде берілген Marketplace-тің тестілік namespace атаулары.
   * Marketplace-тің тестілік namespace-теріне қолжетімділік қажет пайдаланушының электрондық поштасы.

{include(/kz/_includes/_saas_add_update.md)[tags=test_service]}

1. Қажет болса, сервис конфигурациясына өзгерістер енгізіңіз:

   1. Брокер бар директорияның `resources` ішкі директориясына өтіңіз.
   1. `catalog_<ИМЯ_СЕРВИСА>.json` және `plan_mapping.json` файлдарын өңдеңіз.
   1. Тестілеуді тағы бір рет қайталаңыз.

1. Егер тестілеу мен жөндеу уақытына сервис конфигурациясында құнның тестілік мәндерін көрсеткен болсаңыз, `catalog_<ИМЯ_СЕРВИСА>.json` файлын өңдеп, онда нақты мәндерді көрсетіңіз.

## {heading(8. Сервисті жариялаңыз)[id=saas_publish_service]}

[marketplace@cloud.vk.com](mailto:marketplace@cloud.vk.com) мекенжайына хат жіберіңіз. Хатта сервис ID-ін және оның ревизиясын көрсетіңіз.

Сервис конфигурациясына модерация жүргізіледі. Осыдан кейін сервис дүкенде жарияланады.

Жарияланатын сервис ревизиясы дүкеннің тестілік namespace-інен `metadata` параметрінде көрсетілген [сервис конфигурациясының JSON-файлындағы](../../reference/saas-apps-reference/saas-param) ашық namespace-ке ауыстырылады. Сервис осы ашық namespace-ке қолжетімділігі бар барлық пайдаланушыларға қолжетімді болады.

{note:warn}

Ашық namespace-терде тек соңғы жарияланған сервис ревизиясы ғана қолжетімді.

{/note}
