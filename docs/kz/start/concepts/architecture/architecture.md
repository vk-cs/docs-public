VK Cloud — бұл «pay as you go» қағидаты бойынша есептеу ресурстарына (виртуалды машиналарға, дерекқорларға, сақтау жүйелеріне және басқа сервистерге) желілік қолжетімділікті ұсынатын бұлттық платформа: ақы тек тұтынылған ресурстар үшін алынады.

Сіз VK [деректер орталықтарында](#az) орналастырылған виртуалды ресурстарды жалға аласыз. Бұл өзіңіздің физикалық серверлеріңізді сатып алуға және оларға қызмет көрсетуге шығынданбауға мүмкіндік береді, сондай-ақ VK Cloud техникалық қолдауына қол жеткізуді қамтамасыз етеді.

## Орналастыру модельдері

VK Cloud келесі орналастыру модельдерін ұсынады:

- **Қоғамдық бұлт (Public Cloud)** — ортақ ресурстары бар (серверлер, сақтау жүйелері, желілер) көппайдаланушылық платформа түріндегі бұлтты орналастыру моделі. Пайдаланушылардың деректері виртуализация механизмі арқылы бір-бірінен оқшауланады, бірақ физикалық тұрғыдан бір серверлерде орналасуы мүмкін. VK Cloud инфрақұрылымды, соның ішінде деректер орталықтарына қызмет көрсетуді, жабдық сатып алуды, істен шығуға төзімділікті қамтамасыз етуді және желіні басқаруды жүзеге асырады. 

   Артықшылықтары: ауқымдалғыштық, нақты пайдалану бойынша төлем, жаңа технологияларға қол жеткізу.

- **Жеке бұлт (Private Cloud)** — ресурстар әр клиентке (әдетте, ұйымдарға) бөлек бөлінетін орналастыру моделі. Бір ұйымның қызметкерлері оның инфрақұрылымын бірлесіп пайдалана алады, бірақ әртүрлі ұйымдардың физикалық және виртуалды ресурстарды ортақ пайдалануы болмайды. Жеке бұлтта негізгі инфрақұрылымды дәл клиенттің өзі (ұйым) басқарады және оған қызмет көрсетеді.

   Артықшылықтары: толық оқшаулау және деректерді бақылау, жоғары қауіпсіздік. VK ұсынатын Private Cloud туралы толығырақ [Жеке инсталляциялар](/kz/on-premises) порталынан қараңыз.

## Бұлттық қызметтерді ұсыну модельдері

VK Cloud-та виртуализация, мониторинг, контейнерлер және деректерді сақтау сервистері орналастырылуы мүмкін. архитектура сервистердің төрт тобына негізделген:

- **Infrastructure as a Service (IaaS)**. IaaS-сервистері [OpenStack](https://www.openstack.org/software) және меншікті әзірлеме компоненттері негізінде құрылған. Ресурстарды динамикалық бөлуді, ауқымдауды және істен шығуға төзімділікті басқарады. PaaS-сервистерімен бір бұлттық деректер орталығында орналастырылған. IaaS базалық компоненттерді ұсынады: виртуалды серверлер, желі, деректерді сақтау жүйелері, бөлінген жабдыққа қол жеткізу.

   {cut(IaaS-сервистер тізімі)}

  - [Cloud Servers](/kz/computing/iaas) — жария IP-адрестерді пайдалану мүмкіндігі бар виртуалды машиналарды және шектеусіз трафикпен 1 Гбит/с байланыс арнасын ұсынады (NVIDIA GPU бар виртуалды машиналар да қолжетімді). Ресурстарды виртуализациялау мен интеграциялау үшін жауап береді, KVM гипервизорын пайдаланады.
  - [VK Object Storage](/kz/storage/s3) — S3-пен үйлесімді объектілік сақтау жүйесін ұсынады. Сақтау жүйесіне түсетін жұмыс жүктемесі барлық сақтау түйіндері арасында бөлінеді. Бапталатын архитектурасы бар сақтау жүйесін орналастыруға болады.
  - [Cloud Networks](/kz/networks/vnet) — SDN (Software Defined Network) технологиясының көмегімен таңдалған [жоба](/kz/tools-for-using-services/account/concepts/projects) аясында желілік өзара әрекеттесуді қамтамасыз етеді. OpenStack Neutron және меншікті әзірленген Sprut БҚ негізінде жұмыс істейді. Құрамына келесі компоненттер кіреді:

    - [DNS](/kz/networks/dns) — VK Cloud платформасының сервистері үшін атауларды шешуді қамтамасыз ететін жария және жеке DNS-ті қолдайды.
    - [CDN](/kz/networks/cdn) — контентті серверлеріңізден пайдаланушыларға ең аз кідіріспен жеткізуді ұйымдастырады.
    - [Load Balancer](/kz/networks/balancing/concepts/load-balancer) — қолданбалардың істен шығуға төзімділігін және икемді ауқымдалуын қамтамасыз ете отырып, инфрақұрылымға түсетін жүктемені бөледі.
    - [AntiDDoS](/kz/security/firewall-and-protection-ddos#antiddos) — DDoS-шабуылдарды бұғаттау үшін жобада орналастырылған VK Cloud ресурстарына келетін трафикті сүзгіден өткізеді.
    - [WAF](/kz/security/firewall-and-protection-ddos#waf) (Web Application Firewall) — желілік шабуылдарды анықтау және бұғаттау үшін кіріс және шығыс трафикті сүзгілеу ережелерін баптайды.

  {/cut}

- **Platform as a Service (PaaS)**. Ашық бастапқы коды бар шешімдерді (Kubernetes) және технологиялық серіктестердің шешімдерін қамтиды. VK Cloud сервистері мен жекелеген мәндері үшін кірістірілген мониторинг қолдау табады. PaaS моделі IaaS үстінен құрылады. Ол жабдық пен операциялық жүйелерді қамтитын базалық инфрақұрылымды басқаруды талап етпейді және барлық күш-жігерді қолданбаларды әзірлеуге және басқаруға жұмылдыруға мүмкіндік береді. 

  {cut(PaaS-сервистер тізімі)}

  - [Cloud Containers](/kz/kubernetes/k8s) — сервистер мен қолданбаларды іске қосуға болатын Kubernetes кластерлерін құруға және басқаруға мүмкіндік береді.
  - [Cloud Databases](/kz/dbs/dbaas) — ауқымдалатын СУБД ұсынады: MySQL, PostgreSQL, Postgres Pro, ClickHouse, MongoDB, Redis, Tarantool, OpenSearch, Greenplum негізіндегі Arenadata DB.
  - [Cloud Spark](/kz/ml/spark-to-k8s) — деректерді үлестірілген пакеттік және ағындық өңдеуді, Machine Learning-пен және аналитикамен жұмысты қамтамасыз етеді.
  - [Cloud ML Platform](/kz/ml/mlplatform) — ML-әзірлеудің толық цикліне арналған сервистерді қолдайды.
  - [Cloud Voice](/kz/ml/cloud-voice) — машиналық оқыту негізінде сөйлеуді тану және синтездеу үшін REST API ұсынады.
  - [Vision](/kz/ml/vision) — машиналық оқыту негізінде беттер мен объектілерді тану үшін REST API ұсынады.
  - [Cloud Alerting](/kz/monitoring-services/alerting) —  VK Cloud сервистерінің негізгі метрикалары өзгергені туралы хабарландыруларды баптайды.
  - [Cloud Logging](/kz/monitoring-services/logging) — VK Cloud-тағы сервистер журналдарын агрегаттайды және талдайды.
  - [Cloud Monitoring](/kz/monitoring-services/monitoring) — PaaS-сервистеріне тән метрикалардың мониторингін қамтамасыз етеді, мысалы, K8s-контейнерлерінің подтары бойынша аналитика, PostgreSQL СУБД транзакцияларының статистикасы.
  - [Cloud Desktop](/kz/computing/cloud-desktops/concepts/about) — әлемнің кез келген нүктесінен қолжетімді басқарылатын виртуалды жұмыс орындары.
  - [Cloud Kafka](/kz/data-platform/kafka/concepts/about) — АТ-жүйелердің әртүрлі модульдері арасында нақты уақыт режимінде деректер алмасуды қамтамасыз етеді.
  - [Cloud Flink](/kz/data-platform/flink/concepts/about) — көздерден мақсатты жүйелерге деректерді ағындық өңдеу және беру.
  - [Cloud Trino](/kz/data-platform/trino/concepts/about) — қашықтағы көздерден үлкен деректерді өңдеуге арналған SQL-қозғалтқышы.
  - [Marketplace](/kz/applications-and-services/marketplace) — виртуалды машиналар негізінде веб-әзірлеу және әкімшілендіру орталарын жылдам орналастыруға мүмкіндік береді. OpenVPN, VK Testers, Redash, Grafana, Milvus, Nexus, Joomla, WordPress және басқа да қолданбаларды қамтиды.

  {/cut}

- **Software as a Service (SaaS)**. Интернет арқылы жазылым бойынша қолжетімді дайын БҚ. Инфрақұрылымды немесе платформаны басқаруды талап етпейді. VK Cloud-тағы SaaS-сервисінің мысалы — [1С:Готовое рабочее место](/kz/applications-and-services/1cgrm). Сервис [1С сервистерін](/kz/applications-and-services/1cgrm/about#description) орналастыру үшін ресурстар мен БҚ ұсынады.

- **Common&Security-сервистері** пайдаланушылардың қауіпсіз жұмысын қамтамасыз етеді және VK Cloud ресурстарын пайдалану кезінде рөлдік модельді қолдайды.

  {cut(Common&Security-сервистер тізімі)}

  - [Billing](/kz/intro/billing) — ресурстарды пайдалану есебін және шығындарды бақылауды жүргізеді, қаржылық есептерді қалыптастырады, сервистердің ақысын төлеу кезінде төлем жүйелерімен өзара әрекеттесуді қамтамасыз етеді.
  - [Cloud Audit](/kz/monitoring-services/event-log) — VK Cloud-тағы пайдаланушылар әрекеттерінің аудит журналын қалыптастырады.
  - IAM — Keystone-мен бірге пайдаланушылар мен сервистердің аутентификациясы мен авторизациясын басқарады.
  - Monitoring — бұлттық сервистер мен пайдаланушы қолданбаларының мониторингін қамтамасыз етеді.
  - Keystone — API арқылы клиенттің аутентификациясын, қызметтерді анықтауды және үлестірілген мультитенантты авторизацияны қамтамасыз етеді.

  {/cut}

## VK Cloud сервистерін басқару тәсілдері

Бұлттық сервистер API көмегімен басқарылады. Пайдаланушылар API-мен бір немесе бірнеше [құрал](/kz/tools-for-using-services) арқылы өзара әрекеттеседі:

  - VK Cloud жеке кабинеті;
  - OpenStack CLI;
  - бөлек провайдері бар Terraform.

VK Cloud өз сервистері үшін [техникалық қолдау](/kz/start/support/support-info) ұсынады. Серіктестік шешімдерге (мысалы, AntiDDoS, WAF) қолдау серіктеспен бірлесіп көрсетіледі.

## Істен шығуға төзімділік

Физикалық істен шығуға төзімділік VK Cloud-ты РФ аумағындағы Tier III сенімділік деңгейіндегі үш деректер орталығында орналастыру арқылы іске асырылған. Мұндай орналастыру кезінде ДЦ-лардың 98%-дан жоғары SLA-ын сақтау және жалпы [SLA](/kz/start/support/sla) 99.95% қамтамасыз етіледі.

Желілік істен шығуға төзімділік деректер орталықтарының 200 ГБ/с өткізу қабілеттілігі бар арналар арқылы байланысуымен қамтамасыз етіледі. Әр сервердің магистральдық провайдерлермен байланысы екі тәуелсіз арна арқылы, екі маршрутизатор арқылы жүзеге асырылады.

Логикалық істен шығуға төзімділік [өңірлерге](/kz/tools-for-using-services/account/concepts/regions) біріктірілген қолжетімділік аймақтарының (availability zone) көмегімен іске асырылады.

{note:info}

VK Cloud қолжетімділік аймақтарының, сервистерінің және инфрақұрылымының күйін [Сервистер жұмысының күйі](https://status.kz.cloud.vk.com) бетінде тексеруге болады.

{/note}

## Қауіпсіздік

VK Cloud қауіпсіздігі VK Cloud сервистеріне тұрақты мониторинг жүргізу және сыртқы аудиттер өткізу арқылы қамтамасыз етіледі. VK Cloud [HackerOne](https://www.hackerone.com) қауіпсіздік аудиті бағдарламасына қатысады. VK Cloud ақпараттық жүйесі дербес деректермен жұмыс істеу үшін Ресей ФСТЭК-інің 21-бұйрығы талаптарына сәйкес [аттестатталған](https://cloud.vk.com/cloud-platform/certificates).

VK Cloud-тың барлық өңірлеріндегі қолжетімділік аймақтарының инфрақұрылымы РФ-ның «Дербес деректер туралы» №152-ФЗ заңына сәйкес [қорғалған](/kz/intro/it-security/compliance-152fz).

Толық ақпарат [Платформа қауіпсіздігі](/kz/intro/it-security) бөлімінде келтірілген.

## VK Cloud-қа миграция

VK Cloud [үшінші тарап БҚ](/kz/intro/migration) немесе [платформа құралдары](/kz/intro/migration/migrate-hystax-mr) арқылы миграцияны қолдайды. Виртуалды ресурстарды көшіру OSS-шешімдері (Open Source Solutions) негізінде сервистерді іске асырудың арқасында арзанырақ болады.
