AWS CLI арқылы көлемі 100 МБ-тан асатын файлдар интернет-арнаның өткізу қабілетінен едәуір төмен жылдамдықпен жүктеледі.

Файлдарды бір TCP қосылымы арқылы жібергенде мәселе желідегі жоғары кідіріске (latency) байланысты туындауы мүмкін. Интернет-арнаның өткізу қабілетін барынша пайдалану үшін файлды бір уақытта бірнеше ағын арқылы жүктеңіз. Ол үшін [құрама жүктеуді](/kz/storage/s3/concepts/features#object_uploading) пайдаланыңыз және жағдайыңызға ең қолайлы бір уақытта жүктелетін файл бөліктерінің саны мен олардың өлшемін таңдаңыз:

- Әр бөліктің өлшемін ұлғайту API-сұраулардың жалпы санын және қызметтік трафикті азайтады, бұл трафиктің жалпы көлемін қысқартады.
- Параллель жүктелетін бөліктер санын ұлғайту файлды жүктеудің жалпы уақытын қысқартады. Алайда бұл тәсіл процессорыңыздың қуатымен және интернет-арнаның өткізу қабілетімен шектеледі.

### Шешім

1. Бакетке объектілерді AWS CLI арқылы жүктеу үшін `aws s3 cp` командасы қолданылатынына, ал `aws s3api put-object` қолданылмайтынына көз жеткізіңіз. `aws s3 cp` командасы өлшемі `multipart_threshold` параметрінің мәнінен асатын объектілер үшін (әдепкі бойынша 8 МБ) құрама жүктеуді автоматты түрде қолданады.

1. Құрама жүктеу параметрлерін оңтайландырыңыз:

   ```console
   aws configure set default.s3.<ИМЯ_ПАРАМЕТРА> <ЗНАЧЕНИЕ>
   ```

   Мұнда:

   - `<ИМЯ_ПАРАМЕТРА>` — құрама жүктеу параметрінің атауы:
      - `multipart_threshold`: құрама жүктеу үшін қажетті файлдың ең аз өлшемі (әдепкі бойынша 8 МБ).
      - `multipart_chunksize`: құрама жүктеу кезіндегі файл бөліктерінің өлшемі (әдепкі бойынша 8 МБ).
      - `max_concurrent_requests`: параллель жүктелетін файл бөліктерінің саны (әдепкі бойынша 10).
   - `<ЗНАЧЕНИЕ>` — құрама жүктеу параметрінің жаңа мәні.

   {cut(Командалар мысалдары)}

   ```console
   aws configure set default.s3.multipart_chunksize 16MB
   ```

   ```console
   aws configure set default.s3.max_concurrent_requests 20
   ```

   {/cut}

   {note:warn}
   Объект бөліктерінің ең көп саны — 10000. `multipart_chunksize` параметрінің мәнін таңдағанда, ең үлкен файл үшін бөліктердің жалпы саны осы шектеуден аспайтынына көз жеткізіңіз.
   {/note}
