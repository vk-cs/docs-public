# {heading(AWS CLI арқылы үлкен файлдарды баяу жүктеу)[id=s3-slow-load-problem]}

{include(/kz/_includes/_translated_by_ai.md)}

AWS CLI арқылы өлшемі 100 МБ-тан үлкен файлдарды жүктеу жылдамдығы интернет-арнаның өткізу қабілетінен едәуір төмен болады.

Файлдарды бір TCP-қосылым арқылы жіберген кезде мәселе желінің жоғары кідірісіне (latency) байланысты туындауы мүмкін. Интернет-арнаның өткізу қабілетін барынша пайдалану үшін файлды бір уақытта бірнеше ағынмен жүктеңіз. Ол үшін {linkto(../../concepts/features#s3-concepts-features-object-uploading)[text=құрамдас жүктеуді]} пайдаланып, жағдайыңызға ең қолайлы түрде бір уақытта жүктелетін файл бөліктерінің санын және олардың өлшемін таңдаңыз:

- Әр бөліктің өлшемін ұлғайту API сұрауларының жалпы санын және қызметтік трафикті азайтады, бұл трафиктің жалпы көлемін қысқартады.
- Параллель жүктелетін бөліктер санын көбейту файлды жүктеудің жалпы уақытын қысқартады. Алайда бұл тәсіл процессорыңыздың қуатымен және интернет-арнаның өткізу қабілетімен шектеледі.

### {heading(Шешімі)[id=s3-slow-load-problem-resolve]}

1. AWS CLI арқылы бакетке объектілерді жүктеу үшін `aws s3api put-object` емес, `aws s3 cp` командасы қолданылатынына көз жеткізіңіз. `aws s3 cp` командасы өлшемі `multipart_threshold` параметрінің мәнінен (әдепкі бойынша 8 МБ) асатын объектілер үшін құрамдас жүктеуді автоматты түрде қолданады.

1. Құрамдас жүктеу параметрлерін оңтайландырыңыз:

   ```console
   aws configure set default.s3.<ИМЯ_ПАРАМЕТРА> <ЗНАЧЕНИЕ>
   ```

   Мұнда:

   - `<ИМЯ_ПАРАМЕТРА>` — құрамдас жүктеу параметрінің атауы:

     - `multipart_threshold`: құрамдас жүктеу үшін қажет файлдың ең кіші өлшемі (әдепкі бойынша 8 МБ).
     - `multipart_chunksize`: құрамдас жүктеу кезіндегі файл бөліктерінің өлшемі (әдепкі бойынша 8 МБ).
     - `max_concurrent_requests`: параллель жүктелетін файл бөліктерінің саны (әдепкі бойынша 10).

   - `<ЗНАЧЕНИЕ>` — құрамдас жүктеу параметрінің жаңа мәні.

   {cut(Командалар мысалдары)}

   ```console
   aws configure set default.s3.multipart_chunksize 16MB
   ```
   
   ```console
   aws configure set default.s3.max_concurrent_requests 20
   ```

   {/cut}

   {note:warn}
   Объект бөліктерінің ең көп саны — 10000. `multipart_chunksize` параметрінің мәнін таңдағанда, ең үлкен файл үшін бөліктердің жалпы саны осы шектеуден аспайтынына көз жеткізіңіз.
   {/note}
