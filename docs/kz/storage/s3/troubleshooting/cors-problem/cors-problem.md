Веб-қосымша VK Object Storage бакетіне файл жүктеуге тырысады, бірақ сұрауды браузер бұғаттайды. Әзірлеуші консолінде [CORS](/kz/storage/s3/concepts/access/s3-cors) ережелеріне қатысты қате көрсетіледі, мысалы, `Access to XMLHttpRequest at '...' from origin '...' has been blocked by CORS policy`.

Мәселе веб-браузердің Same-Origin Policy (SOP) қауіпсіздік механизміне байланысты туындайды. Бұл механизм бір көздегі (origins) скриптке басқа көзге сұрау жасауға тыйым салады, егер бұған анық түрде рұқсат берілмесе. Мұндай өзара әрекеттесуге рұқсат беру үшін VK Object Storage арнайы HTTP тақырыптарын қайтаруы тиіс.

### Шешім

[CORS](/kz/storage/s3/concepts/access/s3-cors) ережелерін тексеріп, түзетіңіз:

1. [Өтіңіз](https://kz.cloud.vk.com/app) VK Cloud жеке кабинетіне.

1. Қажетті бакет орналасқан жобаны таңдаңыз.

1. **Объектілік қойма → Бакеттер** бөліміне өтіңіз.

1. Қажетті бакет үшін ![Мәзір](/kz/assets/more-icon.svg "inline") батырмасын басып, **Баптаулар** тармағын таңдаңыз.

1. **CORS** қойындысына өтіңіз.

1. [CORS ережесін қосыңыз немесе өңдеңіз](/kz/storage/s3/instructions/access-management/cors):

   - `AllowedOrigins`: веб-қосымшаңыздың доменін көрсетіңіз. `*` таңбасын пайдалану кез келген доменнен қол жеткізуге рұқсат береді.
   - `AllowedMethods`: веб-қосымшаңыз жүктеу үшін пайдаланатын HTTP әдісін көрсетіңіз, мысалы, `PUT` немесе `POST`.
   - `AllowedHeaders`: веб-қосымшадан келетін сұраулардың рұқсат етілген тақырыптарын көрсетіңіз, мысалы, `Content-Type`, `Authorization`, `x-amz-acl`. `*` таңбасын пайдалану кез келген тақырыптары бар сұрауларға рұқсат береді.
