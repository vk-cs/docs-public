VK Object Storage — S3 протоколына негізделген [деректердің объектілік қоймасы](/kz/computing/iaas/concepts/data-storage/storage-types). Ол сирек жүгінетін үлкен көлемді деректерді сақтауға қолайлы. VK Object Storage-пен VK Cloud жеке кабинетінде, AWS CLI пәрмен жолы интерфейсі арқылы, сондай-ақ [басқа құралдардың](#instruments) көмегімен жұмыс істеуге болады.

VK Object Storage келесі ерекшеліктерге ие:

- жоғары ауқымдалғыштық пен қолжетімділік;
- қолданбалармен ықпалдастырудың қарапайымдылығы;
- VK Cloud ішіндегі басқа [қойма түрлерімен](/kz/computing/iaas/concepts/data-storage/storage-types) салыстырғанда деректерді сақтау құнының төмендігі.

VK Object Storage ішіндегі деректер [бакеттерде](#bucket) тәуелсіз [объектілер](#object) және олардың [метадеректері](#metadata) түрінде сақталады.

## {heading(Бакет)[id=bucket]}

*Бакет* — [объектілерді](#object) сақтауға арналған контейнер. Бакетке кез келген мөлшерде объект жүктеуге болады. Бакеттегі объектілердің жалпы көлемі де шектелмейді. Бакетті жасаған пайдаланушы бакетке және ондағы объектілерге толық қол жеткізу құқықтарын алады және бұл құқықтарды басқалармен бөлісе алады.

Әр бакеттің оның ішіндегі объектілерге қол жеткізу үшін пайдаланылатын атауы бар. Атау белгілі бір [талаптарға](#bucket_naming) сай болуы керек. Бакет жасалғаннан кейін оның атауын өзгертуге болмайды.

Жобада [бірнеше бакет](/kz/tools-for-using-services/account/concepts/quotasandlimits#object_storage_limits) жасауға болады, бірақ бір бакетті екіншісінің ішіне кірістіруге болмайды. Әр бакеттің ішінде директориялар иерархиясын құруға болады. Ол үшін объект атауларына [префикстер](#object_key) қосу керек, оларды VK Object Storage директория атаулары ретінде түсіндіреді.

Бакетті статикалық сайттарды хостингтеу үшін, [оған өз доменіңізді байланыстыру арқылы](../../instructions/buckets/manage-bucket#bind_domain), сондай-ақ [CDN сервисінде дереккөз ретінде](../../instructions/buckets/manage-bucket#use_cdn) пайдалануға болады.

VK Object Storage ішінде мыналарға қолдау көрсетіледі:

- бакеттен объектілерді берілген ережелер бойынша [автоматты түрде жою](/kz/storage/s3/concepts/lifecycle);
- бакеттегі оқиғалар туралы [хабарландыруларды баптау](../features#bucket_event_notifications);
- бакет мазмұны мен жергілікті директорияны [синхрондау](../../instructions/buckets/manage-bucket#sync_local_directory_and_bucket);
-  бакеттегі ресурстарға [CORS](../../instructions/access-management/cors) технологиясы бойынша кросс-домендік сұраулар.

### {heading(Бакеттерді атау ережелері)[id=bucket_naming]}

Бакет атауы:

- бүкіл VK Object Storage сервисі бойынша бірегей болуы керек.
- 4-тен 63-ке дейін таңбадан тұруы керек.
- әріппен немесе цифрмен басталып, аяқталуы керек.

Бакет атауы мынандай болмауы керек:

- VK Object Storage сервисінің басқа бакеттерінің атауымен сәйкес келмеуі керек.
- `xn--` префиксінен басталмауы керек.
- `-ol-s3` немесе `s3alias` суффиксімен аяқталмауы керек.
- `-` және `.` таңбаларымен басталмауы немесе аяқталмауы керек.
- қатарынан екі нүктені `..`, дефис пен нүктені `-.`, нүкте мен дефисті `.-` қамтымауы керек.
- тек кіші латын әріптерінен, цифрлардан және мына таңбалардан тұруы керек: `.`, `-`.

{note:warn}

  Мысалдар: `my..bucket`, `my-.bucket`, `my.-bucket`.

- IP мекенжайы пішімінде немесе соған ұқсас пішімде болмауы керек, мысалы `192.168.5.4`.

{/note}

  Бұл шектеу веб-сайттарды статикалық хостингтеу үшін қолданылатын сегменттерге әсер етпейді, өйткені веб-сайттардың статикалық хостингі тек HTTP арқылы қолжетімді.

Бакет атауы ондағы объектілердің URL мекенжайларында болады, сондықтан атауда құпия ақпаратты пайдаланбаңыз. Егер объектіге жария қолжетімділік берсеңіз, ол объектіге сілтемеде көрсетіледі.

{cut(Ұсынылған ережелерге сәйкес келетін бакет атауларының мысалдары)}

- `docexamplebucket1`
- `log-delivery-march-2020`
- `my-hosted-content`

{/cut}

{cut(Жарамсыз бакет атауларының мысалдары)}

- `doc_example_bucket` (астын сызу таңбаларын қамтиды)
- `DocExampleBucket` (бас әріптерді қамтиды)
- `doc-example-bucket-` (дефиспен аяқталады)

{/cut}

## {heading(Объект)[id=object]}

*Объект* — қоймаға жүктелген файл және оның [метадеректері](#metadata). Файл кез келген форматта болуы мүмкін, мысалы: медиафайл, резервтік көшірме, BigData деректер массиві.

VK Object Storage ішіндегі объектілер [бакеттерде](#bucket) сақталады. Файл өлшеміне байланысты объектіні бакетке [стандартты немесе құрамдас жүктеу](../features#object_uploading) қолданылады.

Объект жүктелген кезде оған мыналар тағайындалады:

- [объект кілті](#object_key) — объектіге қол жеткізілетін арнайы идентификатор;
- объект қасиеттерін сипаттайтын [метадеректер](#metadata);
- [қолжетімділік деңгейі](../../instructions/objects/manage-object#manage_access);
- [сақтау класы](#storage_class).

VK Object Storage ішінде объектілерді жүктеу, жүктеп алу, көшіру және жою операцияларымен қатар мыналарға қолдау көрсетіледі:

- объект үшін [қолжетімділік деңгейін өзгерту](../../instructions/objects/manage-object#manage_access);
- объектіге [қолтаңбаланған сілтеме](../../instructions/access-management/signed-url) арқылы қол жеткізу беру;
- объектіні жоюдан немесе қайта жазудан уақытша және тұрақты [бұғаттау](/kz/storage/s3/concepts/objects-lock).

## {heading(Объект кілті)[id=object_key]}

*Объект кілті* — бұл объект атауы мен атауға қосылатын префикстердің комбинациясы. Префикстер бір-бірінен және объект атауынан `/` бөлгіші арқылы ажыратылады. Префикстер міндетті емес.

Объект атауын және префикстерді таңдағанда, төменде келтірілген ережелерді ұстану ұсынылады.

### {heading(Объект кілтін құру бойынша ұсынымдар)[id=object_key_rules]}

Объект кілті Юникод таңбаларынан тұрады. UTF-8 кодтауындағы кілт ұзындығы 1024 байттан аспауы керек.

Объект кілтінде кез келген Юникод таңбасын пайдалана аласыз. Алайда кейбір таңбалар бірқатар қолданбалар мен веб-протоколдарда проблемалар тудыруы мүмкін. Мәселелердің алдын алу үшін мына ұсынымдарды орындаңыз:  

- Объект кілтінде тек латын әріптерін, цифрларды және `/`, `!`, `-`, `_`, `.`, `*`, `'`, `(`, `)` арнайы таңбаларын пайдалану ұсынылады. `/` таңбасы префикстер мен объект атауын бір-бірінен ажыратады.
- Объект кілті:

  - қиғаш сызықпен `/` аяқталмауы керек;
  - қатарынан екі нүктені `..` қамтымауы керек;
  - тек бір нүктеден `.` ғана тұрмауы керек;
  - тек бос орыннан тұрмауы немесе басында не соңында бос орындар болмауы керек.

- Объект кілтінде төменде келтірілген таңбаларды пайдалану ұсынылмайды. Олардың кейбіреулері URL мекенжайларында қолданылғанда кодтауды қажет етеді, ал кейбіреулері басып шығарылмайтын таңбалар болып табылады және браузеріңізде өңделмеуі мүмкін. Кілтінде осындай таңбалары бар объектіні дұрыс адрестеуді қамтамасыз ету үшін кодта объект кілтін қосымша өңдеу қажет.

  Қажетсіз таңбалар:

  - `$`, `&`, `@`, `=`, `;`, `:`, `+`, `\`, `%`, `,`, `?`, `#` арнайы таңбалары;
  - ASCII-кодтары `00h`–`1Fh` (ондық кодтары `0`–`31`) болатын таңбалар;
  - ASCII-коды `7Fh` (ондық коды `127`) болатын `DEL` таңбасы;
  - бос орын, өйткені кейбір жағдайларда, әсіресе бірнеше бос орын болғанда, бос орындардың маңызды тізбектері жоғалуы мүмкін;
  - Юникодтың барлық көрінбейтін таңбалары.

{cut(Ұсынымдарға сәйкес келетін объект кілттерінің мысалдары)}

- `Development/Projects.xls`
- `Finance/statement1.pdf`
- `Finance/taxdocument.pdf`
- `s3-dg.pdf`

`Development/` және `Finance/` префикстері негізінде VK Object Storage бакетте директориялар жасайды. `Development` директориясын ашқанда, оның ішінде `Projects.xlsx` объектісін көресіз, `Finance` директориясынан `statement1.pdf` және `taxdocument.pdf` объектілерін табасыз. `s3-dg.pdf` атауында префикс жоқ, сондықтан бұл объект бакеттің түбірлік деңгейінде орналастырылады.

{/cut}

## {heading(Объект метадеректері)[id=metadata]}

VK Object Storage ішіндегі әр объектінің *метадеректері* бар. Бұл объект қасиеттерін сипаттайтын кілт/мән жұптары. Сервис тағайындайтын метадеректер (мысалы, сақтау класы немесе жасалу күні) және пайдаланушылық метадеректер ажыратылады. Кейбір сервистік метадеректер пайдаланушыға басқару үшін қолжетімді, атап айтқанда объектінің сақтау класы.

Объектінің метадеректерін оны жүктеу кезінде орната аласыз. Жүктелгеннен кейін объектінің метадеректерін тікелей өзгертуге болмайды. Мұны істеудің жалғыз жолы — объектіні [көшіру](../../instructions/objects/manage-object#copy_object). Көшіру кезінде объект кілтін бір уақытта дереккөз және мақсат ретінде көрсету және көшірме үшін жаңа метадеректерді беру қажет. Осылайша сіз мыналарды өзгерте аласыз:

- пайдаланушылық метадеректерді,
- пайдаланушы басқаратын жүйелік метадеректерді, мысалы сақтау класын.

Бұл ретте сервис басқаратын метадеректер қалпына келтіріледі. Мысалы, объектінің жасалған күні қалпына келтіріледі.

## {heading(Лимиттер)[id=instruments]}

VK Object Storage объектілік қоймасымен жұмыс істеу үшін келесі құралдар қолжетімді:

- [Пәрмен жолы интерфейстері](../../connect/s3-cli): AWS CLI, s3cmd CLI.
- [Файл менеджерлері](../../connect/s3-file-managers): CyberDuck, WinSCP, Диск-О:, s3fs.
- [Әзірлеуші құралдар жинақтары](../../connect/s3-sdk): Java, .NET., Javascript, Ruby, Python (Boto), PHP, Go үшін SDK, Powershell үшін AWS құралдары, iOS және Android үшін Mobile SDK.
- [VK Object Storage API](/kz/tools-for-using-services/api/api-spec/s3-rest-api).

## {heading(Сақтау кластары)[id=storage_class]}

Сақтау кластары деректердің мақсаты мен [сақтау құны](https://cloud.vk.com/pricelist) бойынша ерекшеленеді және бакеттер, объектілер және құрамдас жүктеулер үшін қолжетімді. VK Cloud ішінде бірнеше сақтау класына қолдау көрсетіледі:

- `Hotbox` — жоғары жүктемелі сайттар, қолданбалар, медиа сервистер, онлайн-БАҚ, көппайдаланушылық контенті бар сайттар және мобильді қолданбалар үшін көп мөлшердегі файлдарды сақтау және жылдам таратуға арналған. API-де `STANDARD` атауымен белгіленеді.
- `Icebox` — сирек пайдаланылатын деректерге арналған: бэкаптар, логтар, медиаконтент, ғылыми, статистикалық деректер, сондай-ақ жұмыс архивтері. API-де `STANDARD_IA` атауымен белгіленеді.
- `BackupBucket` (тек бакеттер үшін) — автоматты түрде де, қолмен де жасалған [инстанстардың резервтік көшірмелері](/kz/storage/backups) үшін. Мұндай кластағы бакеттерді өздігінен жасауға немесе жоюға болмайды. Резервтік көшірмелер жойылғаннан кейін BackupBucket бакеті бос болып қалуы мүмкін. Ол жобада резервтік көшіру жоспарларымен әрі қарай жұмыс істеу үшін объектілік қоймаға қажет. API-де `BACKUP` атауымен белгіленеді.
