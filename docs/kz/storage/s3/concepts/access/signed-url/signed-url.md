# {heading(Қолтаңбаланған URL арқылы қолжетімділік)[id=s3-concepts-signed-url]}

{include(/kz/_includes/_translated_by_ai.md)}

Әдепкі бойынша жобадағы барлық объектілер мен бакеттер жекеше болады. Алайда алдын ала қолтаңбаланған URL-адресті жасап, басқа пайдаланушыларға бакеттен объектілерді жүктеп алуға немесе оған объектілерді жүктеуге рұқсат бере аласыз. Онда мыналар көрсетіледі:

- Қолжетімділіктің жария кілті. Мысалы, жария {linkto(../../../concepts/access/account-and-keys#s3-concepts-account-and-keys-access-keys)[text={var(s3)} аккаунтына байланыстырылған қолжетімділік кілті]}, немесе жария {linkto(../../../concepts/access/account-and-keys#s3-concepts-account-and-keys-bucket-access-keys)[text=бакет қолжетімділік кілті]}.
- Бакет атауы.
- Объект кілті.
- URL-адрестің жарамдылық мерзімі.
- Қолжетімділіктің құпия кілтімен шифрланған қолтаңба.

{note:info}
Әдепкі бойынша алдын ала қолтаңбаланған URL-адрес онда көрсетілген объектіні жүктеп алуға мүмкіндік береді. Сондай-ақ бакетке объектілерді жүктеу үшін URL-адрестерді жасай аласыз. Толығырақ — [Amazon AWS ресми құжаттамасында](https://docs.amazonaws.cn/en_us/AmazonS3/latest/userguide/PresignedUrlUploadObject.html).
{/note}

Қолтаңбаланған URL-адресі бар кез келген адам онда көрсетілген объектіге қол жеткізеді. Бұл үшін авторизация талап етілмейді. Қолтаңбаланған URL-адресті бірнеше рет пайдалануға болады, бірақ тек онда көрсетілген жарамдылық мерзімі ішінде.

Қолтаңбаланған URL-адрестің мүмкіндіктері оны қолтаңбалаған адамның құқықтарымен шектеледі. Мысалы, егер осы URL-адрес қолтаңбаланған {var(s3)} аккаунты жойылса, URL-адрес жұмысын тоқтатады.

{cut(Қолтаңбаланған URL-дің жалпы түрі)}

```console
https://hb.ru-msk.vkcloud-storage.ru/<ИМЯ_БАКЕТА>/<КЛЮЧ_ОБЪЕКТА>?
     X-Amz-Algorithm=<АЛГОРИТМ>
    &X-Amz-Expires=<ВРЕМЯ_ДЕЙСТВИЯ>
    &X-Amz-SignedHeaders=<СПИСОК_ЗАГОЛОВКОВ>
    &X-Amz-Date=<ДАТА_И_ВРЕМЯ>
    &X-Amz-Credential=<КЛЮЧ_ДОСТУПА>/<YYYYMMDD>/ru-msk/s3/aws4_request
    &X-Amz-Signature=<ПОДПИСЬ>
```

Мұнда:

- `<КЛЮЧ_ОБЪЕКТА>` — қолжетімділік берілетін объект кілті.
- `<ИМЯ_БАКЕТА>` — объект орналасқан бакеттің атауы.
- `<АЛГОРИТМ>` — қолтаңба нұсқасы және оны есептеу алгоритмі, мысалы `AWS4-HMAC-SHA256`.
- `<ВРЕМЯ_ДЕЙСТВИЯ>` — URL-адрестің секундтармен берілген жарамдылық уақыты. `X-Amz-Date` тақырыбында көрсетілген сәттен бастап есептеледі. Ең жоғары мәні — 604800 секунд (7 күн).
- `<СПИСОК_ЗАГОЛОВКОВ>` — қолтаңбаланатын сұрау тақырыптарының тізімі. Тақырыптар `;` таңбасымен бөлінеді. Міндетті түрде `Host` тақырыбы және сұрауда қолданылатын барлық `X-Amz-*` тақырыптары қолтаңбалануы тиіс.
- `<ДАТА_И_ВРЕМЯ>` — URL-адрестің жарамдылық мерзімі аяқталатын күн мен уақыт ISO8601 форматында, мысалы `20180719T000000Z`. Күн `X-Amz-Credential` тақырыбындағы күнмен мәні бойынша сәйкес келуі тиіс.
- `<КЛЮЧ_ДОСТУПА>` — Access Key ID, қолжетімділік кілтінің идентификаторы.
- `<YYYYMMDD>` — `X-Amz-Credential` тақырыбына қою үшін `YYYYMMDD` форматындағы күн. `X-Amz-Date` тақырыбында орнатылған күнмен мәні бойынша сәйкес келуі тиіс.  
- `<ПОДПИСЬ>` — сұраудың қолтаңбасы. Сұрау қолжетімділіктің құпия кілті арқылы қолтаңбаланады.

{/cut}
