Әдепкі бойынша жоба ішіндегі барлық объектілер мен бакеттер жекеменшік болып табылады. Алайда сіз басқа пайдаланушыларға бакеттен объектілерді жүктеп алуға немесе оған объектілерді жүктеуге алдын ала қолтаңбаланған URL-мекенжайын жасау арқылы рұқсат бере аласыз. Онда мыналар көрсетіледі:

- Ашық қолжетімділік кілті. Мысалы, ашық [VK Object Storage аккаунтына байланыстырылған қолжетімділік кілті](../account-and-keys#account_access_keys) немесе ашық [бакет қолжетімділік кілті](../account-and-keys#bucket_access_keys).
- Бакет атауы.
- Объект кілті.
- URL-мекенжайының қолданылу мерзімі.
- Құпия қолжетімділік кілтімен шифрланған қолтаңба.

{note:info}
Әдепкі бойынша алдын ала қолтаңбаланған URL-мекенжайы онда көрсетілген объектіні жүктеп алуға мүмкіндік береді. Сондай-ақ бакетке объектілерді жүктеу үшін URL-мекенжайларын да жасай аласыз. Толығырақ — [Amazon AWS ресми құжаттамасында](https://docs.amazonaws.cn/en_us/AmazonS3/latest/userguide/PresignedUrlUploadObject.html).
{/note}

Қолтаңбаланған URL-мекенжайы бар кез келген адам онда көрсетілген объектіге қол жеткізе алады. Ол үшін авторизация талап етілмейді. Қолтаңбаланған URL-мекенжайын бірнеше рет пайдалануға болады, бірақ тек онда көрсетілген қолданылу мерзімі ішінде ғана.

Қолтаңбаланған URL-мекенжайының мүмкіндіктері оған қол қойған тұлғаның құқықтарымен шектеледі. Мысалы, бұл URL-мекенжайына қол қойған VK Object Storage аккаунты жойылса, URL-мекенжайы жұмысын тоқтатады.

{cut(Қолтаңбаланған URL-дің жалпы түрі)}

```console
https://hb.ru-msk.vkcloud-storage.ru/<ИМЯ_БАКЕТА>/<КЛЮЧ_ОБЪЕКТА>?
     X-Amz-Algorithm=<АЛГОРИТМ>
    &X-Amz-Expires=<ВРЕМЯ_ДЕЙСТВИЯ>
    &X-Amz-SignedHeaders=<СПИСОК_ЗАГОЛОВКОВ>
    &X-Amz-Date=<ДАТА_И_ВРЕМЯ>
    &X-Amz-Credential=<КЛЮЧ_ДОСТУПА>/<YYYYMMDD>/ru-msk/s3/aws4_request
    &X-Amz-Signature=<ПОДПИСЬ>
```

Мұнда:

- `<КЛЮЧ_ОБЪЕКТА>` — қолжетімділік берілетін объектінің кілті.
- `<ИМЯ_БАКЕТА>` — объект орналасқан бакеттің атауы.
- `<АЛГОРИТМ>` — қолтаңба нұсқасы және оны есептеу алгоритмі, мысалы `AWS4-HMAC-SHA256`.
- `<ВРЕМЯ_ДЕЙСТВИЯ>` — URL-мекенжайының секундпен берілетін қолданылу уақыты. `X-Amz-Date` тақырыбында көрсетілген сәттен бастап есептеледі. Ең үлкен мәні — 604800 секунд (7 күн).
- `<СПИСОК_ЗАГОЛОВКОВ>` — сұраудың қол қойылатын тақырыптарының тізімі. Тақырыптар `;` символымен бөлінеді. Міндетті түрде `Host` тақырыбы және сұрауда қолданылатын барлық `X-Amz-*` тақырыптары қол қойылуы тиіс.
- `<ДАТА_И_ВРЕМЯ>` — URL-мекенжайының қолданылу мерзімі аяқталатын күн мен уақыт ISO8601 форматында, мысалы `20180719T000000Z`. Күн `X-Amz-Credential` тақырыбындағы күнмен сәйкес келуі керек.
- `<КЛЮЧ_ДОСТУПА>` — Access Key ID, қолжетімділік кілтінің идентификаторы.
- `<YYYYMMDD>` — `X-Amz-Credential` тақырыбына қоюға арналған `YYYYMMDD` форматындағы күн. Ол `X-Amz-Date` тақырыбында орнатылған күнмен сәйкес келуі керек.  
- `<ПОДПИСЬ>` — сұраудың қолтаңбасы. Сұрау құпия қолжетімділік кілті көмегімен қолтаңбаланады.

{/cut}

Қолтаңбаланған URL-мекенжайын [құрастыруға](../../../instructions/access-management/signed-url) AWS CLI немесе SDK көмегімен болады.
