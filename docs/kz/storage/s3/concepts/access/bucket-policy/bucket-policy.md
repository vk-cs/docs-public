# {heading(Қолжетімділік саясаттары (bucket policy))[id=s3-concepts-bucket-policy]}

{include(/kz/_includes/_translated_by_ai.md)}

## {heading(Жалпы ақпарат)[id=s3-concepts-bucket-policy-general-info]}

Бакет саясаты (bucket policy) — S3-бакет мазмұнына қолжетімділікті бүкіл бакет деңгейінде басқару механизмі. Саясаттар JSON форматында беріледі және көрсетілген субъектілер (пайдаланушылар, топтар, есептік жазбалар) үшін бакет объектілеріне қатысты белгілі бір әрекеттерді (мысалы, оқу, жазу, жою) орындауға рұқсаттарды анықтайды.

## {heading(Негізгі ерекшеліктері)[id=s3-concepts-bucket-policy-key-features]}

Бакет саясатының көмегімен мыналарды жасауға болады:

- Бакет ішіндегі объектілерге қолжетімділікті басқару.
- Бакеттің сырттан оқуға немесе жазуға қолжетімділігін бақылау.
- Жекелеген пайдаланушыларға, топтарға немесе қолданбаларға қолжетімділік құқықтарын тағайындау.
- IP-адрестер, жол префикстері немесе басқа шарттар бойынша қолжетімділікті шектеу.
- `Principal` идентификаторын пайдалану.
- `IpAddress` немесе `UserAgent` шарттары бойынша шектеулерді орнату.

## {heading(Пайдаланушы түрлері)[id=s3-concepts-bucket-policy-user-types]}

Бакет саясаты келесі пайдаланушы түрлерін қамтиды:

- Қарапайым пайдаланушы (IAM-пайдаланушы) — жоба шеңберінде бакет ішіндегі объектілерге толық қолжетімділікке ие.
- Префикстік пайдаланушы — объектілерге қолжетімділік `<bucket>/<prefix>` префиксімен шектеледі.
- ROOT-пайдаланушы — жүйелік резервтік көшірмелерге (RO-бакеттерге) қолжетімділігі бар арнайы сервистік пайдаланушы.
- Сервистік есептік жазба — қолданбаларда жұмыс істеу үшін ұсынылатын пайдаланушы.

## {heading(Principal форматы)[id=s3-concepts-bucket-policy-principal]}

Бакет саясатында `Principal` идентификаторы әрекеттердің нақты кімге рұқсат етілгенін немесе тыйым салынғанын анықтайды. Мүмкін нұсқалар:

- Барлық пайдаланушылар (жария қолжетімділік):

  ```json
  "Principal": "*"
  ```

  Қолжетімділік аутентификациясыз (анонимді) беріледі. Мысалы, статикалық сайттарды жариялау үшін қолданылады.

- Жобаның барлық пайдаланушылары (жобаның root-қолжетімділігі):

  ```json
  "Principal": {
      "AWS": "<ID_ПРОЕКТА>:root"
  }
  ```

  Қолжетімділікті жобаның барлық IAM-пайдаланушылары және сервистік есептік жазбалары алады.

- Нақты пайдаланушы:

  ```json
  "Principal": {
      "AWS": "<ID_ПРОЕКТА>:user/<ИМЯ_ПОЛЬЗОВАТЕЛЯ>"
  }
  ```

- Сервистік есептік жазба:

  ```json
  "Principal": {
      "AWS": "<ID_ПРОЕКТА>:serviceaccount/<ИМЯ_СЕРВИСНОГО_ПОЛЬЗОВАТЕЛЯ>"
  }
  ```

  Сервистік есептік жазба қолданбаға немесе сервиске құқық беру үшін пайдаланылады.

### {heading(Қолдану ережелері мен шектеулер)[id=s3-concepts-bucket-policy-restriction-rules]}

Бакет саясатында келесі ережелер мен шектеулер қолданылады:

- Барлық саясаттар айқын тыйым салу немесе рұқсат беру (Explicit Deny/Allow) логикасы бойынша өңделеді. Тыйым салудың басымдығы жоғары.
- Жобалар арасындағы қолжетімділік басқа жобаның `Principal` идентификаторын көрсету арқылы іске асырылады. Екі нұсқа мүмкін:

  - Басқа жобаға ROOT-құқықтар беріледі — жобаның барлық пайдаланушылары қолжетімділік алады.
  - Қолжетімділік нақты пайдаланушыға беріледі.

- Бакет саясатын тек бакет жасалған өңірде қолдануға болады.
- Ресурстың бірегей атауы (ARN) бакет атауында `*` таңбаларын қамтымауы тиіс.

### {heading(ACL-мен өзара әрекеттесу)[id=s3-concepts-bucket-policy-interact-acl]}

Бакет саясаты ACL-ден басым. ACL тек саясат бойынша қолжетімділік алынғаннан кейін ғана тексеріледі.
