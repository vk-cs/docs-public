VK Object Storage жүйесінде бакеттер мен объектілерге қол жеткізу үшін бір-біріне тәуелсіз екі түрлі кілт пайдаланылады:

- [VK Object Storage аккаунтына байланыстырылған қолжетімділік кілттері](#account_access_keys). Аккаунттың барлық бакеттеріне қол жеткізуге мүмкіндік береді.
- [Бакетке байланыстырылған қолжетімділік кілттері](#bucket_access_keys). Бір бакетке немесе сол бакеттегі объектілердің бір бөлігіне қол жеткізуге мүмкіндік береді.

## {heading(VK Object Storage аккаунты және оның қолжетімділік кілттері)[id=account_access_keys]}

VK Cloud жеке кабинетінде VK Object Storage сервисімен жұмыс істеу үшін қолжетімділік кілттері қолданылмайды. Бірақ егер сіз бакеттер мен объектілермен [CLI](/kz/tools-for-using-services/cli/aws-cli), [API](/kz/tools-for-using-services/api/api-spec/s3-rest-api) немесе [файл менеджерлері](../../../connect/s3-file-managers) арқылы жұмыс істегіңіз келсе, жоба ішінде VK Object Storage аккаунтын және оған байланыстырылатын қолжетімділік кілттерін [жасау](../../../instructions/access-management/access-keys#create_account_and_access_key) қажет.

Сіз бірнеше VK Object Storage аккаунтын жасай аласыз. Олардың бір жоба ішіндегі саны [лимиттермен](/kz/tools-for-using-services/account/concepts/quotasandlimits#object_storage_limits) шектелген. Аккаунтпен бірге оған арналған қолжетімділік кілті автоматты түрде жасалады. Аккаунтқа арналған қолжетімділік кілті — бұл ашық кілт идентификаторынан (Access Key ID) және құпия кілттен (Secret Key) тұратын жұп. Құпия кілт тек жұпты жасау кезінде ғана қолжетімді, оны кейін қалпына келтіру мүмкін емес, бірақ сіз жаңа қолжетімділік кілтін [жасай](../../../instructions/access-management/access-keys#add_new_access_key) аласыз. Бір аккаунтқа екі кілтке дейін байланыстыруға болады.

VK Object Storage аккаунттарын және оларға байланыстырылған кілттерді басқару тек VK Cloud жеке кабинетінде қолжетімді.

## {heading(Бакетке қолжетімділік кілттері)[id=bucket_access_keys]}

Бакетке қолжетімділік кілттері аккаунтқа емес, нақты бір бакетке байланысады. Олар пайдаланушыға тек осы бакеттегі объектілермен немесе оның бір директориясындағы объектілермен жұмыс істеу құқығын береді, бірақ жоба ішіндегі басқа бакеттер мен объектілерге қолжетімділік бермейді.

Бакетке арналған қолжетімділік кілті — бұл ашық кілт идентификаторынан (Access Key ID) және құпия кілттен (Secret Key) тұратын жұп. Құпия кілт тек жұпты жасау кезінде ғана қолжетімді, оны кейін қалпына келтіру мүмкін емес, бірақ сіз жаңа қолжетімділік кілтін [жасай](../../../instructions/access-management/bucket-keys#add-keys) аласыз.

Бакетке қолжетімділік кілтін қосқанда, параметр ретінде VK Object Storage бакеттегі директория атауы ретінде интерпретациялайтын [объект атауына префикс](../../about#object_key) көрсетуге болады. Мұндай кілт префикстік (prefix access key) деп аталады. Префикстік кілт тек префикспен берілген бакет директориясындағы объектілерге ғана қолжетімділік береді. Префикс көрсетілмей жасалған кілт бакеттегі барлық объектілерге қолжетімділік береді.

Аккаунтқа байланыстырылған кілттер сияқты, бакетке қолжетімділік кілттері де ондағы объектілермен CLI, API немесе файл менеджерлері арқылы жұмыс істеуге мүмкіндік береді. Бакетке қолжетімділік кілттерін [жеке кабинет арқылы](../../../instructions/access-management/bucket-keys)немесе [API көмегімен](/kz/tools-for-using-services/api/api-spec/s3-rest-api/pak-api) басқаруға болады.
