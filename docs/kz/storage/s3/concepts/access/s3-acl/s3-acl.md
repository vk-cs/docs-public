ACL (Access Control List, қолжетімділікті басқару тізімі) VK Object Storage сервисінде қандай тіркелгі жазбаларына немесе сақтау топтарына бакетке не объектіге қолжетімділік берілетінін анықтайды. Әрбір бакетте немесе объектіде 100 элементке дейін қамтитын өз ACL-і болады. Әр элементте [құқық алушы](#permittees) және оған берілетін [рұқсат түрі](#permissons) көрсетіледі. Бакетке немесе объектіге сұрау түскенде, VK Object Storage сұрауды жіберушіде қажетті құқықтардың бар-жоғын тиісті ACL-дерден тексереді.

VK Object Storage сервисі [стандартты ACL-дердің](#standard_acl) жиынтығын қолдайды — бұл құқық алушылар мен оларға тағайындалған рұқсаттардың алдын ала дайындалған тізімдері. Бакет жасалған кезде VK Object Storage әдепкі бойынша ол үшін `private` стандартты ACL негізінде қолжетімділікті басқару тізімін жасайды. Бұл ACL бакет иесіне толық басқару құқығын береді және қалған пайдаланушыларға қолжетімділікті тыйым салады.

Объектінің ACL-і келесі түрде тағайындалуы мүмкін:

- Жаңа объектіні бакетке жеке кабинет арқылы жүктеген кезде, сіз оған [стандартты ACL-дердің](#standard_acl) бірін таңдайсыз.
- API арқылы жүктеген кезде, объектінің ACL-ін сұрау денесінде немесе тақырыптарында көрсете аласыз.

Кейіннен бакет немесе объект үшін ACL-ді өзгертуге болады. Бакеттің ACL-і мен оның ішіндегі объектінің ACL-і бір-бірінен өзгеше болуы мүмкін.

ACL [конфигурация файлы](/kz/tools-for-using-services/api/api-spec/s3-rest-api/acl-api#acl_config_file) түрінде XML форматында беріледі. Бакеттің немесе объектінің ағымдағы ACL-ін алу немесе оны өзгерту [AWS CLI немесе API көмегімен](../../../instructions/access-management/acl) мүмкін.

## {heading(Құқық алушылар)[id=permittees]}

Бакеттер мен объектілерге қолжетімділік құқықтары келесі алушылар санаттарына берілуі мүмкін:

- VK Cloud пайдаланушысы.
- VK Cloud жобасы.
- Алдын ала анықталған метатоп. Мысалы, VK Cloud ішіндегі барлық қолданыстағы тіркелгі жазбаларын қамтитын `AuthenticatedUsers` метатобы.
- Бүкіл әлем, соның ішінде анонимді қолжетімділік. `AllUsers` метатобына сәйкес келеді. Объектінің толық URL-мекенжайын білетін кез келген адам оған қол жеткізе алады.

{note:warn}

Метатоптарды тек қажет болған жағдайда пайдаланыңыз.

Бакетке және ондағы объектілерге белгісіз тұлғалар тобына қолжетімділік ашсаңыз, бұл ресурстармен орындалатын операцияларды, соның ішінде ақылы операцияларды да бақылай алмайсыз. Бакетпен орындалатын операциялар үшін төлем оның иесіне жүктеледі.

{/note}

Кестеде келтірілген операциялар туралы ақпаратты [VK Object Storage API сипаттамасы бар бөлімнен](/kz/tools-for-using-services/api/api-spec/s3-rest-api) немесе [AWS CLI командалары анықтамалығынан](https://docs.aws.amazon.com/cli/latest/reference/s3api) табуға болады.

## {heading(Стандартты ACL-дер)[id=standard_acl]}

VK Object Storage қолжетімділікті басқарудың алдын ала бапталған тізімдерін — стандартты ACL-дерді ұсынады. Бұл тізімдерді бакетке, объектіге немесе олардың ағымдағы ACL-деріне қолжетімділікке арналған API сұрауларында қолдануға болады.

Стандартты ACL сұрауда `x-amz-acl` HTTP-тақырыбы арқылы көрсетіледі. Тек бір стандартты ACL көрсетуге болады. VK Object Storage стандартты ACL қамтитын сұрауды алған кезде, ол алдын ала бапталған рұқсаттарды тиісті бакеттің немесе объектінің ACL-іне қосады.

[cols="2,1,4", options="header"]
|===
|Стандартты ACL
|Қолданылу аймағы
|Алдын ала бапталған рұқсаттар

|`private`

(әдепкі бойынша)
|Бакет және объект
|Иесінің толық құқықтары бар (`FULL_CONTROL`). Қалған пайдаланушыларда қолжетімділік құқықтары жоқ

|`public-read`
|Бакет және объект
|Иесінің толық құқықтары бар (`FULL_CONTROL`). `AllUsers` метатобының оқу құқығы (`READ`) бар

|`public-read-write`
|Бакет және объект
|Иесінің толық құқықтары бар (`FULL_CONTROL`). `AllUsers` метатобының оқу (`READ`) және жазу (`WRITE`) құқықтары бар.

Бұл ACL-ді бакет үшін пайдалану ұсынылмайды

|`aws-exec-read`
|Бакет және объект
|Иесінің толық құқықтары бар (`FULL_CONTROL`)

|`authenticated-read`
|Бакет және объект
|Иесінің толық құқықтары бар (`FULL_CONTROL`). `AuthenticatedUsers` метатобының оқу құқықтары (`READ`) бар

|`bucket-owner-read`
|Объект
|Объект иесінің толық құқықтары бар (`FULL_CONTROL`). Бакет иесінің оқу құқығы (`READ`) бар.

Егер бұл стандартты ACL бакет жасау кезінде көрсетілсе, VK Object Storage оны елемейді

|`bucket-owner-full-control`
|Объект
|Объект иесі мен бакет иесінің объектіге толық құқықтары (`FULL_CONTROL`) бар.

Егер бұл стандартты ACL бакет жасау кезінде көрсетілсе, VK Object Storage оны елемейді
|===
