# {heading(Қолжетімділікті басқару (ACL))[id=s3-concepts-acl]}

{include(/kz/_includes/_translated_by_ai.md)}

{var(s3)} сервисіндегі ACL (Access Control List, қолжетімділікті басқару тізімі) қойманың қандай есептік жазбаларына немесе топтарына бакетке немесе объектіге қолжетімділік берілетінін анықтайды. Әр бакетте немесе объектіде өз ACL-і бар, ол 100 элементке дейін қамтуы мүмкін. Әр элементте {linkto(#s3-concepts-acl-permittees)[text=құқық алушы]} және оған берілетін {linkto(#s3-concepts-acl-permissons)[text=рұқсат түрі]} көрсетіледі. Бакетке немесе объектіге сұрау түскен кезде {var(s3)} тиісті ACL-дерде сұрауды жіберушіде қажетті құқықтардың бар-жоғын тексереді.

{var(s3)} сервисі {linkto(#s3-concepts-acl-pre-set)[text=стандартты ACL]} жиынтығын қолдайды — құқық алушылардың және оларға тағайындалған рұқсаттардың алдын ала дайындалған тізімдері. Бакет жасалған кезде {var(s3)} әдепкі бойынша `private` стандартты ACL-ін қолданады. Бұл ACL бакет иесіне бакетке толық бақылау береді және қалған пайдаланушыларға қолжетімділікті тыйым салады.

Объектінің ACL-і мына кезде тағайындалуы мүмкін:

- Жеке кабинет арқылы бакетке жаңа объект жүктегенде, ол үшін {linkto(#s3-concepts-acl-pre-set)[text=стандартты ACL]} ішінен біреуін таңдайсыз.
- API көмегімен жүктегенде, сұраудың денесінде немесе тақырыптарында объектінің ACL-ін көрсете аласыз.

Егер ACL көрсетілмесе, жаңа бакетке немесе объектіге қолжетімділігі барынша шектелген `private` стандартты ACL-і тағайындалады: бакет жасалғанда қолжетімділікті тек бакет иесі алады, объект қосылғанда қолжетімділікті тек объект иесі алады.

Кейін бакет немесе объект үшін ACL-ді өзгертуге болады. Бакеттің ACL-і мен оның ішіндегі объектінің ACL-і әртүрлі болуы мүмкін.

ACL XML форматындағы {linkto(../../../api/acl#api-spec-s3-acl-request-body)[text=конфигурация файлы]} түрінде болады. Бакеттің немесе объектінің ағымдағы ACL-ін алу немесе оны өзгерту {linkto(../../../instructions/access-management/acl#s3-instructions-acl)[text=AWS CLI немесе API көмегімен]} мүмкін.

## {heading(Құқық алушылар)[id=s3-concepts-acl-permittees]}

Бакеттер мен объектілерге қолжетімділік құқықтары келесі алушылар санаттарына берілуі мүмкін:

- {ifdef(public)}{var(cloud)}{/ifdef}{ifdef(s3,s3-pdf)}IAM Only{/ifdef} пайдаланушысы.
- {ifdef(public)}{var(cloud)}{/ifdef}{ifdef(s3,s3-pdf)}IAM Only{/ifdef} жобасы.
- Алдын ала анықталған метатоп. Мысалы, барлық бар {ifdef(public)}{var(cloud)}{/ifdef}{ifdef(s3,s3-pdf)}IAM Only{/ifdef} есептік жазбаларын қамтитын `AuthenticatedUsers` метатобы.
- Бүкіл әлем, соның ішінде анонимді қолжетімділік. `AllUsers` метатобына сәйкес келеді. Объектінің толық URL-адресін білетін кез келген адам оған қол жеткізе алады.

{note:warn}
Метатоптарды тек қажеттілік болған жағдайда ғана пайдаланыңыз.

Бакетке және ондағы объектілерге белгісіз адамдар тобына қолжетімділік ашсаңыз, бұл ресурстармен орындалатын операцияларды{ifdef(public)}, соның ішінде ақылы операцияларды да, бақылай алмайсыз. Бакетпен жасалатын операциялардың ақысын оның иесі төлейді{/ifdef}.
{/note}

## {heading(Стандартты ACL)[id=s3-concepts-acl-pre-set]}

{var(s3)} алдын ала бапталған қолжетімділікті басқару тізімдерін — стандартты ACL-дерді ұсынады. Бұл тізімдерді бакетке, объектіге немесе олардың ағымдағы ACL-деріне қолжетімділікке арналған API-сұрауларда пайдалануға болады.

Стандартты ACL сұрауда `x-amz-acl` HTTP-тақырыбы арқылы көрсетіледі. Тек бір стандартты ACL көрсетуге болады. {var(s3)} стандартты ACL бар сұрауды алған кезде, ол тиісті бакеттің немесе объектінің ACL-іне алдын ала бапталған рұқсаттарды қосады( {ifdef(s3-pdf)}{linkto(#tab_acl)[text=%number кестеде]}{/ifdef}).
