CORS (Cross-Origin Resource Sharing) — кросс-домендік сұрауларды өңдеу ережелерін орнатуға мүмкіндік беретін механизм. Әдепкі бойынша браузерлер бір доменде орналасқан веб-қосымша басқа доменде орналасқан дереккөз үшін API шақырса, қолжетімділікті бұғаттайды. Бұл ресурстарға рұқсатсыз қол жеткізуді болдырмау үшін жасалады. VK Object Storage сервисіндегі CORS қолдауының арқасында сіз бұл шектеуді айналып өтіп, басқа домендерде орналасқан клиенттік веб-қосымшаларға өз бакеттеріңізге таңдаулы түрде қолжетімділік бере аласыз.

Бакет үшін CORS механизмін екі тәсілмен қосуға болады:

- Жеке кабинетте әрбір CORS ережесін жеке-жеке [орнату](../../../instructions/access-management/cors#add_rule_cors).
- API көмегімен барлық CORS ережелерінің [XML конфигурациясын жүктеу](/kz/tools-for-using-services/api/api-spec/s3-rest-api/cors-api#set_bucket_cors).

{note:warn}
Екінші тәсілді пайдаланғанда, бар CORS конфигурациясы қайта жазылады. Бұл жеке кабинетте бұрын қосылған ережелер жоғалатынын білдіреді.
{/note}

Әр бакет үшін өз CORS ережелері бапталады, оларда келесі параметрлер көрсетіледі:

- `AllowedOrigins` — бакетке сұрауларға рұқсат берілген домендер.
- `AllowedMethods` — сұрауларда рұқсат етілген HTTP әдістері.
- (Опционалды) `MaxAgeSeconds` — пайдаланушы браузері [алдын ала сұраудың](#cors_request) нәтижелерін кэште сақтай алатын секундпен берілетін уақыт.
- (Опционалды) `AllowedHeaders` — сұрауларда рұқсат етілген тақырыптар.
- (Опционалды) `ExposeHeaders` — сұрауға жауапта пайдаланушыға қолжетімді болатын тақырыптар.

## {heading(CORS сұрауларының ерекшеліктері)[id=cors_request]}

CORS сұрауы — `Origin` тақырыбын қамтитын HTTP сұрау. Бұл тақырыпта сұрауды бастамашы домен көрсетіледі. Егер CORS сұрауы деректердің өзгеруіне әкелуі мүмкін болса, ол қауіпті емес деп емес, қауіпті деп қарастырылады және екі кезеңде орындалады:

- Сұралатын ресурс үшін CORS шектеулерін тексеретін алдын ала сұрау (preflight request).
- Ресурсты алуға бағытталған негізгі сұрау.

Алдын ала сұрау `OPTIONS` әдісін пайдаланады және негізгі сұраудың параметрлерін қамтиды: тақырыптар, HTTP әдісі және домен (`Origin`).

VK Object Storage браузерден алдын ала сұрау алған кезде, сұралатын бакет үшін CORS конфигурациясын талдайды және сұрауды қабылдауы немесе қабылдамай тастауы мүмкін. Сұрау қабылдануы үшін келесі шарттар орындалуы керек:

- `Origin` тақырыбында көрсетілген домен бакет CORS конфигурациясындағы `AllowedOrigins` тізіміне енгізілуі керек.

- Сұрауда көрсетілген HTTP әдісі бакет CORS конфигурациясындағы `AllowedMethods` тізіміне енгізілуі керек.

- Сұрауда көрсетілген тақырыптар бакет CORS конфигурациясындағы `AllowedHeaders` тізіміндегі тақырыптарға сәйкес келуі керек.

Егер алдын ала сұрау қабылданбаса, браузер негізгі сұрауды жіберуді бұғаттайды.

Егер алдын ала сұрау қабылданса, VK Object Storage бакеттің CORS конфигурациясы туралы ақпаратты қайтарады. Браузер бұл ақпаратты CORS конфигурациясындағы `MaxAgeSeconds` параметрінде көрсетілген уақыт аралығында кэште сақтай алады.

Алдын ала сұрауға жауаптарды кэштеу VK Object Storage жүйесімен жұмыс істеу өнімділігін айтарлықтай жақсарта алады, себебі қажетті сұраулар санын азайтады. Сол бір бакетке қайта сұрау жасалғанда, егер браузер кэшінде бакеттің CORS конфигурациясы туралы деректер бұрыннан бар болса, алдын ала сұрау қажет болмайды.
