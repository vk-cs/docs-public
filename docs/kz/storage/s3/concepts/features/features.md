## {heading(Объектілерді стандартты және көпбөлікті жүктеу)[id=object_uploading]}

VK Object Storage бакетіне кез келген файл түрін, мысалы, кескіндерді, сақтық көшірмелерді, бейнелерді жүктеуге болады.

Файл көлеміне қарай әртүрлі жүктеу тәсілдері ұсынылады:

- 1 ГБ дейін — [стандартты жүктеу](../../instructions/objects/upload-object#standard_upload) кез келген ыңғайлы тәсілмен: жеке кабинет, файл менеджерлері, CLI, SDK немесе API арқылы.
- 1 ГБ-тан жоғары — CLI, SDK немесе API.
- 32 ГБ-тан жоғары — тек [көпбөлікті жүктеу](../../instructions/objects/upload-object#multipart_upload) арқылы CLI, SDK немесе API көмегімен.

Көпбөлікті жүктеу кезінде деректер бірнеше бөлікке бөлінеді және жеке-жеке әрі кез келген ретпен жүктеледі. Егер қандай да бір бөлікті беру сәтсіз болса, оны басқа бөліктерге әсер етпей қайта жіберуге болады. Барлық бөліктер жүктелген кезде, сақтау жүйесі оларды берілген ретпен біріктіріп, объект жасайды.

Көпбөлікті жүктеудің артықшылықтары:

- Үлкен файлдың бөліктерін қатар жүктеуге болады, бұл жүктеу уақытын қысқартады.
- Егер қандай да бір бөлік желі қатесіне байланысты жүктелмесе, оны үлкен файлға қарағанда тезірек қайта жүктеуге болады.
- Жүктеуді кідіртіп, кейінірек жалғастыруға болады.
- Жүктеуді файлдың соңғы өлшемі белгілі болмай тұрып бастауға болады, яғни файлды жасалуына қарай жүктеуге болады.

{note:info}

Жүктелетін файл өлшеміне қойылатын шектеулер:

- Құрастырылмаған объектінің немесе көпбөлікті объект бөлігінің ең үлкен өлшемі — 32 ГБ.
- Көпбөлікті объектінің ең үлкен өлшемі — 320 ТБ.
- Көпбөлікті жүктеу бөліктерінің саны — 1-ден 10000-ға дейін.

{/note}

## {heading(Объектілерді көшіру)[id=objects_copying]}

VK Object Storage сақталатын объектілердің көшірмелерін жасауға мүмкіндік береді. Көшіру [CLI, SDK, файл менеджері арқылы](../../instructions/objects/manage-object#copy_object) немесе [API](/kz/tools-for-using-services/api/api-spec/s3-rest-api/object-api#copy) арқылы орындалады.

Көшіру арқылы сіз мыналарды істей аласыз:

- Сол бакетте немесе басқа бакетте объектілердің қосымша көшірмелерін жасау.
- Объектілердің атауын өзгерту.

  Бакеттегі объектілердің атауын тікелей өзгертуге қолдау көрсетілмейді. Бұл шектеуді жаңа атаумен объект көшірмесін жасап, содан кейін бастапқы объектіні жою арқылы айналып өтуге болады. [файл менеджерлерінде](../../connect/s3-file-managers) бұл әрекеттер автоматты түрде орындалады.

- Объект метадеректерін өзгерту.

  Объект бакетке жүктелген кезде оған кілт/мән форматындағы метадеректер тағайындалады. Оларды жүктегеннен кейін өзгертудің жалғыз жолы — көшіру сұрауында жаңа метадеректерді көрсетіп, сол атаумен объект көшірмесін жасау. Осылайша пайдаланушы метадеректерін және кейбір жүйелік метадеректерді, мысалы, [сақтау класын](../about#storage_class) өзгертуге болады.

## {heading(Бакеттегі оқиғалар туралы хабарландырулар)[id=bucket_event_notifications]}

VK Object Storage вебхуктар (webhooks) көмегімен бакеттегі оқиғалар туралы хабарландырулар жіберуді баптауға мүмкіндік береді. Әр вебхукта оқиға түрі және бакетте көрсетілген түрдегі оқиға болған кезде хабарлама немесе HTTP/HTTPS сұрауы жіберілетін URL мекенжайы көрсетіледі. Оқиғалардың келесі түрлері үшін вебхуктар жасауға болады:

- объектіні жасау;
- объектіні көшіру;
- объектіні жою;
- көпбөлікті жүктеуді аяқтау.

Вебхуктарды пайдалану мыналарға мүмкіндік береді:

- Пайдаланушыны бакетте болған оқиға туралы хабардар етуге;
- Жүктеуден кейін файлдарды өңдеу мен түрлендіруді баптауға;
- Сыртқы жүйелермен өзара әрекеттесуді баптауға;
- Объектілік қойма үшін логтауды қосуға.

Бакетке арналған вебхуктар конфигурациясы [жеке кабинет арқылы](../../instructions/buckets/manage-bucket#manage_webhooks) немесе [API көмегімен](/kz/tools-for-using-services/api/api-spec/s3-rest-api/webhook-api) жасалады.

## {heading(Объектілерді тегтермен таңбалау)[id=s3_api]}

VK Object Storage стандартты HTTP сұраулары арқылы бакеттер мен объектілерді бағдарламалық түрде басқаруға арналған RESTful XML [API](/kz/tools-for-using-services/api/api-spec/s3-rest-api) ұсынады. VK Object Storage API [Amazon Web Service S3 API](https://docs.aws.amazon.com/AmazonS3/latest/userguide/Welcome.html) жүйесімен өзара әрекеттесуге арналған және онымен үйлесімді интерфейске ие. VK Object Storage API-ге қол жеткізу үшін AWS S3 сервисі үшін жасалған протокол пайдаланылады.

API-ге сұрауларды [CLI](../../connect/s3-cli), [SDK](../../connect/s3-sdk) немесе [файл менеджерлері](../../connect/s3-file-managers) арқылы жібере аласыз.

VK Object Storage сервисінде AWS S3 API-дің кейбір әдістері іске асырылмаған. Мұндай әдістерге арналған сұраулар XML форматындағы `NotImplemented` қатесі туралы S3-пен үйлесімді жауап қайтарады.
