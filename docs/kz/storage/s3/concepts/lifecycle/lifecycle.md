VK Object Storage бакеттегі объектілердің өмірлік циклін (lifecycle) баптауға мүмкіндік береді. Өмірлік цикл — бұл берілген ережелер бойынша объектілерді бакеттен автоматты түрде жою.

Ережелер бір бакет деңгейінде орнатылады. Бакеттер тобы немесе жоба үшін өмірлік циклді баптау мүмкін емес.

[Жеке кабинетте](../../instructions/buckets/manage-bucket#manage_lifecycle) әр ережені жеке-жеке беруге болады, ал [API](/kz/tools-for-using-services/api/api-spec/s3-rest-api/lifecycle-api) және AWS CLI көмегімен өмірлік цикл ережелерінің конфигурациясын баптауға болады. [Ережелер конфигурациясын](#lifecycle_config) баптау дегеніміз — барлық ережелер XML немесе JSON форматындағы бір файлда беріледі, ал бұл файл VK Object Storage жүйесіне [API](/kz/tools-for-using-services/api/api-spec/s3-rest-api/lifecycle-api) немесе AWS CLI арқылы жіберіледі.

Өмірлік цикл ережелерін жасау ерекшеліктері:

- Конфигурацияда кемінде бір ереже болуы керек.

  Объект кілтінің [префиксі](/kz/storage/s3/concepts/about#object_key) бойынша сүзгілеуге қолдау көрсетіледі. Кілтке арналған префикс мысалдары: `image`, `image/`, `image/photo`. Сүзгі тек бір префиксті ғана қамтуы мүмкін. Әрбір префикс үшін жеке ереже жасау қажет.

- Объектінің өмір сүру уақыты (`Expiration` параметрі). Объектінің қашан жойылуы керектігін анықтайды: 

   - жүктелгеннен кейін берілген күн саны өткен соң;
   - көрсетілген уақытта (тек API және AWS CLI көмегімен).

Ереже қосылғаннан кейін ол бакеттегі тиісті объектілерге автоматты түрде қолданылады. Объектілер фондық режимде жойылады: өмір сүру мерзімі аяқталғаннан кейін олар қолжетімсіз болады, содан кейін кезек тәртібімен жойылады. Объектілер бірден емес, ереже қолданылғаннан кейінгі келесі 24 сағат ішінде жойылуы мүмкін.

Шектеулер:

- Әр бакет үшін 50 ережеден артық орнатуға болмайды.
- API арқылы ережелерді жасағанда, ережелері бар XML файлының өлшемі 512 КБ-тан аспауы керек.

Өмірлік цикл ережелері әдетте VK Object Storage сервисінің өнімділігіне әсер етпейді, өйткені олар фондық режимде жұмыс істейді.

## {heading(Өмірлік цикл ережелерінің конфигурациясы)[id=lifecycle_config]}

Өмірлік цикл конфигурациясы файлда JSON форматында (AWS CLI үшін) немесе XML форматында (API үшін) сипатталады.

{cut(JSON форматындағы префиксі және бірнеше тегі бар конфигурация мысалы)}

```json
{
  "Rules": [
    {
      "Expiration": {
        "Days": 1
      },
      "ID": "string",
      "Status": "string",
      "Filter": {
        "Prefix": "string",
        "And": {
          "Prefix": "string"
        }
      }
    }
  ]
}
```

[cols="2,1,3", options="header"]
|===

|Параметр
|Түрі
|Сипаттамасы

|`Rules`
|array of objects
|Өмірлік цикл ережелерінің тізімі

|`Rules.Expiration`
|object
|Объектінің өмір сүру мерзімі

|`Rules.Expiration.Days`
|integer
|Осы мерзімнен кейін объект жойылатын күндер саны

|`Rules.ID`
|string
|Ереженің бірегей идентификаторы

|`Rules.Status`
|string
|Ереже күйі. Мүмкін мәндер: 

- `Enabled` — ереже қолданылады; 
- `Disabled` — ереже қолданылмайды

|`Rules.Filter`
|object
|Ереже қолданылуы тиіс объектілер бойынша сүзгі

|`Rules.Filter.And`
|object
|[Объект префиксі](/kz/storage/s3/concepts/about#object_key). Егер сізді тек белгілі бір префиксі бар объектілер тізімі қызықтырса, префиксті көрсетіңіз

|`Rules.Filter.And.Prefix`
|string
|«ЖӘНЕ» логикалық операторы. Бір ережеде әртүрлі префикстер бойынша сүзгілеуді біріктіру үшін қолданылады

|`Rules.Filter.And.Tags`
|array of objects
|[Объект префиксі](/kz/storage/s3/concepts/about#object_key). Егер сізді тек белгілі бір префиксі бар объектілер тізімі қызықтырса, префиксті көрсетіңіз

|===
