VK Object Storage объектілерді файлдық менеджерлердің көмегімен басқаруды қолдайды. Мұндай менеджерлер қойма объектілерімен құрылғыңыздағы директориялар мен файлдар сияқты жұмыс істеуге мүмкіндік береді. Бұл көп файлмен немесе әртүрлі қоймалармен бір уақытта жұмыс істеу үшін ыңғайлы.

## CyberDuck

CyberDuck әртүрлі қоймаларға бір уақытта қосылуға мүмкіндік береді. Әр қосылым бөлек терезеде ашылады. Файлдарды құрылғы мен қойма арасында да, әртүрлі қоймалар немесе бір қойманың бакеттері арасында да көшіруге болады. macOS және Windows үшін қолжетімді.

1. [Аккаунт пен қатынау кілтін жасаңыз](../../instructions/access-management/access-keys) VK Object Storage үшін VK Cloud жеке кабинетінде. Құпия қатынау кілтін (**Secret Key**) сақтап қойыңыз.
1. CyberDuck-ты [орнатыңыз](https://cyberduck.io/download).
1. Объектілік қоймаға қосылуды баптаңыз:
    1. **Новое подключение** түймесін басыңыз.
    1. Қосылу параметрлерін көрсетіңіз:

        - Қосылу түрі: `Amazon S3`.
        - **Сервер**: домендік атауды көрсетіңіз, аккаунттың [аймағына](../../../../tools-for-using-services/account/concepts/regions) сәйкес келетін.
            - `hb.vkcloud-storage.ru` немесе `hb.ru-msk.vkcloud-storage.ru` — Мәскеу аймағы үшін;
            - `hb.kz-ast.vkcloud-storage.ru` — Қазақстан аймағы үшін.
        - **Порт**: `443`.
        - **Access Key ID**: VK Object Storage аккаунтын жасау кезінде алынған қатынау кілтінің идентификаторы.
        - **Secret Access Key**: VK Object Storage аккаунтын жасау кезінде алынған құпия қатынау кілті. Құпия кілт **Access Key ID** өрісінде көрсеткен кілт идентификаторына сәйкес болуы керек.
    1. **Подключиться** түймесін басыңыз.

Сәтті қосылғаннан кейін аккаунт үшін қолжетімді бакеттер CyberDuck ішінде көрсетіледі.

{note:info}

Cyberduck арқылы жасалатын бакеттер мен объектілер әдепкі бойынша Cyberduck баптауларында көрсетілген [сақтау класына](../../concepts/about#storage_class) және [қатынау түріне](../../concepts/access/s3-acl#permissons) ие болады.

{/note}

## WinSCP

WinSCP әртүрлі қоймаларға бір уақытта қосылуға мүмкіндік береді. Әр қосылым немесе жергілікті директория бөлек қойындыда ашылады. Файлдарды құрылғы мен қойма арасында да, әртүрлі қоймалар немесе бір қойманың бакеттері арасында да көшіруге болады. Windows үшін қолжетімді.

1. [Аккаунт пен қатынау кілтін жасаңыз](../../instructions/access-management/access-keys) VK Object Storage үшін VK Cloud жеке кабинетінде. Құпия қатынау кілтін (**Secret Key**) сақтап қойыңыз.
1. WinSCP-ты [орнатыңыз](https://winscp.net/eng/download.php).
1. Объектілік қоймаға қосылуды баптаңыз:
    1. Негізгі мәзірден **Қойындылар** → **Қосылу** → **Қосылымдарды басқару** тармағын таңдаңыз.
    1. Қосылу баптауларын көрсетіңіз:

        - **Протокол передачи**: `Amazon S3`;
        - **Имя хоста**: домендік атауды көрсетіңіз, аккаунттың [аймағына](../../../../tools-for-using-services/account/concepts/regions) сәйкес келетін.
            - `hb.vkcloud-storage.ru` немесе `hb.ru-msk.vkcloud-storage.ru` — Мәскеу аймағы үшін;
            - `hb.kz-ast.vkcloud-storage.ru` — Қазақстан аймағы үшін.
        - **Порт**: `443`.
        - **Идентификатор ключа доступа**: VK Object Storage аккаунтын жасау кезінде алынған кілт идентификаторы.
        - **Секретный ключ доступа**: VK Object Storage аккаунтын жасау кезінде алынған құпия қатынау кілті. Құпия кілт **Идентификатор ключа доступа** өрісінде көрсеткен кілт идентификаторына сәйкес болуы керек.
    1. **Войти** түймесін басыңыз.

Сәтті қосылғаннан кейін аккаунт үшін қолжетімді бакеттер WinSCP қойындысында көрсетіледі.

{note:info}

WinSCP арқылы жасалатын бакеттердің [сақтау класы](../../concepts/about#storage_class) `Hotbox` болады. WinSCP арқылы жасалатын объектілер бакеттен мұраланатын сақтау класына ие болады.

{/note}

## Диск-О:

Диск-О: әртүрлі қоймаларға бір уақытта қосылуға мүмкіндік береді. Әр қосылым жергілікті файлдық жүйеде бөлек диск ретінде ашылады. Диск-O: бүкіл VK Object Storage жүйесіне емес, бакетке қосылады. Бірнеше бакетпен жұмыс істеу үшін бөлек қосылымдар жасау керек. Файлдарды құрылғы мен қойма арасында да, әртүрлі қоймалар немесе бір қойманың бакеттері арасында да көшіруге болады. macOS және Windows үшін қолжетімді.

1. [Аккаунт пен қатынау кілтін жасаңыз](../../instructions/access-management/access-keys) VK Object Storage үшін VK Cloud жеке кабинетінде. Құпия қатынау кілтін (**Secret Key**) сақтап қойыңыз.
1. Қажетті бакетті таңдаңыз немесе [жаңасын жасаңыз](../../instructions/buckets/create-bucket).
1. Диск-О:-ны [орнатыңыз](https://disk-o.cloud/ru).
1. Объектілік қойма бакетіне қосылуды баптаңыз:
    1. **Подключить** түймесін басыңыз.
    1. **MSC S3** таңдаңыз.
    1. Қосылу баптауларын көрсетіңіз:

        - **Выберите хранилище**: `Горячие данные` — [сақтау класы](../../concepts/about#storage_class) Hotbox болатын бакет үшін, `Холодные данные` — [сақтау класы](../../concepts/about#storage_class) Icebox болатын бакет үшін;
        - **Введите бакет**: бар бакеттің атауы.
        - **Access Key**: VK Object Storage аккаунтын жасау кезінде алынған кілт идентификаторы.
        - **Secret Key**: VK Object Storage аккаунтын жасау кезінде алынған құпия қатынау кілті. Құпия кілт **Access Key** өрісінде көрсеткен кілт идентификаторына сәйкес болуы керек.
    1. **Подключить** түймесін басыңыз.

Сәтті қосылғаннан кейін бакет объектілері жергілікті файлдық жүйеде және Диск-О: жылдам қол жеткізу терезесінде қолжетімді болады. Егер бакетте объектілер болмаса, қосылған диск бос болады. Диск директориясына 1 ГБ-қа дейінгі файлды қосып, VK Cloud жеке кабинеті арқылы бакетте объект пайда болғанына көз жеткізіңіз.

{note:info}

Диск-О: арқылы жасалатын объектілер бакеттен мұраланатын сақтау класына ие болады.

{/note}

## s3fs

s3fs файлдық менеджері бакетті FUSE арқылы монттауға және VK Object Storage объектілерімен жергілікті файлдық жүйеңізде немесе CLI арқылы жұмыс істеуге мүмкіндік береді. Linux, macOS, Windows және FreeBSD үшін қолжетімді.

1. [Аккаунт пен қатынау кілтін жасаңыз](../../instructions/access-management/access-keys) VK Object Storage үшін VK Cloud жеке кабинетінде. Құпия қатынау кілтін (**Secret Key**) сақтап қойыңыз.
1. Қажетті бакетті таңдаңыз немесе [жаңасын жасаңыз](../../instructions/buckets/create-bucket).
1. s3fs орнатыңыз.

    {tabs}

    {tab(Debian 9 или новее)}

    Команданы орындаңыз:

    ```console
    sudo apt install s3fs
    ```

    {/tab}

    {tab(Ubuntu 16.04 или новее)}

    Команданы орындаңыз:

    ```console
    sudo apt install s3fs
    ```

    {/tab}

    {tab(CentOS 7 или новее)}

    EPEL ішінде команданы орындаңыз:

    ```console
    sudo yum install epel-release
    sudo yum install s3fs-fuse
    ```

    {/tab}

    {tab(SUSE 12 и openSUSE 42.1)}

    Команданы орындаңыз:

    ```console
    sudo zypper install s3fs
    ```

    {/tab}

    {tab(macOS)}

    Homebrew ішінде команданы орындаңыз:

    ```console
    brew cask install osxfuse
    brew install s3fs
    ```

    {/tab}

    {/tabs}

1. Объектілік қоймаға қосылуды баптаңыз:

    1. Кілт идентификаторы мен құпия кілтті `~/.passwd-s3fs` файлына `<ИДЕНТИФИКАТОР_КЛЮЧА>:<СЕКРЕТНЫЙ_КЛЮЧ>` пішімінде сақтаңыз:

        ```console
        echo <ИДЕНТИФИКАТОР_КЛЮЧА>:<СЕКРЕТНЫЙ_КЛЮЧ> >  ~/.passwd-s3fs
        ```
    1. `~/.passwd-s3fs` файлына қатынауды шектеңіз:

        ```console
        chmod 600  ~/.passwd-s3fs
        ```
    1.  Бакет монтталатын директорияны таңдап, осы директорияға оқу және жазу құқықтары бар екеніне көз жеткізіңіз.
    1.  Команданы орындаңыз:

        ```console
        s3fs <ИМЯ_БАКЕТА> <ПУТЬ> -o passwd_file=~/.passwd-s3fs -o url=<ENDPOINT_URL> -o use_path_request_style
        ```
        Мұнда:

        - `<ИМЯ_БАКЕТА>` — VK Object Storage жүйесіндегі бакет атауы;
        - `<ПУТЬ>` — бакет мазмұны көрсетілетін директорияға дейінгі жергілікті файлдық жүйедегі жол;
        - `<URL_СЕРВИСА>` — аккаунттың [аймағына](../../../../tools-for-using-services/account/concepts/regions) сәйкес болуы керек:
            - `https://hb.kz-ast.vkcloud-storage.ru` — Қазақстан аймағы үшін.
            - `https://hb.kz-ast.vkcloud-storage.ru` — для региона Казахстан.

Егер қосылу сәтті болса, `<ПУТЬ>` директориясында қосылған бакеттің барлық объектілері көрсетілуі керек. Егер бакетте объектілер болмаса, директория бос болады. Директорияға 1 ГБ-қа дейінгі файлды қосып, VK Cloud жеке кабинеті арқылы бакетте объект пайда болғанына көз жеткізіңіз.

{note:info}

s3fs арқылы жасалатын объектілер бакеттен мұраланатын [сақтау класына](../../concepts/about#storage_class) ие болады.

{/note}
