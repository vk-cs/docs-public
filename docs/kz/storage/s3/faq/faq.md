{cut(VK Object Storage жүйесіне сұраулар лимиті қандай?)}

* Қалыпты сұраулар — секундына 1000 сұраудан артық емес.
* Тізімдеуге арналған сұраулар — секундына 250 сұраудан артық емес.

Сұраулар лимиті әрбір бакет үшін жеке орнатылады. Егер сұраулар саны лимиттен асып кетсе, сервис `503` HTTP жауап кодын қайтарады.

[VK Object Storage сервисінің лимиттері](../../../tools-for-using-services/account/concepts/quotasandlimits#object_storage_limits) туралы толығырақ.
{/cut}

{cut(VK Object Storage бакетінің жалпы өлшемін қалай көруге болады?)}
Бакеттің жалпы өлшемін AWS CLI көмегімен көруге болады:

1. Егер әлі орнатылмаған болса, [AWS CLI](../connect/s3-cli) орнатып, баптаңыз.

1. Консольде келесі пәрменді орындаңыз:

   ```console
   aws s3 ls s3://<ИМЯ_БАКЕТА> --summarize --human-readable --recursive --endpoint-url <ENDPOINT_URL>
   ```
   Мұнда:

   - `<ИМЯ_БАКЕТА>` — жалпы өлшемін көру қажет бакеттің атауы.
   - `<ENDPOINT_URL>` — аккаунттың [аймағына](/kz/tools-for-using-services/account/concepts/regions) сәйкес келуі тиіс:
      - `https://hb.vkcloud-storage.ru` немесе `https://hb.ru-msk.vkcloud-storage.ru` — Мәскеу аймағы үшін;
      - `https://hb.kz-ast.vkcloud-storage.ru` — Қазақстан аймағы үшін.


   Бакеттің жалпы өлшемі пәрмен жауабындағы `Total Size` өрісінде көрсетіледі.
{/cut}

Деректерді резервтік көшіру VK Object Storage функциялары арқылы орындалады және жеке сервис ретінде ұсынылмайды.

{cut(VK Object Storage объектілерін жеке кабинетке кірмей қалай жүктеп алуға болады?)}

{cut(Мен бакеттегі объектілер үшін өмірлік цикл ережелерін орнаттым. Тазалау қаншалықты жиі жүргізіледі?)}

- объект орналасқан бакет үшін өмірлік цикл ережелері қосылған кезде.

Объектілер фондық режимде жойылады: олар белгіленген уақытқа жеткенде қолжетімсіз болады және кезек тәртібімен жойылады.

Егер өмірлік цикл операцияларын орындау кезегі үлкен болса, жою ұзаққа созылуы мүмкін.

{/cut}
