# {heading(Объектілерді шифрлау)[id=s3-instructions-encryption]}

{include(/kz/_includes/_translated_by_ai.md)}

{var(s3)} жүйесінде объектілерді екі режимде шифрлауға болады:

- Сервер жағындағы шифрлау (Server-side encryption, SSE), кілттерді басқару мүмкіндігінсіз (SSE-E).
- Клиент басқаратын кілттер көмегімен шифрлау (SSE-C).

{note:warn}
Шифрлау параметрлеріне қарамастан, өлшемі 21 байттан кіші объектілер шифрланбаған түрде сақталады.
{/note}

## {heading(Сервер жағындағы шифрлау)[id=s3-instructions-encryption-server]}

SSE-E Envelope Encryption сұлбасы бойынша орындалады. Әр объект шифрлау кілттерін басқару қызметі (Key Management Service, KMS) арқылы бірегей кілтпен шифрланады. {var(s3)} жүйесінде KMS ретінде Hashicorp Vault қолданылады. Қосымша қорғаныс ретінде SSE кілтті өзі тұрақты түрде ауыстырып отыратын түбірлік кілтпен шифрлайды. Деректерді шифрлау үшін 256-биттік Advanced Encryption Standard (AES-256) қолданылады.

Объектілер бакетке сақтаудың алдында шифрланады және одан жүктеп алу кезінде дешифрланады. Сервер жағындағы шифрлау шифрлаудың екі түрін қарастырады:

- Әдепкі бойынша бакетті шифрлау. Объектіні жүктеу және жүктеп алу тақырыптарды көрсетусіз орындалады. Шифрлау барлық жаңа объектілерге қолданылады, ал бұрын жүктелген объектілер өзгеріссіз қалады. Бакет шифрлауы өшірілген кезде тиісінше: барлық жаңа объектілер шифрлаусыз жүктеледі.
- Шифрланбаған бакетке жүктелетін объектіні шифрлау. Объектіні жүктеген кезде `x-amz-server-side-encryption` тақырыбын көрсетіңіз: AES-256. Объектіні жүктеп алу кезінде бұл тақырып көрсетілмейді.

Бакетті әдепкі бойынша шифрлауды орнату үшін келесі команданы орындаңыз:

```console
aws s3api put-bucket-encryption \
   --bucket <ИМЯ_БАКЕТА> \
   --endpoint-url=<URL_АДРЕС_СЕРВИСА> \
   --server-side-encryption-configuration '{
      "Rules": [
         {
         "ApplyServerSideEncryptionByDefault": {
  	        "SSEAlgorithm": "AES256"}}]}'
```

Бакет шифрлауын жою үшін келесі команданы орындаңыз:

```console
aws s3api delete-bucket-encryption \
   --bucket <ИМЯ_БАКЕТА> \
   --endpoint-url=<URL_АДРЕС_СЕРВИСА>
```

Бакет шифрлауының конфигурациясын алу үшін келесі команданы орындаңыз:

```console
aws s3api get-bucket-encryption \
   --bucket <ИМЯ_БАКЕТА> \
   --endpoint-url=<URL_АДРЕС_СЕРВИСА>
```

Шығыс мысалы:

```console
{
   "ServerSideEncryptionConfiguration": {
      "Rules": [
         {
            "ApplyServerSideEncryptionByDefault": {
               "SSEAlgorithm": "AES256"
            }
         }
      ]
   }
}
```

## {heading(Пайдаланушы басқаратын кілттер көмегімен шифрлау)[id=s3-instructions-encryption-key]}

SSE-C шифрлауы пайдаланушы ұсынған кілтпен орындалады. Кілт тек объектіні шифрлау және дешифрлау үшін қолданылады. Сұраудан кейін кілт жойылады және объектінің метадеректерінде сақталмайды. Шифрлау кілті API арқылы сұрау кезінде көрсетіледі.

Пайдаланушы кілттерімен шифрлауды пайдалану үшін:

1. AES-256 алгоритмімен шифрлау кілтін генерациялаңыз. Ол үшін MD5 хешін есептеңіз.
1. Объектіні жүктеп, Base64 форматында кодталған кілт пен хешті тақырыптарда беріңіз:

   - `x-amz-server-side-encryption-customer-key`.
   - `x-amz-server-side-encryption-customer-key-MD5`.
   - `x-amz-server-side-encryption-customer-algorithm: AES256`.
