VK Cloud келесі сущностер үшін [сақтау кластарының](../../concepts/about#storage_class) өзгертілуін қолдайды:

- бакет,
- объект,
- multipart жүктеу.

{note:err}

Сақтау класын өзгерту кезінде трафик беріледі, ол [тарифтеледі](../../tariffication).

{/note}

## Бакеттің сақтау класын өзгерту

{note:warn}

Бакеттің сақтау класын өзгерткен кезде оған бұрыннан кіретін объектілердің сақтау класы өзгеріссіз қалады. Жаңа класс кейін қосылатын жаңа объектілерге ғана қолданылады.

{/note}

{tabs}

{tab(Жеке кабинет)}

{include(/kz/_includes/_s3-open-bucket.md)}

1. **Сақтау класы** қойындысына өтіңіз.
1. Қажетті класты белгілеп, **Өзгерістерді сақтау** батырмасын басыңыз.
   
   Енді осы бакетке қосатын жаңа объектілер тиісті сақтау класына ие болады. Бұрын қосылған объектілердің сақтау класы өзгермейді. 
   
   Әр объектінің сақтау класын бакет объектілері тізіміндегі **Сақтау класы** бағанынан [қарауға болады](/kz/storage/s3/instructions/objects/manage-object#obektiler_tizimin_karau).

{/tab}

{/tabs}

## Объектінің сақтау класын өзгерту

{tabs}

{tab(AWS CLI)}

1. Егер әлі орнатылмаған болса, [AWS CLI](../../connect/s3-cli) орнатып, баптаңыз.

1. Консольде команданы орындаңыз:

   - бакеттегі бір объект үшін:

   ```console
   aws s3 cp s3://<ИМЯ_БАКЕТА>/<КЛЮЧ_ОБЪЕКТА> s3://<ИМЯ_БАКЕТА>/ --storage-class <КЛАСС_ХРАНЕНИЯ> --endpoint-url <URL_СЕРВИСА>
   ```

   - бакеттегі барлық объектілер үшін:

   ```console
   aws s3 cp s3://<ИМЯ_БАКЕТА>/ s3://<ИМЯ_БАКЕТА>/ --recursive --storage-class <КЛАСС_ХРАНЕНИЯ> --endpoint-url <URL_СЕРВИСА>
   ```

   Мұнда:

   - `<ИМЯ_БАКЕТА>` — қажетті объект немесе объектілер орналасқан бакет атауы.
   - `<КЛЮЧ_ОБЪЕКТА>` — объектінің толық атауы, оған дейінгі жолды қоса алғанда.
   - `<КЛАСС_ХРАНЕНИЯ>` — объектілерге тағайындалатын сақтау класы.
   - `<URL_СЕРВИСА>` — VK Object Storage сервисінің домені, ол аккаунттың [аймағына](/kz/tools-for-using-services/account/concepts/regions) сәйкес болуы тиіс:
      - `https://hb.vkcloud-storage.ru` немесе `https://hb.ru-msk.vkcloud-storage.ru` — Мәскеу аймағының домені;
      - `https://hb.kz-ast.vkcloud-storage.ru` — Қазақстан аймағының домені.

{/tab}

## Multipart жүктеудің сақтау класын өзгерту

Сақтау класы multipart жүктеуді инициализациялау немесе аяқтау кезеңінде (`InitiateMultipart` және `CompleteMultipart` операциялары) беріледі. Егер класс көрсетілмесе, бакеттің сақтау класы тағайындалады.

Multipart жүктеу арқылы бакетке орналастырылған объектінің сақтау класын өзгерту үшін:

{tabs}

{tab(AWS CLI)}

1. Егер әлі орнатылмаған болса, [AWS CLI](../../connect/s3-cli) орнатып, баптаңыз.
1. Multipart жүктеудің аяқталуын күтіңіз.
1. Консольде команданы орындаңыз:

   - бакеттегі бір объект үшін:

