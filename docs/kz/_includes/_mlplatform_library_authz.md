{includetag(intro_part)}

Python-библиотека Cloud ML Platform ұсынатын әдістердің көпшілігін пайдалану үшін _қолжеткізу токені_ арқылы авторизациядан өту қажет.

Қолжеткізу токендерінің (refresh-токендердің) қолданылу аясын анықтайтын келесі рөлдердің бірі болады:

- `Пайдаланушы`. Бұл рөлдегі токен кітапханада қолжетімді әдістердің көпшілігін пайдалануға мүмкіндік береді.
- `Әкімші`. Бұл рөлдегі токен кітапханада қолжетімді барлық әдістерді пайдалануға мүмкіндік береді. Мысалы, осы рөлдегі қолжеткізу токені болса, басқа токендерді басқаруға болады.

{/includetag}

{includetag(main_part)}

Қолжеткізу құқықтарына байланысты токенді әртүрлі тәсілмен алуға болады:

- VK Cloud жеке кабинеті арқылы (әдетте `Әкімші` рөлі бар қолжеткізу токені);
- Cloud ML Platform кітапханасы арқылы.

## {heading(Жеке кабинетте қолжеткізу токенін жасау)[id=create_token_management]}

Егер пайдаланушының VK Cloud жеке кабинетіне қолжетімділігі болса, ол қолжеткізу токенін өзі жасап, екі рөлдің бірін таңдай алады. Мұндай токеннің әрекет ету мерзімі шектелмеген.

1. VK Cloud жеке кабинетіне [өтіңіз](https://kz.cloud.vk.com/app/).
1. Токен жасалуы керек жобаны таңдаңыз.
1. **ML Platform → Токендер** бөліміне өтіңіз.
1. **Қолжеткізу токенін жасау** түймесін басыңыз.
1. Ашылған терезеде токен параметрлерін көрсетіңіз:

    - **Атауы**: кез келген болуы мүмкін.
    - **Рөлі**: таңдалған рөл (`Пайдаланушы` немесе `Әкімші`).

1. **Жасау** түймесін басыңыз.
1. Ашылған терезеде токен мәнін көшіріп, оны өз құрылғыңызға сақтаңыз. Терезе жабылғаннан кейін токен мәнін қалпына келтіру мүмкін болмайды. Егер ол жоғалса, жаңа токен жасаңыз.
1. **Дайын** түймесін басыңыз.

{note:err}

Қолжеткізу токеніңізді ешкімге бермеңіз.

{/note}

## {heading(Cloud ML Platform кітапханасы арқылы қолжеткізу токенін жасау)[id=create_token_console]}

Егер пайдаланушының VK Cloud жеке кабинетіне қолжетімділігі болмаса, қолжеткізу токенін тіркеу токені арқылы кітапхана көмегімен жасауға болады:

1. `Әкімші` рөлі бар қолжеткізу токенінің иесі кітапхана арқылы пайдаланушы үшін [тіркеу токенін (register-токен)](#create_registration_token) жасайды. Ол бір реттік және өмір сүру уақыты (TTL) шектеулі болады. Тіркеу токенінде қолжеткізу токенінің барлық параметрлері беріледі: атауы, рөлі, өмір сүру уақыты.

2. Пайдаланушы тіркеу токені арқылы кітапханада [қолжеткізу токенін (refresh-токен)](#create_access_token) жасап, кітапхана әдістерін шақырған кезде авторизациядан өтеді. Қолжеткізу токенінің барлық параметрлерін әкімші тіркеу токенінде алдын ала орнатады, оларды өзгертуге болмайды.

### {heading(Тіркеу токенін жасау)[id=create_registration_token]}

Тіркеу токені тек Cloud ML Platform кітапханасы арқылы жасалады.

{note:err}

Қарапайымдылық үшін Python скрипті мысалында қолжеткізу токенінің мәні көрсетілген, ал тіркеу токенінің мәні `print()` көмегімен шығарылады.

Production-ортада токендермен ашық түрде жұмыс істемеңіз. Сезімтал деректермен жұмыс істеу үшін орта айнымалыларын, құпиялар қоймаларын немесе басқа құралдарды пайдаланыңыз.

{/note}

1. Егер мұны әлі жасамаған болсаңыз, [кітапхананы орнатыңыз](/kz/ml/mlplatform/mlplatform-lib/lib-install).
1. Егер мұны әлі жасамаған болсаңыз, `Әкімші` рөлі бар [қолжеткізу токенін жасаңыз](#create_token_management).
1. Python скриптін орындаңыз:

   ```python
   from mlplatform_client import MLPlatform
   from mlplatform_client.serializers.auth import MLPTokenType

   ADMIN_REFRESH_TOKEN = "<ТОКЕН_АДМИНИСТРАТОРА>"

   mlp = MLPlatform(ADMIN_REFRESH_TOKEN)

   register_token = mlp.create_register_token(
         client_name="<ИМЯ_ТОКЕНА_ДОСТУПА>",
         access_ttl="<ВРЕМЯ_ЖИЗНИ_РЕГИСТРАЦИОННОГО_ТОКЕНА>",
         refresh_ttl="<ВРЕМЯ_ЖИЗНИ_ТОКЕНА_ДОСТУПА>",
         token_type=<РОЛЬ_ТОКЕНА_ДОСТУПА>)

   print(register_token)
   ```

   Мұнда:
    - `<ТОКЕН_АДМИНИСТРАТОРА>`: `Әкімші` рөлі бар қолжеткізу токенінің мәні.
    - `client_name`: осы тіркеу токені арқылы жасалатын қолжеткізу токенінің атауы.
    - `access_ttl`: тіркеу токенінің өмір сүру уақыты, мысалы, `2h45m30s`. Өмір сүру мерзімі біткен тіркеу токенін қолжеткізу токенін жасау үшін пайдалану мүмкін болмайды.
    - `refresh_ttl`: қолжеткізу токенінің өмір сүру уақыты, мысалы, `2h`. Егер бұл параметр көрсетілмесе, қолжеткізу токені мерзімсіз әрекет етеді.

    - `token_type`: қолжеткізу токенінің рөлі:

        - `MLPTokenType.USER` мәні `Пайдаланушы` рөліне сәйкес келеді;
        - `MLPTokenType.ADMIN` мәні `Әкімші` рөліне сәйкес келеді.

   Осы скрипт орындалғаннан кейін жасалған тіркеу қолжеткізу токенінің мәні шығарылады.

1. Басқа пайдаланушы берілген параметрлері бар қолжеткізу токенін жасау үшін осы тіркеу токенін пайдалана алатындай, оған бұл тіркеу токенін беріңіз.

### Тіркеу токенін пайдаланып қолжеткізу токенін жасау

{note:err}

Қарапайымдылық үшін Python скрипті мысалында тіркеу токенінің мәні көрсетілген, ал қолжеткізу токенінің мәні `print()` көмегімен шығарылады.

Production-ортада токендермен ашық түрде жұмыс істемеңіз. Сезімтал деректермен жұмыс істеу үшін орта айнымалыларын, құпиялар қоймаларын немесе басқа құралдарды пайдаланыңыз.

{/note}

1. Егер мұны әлі жасамаған болсаңыз, [кітапхананы орнатыңыз](/kz/ml/mlplatform/mlplatform-lib/lib-install).
1. Тіркеу токенінің бар екеніне көз жеткізіңіз. Егер ол жоқ болса, `Әкімші` рөлі бар қолжеткізу токенінің иесінен [тіркеу токенін жасауды](#create_registration_token) сұраңыз.
1. Python скриптін орындаңыз:

   ```python
   from mlplatform_client import MLPlatform

   REGISTER_TOKEN = "<ЗНАЧЕНИЕ_РЕГИСТРАЦИОННОГО_ТОКЕНА>"

   mlp = MLPlatform()
   refresh_token = mlp.create_refresh_token(REGISTER_TOKEN)

   print(refresh_token)
   ```

   Осы скрипт орындалғаннан кейін жасалған қолжеткізу токенінің мәні шығарылады.

   Қолжеткізу токенінің параметрлерін (атауы, рөлі және өмір сүру уақыты) тіркеу токені анықтайды.

1. Токеннің мәнін көшіріп, оны өз құрылғыңызға сақтаңыз.

{note:err}

Қолжеткізу токеніңізді ешкімге бермеңіз.

{/note}

## {heading(Токен арқылы авторизация)[id=authorisation_by_token]}

{note:err}

Қарапайымдылық үшін Python скрипті мысалында қолжеткізу токенінің мәні көрсетілген. Production-ортада токендермен ашық түрде жұмыс істемеңіз. Сезімтал деректермен жұмыс істеу үшін орта айнымалыларын, құпиялар қоймаларын немесе басқа құралдарды пайдаланыңыз.

{/note}

1. [Жеке кабинет арқылы](#create_token_management) немесе [Cloud ML Platform кітапханасы арқылы](#create_token_console) қолжеткізу токенін жасаңыз.

   Қандай әдістерді пайдалану жоспарланғанына байланысты белгілі бір рөлі бар қолжеткізу токені қажет болуы мүмкін. [Кітапхана әдістерінің анықтамалығында](/kz/ml/mlplatform/mlplatform-lib/lib-reference) нақты әдісті шақыру үшін токенде қандай рөл болуы керектігі көрсетіледі.

1. MLPlatform класының объектісін жасап, конструкторға алынған токеннің мәнін беріңіз.

   **Мысал:**

   ```python
   from mlplatform_client import MLPlatform

   REFRESH_TOKEN = "<ЗНАЧЕНИЕ_ТОКЕНА_ДОСТУПА>"
   mlp = MLPlatform(REFRESH_TOKEN)
   ```

Енді осы объектіні пайдаланып, қажет әдістерді шақыра аласыз. Объектіні жасау кезінде көрсетілген токен әдісті шақырған кезде авторизация үшін пайдаланылады.

Егер авторизация сәтсіз болса, келесіге ұқсас қате туралы хабар шығады:

- Егер токен рөлі әдісті шақыру үшін жеткіліксіз болса:

  ```text
  mlplatform_client.core.exceptions.ForbiddenException: <Response [403]>. Info: {"status":403,"warnorCode":"ForbiddenException","warnorParams":{"field":"KtoTam token does not contain the following scopes: ['ml-platform:Manage']"}}
  ```

- Егер токеннің өмір сүру уақыты аяқталса:

  ```text
  mlplatform_client.core.exceptions.ForbiddenException: <Response [403]>. Info: {"status":403,"warnorCode":"ForbiddenException","warnorParams":{"field":"Forbidden"}}
  ```
