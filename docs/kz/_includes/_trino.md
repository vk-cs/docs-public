1. Дереккөзді таңдаңыз:

    - `PostgreSQL` — PostgreSQL деректер базасына қосылу.
    - `Greenplum` — Greenplum деректер базасына қосылу.
    - `Clickhouse` — ClickHouse деректер базасына қосылу.
    - `S3 VK Cloud бар Apache Iceberg` — сол жобадағы VK Object Storage-ге қосылу.
    - `сыртқы S3 бар Apache Iceberg` — сыртқы S3 объектілік сақтау орнына қосылу.
    - `деректері HDFS-те сақталатын Hive Metastore` — HDFS сақтау орны бар Hive Metastore серверіне қосылу.
    - `деректері сыртқы S3-те сақталатын Hive Metastore` — S3 сақтау орны бар Hive Metastore серверіне қосылу.
    - `деректері VK Cloud S3-те сақталатын Hive Metastore` — [VK Object Storage](/kz/storage/s3/concepts/about) сақтау орны бар Hive Metastore серверіне қосылу.

   {tab(PostgreSQL, Greenplum, Clickhouse)}

    - **Атауы**: қосылым атауы.
    - **db_name**: Trino қосылатын деректер базасының атауы.
    - **hostname**: қосылуға арналған сервер мекенжайы. Мысал: `postgresql-primary.postgresqlcluster-gttxnz96.svc:5432`.
    - **username**: деректер базасы пайдаланушысының есептік жазба атауы.
    - **password**: деректер базасы пайдаланушысының есептік жазба құпиясөзі.
    - **tls_crt**: TLS-сертификат файлына жол. TLS-сертификатты пайдалану деректерді тасымалдау кезінде қорғауға және қосылым қауіпсіздігін арттыруға көмектеседі.

   {tab(S3 VK Cloud)}

    - **Атауы**: қосылым атауы.
    - **s3_bucket**: Trino қосылатын VK Object Storage бакетінің атауы.
    - **s3_folder**: Trino қосылатын VK Object Storage бакетіндегі директория атауы.
    - **parquet.small-file-threshold**: файлдың шекті өлшемі. Өнімділікті жақсарту және сақтау орнындағы файлдар санын азайту үшін өлшемі кішірек файл арнайы түрде өңделуі мүмкін (мысалы, басқа файлдармен біріктірілуі мүмкін).

   {/tab}

    - **s3_secret_key**: сақтау орнына қол жеткізуге арналған жеке кілт.
    - **s3_endpoint**: S3 сақтау орнының URL мекенжайы.
    - **s3_bucket**: Trino қосылатын S3 бакетінің атауы.
    - **s3_folder**: Trino қосылатын S3 бакетіндегі директория атауы.
    - **s3_region**: S3 сақтау орны орналасқан аймақ.
    - **parquet.small-file-threshold**: файлдың шекті өлшемі. Өнімділікті жақсарту және сақтау орнындағы файлдар санын азайту үшін өлшемі кішірек файл арнайы түрде өңделуі мүмкін (мысалы, басқа файлдармен біріктірілуі мүмкін).

   {tab(Hive Metastore (HDFS))}

    - **Атауы**: қосылым атауы.
    - **Hive Metastore Thrift сервисінің hostname:port мәні**: `<FQDN>:<ПОРТ>` немесе `<IP-АДРЕС>:<ПОРТ>` форматындағы кілт/мән жұбы. Деректер Hive Metastore Thrift сервисі үшін көрсетіледі.
    - **Cloud Trino сервисінің principal-ына арналған base64 форматындағы Keytab файлы**: base64 форматына қайта кодталған Keytab файлының мазмұны.

      HDFS сақтау орны бар Hive Metastore кластеріне қосылу үшін Kerberos есептік деректері пайдаланылады. Жаңа клиент principal-ын жасау және оған Keytab файлын алу үшін Hive Metastore әкімшісіне жүгініңіз. Бұл файлды base64 форматына қайта кодтау керек.

    - **KDC сервисінің hostname:port мәні**: `<FQDN>:<ПОРТ>` немесе `<IP-АДРЕС>:<ПОРТ>` форматындағы кілт/мән жұбы. Деректер KDC (Key Distribution Center) сервисі үшін көрсетіледі.
    - **Hive Metastore сервисінің principal атауы**: Hive Metastore жұмыс істейтін principal атауы.
    - **Клиенттің principal атауы**: жасалған Keytab файлы үшін пайдаланылған principal атауы.
    - **User Impersonation функциясын қосу**: Cloud Trino-да сұрауды бастамалаған пайдаланушы атынан Hive Metastore-ға сұрауларды орындау үшін опцияны қосыңыз.
    - **Сыртқы Hive кестелеріне жазу**: Cloud Trino Hive Metastore-ға қосылған сыртқы (external) кестелерге жазба жүргізе алуы үшін опцияны қосыңыз. Опция өшірулі болса, тек басқарылатын (managed) кестелерге жазу қолжетімді болады.
