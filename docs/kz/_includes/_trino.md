1. Дереккөзді таңдаңыз:

    - `PostgreSQL` — PostgreSQL деректер базасына қосылу.
    - `Greenplum` — Greenplum деректер базасына қосылу.
    - `Clickhouse` — ClickHouse деректер базасына қосылу.
    - `MySQL` — MySQL дерекқорына қосылу.
    - `MariaDB` — MariaDB дерекқорына қосылу.
    - `Redis` — Redis дерекқорына қосылу.
    - `Деректерді S3 VK Cloud-та сақтау арқылы Iceberg` — сол жобадағы VK Object Storage қызметіне қосылу.
    - `Деректерді сыртқы S3-те сақтау арқылы Iceberg` — сыртқы S3 объектілік қоймасына қосылу.
    - `деректері HDFS-те сақталатын Hive Metastore` — HDFS сақтау орны бар Hive Metastore серверіне қосылу.
    - `деректері сыртқы S3-те сақталатын Hive Metastore` — S3 сақтау орны бар Hive Metastore серверіне қосылу.
    - `деректері VK Cloud S3-те сақталатын Hive Metastore` — [VK Object Storage](/kz/storage/s3/concepts/about) сақтау орны бар Hive Metastore серверіне қосылу.

   {tab(Hive Metastore (HDFS))}

      HDFS қоймасы бар Hive Metastore кластеріне қосылу үшін Kerberos есептік деректері қолданылады. Жаңа клиент принципалын (principal) жасау және оған арналған Keytab-файлды алу үшін Hive Metastore әкімшісіне жүгініңіз. Бұл файлды base64 форматына қайта кодтау қажет.

      {cut(base64 форматындағы мәнді қалай алуға болады)}

      {tabs}

      {tab(macOS)}

      Команданы орындаңыз:

       ```bash
       base64 -i <ПУТЬ_ДО_KEYTAB-ФАЙЛА>
       ```

      {/tab}

      {tab(Linux)}

      Команданы орындаңыз:

       ```bash
       base64 -w 0 <ПУТЬ_ДО_KEYTAB-ФАЙЛА>
       ```

      {/tab}

      {tab(Windows)}

      PowerShell ішінде команданы орындаңыз:

       ```ps
       [Convert]::ToBase64String([IO.File]::ReadAllBytes("<ПУТЬ_ДО_KEYTAB-ФАЙЛА>"))
       ```

      {/tab}

      {/tabs}

      {/cut}

    - **Hive Metastore каталогының атауы**: Hive Metastore әкімшісінен сұратыңыз. Hive Metastore каталогтарының тізімін алу үшін мына сұрау пайдаланылады:

       ```sql
       SHOW CATALOGS;
       ```

      Дәл осы каталог атауы Cloud Trino ішінде Hive Metastore қосылымына сұраулар жасау үшін пайдаланылады.

    - **KDC сервисінің hostname:port мәні**: `<FQDN>:<ПОРТ>` немесе `<IP-АДРЕС>:<ПОРТ>` форматындағы кілт/мән жұбы. Деректер KDC (Key Distribution Center) сервисі үшін көрсетіледі.
    - **Hive Metastore сервисінің principal атауы**: Hive Metastore жұмыс істейтін principal атауы.
    - **Клиенттің principal атауы**: жасалған Keytab файлы үшін пайдаланылған principal атауы.
