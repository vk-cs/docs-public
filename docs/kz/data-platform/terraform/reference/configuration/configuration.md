# {heading(Конфигурация файлдары)[id=terraform_configuration]}

{include(/kz/_includes/_translated_by_ai.md)}

Ресурстарды жасау кезінде Terraform конфигурация файлдарымен жұмыс істейді. Олардың кеңейтімі `.tf`.

Әрбір {var(data-p)} жобасы үшін бөлек жұмыс директориясын жасау ұсынылады. Жұмыс директориясында келесі файлдар орналастырылады:

- провайдер конфигурациясының файлы (жобаға тән);
- жасауды жоспарлап отырған ресурстардың конфигурация файлдары.

Terraform дұрыс жұмыс істеуі үшін қосымша файл — Terraform айнасының конфигурация файлы қажет.

## {heading(Terraform провайдері конфигурациясының файлы)[id=terraform_configuration_provider]}

Файлда Terraform-провайдерлер туралы ақпарат және жоба ішіндегі пайдаланушыны аутентификациялауға арналған деректер бар.

Файл атауы кез келген болуы мүмкін. Файлда {var(data-p)} жүйесіне қосылудың негізгі баптаулары бар екенін көрсету үшін `provider.tf` атауын пайдалану ұсынылады.

Файл барлық Terraform жұмыс директорияларында орналастырылуы тиіс.

Файлды жеке кабинеттегі **Жоба баптаулары** бетінің **Terraform** қойындысынан [жүктеп алуға](../../quick_start#podgotovitelnye_shagi) болады. Жүктелген файлдың атауы `vkcs_provider.tf` болады және бір провайдер (`vkcs`) туралы деректерді қамтиды. Жүктеп алғаннан кейін файлды өңдеңіз — `password` параметрінде тіркелгіңіздің құпиясөзін көрсетіңіз.

Файл мазмұнының мысалы:

```console
terraform {
    required_providers {
        vkcs = {
            source = "vk-cs/vkcs"
        }
    }
}

provider "vkcs" {
    # Your user account.
    username = "<user_email>"

    # The password of the account
    password = "<user_password>"

    # The tenant token can be taken from the project Settings tab - > API keys.
    # Project ID will be our token.
    project_id = "<project_ID>"

    # Region name
    region = "<Region>"

    auth_url = "<auth_url>"
}
```

Файл келесі секциялардан тұрады:

- `terraform.required_providers {}` — пайдаланылатын провайдерлер тізімі. {var(data-p)} жүйесімен жұмыс істеу үшін `vkcs` провайдері, оның көзі және нұсқалары көрсетілуі тиіс. Егер қосымша провайдерлерді пайдалануды жоспарласаңыз, оларды осы секцияға қосыңыз.
- `provider "vkcs" {}` — жоба ішіндегі аутентификация баптаулары:

    - `password`: тіркелгіңіздің құпиясөзін көрсетіңіз;
    - `username`, `project_id`, `region`: жеке кабинет арқылы жүктелген файлда көрсетілген мәндер ағымдағы жобамен жұмыс істеуге жарамды. Бұл мәндер **Жоба баптаулары** бетінің **Terraform** қойындысында да көрсетілген.

Кейбір параметрлердің мәндері аймаққа байланысты:

- `region`:

    - Мәскеу аймағы үшін: `RegionOne`;
    - Қазақстан аймағы үшін: `kz`;

- `auth_url`:

    - Мәскеу аймағы үшін: `https://infra.mail.ru:35357/v3/`;
    - Қазақстан аймағы үшін: `https://kz.infra.mail.ru:35357/v3/`.

## {heading(Ресурстар конфигурациясының файлдары)[id=terraform_configuration_resource]}

Ресурстар конфигурациясының файлдары Terraform көмегімен жасалатын ресурстарды, олардың баптауларын және тәуелділіктерін сипаттайды.

Конфигурация файлдарының атаулары кез келген болуы мүмкін. Бұл құжаттамадағы мысалдарда мынадай атаулау қағидалары қолданылады:

- `variables.tf` — конфигурацияда пайдаланылатын айнымалылар. Жиі өзгеретін параметрлерді айнымалыларға шығару жаңа жоба үшін инфрақұрылым конфигурациясын оңай өзгертуге мүмкіндік береді.
- `main.tf` — белгілі бір сценарийде жасалатын негізгі ресурстың сипаттамасын қамтиды. Мысалы, [Spark кластерін жасау](../../tf-spark) кезінде кластер конфигурациясын `main.tf` файлында орналастыру ұсынылады.

Кейде файлдар ресурс түрлері бойынша аталуы мүмкін, мысалы:

- `network.tf` — бұлттық желінің сипаттамасы.
- `kubernetes.tf` — Kubernetes ресурстарының сипаттамасы.

Әдетте ресурстар конфигурациясында мыналар пайдаланылады:

- `resource "resource_type" "resource_name" {}` — жасалатын [ресурс](https://www.terraform.io/language/resources/syntax), мысалы, желі, ішкі желі, Kubernetes кластері немесе дерекқор кластері.
- `data "data_type" "data_name" {}` — бұлтта немесе жергілікті түрде бар, Terraform конфигурациясынан тыс көрсетілген [деректерді](https://www.terraform.io/language/data-sources) пайдалануға мүмкіндік береді. Мысалы, ВМ конфигурациясын, кластер үлгісін/нұсқасын және т.б.
- `variable "parameter" {}` — [кіріс айнымалылары](https://www.terraform.io/language/values/variables). Конфигурацияда қолданылатын айнымалыларды жариялау үшін пайдаланылады (`variables.tf`).
- `output "parameter" {}` — [шығыс айнымалылары](https://www.terraform.io/language/values/outputs). Деректерді командалық жолға шығарады.

Ресурстарды жасау ретін және олардың тәуелділіктерін анықтау үшін `depends_on` мета-аргументін пайдалануға болады. Мета-аргумент жасалатын ресурс тәуелді болатын ресурсты көрсетеді:

```console
depends_on = [
    vkcs_kubernetes_cluster.k8s-cluster,
]
```
Алдымен мета-аргумент арқылы көрсетілген ресурс, содан кейін тәуелділігі көрсетілген ресурс жасалады.

## {heading(Terraform айнасы конфигурациясының файлы)[id=terraform_configuration_mirror]}

Бұл файл Terraform іске қосу үшін қажет, ол {var(cloud)} ұсынған Terraform айнасының мекенжайын қамтиды. Файлдың атауы Windows үшін `terraform.rc`, ал басқа ОЖ үшін `.terraformrc` болуы тиіс. Файл мазмұны барлық жобалар мен аймақтар үшін бірдей.

Файлды жеке кабинеттегі **Жоба баптаулары** бетінің **Terraform** қойындысынан [жүктеп алуға](../../quick_start#podgotovitelnye_shagi) болады. Жүктеп алғаннан кейін файлға қандай да бір өзгерістер енгізудің қажеті жоқ.

Файл келесі жерде орналастырылуы тиіс:

- Windows үшін — ағымдағы пайдаланушының **Application Data** директориясында.
- Басқа ОЖ үшін — пайдаланушының үй директориясында.

{var(cloud)} жобаларына арналған файл мазмұны:

```yaml
provider_installation {
    network_mirror {
        url = "https://terraform-mirror.mcs.mail.ru"
        include = ["registry.terraform.io/*/*"]
    }
    direct {
        exclude = ["registry.terraform.io/*/*"]
    }
}
```
