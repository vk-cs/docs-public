## {heading(Дайындық қадамдары)[id=terraform_quick_start_preparations]}

1. Terraform-ды {var(cloud)} ресми [айнасынан](https://hashicorp-releases.mcs.mail.ru/terraform) орнатыңыз.
1. {var(data-p)} жеке кабинетіне өтіңіз.
1. Егер бұл әлі жасалмаса, екі факторлы аутентификацияны баптап, API арқылы қолжетімділікті белсендіріңіз.

1. Бет тақырыбындағы пайдаланушы атына басып, ашылмалы тізімнен **Жоба баптаулары** тармағын таңдаңыз.

1. **Terraform** қойындысына өтіңіз. Аттас батырмаларды басып, негізгі [Terraform конфигурация файлын](../reference/configuration#fayl_konfiguracii_provaydera_terraform) және [Terraform айнасының конфигурация файлын](../reference/configuration#fayl_konfiguracii_zerkala_terraform) жүктеп алыңыз.

   `vkcs_provider.tf` және `terraform.rc` атауларымен файлдар жүктеледі.

1. Файлдармен келесі әрекеттерді орындаңыз:

   {tabs}

   {tab(Windows)}

    1. Windows шолғышының мекенжай жолағына `%APPDATA%` енгізіп, ашылған директорияға `terraform.rc` файлын көшіріңіз.
    1. `vkcs_provider.tf` файлын платформамен жұмыс істеуді жоспарлап отырған жұмыс директориясына көшіріңіз.

       Әрбір {var(data-p)} жобасы үшін бөлек жұмыс директориясын жасау ұсынылады.

   {/tab}

   {tab(Басқа ОЖ)}

    1. Terraform айнасының конфигурация файлын `terraform.rc` атауынан `.terraformrc` атауына қайта атаңыз.
    1. `.terraformrc` файлын пайдаланушының үй директориясының түбіріне көшіріңіз.
    1. `vkcs_provider.tf` файлын платформамен жұмыс істеуді жоспарлап отырған жұмыс директориясына көшіріңіз.

       Әрбір {var(cloud)} жобасы үшін бөлек жұмыс директориясын жасау ұсынылады.

   {/tab}

   {/tabs}

   {note:info}

   Екі конфигурация файлын да өзіңіз жасай аласыз. Сондай-ақ жүктелген файлдарды өңдей аласыз — мысалы, қосымша Terraform-провайдерді қосу үшін. Файлдардың мазмұны [Конфигурация файлдары](../reference/configuration) мақаласында сипатталған.

   {/note}

## {heading(Terraform инициализациясы)[id=terraform_quick_start_initialization]}

Жобамен жұмыс істеуді жоспарлап отырған директорияда келесі команданы орындаңыз:

```console
terraform init
```

Terraform жұмысына қажет қосымша файлдар жасалады.

## {heading(Terraform көмегімен ресурстарды жасау)[id=terraform_quick_start_create_resource]}

1. Жұмыс директориясында ресурстар конфигурациясын жасаңыз — мысалы, [Cloud Spark кластерін жасау](../tf-spark/) үшін конфигурацияны.

1. Келесі команданы орындаңыз:

    ```console
    terraform apply
    ```

   Растауды сұрағанда, `yes` енгізіңіз.

1. Операцияның аяқталуын күтіңіз.

Жасалған ресурстар сіздің жеке кабинетіңізде пайда болады.

## {heading(Terraform жаңарту)[id=terraform_quick_start_update]}

Егер `vkcs_provider.tf` файлындағы провайдер нұсқасы ресурсты жасауға қажет нұсқадан төмен болса, конфигурацияны валидациялау кезінде қате пайда болады. Оны болдырмау үшін провайдерді [соңғы нұсқаға](https://github.com/vk-cs/terraform-provider-vkcs) немесе ресурс сипаттамасында көрсетілген нұсқаға дейін жаңартыңыз:

1. `vkcs_provider.tf` файлын ашып, нұсқаны қажетті мәнге өзгертіңіз, мысалы: `version = "~> 0.8.0"`.
1. Терминалда келесі команданы орындаңыз:

    ```console
    terraform init --upgrade
    ```
