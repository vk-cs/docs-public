{includetag(intro)}

Apache Spark — үлкен деректерді үлестірілген түрде өңдеуге арналған сервис. Ол Java, Scala, Python және R тілдеріндегі API-интерфейстерден, сондай-ақ SQL үшін [Spark SQL](https://spark.apache.org/docs/latest/sql-programming-guide.html), [Pandas API](https://spark.apache.org/docs/latest/api/python/getting_started/quickstart_ps.html), машиналық оқытуға арналған [MLlib](https://spark.apache.org/docs/latest/ml-guide.html), графтарды өңдеуге арналған [GraphX](https://spark.apache.org/docs/latest/graphx-programming-guide.html) және ағындық өңдеуге арналған [Structured Streaming](https://spark.apache.org/docs/latest/structured-streaming-programming-guide.html) өңдеу құралдарынан тұрады. Spark көбінесе Hadoop-кластерінің құрамында пайдаланылады.

Cloud Spark — {var(cloud)} ұсынған [Apache Spark Operator](https://github.com/GoogleCloudPlatform/spark-on-k8s-operator) және [PaaS Kubernetes](/kz/kubernetes/k8s) негізіндегі шешім. Ол Hadoop-кластерін пайдаланбай, Docker Registry-ден алынған образдың көмегімен Kubernetes ішінде Spark-ты орналастыруға мүмкіндік береді.

## {heading(Сервис қандай міндеттерге жарайды)[id=use_cases]}

- Үлкен деректерді үлестірілген түрде өңдеу.
- Объектілік сақтау орнынан деректерді оқып, оларды кейін өңдеу үшін БД-ға экспорттау (ClickHouse, Greenplum, PostgreSQL). Сондай-ақ деректерді БД-дан объектілік сақтау орнына жіберуге болады.
- Үлкен деректерді пайдаланып ML-модельдерді үлестірілген түрде оқыту.
- [GraphX](https://spark.apache.org/docs/latest/graphx-programming-guide.html) компонентін пайдалана отырып графтық есептеулерді орындау.

{includetag(login_password)}

1. Spark-қа қол жеткізу үшін әкімші логинін көрсетіңіз. Логинге қойылатын талаптар:

    - тек сандар, латын әріптері және `_` символы рұқсат етіледі;
    - бірінші символ — кез келген регистрдегі латын әрпі немесе `_`;
    - рұқсат етілмейтін атаулар: `os_admin`, `root`, `dataplatform_moth`;
    - 50 таңбадан аспауы керек.

1. Spark-қа қол жеткізу үшін әкімші құпиясөзін көрсетіңіз. Құпиясөзді орнату үшін **Генерировать** түймесін басыңыз немесе өз құпиясөзіңізді көрсетіңіз.

    Құпиясөзге қойылатын талаптар:

    - кемінде 16 таңба;
    - латын әліпбиінің кемінде бір бас және бір кіші әрпі;
    - кемінде бір сан;
    - мына таңбалардың кемінде бірі: `!`, `?`, `%`, `#`, `/`, `(`, `)`, `-`, `+`, `*`.

    {note:warn}

    Құпиясөзді сақтап қойыңыз. Құпиясөзді қалпына келтіру мүмкіндігі қарастырылмаған.

    {/note}

{/includetag}

{includetag(connection)}

1. **Подключение қосу** түймесін басыңыз.
1. Деректер көзін таңдаңыз:

    - `S3 внешний` — сыртқы S3 объектілік сақтау орнына қосылу.
    - `S3 VK Cloud` — сол жобадағы {var(s3)} сақтау орнына қосылу.
    - `PostgreSQL` — PostgreSQL дерекқорына қосылу.
    - `Iceberg Metastore VK Cloud S3-пен` — сол VK Cloud жобасындағы Cloud Iceberg Metastore данасына қосылу.
    - `Iceberg Metastore сыртқы S3-пен` — сыртқы Cloud Iceberg Metastore каталогына қосылу.

1. Көзге қосылу параметрлерін баптаңыз:
