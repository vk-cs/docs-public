Data Lakehouse (DLH) архитектурасына жеке аналитикалық ДҚ қосу деректерді S3-қоймада сақтау қажеттілігінсіз келіп түсетін оқиғалар ағынын жылдам өңдеуге мүмкіндік береді. Мұндай ДҚ ретінде PostgreSQL пайдаланылуы мүмкін, онда жедел деректер хабарламалар ағыны немесе батчтар түрінде келіп түседі, немесе [Cloud ClickHouse](/kz/data-platform/clickhouse).

Төменде ClickHouse өнімі негізіндегі аналитикалық деректер қоры бар Data Lakehouse сервисінің конфигурация нұсқасы қарастырылады.

Аналитикалық ДҚ-ға деректер сұрауларын [Cloud Trino](/kz/data-platform/trino) сервисі орындайды, сондықтан аналитикалық ДҚ бар Data Lakehouse конфигурациясын өрістету сервисінің базалық архитектурасын алдын ала өрістетуді, сондай-ақ пайдаланылатын Cloud Trino сервисі данасын аналитикалық ДҚ-ға қосымша қосуды қамтиды.

## Дайындық қадамдары

[Өрістетіңіз](../../quick-start) Data Lakehouse сервисінің базалық архитектурасын.

## 1. ClickHouse сервисінің данасын өрістетіңіз

1. **Data Platform → Сервис даналары** бөліміне өтіңіз.
1. **Дана жасау** түймесін басыңыз.
1. **Конфигурация** қадамында:

 1. `Cloud ClickHouse` сервис түрін таңдаңыз.
 1. **Нұсқа** өрісінде әдепкі мәнді қалдырыңыз.
 1. `Кластер` конфигурациясын таңдаңыз.

1. **Келесі қадам** түймесін басыңыз.
1. **Параметрлер** қадамында:

 1. Дана параметрлерін орнатыңыз:

 - **Атауы**: сервис данасы үшін атау көрсетіңіз.
 - **Сипаттамасы**: қажет болса, сипаттама қосыңыз.
 - **Деректер қорының атауы**: Cloud ClickHouse қосылатын деректер қорының атауын көрсетіңіз.
 - **Сыртқы IP тағайындау**: Cloud ClickHouse данасы интернет арқылы қолжетімді болуы үшін опцияны қосыңыз.
 - **Kubernetes кластері**: базалық конфигурацияны өрістету кезеңінде [Cloud Iceberg Metastore сервисі данасын жасау](/kz/data-platform/dlh/quick-start#create_iceberg_instance) кезінде көрсеткен кластерді ашылмалы тізімнен таңдаңыз.

 1. `Базовая` кластер конфигурациясын таңдап, конфигурация үлгілерінің бірін көрсетіңіз:

 * `Малый с репликацией` – тестілік конфигурация үшін қолайлы.
 * `Средний с репликацией` – жүктемесі төмен PROD-кластер үшін.
 * `2-шардовый с репликацией` – жүктемесі орташа PROD-кластер үшін.
 * `3-шардовый с репликацией` – жүктемесі жоғары PROD-кластер үшін.

1. **Келесі қадам** түймесін басыңыз.
1. **Қызмет көрсету** қадамында:

 {include(/kz/_includes/_clickhouse.md)[tags=maintenance]}

1. **Келесі қадам** түймесін басыңыз.
1. **Тіркелгі деректері** қадамында:

 1. **Логин** өрісінде әдепкі мәнді қалдырыңыз. Cloud ClickHouse-қа қол жеткізу үшін әкімші логинін жазып алыңыз, ол сізге кейін қажет болады.
 1. **Құпиясөз** өрісінде әкімші құпиясөзін генерациялау үшін **Генерациялау** түймесін басыңыз. Құпиясөзді сақтап қойыңыз, ол сізге кейін қажет болады

 {note:warn}

 Құпиясөзді қалпына келтіру мүмкіндігі қарастырылмаған.

 {/note}

1. **Жасау** түймесін басыңыз.

Cloud ClickHouse сервисі данасын жасау операциясының аяқталуын күтіңіз.

## 2. Cloud Trino-да Cloud ClickHouse-қа қосылуды баптаңыз

[Қосыңыз](/kz/data-platform/trino/instructions/management#add_connect) базалық конфигурацияны өрістету кезінде жасалған Cloud Trino сервисі данасының баптауларында `Clickhouse` көзі үшін жаңа қосылым. Қосылым параметрлерінде жоғарыда жасалған Cloud ClickHouse сервисі данасының деректерін көрсетіңіз.
