# {heading(Data Lakehouse)[id=dlh_dlh]}

{include(/kz/_includes/_translated_by_ai.md)}

Data Lakehouse (DLH) — {linkto(../dwh#dlh_dwh)[text=%text]} (DWH) және {linkto(../data-lake#dlh_data-lake)[text=%text]} артықшылықтарын біріктіретін, сонымен бірге олардың негізгі кемшіліктерін жоятын заманауи гибридті архитектура. 

Data Lakehouse-та, DWH-тен айырмашылығы, тек кестелік деректерді ғана емес, басқа форматтарды да (суреттер, логтар, JSON-файлдар және т.б.) сақтауға болады. Бұл ретте Data Lakehouse метадеректермен жұмыс істеудің жетілдірілген тетіктері арқылы мұндай деректерді басқаруды тиімді ұйымдастырып, олардың бақылаусыз «деректер батпағына» (data swamp) айналып кетуіне жол бермейді, бұл көбіне Data Lake архитектурасына негізделген қоймаларға тән.

Бұдан бөлек, Data Lakehouse BI-құралдарын «шикі» деректерге тікелей қосуға мүмкіндік береді, бұл Data Lake және DWH архитектураларымен салыстырғанда үш негізгі артықшылық береді:

- Лезде өзектілік — аналитиктер деректердің ағымдағы күйімен, түрлендіруге кететін кідіріссіз жұмыс істейді.
- Өңдеу уақытын қысқарту — алдын ала ETL-түрлендіру кезеңі алынып тасталып, нәтижелерді алу жеделдейді.
- Ресурстарды үнемдеу — дайындалған деректерді түрлендіру және сақтау үшін қосымша қуаттар қажет емес.

## {heading(Data Lakehouse-тағы деректерді сақтау моделі)[id=dlh_dlh_models]}

Data Lakehouse-та әртүрлі типтегі деректерді сақтауды ұйымдастыруда S3-пен үйлесімді объектілік қойма негізгі рөл атқарады. Осы салыстырмалы түрде арзан шешімді сақтау моделіне негізгі қойма ретінде енгізу сақтау құнын айтарлықтай төмендетуге мүмкіндік береді.

S3 пен [Apache Iceberg](https://iceberg.apache.org/) интеграциясы кәдімгі СУБД-лардағыдай ACID-транзакцияларды қолдауды іске асыруға және [Apache Spark](https://spark.apache.org/) және [Trino](https://trino.io/) сияқты есептеу компоненттерімен өзара әрекеттесуді баптауға мүмкіндік береді.

## {heading(Data Lakehouse архитектурасының артықшылықтары)[id=dlh_dlh_advantages]}

- Шексіз масштабталу (көлем бойынша шектеулер жоқ).
- Шығындарды азайту (тек нақты пайдаланылған орын үшін «pay as you go» қағидаты бойынша төлем).
- Алдын ала өңдеуді қажет етпейтін «шикі» деректерді қолдау.
- SQL және AI/ML-ді бір уақытта пайдалану.
- ACID қолдауы.
- Қосымша қондырмаларсыз SQL-сұраулардың жоғары жылдамдығы.
- Деректер көлінің «деректер батпағына» (data swamp) айналуына жол бермейтін метадеректерді басқару.
