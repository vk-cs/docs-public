## {heading(Дананы масштабтау)[id=scaling]}

{tabs}

{tab(Жеке кабинет)}

1. [VK Cloud жеке кабинетіне](https://kz.cloud.vk.com/app/) өтіңіз.
1. **Data Platform → Қызмет даналары** бөліміне өтіңіз.
1. Масштабтау баптауларына келесі тәсілдердің бірімен өтіңіз:

    - Қажетті дана үшін ![ ](/kz/assets/more-icon.svg "inline") белгішесін басып, **Масштабтау** тармағын таңдаңыз.
    - Қажетті дананың атауын басыңыз, **Жалпы ақпарат** немесе **Компоненттер күйі** қойындысында **Масштабтау** түймесін басыңыз.

1. ClickHouse Keeper және ClickHouse түйіндерінің санын көрсетіңіз.
1. **Өзгерістерді сақтау** түймесін басыңыз.

{/tab}

## {heading(Дана параметрлерін қарау және өзгерту)[id=settings]}

{tabs}

{tab(Жеке кабинет)}

{include(/kz/_includes/_clickhouse.md)[tags=open]}

1. Дана бетінде **Баптаулар** қойындысына өтіңіз.
1. Дананың баптаулар кестесінен қажетті параметрді табыңыз, қажет болса іздеу жолағын пайдаланыңыз.
1. Баптаулардың өзгеру тарихын қарау үшін ![ ](/kz/assets/clock-icon.svg "inline") белгішесін басыңыз.
1. Қажетті баптауларды өзгертіңіз:
    1. **Өңдеу** түймесін басыңыз.
    1. Кестедегі қажетті параметрлердің мәндерін өзгертіңіз.
    1. Егер әдепкі мәнге қайту қажет болса, қажетті параметр үшін ![ ](/kz/assets/more-icon.svg "inline") белгішесін басып, **Қалпына келтіру** тармағын таңдаңыз.
    1. **Өзгерістерді сақтау** түймесін басыңыз.

{/tab}

{/tabs}

## {heading(Қосылым қосу)[id=add_connect]}

{tabs}

{tab(Жеке кабинет)}

{include(/kz/_includes/_clickhouse.md)[tags=open]}

1. Дана бетінде **Қосылымдар** қойындысына өтіңіз.
1. **Қосылым қосу** түймесін басыңыз.
1. Деректер көзін таңдаңыз:

    - `s3-int` — сол жобадағы VK Object Storage қоймасына қосылу.
    - `s3-ext` — сыртқы S3 объектілік қоймасына қосылу.
    - `iceberg-metastore-int` — сол VK Cloud жобасындағы Cloud Iceberg Metastore данасына қосылу.
    - `iceberg-metastore-ext` — сыртқы Cloud Iceberg Metastore каталогына қосылу.
    - `сlickhouse` — Cloud ClickHouse дерекқорына қосылу.
    - `postgresql` — PostgreSQL дерекқорына қосылу.
    - `mysql` — MySQL дерекқорына қосылу.

1. Көзге қосылу параметрлерін баптаңыз:

   {tabs}

   {tab(s3-int)}

    - **Атауы**: қосылым атауын көрсетіңіз. Атау кіші латын әрпінен басталып, тек бас және кіші латын әріптерін, цифрларды және `_` таңбасын қамтуы тиіс. Атау Cloud ClickHouse-пен жұмыс істеу кезінде пайдаланылады.
    - **Бакет**: Cloud ClickHouse данасы қосылатын VK Object Storage бакетінің атауы.

   {/tab}

   {tab(s3-ext)}

    - **Атауы**: қосылым атауын көрсетіңіз. Атау кіші латын әрпінен басталып, тек бас және кіші латын әріптерін, цифрларды және `_` таңбасын қамтуы тиіс. Атау Cloud ClickHouse-пен жұмыс істеу кезінде пайдаланылады.
    - **Аймақ**: S3 қоймаңыз орналасқан аймақ.
    - **Access Key**: қоймаға қол жеткізу кілтінің идентификаторы.
    - **Secret Key**: қоймаға қол жеткізудің құпия кілті.
    - **S3 URL**: S3 қоймаңыздың URL мекенжайы.
    - **Бакет**: Cloud ClickHouse данасы қосылатын S3 қоймасындағы бакет атауы.

   {/tab}

   {tab(iceberg-metastore-int)}

    - **Атауы**: қосылым атауын көрсетіңіз. Атау кіші латын әрпінен басталып, тек бас және кіші латын әріптерін, цифрларды және `_` таңбасын қамтуы тиіс. Атау Cloud ClickHouse-пен жұмыс істеу кезінде пайдаланылады.
    - **s3_bucket**: Cloud ClickHouse данасы қосылатын VK Object Storage бакетінің атауы.
    - **s3_folder**: Cloud ClickHouse данасына қолжетімді болатын бакеттегі директорияның атауы.

   {/tab}

   {tab(iceberg-metastore-ext)}

    - **Атауы**: қосылым атауын көрсетіңіз. Атау кіші латын әрпінен басталып, тек бас және кіші латын әріптерін, цифрларды және `_` таңбасын қамтуы тиіс. Атау Cloud ClickHouse-пен жұмыс істеу кезінде пайдаланылады.
    - **s3_region**: сыртқы Cloud Iceberg Metastore каталогыңыз орналасқан аймақ.
    - **s3_access_key**: каталогқа қол жеткізу кезінде аутентификация үшін пайдаланылатын бірегей идентификатор.
    - **s3_secret_key**: каталогқа қол жеткізуге арналған жеке кілт.
    - **s3_endpoint**: Cloud Iceberg Metastore каталогыңыздың URL мекенжайы.
    - **s3_bucket**: Cloud ClickHouse данасы қосылатын бакет атауы.
    - **s3_folder**: Cloud ClickHouse данасына қолжетімді болатын бакеттегі директорияның атауы.

   {/tab}

   {tab(сlickhouse, postgresql)}

    - **Атауы**: қосылым атауын көрсетіңіз. Атау кіші латын әрпінен басталып, тек бас және кіші латын әріптерін, цифрларды және `_` таңбасын қамтуы тиіс. Атау Cloud ClickHouse-пен жұмыс істеу кезінде пайдаланылады.
    - **db_name**: Cloud ClickHouse данасы қосылатын дерекқордың атауы.
    - **hostname**: қосылу серверінің мекенжайы.
    - **username**: дерекқор пайдаланушысының тіркелгі атауы.
    - **password**: дерекқор пайдаланушысының тіркелгі құпиясөзі.

   {/tab}

   {tab(mysql)}

    - **Атауы**: қосылым атауын көрсетіңіз. Атау кіші латын әрпінен басталып, тек бас және кіші латын әріптерін, цифрларды және `_` таңбасын қамтуы тиіс. Атау Cloud ClickHouse-пен жұмыс істеу кезінде пайдаланылады.
    - **db_name**: Cloud ClickHouse данасы қосылатын дерекқордың атауы.
    - **hostname**: қосылу серверінің мекенжайы.
    - **username**: дерекқор пайдаланушысының тіркелгі атауы.
    - **password**: дерекқор пайдаланушысының тіркелгі құпиясөзі.
    - **connection_pool_size**: Cloud ClickHouse данасы мен MySQL ДҚ арасындағы сұрауларды өңдеу үшін бір уақытта пайдаланылуы мүмкін белсенді қосылымдар саны.

   {/tab}

   {/tabs}

1. **Өзгерістерді сақтау** түймесін басыңыз.

{/tab}

## {heading(Дананы сақтық көшірмеден қалпына келтіру)[id=restore]}

{tabs}

{tab(Жеке кабинет)}

{note:warn}

Сақтық көшірмеден қалпына келтіру данадағы барлық деректердің үстінен жазады. Көшірме жасалғаннан кейін енгізілген барлық өзгерістер қайтарымсыз жоғалады.

{/note}

{include(/kz/_includes/_clickhouse.md)[tags=open]}

1. Дана бетінде сақтық көшірмелер қойындысына өтіңіз.
1. Қажетті сақтық көшірме үшін ![ ](/kz/assets/more-icon.svg "inline") белгішесін басып, **Қалпына келтіру** тармағын таңдаңыз.
1. (Қосымша) Дана параметрлерінің мәндерін өзгертіңіз. Параметрлер [дананы жасау](../create) бөлімінде сипатталған.

1. **Жасау** түймесін басыңыз.

   Cloud ClickHouse қызметінің жаңа данасы жасалады.

{/tab}
