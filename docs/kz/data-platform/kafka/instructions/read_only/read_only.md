# {heading(Экземплярды тек оқу режимінен шығару)[id=kafka_read_only]}

{include(/kz/_includes/_translated_by_ai.md)}

Егер Cloud Kafka данасындағы брокерлердің бірінің дискісінде бос орын таусылса, {var(data-p)} барлық брокерлерді автоматты түрде тек оқу режиміне (read-only) ауыстырады. Бұл ақаулардың алдын алуға және дана жұмыс қабілетін сақтауға мүмкіндік береді. Тек оқу режимінде Cloud Kafka-ға хабарламалар жариялау қолжетімсіз болады — жариялау әрекеті кезінде қате пайда болады. Барлық брокерлер тек оқуға ғана қолжетімді болып қалады.

## {heading(Тек оқу режимінің көрсетілуі)[id=kafka_read_only_view]}

Дана тек оқу режиміне ауысқанда **Data Platform** → **Сервис даналары** бөлімінде сервис данасы үшін **Сервис күйі** бағанында `Тек оқу үшін` мәртебесі көрсетіледі. 

Қай брокердің дискісінде бос орын таусылғанын **Data Platform** → **Сервис даналары** бөліміндегі **Компоненттер күйі** қойындысынан білуге болады.

## {heading(Экземплярды тек оқу режимінен шығару)[id=kafka_read_only]}

Дананы тек оқу режимінен шығару үшін брокерлерге арналған дисктердің көлемін ұлғайтыңыз:
1. **Data Platform** → **Сервис даналары** бөліміндегі **Компоненттер күйі** қойындысына өтіңіз.
1. **Дискілерді масштабтау** түймесін басыңыз.
1. **Тік масштабтауды** таңдаңыз.
1. Дисктердің өлшемін ұлғайтыңыз.
1. **Өзгерістерді сақтау** түймесін басыңыз.
