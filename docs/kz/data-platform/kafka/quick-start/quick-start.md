Жылдам бастау сервиспен жұмысты бастауға және оның мүмкіндіктерімен танысуға көмектеседі.

Жылдам бастау қадамдарының барлығын орындағаннан кейін сіз:

1. **Single** конфигурациясындағы Cloud Kafka данасын жасайсыз.
1. Топикке ([topic](https://kafka.apache.org/documentation/#intro_concepts_and_terms)) хабарламалар жіберуді және Cloud Kafka данасының топигінен хабарламалар алуды үйренесіз.

{note:warn}

Cloud Kafka данасы [тарифтеледі](../tariffication) және есептеу ресурстарын тұтынады. Жылдам бастауды аяқтағаннан кейін, егер ол енді қажет болмаса, [сервис данасын жойыңыз](#kafka-quick-start-delete-resources).

{/note}

## 1. Cloud Kafka сервис данасын жасаңыз

1. [Өтіңіз](https://kz.cloud.vk.com/app/) {var(cloud)} жеке кабинетіне.
1. Сервис данасын жасау қажет жобаны таңдаңыз.
1. **Деректер платформасы → Сервис даналары** бөліміне өтіңіз.
1. **Дана жасау** батырмасын басыңыз.
1. **Конфигурация** қадамында:

   1. **Cloud Kafka** сервис түрін таңдаңыз.
   1. **3.5.0** нұсқасын таңдаңыз.
   1. **Жалғыз торап** конфигурациясын таңдаңыз.
   1. **Келесі қадам** батырмасын басыңыз.

1. **Параметрлер** қадамында:

   1. Данаға атау мен сипаттама беріңіз: `vkcloud-demo-kafka` және `Бірінші Cloud Kafka данасы`.
   1. Брокер торабының параметрлерін орнатыңыз:

      - **Виртуалды машинаның түрі:** `Standard-2-4`.
      - **Диск түрі:** `High-IOPS SSD`.
      - **Диск өлшемі:** `50 ГБ`.

   1. **Kubernetes кластері**: **Жаңа кластер жасау** тармағын таңдаңыз.
   1. **Желі**: **Жаңа желі жасау** тармағын таңдаңыз.
   1. SDN ретінде **Sprut** таңдаңыз.
   1. **Ішкі желі мекенжайы**: `10.0.1.0/24` көрсетіңіз.
   1. **Қолжетімділік аймағы**: `Мәскеу (GZ1)`.
   1. **Келесі қадам** батырмасын басыңыз.

1. **Тіркелгі деректері** қадамында:

   1. **Kafka-ға қол жеткізуге арналған әкімші логині**: мысалы, `kafkaadmin`.
   1. **Kafka-ға қол жеткізуге арналған әкімші құпиясөзі**: **Жасау** батырмасын басып, алынған құпиясөзді сақтаңыз.

      {note:err}

      Құпиясөзді қалпына келтіру мүмкін емес.

      {/note}

   1. **Жасау** батырмасын басыңыз.

      Операцияның аяқталуын күтіңіз. Cloud Kafka данасын жасау ұзақ уақыт алуы мүмкін.

## 2. Cloud Kafka данасымен жұмысқа дайындалыңыз

1. Сервис данасымен жұмыс істеу жоспарланған хостта Kafka консольдік клиенттеріне қол жеткізіңіз:

   {tabs}

   {tab(Linux (bash) / macOS (zsh))}

   1. Хостта Java 11 немесе одан жоғары нұсқасы орнатылғанына көз жеткізіңіз:

      ```console
      java --version
      ```

      Егер шығыста `Command 'java' not found` болса немесе нұсқа ескірген болса, мысалы, [OpenJDK](https://openjdk.org/install/) Java Development Kit (JDK) орнатыңыз. JDK-ны толықтай да, тек Java Runtime Environment (JRE) нұсқасын да орнатуға болады.

      Ubuntu 22.04 LTS үшін `apt` арқылы OpenJDK 11 (тек JRE) орнату мысалы:

      ```console
      sudo apt update && sudo apt install openjdk-11-jre
      ```

   1. [Kafka 3.5.0](https://archive.apache.org/dist/kafka/3.5.0/kafka_2.13-3.5.0.tgz) нұсқасының дистрибутиві бар архивті жүктеп алыңыз. Сервис данасындағы және хосттағы Kafka нұсқалары сәйкес келуі керек. cURL көмегімен дистрибутивті жүктеу пәрменінің мысалы:

      ```console
      curl -O https://archive.apache.org/dist/kafka/3.5.0/kafka_2.13-3.5.0.tgz
      ```

   1. Архивті ашып, ашылған файлдар директориясына өтіңіз:

      ```console
      tar -xzf kafka_2.13-3.5.0.tgz && cd kafka_2.13-3.5.0
      ```

   1. Консольдік producer клиенті мен consumer клиентінің нұсқасын алыңыз:

      ```console
      bin/kafka-console-producer.sh --version
      bin/kafka-console-consumer.sh --version
      ```

      Нұсқаның сәтті шығуы клиенттердің дұрыс жұмыс істеп тұрғанын білдіреді.

   1. Келесі мазмұнмен `client-sasl-ssl.properties` конфигурациялық файлын жасаңыз:

      ```ini
      security.protocol=SASL_SSL
      ```

      Бұл файлда producer клиенті мен consumer клиентінен сервис данасында SASL_SSL протоколы арқылы аутентификациядан өтуді талап ететін [баптау](https://kafka.apache.org/documentation/#security_sasl_scram) бар.

   {/tab}

   {/tabs}

1. Сервис данасына қосылу үшін bootstrap-сервердің мекенжайын алыңыз:

   1. [Өтіңіз](https://kz.cloud.vk.com/app/) {var(cloud)} жеке кабинетіне.
   1. Сервис данасы орналасқан жобаны таңдаңыз.
   1. **Деректер платформасы → Сервис даналары** бөліміне өтіңіз.
   1. `vkcloud-demo-kafka` кластерінің атауын басып, оның бетін ашыңыз.
   1. **Жалпы ақпарат** қойындысына өтіңіз.
   1. **Bootstrap** параметрінен мекенжайды жазып алыңыз.

   {note:info}

   Төмендегі мысал үшін `kafka-bootstrap.example.com:9093` мекенжайы қолданылады.

   Оны төмендегі пәрмендерде bootstrap-сервердің нақты мекенжайымен ауыстырыңыз.

   {/note}

## 3. Топикке хабарлама жіберіңіз және оны алыңыз

{tabs}

{tab(Linux (bash) / macOS (zsh))}

1. Консольдік клиент жұмыс істейтін жеке терминал сессиясын жасаңыз.
1. Осы сессияда consumer клиентін іске қосып, `test` топигін дереккөз ретінде көрсетіңіз. Consumer клиенті бұрын алынған bootstrap-сервер мекенжайын және конфигурация файлын пайдаланып Cloud Kafka данасына қосылады да, топиктен хабарламаларды үздіксіз оқып, шығарып отырады. Жаңа топик оған бірінші рет жүгінген кезде автоматты түрде жасалады.

   Ол үшін мына пәрменді орындаңыз:

   ```console
   bin/kafka-console-consumer.sh      --bootstrap-server kafka-bootstrap.example.com:9093      --consumer.config client-sasl-ssl.properties      --topic test
   ```

1. Басқа консольдік клиент жұмыс істейтін тағы бір терминал сессиясын жасаңыз.
1. Осы сессияда producer клиентін іске қосып, `test` топигін қабылдағыш ретінде көрсетіңіз. Producer клиенті бұрын алынған bootstrap-сервер мекенжайын және конфигурация файлын пайдаланып Cloud Kafka данасына қосылады да, енгізілген хабарламаларды топикке жібереді.

   Ол үшін мына пәрменді орындаңыз:

   ```console
   bin/kafka-console-producer.sh      --broker-list kafka-bootstrap.example.com:9093      --producer.config client-sasl-ssl.properties
     --topic test
   ```

1. Producer клиенті бар терминал сессиясында сынақ хабарламасын енгізіп, содан кейін _Enter_ пернесін басыңыз:

   ```text
   This is the test message.
   ```

1. Consumer клиенті бар терминал сессиясында жіберілген сынақ хабарламасының көрсетіліп тұрғанын тексеріңіз:

   ```text
   This is the test message.
   ```

   Хабарламаның болуы Cloud Kafka данасының дұрыс жұмыс істеп тұрғанын білдіреді.

1. Екі терминал сессиясында да CTRL+C пернелер тіркесімін басып, клиенттердің жұмысын аяқтаңыз.

{/tab}

{/tabs}

## {heading(Пайдаланылмайтын ресурстарды жойыңыз)[id=kafka-quick-start-delete-resources]}

Cloud Kafka данасы [тарифтеледі](../tariffication) және есептеу ресурстарын тұтынады. Егер ол сізге енді қажет болмаса, оны жойыңыз.

Сондай-ақ `client-sasl-ssl.properties` конфигурациялық файлын және жүктелген Kafka дистрибутивін жоя аласыз.
