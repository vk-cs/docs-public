# {heading(Нұсқаларды қолдау саясаты)[id=trino_versions]}

{include(/kz/_includes/_translated_by_ai.md)}

Cloud Trino нұсқаларын қолдау саясаты нұсқаларды қолдау ережелерін реттейді және жаңаларына көшу процесін сипаттайды. Саясат жаңартуларды шығарудың негізі болып табылады және Cloud Trino нұсқаларымен жұмысты жоспарлауға мүмкіндік береді.

## {heading(Нұсқалар туралы)[id=trino_versions_general]}

Өзекті нұсқа: `0.468.1`

Cloud Trino нұсқасының нөмірі `X.YYY.Z` үлгісі бойынша қалыптастырылады. Мұнда:
- `X` — Cloud Trino-ның мажорлық нұсқасы;
- `YYY` — [Trino релизінің](https://trino.io/docs/current/release.html) нөмірі;
- `Z` — бір [Trino релизі](https://trino.io/docs/current/release.html) шеңберіндегі Cloud Trino жинағының нөмірі.

Cloud Trino-да нұсқалардың екі түрі қолдау табады: бір өзекті және алдыңғы үш нұсқаға дейін. Қолжетімді нұсқалармен сервистің данасын {linkto(../../instructions/create#trino_create)[text=жасау]} кезінде жеке кабинетте танысуға болады. Қалған нұсқалар *ескірген* болып саналады.

{note:warn}

Cloud Trino-ның ескірген нұсқалары үшін жаңартулар шығарылмайды, ал техникалық қолдау мүмкіндігі шектеулі. Егер сіз ескірген нұсқаны пайдалансаңыз, [жаңартылу](#trino_versions_update) ұсынылады.

{/note}

## {heading(Релиз және қолдау мерзімдері)[id=trino_versions_release_support]}

Cloud Trino-ның жаңа нұсқалары үш айда бір реттен жиі шықпайды. Жаңартуды әзірлеу кезінде {var(cloud)} мамандары дайындау мен тестілеудің қажетті кезеңдерінен соңғы болып өткен [Trino релизі](https://trino.io/docs/current/release.html) пайдаланылады. Мұндай тәсіл сервистің барынша тұрақтылығы мен қауіпсіздігін қамтамасыз етуге мүмкіндік береді.

Cloud Trino-ның жаңа нұсқасы шыққан кезде, қолда бар нұсқалардың ең ескісі ескірген деп танылады және жеке кабинетте таңдау үшін қолжетімсіз болады. Мұндай нұсқамен жаңа дана жасау қолжетімсіз. Бұрыннан жасалған даналар жұмысын жалғастырады.

## {heading(Жаңарту)[id=trino_versions_update]}

Cloud Trino-ның өзекті нұсқаларын пайдалану соңғы релиздердің мүмкіндіктеріне қол жеткізуге мүмкіндік береді, сондай-ақ қауіпсіздік түзетулерін және сервистің тұрақтылығын арттыруға бағытталған патчтарды алуға жағдай жасайды.

Cloud Trino жаңартуы сервистің өзекті нұсқасы бар жаңа дананы жасау арқылы орындалады. Қызметтің бұрыннан бар данасын жаңарту қолдау таппайды.

{note:info}

{var(cloud)} сервистік инфрақұрылымы деңгейіндегі түзетулер мен жақсартулар фондық режимде автоматты түрде орындалады және Cloud Trino нұсқасына тәуелді емес.

{/note}

Жаңарту бойынша ұсыныстар:

- Егер сізде жұмыс жүктемесі болса, сервис данаңыздың параметрлерін {linkto(../../instructions/management#trino_view)[text=біліңіз]} және дәл сондай параметрлері бар (баптаулар, қосылымдар, тіркелгі деректері), бірақ анағұрлым жаңа нұсқадағы жаңа дананы {linkto(../../instructions/create#trino_create)[text=жасаңыз]}. Жүктемені соған ауыстырыңыз және егер ескі сервис данасы енді қажет болмаса, оны {linkto(../../instructions/management#trino_delete)[text=жойыңыз]}.
- Егер сізде жұмыс жүктемесі болмаса, сервис данасының параметрлерін (баптаулар, қосылымдар, тіркелгі деректері) {linkto(../../instructions/management#trino_view)[text=тексеріңіз]} және оны {linkto(../../instructions/management#trino_delete)[text=жою]} алдында сақтап қойыңыз. Мұндай тәсіл жаңа нұсқаны пайдалануды жоспарласаңыз, бірақ жұмыс жүктемесін ұстап тұру қажет болмаған жағдайда пайдалы болуы мүмкін.
- Жаңа нұсқаға көшкеннен кейін Cloud Trino сервис данасына қосылу үшін пайдаланылатын БҚ-ны, мысалы, [Trino CLI](https://trino.io/docs/current/client/cli.html), жаңартыңыз.
