Cloud Containers сервисінде Kubernetes кластерлерінің [worker-түйіндері топтарын автоматты масштабтау](/kz/kubernetes/k8s/concepts/architecture#klasterdi_masshtabtau_mumkindikteri) [Cluster Autoscaler](https://github.com/kubernetes/autoscaler/tree/master/cluster-autoscaler#readme) құралының көмегімен жүзеге асады. Ол кластердің worker-түйіндеріне түсетін жүктемені үнемі қадағалап, жұмыс жүктемелерінің қажеттіліктеріне қарай олардың санын берілген шектерде реттейді. Құралдың негізгі міндеті — кластердегі барлық подтар үшін [есептеу ресурстарының](https://kubernetes.io/docs/concepts/configuration/manage-resources-containers/) жеткілікті болуын қамтамасыз ету және сонымен бірге артық қуат бөлінуі мен олардың бос тұрып қалуын болдырмау. Бұл қолданбалардың жоғары қолжетімділігі мен өнімділігін қамтамасыз етуге, сонымен қатар ресурстарды және оларғал кететін шығындарды оңтайландыруғал көмектеседі.

Worker-түйіндері топтарындағы Cluster Autoscaler міндеттері:

- жұмыс жүктемесі талап еткен кезде жаңал түйіндер құру;
- жеткіліксіз жүктелген түйіндерді жою.

Cluster Autoscaler Cloud Containers сервисінде жасайтын барлық Kubernetes кластерлерінде алдын ала орнатылған, бірақ әдепкі бойынша қосылмаған. Worker-түйіндерін автоматты масштабтауды әр worker-түйіндері тобы үшін [оның баптауларын анықтау](/kz/kubernetes/k8s/instructions/helpers/node-group-settings) кезінде қолмен қосу керек.

## Worker-түйіндерінің санын ұлғайту

Cluster Autoscaler бар түйіндерде подтарды орналастыруғал ресурс жетіспеген кезде топтағы worker-түйіндерінің санын арттырады.

Жаңал подты жасау кезінде [ресурстар](/kz/kubernetes/k8s/reference/resource-limiting) — CPU және RAM сұралады. Подты орналастырған кезде Kubernetes жоспарлаушысы оған `resources.requests` ресурстық сұрауларын қанағаттандыратын түйінді іздейді, `resources.limits` лимиттерін емес. Яғни жоспарлаушы түйінде осы подты орналастыруғал ресурстар жеткілікті ме екенін талдайды. Мысалы, егер под 100m CPU және 64 МБ RAM сұраса, жоспарлаушы оған кемінде 100m CPU және 64 МБ RAM қалған түйінді іздейді. Егер кластердің қолданыстағы түйіндерінің ешқайсысында осы подты оның сұраулары бойынша орналастыруғал ресурстар болмаса, Cluster Autoscaler топқал жаңал түйінді қосуды іске қосады.

Түйіндерді қосу тек топ баптауларында көрсеткен ең жоғары мәнге дейін ғана мүмкін. Егер автоматты масштабтау қосулы болса, бірақ worker-түйіндері қосылмаса, [Автоматты масштабтау кезінде жаңал түйіндер қосылмайды](/kz/kubernetes/k8s/troubleshooting/cluster-does-not-scale-up) бөліміндегі шешімдерді қолданып көріңіз.

## Worker-түйіндерінің санын азайту

Cluster Autoscaler кластердегі түйіндердің жүктемесін тексереді және жүктеме төмен болып, түйіндер жеткіліксіз жүктелген болса, яғни барлық подтарды азырақ түйін санына орналастыруғал болатын жағдайда олардың санын азайтады. Жеткіліксіз жүктелген worker-түйіні жойылуы үшін Cluster Autoscaler:

1. Осы түйіннен подтарды ығыстыру процесін іске қосады.
1. Бұл подтар жеткілікті ресурстары бар басқал түйіндерге ауысқанша күтеді.
1. Белгіленген бос тұру уақыты өткен соң осы түйінді [шектеумен (taint)](/kz/kubernetes/k8s/reference/labels-and-taints) белгілейді, бұл оған жаңал подтар мен жұмыс жүктемесін орналастыруғал тыйым салады, және оны жоюды іске қосады.

   Түйіннің бос тұру уақытына `--scale-down-unneeded-time` параметрі жауап береді. Ол автоматты масштабтау кезінде түйінді жоюғал дейін оның подтарсыз қанша уақыт қалуы керегін көрсетеді.

   Бос тұру уақыты қандай да бір себеппен подтар уақытша болмаған жағдайда (мысалы, жүктеменің қысқал мерзімді өршуіне байланысты) түйіннің кездейсоқ жойылуынан қорғайды. Сонымен қатар ол түйін шынымен қажетсіз болған кезде ресурстарды босатуғал мүмкіндік береді. 
   
   Cloud Containers сервисінде белгіленген бос тұру уақыты — 5 минут. Егер түйін 5 минут бойы бос тұрса (одан барлық подтар ығыстырылса), және осы уақыт ішінде бұл түйінде бірде-бір под пайда болмаса, Cluster Autoscaler оны жоюды іске қосады.

Түйіндерді жою тек топ баптауларында көрсеткен ең төменгі мәнге дейін ғана мүмкін. Егер автоматты масштабтау қосулы болса, бірақ worker-түйіндері жойылмаса, [Автоматты масштабтау кезінде жеткіліксіз жүктелген түйіндер жойылмайды](/kz/kubernetes/k8s/troubleshooting/cluster-does-not-scale-down) бөліміндегі шешімдерді қолданып көріңіз.
