Cloud Containers сервисі VK Cloud платформасында Kubernetes кластерлерімен жұмыс істеуге арналған ортаны қамтамасыз етеді. [OpenStack](https://www.openstack.org/) негізіндегі сервистің архитектурасы пайдаланушыларға жұмыс істеуге кең мүмкіндіктер береді, істен шығуға төзімділікті, масштабталуды және платформаның басқа сервистерімен интеграцияны қамтамасыз етеді.

{cut(Cloud Containers сервисінің VK Cloud басқа компоненттерімен өзара әрекеттесу сызбасы)}

Cloud Containers кластерлердің дұрыс жұмысын бақылайды, ал:

- [Cloud Servers](/ru/computing/iaas) кластер түйіндеріндегі ВМ басқарады.
- [Cloud Networks](/ru/networks/vnet) кластер желілерін басқарады.

![Сервис архитектурасы](/ru/kubernetes/k8s/assets/k8s_arch.png){params[noBorder=true; width=50%]}

{/cut}

## {heading(Кластер топологиялары )[id=cluster_topology]}

Cloud Containers кластері түйіндердің (nodes) екі түрінен тұрады — master-түйіндер және worker-түйіндер:

- _Master-түйіндер_ бүкіл кластердің күйі туралы ақпаратты сақтайды және worker-түйіндер арасында жұмыс жүктемесін бөлуді басқарады. Пайдаланушыларға master-түйіндерді басқару қолжетімді емес, ол VK Cloud платформасы жағында жүзеге асырылады.

  Kubernetes кластерін [құрған кезде](/ru/kubernetes/k8s/instructions/create-cluster), Cloud Containers оның master-түйіндері үшін ең аз сәйкес келетін [конфигурация үлгісін](/ru/computing/iaas/concepts/vm/flavor) таңдайды. Әдепкі бойынша бұл Intel Cascade Lake процессоры, 2 CPU және 6 ГБ жедел жады бар ВМ. Master-түйіндердің [диск түрі](/ru/kubernetes/k8s/concepts/storage#supported_storage_types) — 20 ГБ көлеміндегі High-IOPS SSD.

  Master-түйіндерде әдепкі бойынша [автоматты масштабтау](/ru/kubernetes/k8s/concepts/scale#tipy_masshtabirovaniya) қосылған, сондықтан кластерге түсетін жүктеме өзгерген кезде оның есептеу ресурстарының саны автоматты түрде өзгертіледі.

- _Worker-түйіндер_ жұмыс жүктемесін ([workload](https://kubernetes.io/docs/concepts/workloads/)) орындайды. Олар worker-түйіндер топтарына біріктірілуі мүмкін. Істен шығуға төзімділікті арттыру үшін топтарды әртүрлі [қолжетімділік аймақтарына](/ru/start/concepts/architecture#az) орналастырыңыз. Worker-түйіндерді басқару да VK Cloud платформасы жағында жүзеге асырылады, бірақ сонымен бірге олар пайдаланушылар жобаларымен желілік байланысқа ие.

Кластердің істен шығуға төзімділігі master-түйіндер санына және олардың қолжетімділік аймақтары бойынша бөлінуіне байланысты. Ықтимал конфигурациялар:

- Бір master-түйіні бар Cloud Containers кластері.

  Мұндай кластер істен шығуға төзімді емес: worker-түйіндер бірнешеу болып, олар топтарға біріктірілгеннің өзінде, жалғыз master-түйін жоғалған жағдайда кластер жұмысқа жарамсыз болады.

  {cut(Бір master-түйіні бар кластер үшін түйіндердің өзара әрекеттесу сызбасы)}
  ![Бір master-түйіні бар кластер](/ru/kubernetes/k8s/assets/cluster_types_1.png){params[noBorder=true; width=80%]}
  {/cut}

- 3 немесе 5 master-түйіннен тұратын стандартты Cloud Containers кластері.

  Мұндай кластер қолжетімділік аймағы деңгейінде істен шығуға төзімді: қолжетімділік аймағы тұрақты жұмыс істеп тұрған және бірнеше master-түйін жоғалған жағдайда, master-түйіндердің жартысынан көбі жұмыс істеп тұрғанша, ол жұмыс қабілеттілігін сақтайды. Бір master-түйін қалған кезде кластер жұмысын тоқтатады.

  {cut(Стандартты кластер үшін түйіндердің өзара әрекеттесу сызбасы)}  
  ![Стандартты кластер](/ru/kubernetes/k8s/assets/cluster_types_2.png){params[noBorder=true; width=80%]}
  {/cut}

- 3 немесе 5 master-түйіннен тұратын істен шығуға төзімді Cloud Containers кластері.

  Істен шығуға төзімді кластерлердің master-түйіндері аймақтың барлық қолжетімділік аймақтары бойынша таратылады. Мұндай кластер барынша істен шығуға төзімді: бір қолжетімділік аймағы істен шыққан кезде жүктеме басқа қолжетімділік аймақтарында орналасқан master-түйіндер арасында бөлінеді. Алайда, стандартты кластерлердегідей, істен шығуға төзімді кластер master-түйіндердің жартысынан көбі жұмыс істеп тұрғанша жұмыс қабілеттілігін сақтайды және бір master-түйін қалған кезде жұмысын тоқтатады.

  {cut(Істен шығуға төзімді кластер үшін түйіндердің өзара әрекеттесу сызбасы)}
  ![Істен шығуға төзімді кластер](/ru/kubernetes/k8s/assets/cluster_types_3.png){params[noBorder=true; width=80%]}
  {/cut}

Таңдалған кластер топологиясына қарамастан, master-түйіндер кластер күйі туралы ақпаратты сақтау үшін «кілт-мән» үлестірілген сақтау орнын [etcd](https://etcd.io/) пайдаланады:

- Бір master-түйіні бар кластерде бір `etcd` данасы болады.
- Бірнеше master-түйіні бар кластерлерде істен шығуға төзімділік үшін кластерлік режимде жұмыс істейтін бірнеше `etcd` данасы бар.
- Әрбір `etcd` данасы үшін бөлек жоғары өнімді SSD-диск (High-IOPS) бөлінеді. Бұл ең аз кідіріспен кластердің API-эндпоинтімен барынша жылдам өзара әрекеттесуді ұйымдастыруға мүмкіндік береді.

Worker-түйіндер деңгейінде істен шығуға төзімділікті қамтамасыз ету үшін әртүрлі қолжетімділік аймақтарында бірнеше worker-түйіндер тобын құрып, қолданба репликаларын осы түйіндерде репликалар да әртүрлі қолжетімділік аймақтарында болатындай етіп орналастыру ұсынылады.

## Кластер ортасы

Master- және worker-түйіндерде AlmaLinux операциялық жүйесі пайдаланылады (Kubernetes 1.31 нұсқасынан бастап).

Кластер контейнерлерді Kubernetes [Container Runtime Interface](https://kubernetes.io/docs/concepts/architecture/cri/) (CRI) арқылы CRI-O көмегімен іске қосады.

Толығырақ [Қолжетімді Kubernetes нұсқалары және нұсқаларды қолдау саясаты](../versions) бөлімінде.

## {heading(Kubernetes API-мен интеграция)[id=kubernetes-api-integration]}

Кластермен барлық өзара әрекеттесу [Kubernetes API](https://kubernetes.io/ru/docs/concepts/overview/kubernetes-api/) арқылы жүзеге асырылады.

Cloud Containers кластерлерінің API-эндпоинті [жеке жүктеме теңгергішінің](../network) артында орналасқан, сондықтан кластер API-іне master-түйіндер санына қарамастан бірдей IP-мекенжай бойынша қол жеткізуге болады.

## {heading(VK Cloud платформасымен интеграция)[id=integration]}

VK Cloud платформасымен интеграция стандартты Kubernetes интерфейстері арқылы жүзеге асырылады:

- [Container Storage Interface](https://kubernetes-csi.github.io/docs/) (CSI) — деректерді сақтау сервистерімен интеграция.

  Кластерлерде VK Cloud сақтау орнын тұрақты томдар ([persistent volumes](https://kubernetes.io/docs/concepts/storage/persistent-volumes/)) түрінде пайдалануға мүмкіндік береді.
  Persistent Volume Claim (PVC) пайдалану қолжетімді.

  Интеграция OpenStack Cinder API көмегімен жүзеге асырылады. Толығырақ [Кластердегі сақтау орны](../storage) бөлімінде.

- [Container Network Interface](https://kubernetes.io/docs/concepts/extend-kubernetes/compute-storage-net/network-plugins/) (CNI) — желілік ішкі жүйелермен интеграция.

  Cloud Containers сервисінде жасайтын Kubernetes кластерлерінде осы интерфейсті қолдайтын плагиндер іске асырылған: [Calico](https://projectcalico.docs.tigera.io/about/about-calico) және [Cilium](https://docs.cilium.io/en/stable/index.html) (тек екінші буын кластерлері үшін қолжетімді). Олар мыналарды қамтамасыз етеді:

  - контейнерлер, [подтар](../../reference/pods) және кластер түйіндері арасындағы желілік байланыс;
  - Kubernetes [желілік саясаттарын](https://kubernetes.io/docs/concepts/services-networking/network-policies/) (Network Policies) қолдану және сақтау.

  Calico және Cilium VK Cloud платформасымен SDN Sprut көмегімен интеграцияланады. Толығырақ [Кластердегі желі](../network) бөлімінде.

## Open Policy Agent кірістірілген қолдауы

Cloud Containers кластерлеріне [Open Policy Agent Gatekeeper](../../reference/gatekeeper/) кірістірілген. Ол Kubernetes ресурстары үшін [қауіпсіздік саясаттарын](../security-policies) қолдануға мүмкіндік береді. Сондай-ақ мұндай кластерлерде [әдепкі қауіпсіздік саясаттары](/ru/kubernetes/k8s/concepts/security-policies#default_policies) әрекет етеді.

## Кластерді масштабтау мүмкіндіктері

Cloud Containers кластерінде [master-түйіндер мен worker-түйіндерді масштабтау мүмкіндіктері](../scale) кірістірілген.

Соның ішінде кластер түйіндерін автоматты масштабтау қолдау табады, мұнда түйіндер саны жұмыс жүктемесінің қажеттіліктеріне байланысты автоматты түрде реттеледі:

- Master-түйіндер үшін ол әдепкі бойынша қосылған және оны өшіру мүмкін емес.
- Worker-түйіндер үшін ол [Cluster Autoscaler](/ru/kubernetes/k8s/concepts/cluster-autoscaler) көмегімен жүзеге асырылады және [әрбір түйін тобы үшін параметрлерді](/ru/kubernetes/k8s/instructions/helpers/node-group-settings) анықтау кезінде қолмен қосылады.
