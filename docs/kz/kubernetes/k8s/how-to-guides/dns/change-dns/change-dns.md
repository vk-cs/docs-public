# {heading(Kubernetes кластері желісіндегі DNS серверлерінің мекенжайларын өзгерту)[id=k8s-change-dns]}

{include(/kz/_includes/_translated_by_ai.md)}

Мақалада кластердегі хосттардың домендік атауларын шешу үшін пайдаланылатын DNS серверлерінің IP мекенжайларын өзгерту мысалы көрсетілген.

## {heading(Дайындық қадамдары)[id=k8s-change-dns-prepare]}

1. {linkto(/ru/tools-for-using-services/cli/openstack-cli#tools-cli-openstack)[text=OpenStack CLI орнатыңыз және аутентификациядан өтіңіз]}, егер бұл әлі жасалмаған болса.
1. {var(cloud)} жеке кабинетіне [өтіңіз](https://kz.cloud.vk.com/app/).
1. **Kubernetes кластерлері** → **Kubernetes кластерлері** бөліміне өтіңіз.
1. DNS баптауларын өзгерту қажет кластердің атауын басыңыз.
1. **Жалпы деректер** қойындысында кластер орналасқан желі мен ішкі желінің атауын біліңіз.
1. Кластерге тағайындалғандарының орнына пайдаланылатын DNS серверлерінің IP мекенжайларын DNS провайдеріңізден біліңіз.

## {heading(1. Желі баптауларында DNS серверлерінің мекенжайларын өзгертіңіз)[id=k8s-change-dns-network-settings]}

1. Өңдеу қажет ішкі желінің ID идентификаторын алыңыз:

   ```console
   openstack subnet list
   ```

1. Тағайындалған барлық IP мекенжайларын тазалаңыз:

   ```console
   openstack subnet set --no-dns-nameservers <ID_ІШКІ_ЖЕЛІЛЕР>
   ```

   Мұнда `<ID_ІШКІ_ЖЕЛІЛЕР>` — өңдегіңіз келетін ішкі желінің идентификаторы.

1. Жаңа IP мекенжайын көрсетіңіз:

   ```console
   openstack subnet set --dns-nameserver <МЕКЕНЖАЙ_DNS> <ID_ІШКІ_ЖЕЛІЛЕР>
   ```

   Бірден бірнеше IP мекенжайын көрсетуге болады, бұл жағдайда әр мекенжайдың алдында `--dns-nameserver` параметрін қайталаңыз. Команда мысалы:

   ```console
   openstack subnet set --dns-nameserver <МЕКЕНЖАЙ_DNS_1> --dns-nameserver <МЕКЕНЖАЙ_DNS_2> --dns-nameserver <МЕКЕНЖАЙ_DNS_3> <ID_ІШКІ_ЖЕЛІЛЕР>
   ```

## {heading(2. Кластерді қайта іске қосыңыз)[id=k8s-change-dns-restart-cluster]}

Өзгерістер күшіне енуі үшін:

1. {linkto(../../../instructions/manage-cluster#k8s-manage-cluster-stop)[text=Тоқтатыңыз]} кластеріңізді және барлық тораптардың толық тоқтауын күтіңіз.
1. {linkto(../../../instructions/manage-cluster#k8s-manage-cluster-start)[text=Қайта іске қосыңыз]} кластерді және барлық тораптардың инициализациялануын күтіңіз.

{note:warn}
Қайта іске қосу кластердегі сервистердің уақытша қолжетімсіз болуына әкеледі. Кластерді қолданбаның белсенділігі ең төмен уақытта тоқтатып, іске қосыңыз.
{/note}

Кластер іске қосылған кезде оның барлық тораптары жаңартылған DNS баптауларын алады.
