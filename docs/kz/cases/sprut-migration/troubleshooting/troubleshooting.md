## Default қауіпсіздік тобы атауына қатысты мәселе

Егер көшіру үшін [OpenStack Terraform Provider](https://docs.comcloud.xyz/providers/terraform-provider-openstack/openstack/latest) пайдалансаңыз, `default` қауіпсіздік тобының атауына қатысты мәселе туындауы мүмкін.

### Мәселе қалай көрінеді

Екі SDN (Neutron және Sprut) бар жобада желілерде әдепкі бойынша `default` атауымен қауіпсіздік тобы болады.

OpenStack Terraform Provider қауіпсіздік топтарының ID мәндерімен емес, тек атауларымен жұмыс істейді. Соның салдарынан қақтығыс туындайды, өйткені провайдер қай `default` қауіпсіздік тобын пайдалану керегін бірмәнді түрде анықтай алмайды.

### Мәселені шешудің ықтимал жолдары

- [terraform-provider](https://github.com/vk-cs/terraform-provider-vkcs) VK Cloud пайдаланыңыз, өйткені ол қауіпсіздік топтары үшін ID қолдануға мүмкіндік береді.
- Terraform кодында қауіпсіздік тобының атауын өзгертіңіз:

    1. ВМ жасау кезінде манифесте `default` қауіпсіздік тобын көрсетіңіз. ВМ сәтті жасалады, бірақ `terraform plan` командасын орындағанда `default-sprut` қауіпсіздік тобы қайтарылады, ал state-файл жаңартыла алмайды.
    1. Манифестегі қауіпсіздік тобын қолмен `default-sprut` деп өзгертіңіз.

- Жобаны SDN Sprut жүйесіне толық көшіріп, содан кейін жоба ішіндегі SDN Neutron жүйесін [техникалық қолдау](/kz/contacts) арқылы өшіріңіз.
- Егер виртуалды машиналар сирек жасалса, state-файлды қолмен өзгертіңіз. State-файлда `security group = "default-sprut"` деп көрсетіп, оны сақтаңыз.
- Әдепкі қауіпсіздік топтарын пайдаланбаңыз.
