## Дайындық қадамдары

1. Көшіру үшін жобаны таңдаңыз.
1. [Қараңыз](/kz/tools-for-using-services/account/instructions/project-settings/manage#sdn_view), жобаңызға қандай SDN қосылған. Егер SDN Sprut қосылмаған болса, [техникалық қолдау қызметіне](/kz/contacts) хабарласыңыз.
1. Көшіруді орындау үшін әкімшінің жұмыс орнын дайындаңыз:

    1. Интернетке қолжетімді ішкі желіде [ВМ жасаңыз](/kz/computing/iaas/instructions/vm/vm-create). Келесі параметрлерді көрсетіңіз:

        - **Операциялық жүйе**: Ubuntu таңдаңыз.
        - **Firewall баптаулары**: ВМ-ге CLI арқылы қол жеткізу үшін `ssh` көрсетіңіз.
        - **Сыртқы IP тағайындау**: опцияны қосыңыз.
        - Қалған параметрлерді өз қалауыңыз бойынша таңдаңыз.

    1. ВМ жасалғанша күтіңіз және оған [SSH арқылы қосылыңыз](/kz/computing/iaas/instructions/vm/vm-connect/vm-connect-nix).

    1. Пакеттер индекстерін келесі пәрменді орындап жаңартыңыз:

        ```console
        sudo apt-get update
        ```

    1. [jq](https://jqlang.github.io/jq/) утилитасын орнатыңыз:

        ```console
        sudo apt-get install -y jq
        ```

    1. [OpenStack клиентін](/kz/tools-for-using-services/cli/openstack-cli#1_openstack_klientin_ornatynyz) орнатып, жобада [аутентификациядан өтіңіз](/kz/tools-for-using-services/cli/openstack-cli#3_autentifikaciyadan_otiniz).

    1. Көшіруді орындауға арналған скрипттері бар репозиторийді көшіріңіз:

        ```console
        git clone https://github.com/vk-cs/neutron-2-sprut.git
        ```
    1. `neutron-2-sprut` директориясына өтіңіз.
    1. Осы директориядағы `.sh` кеңейтімі бар барлық файлдарға орындау құқығын қосатын пәрменді орындаңыз:

        ```console
        chmod +x .sh
        ```

## 1. Көшіру үшін жоба инфрақұрылымы туралы деректерді жинаңыз

Жобаның желілік инфрақұрылымын бағалап, көшіру қажет мәндерді анықтаңыз:

1. Әкімші ВМ-ге SSH арқылы қосылыңыз.
1. ВМ тізімін алыңыз:

    ```console
    openstack server list
    ```
   Жауаптан келесі мәліметтерді тауып, сақтаңыз:

    - бірнеше желілік интерфейсі бар виртуалды машиналардың ID-лері;
    - тағайындалған Floating IP-адрестері бар виртуалды машиналардың ID-лері мен IP-адрестері;
    - көшіру қажет қалған барлық ВМ-дердің ID-лері.
1. Маршрутизаторлар тізімін алыңыз:

    ```console
    openstack router list
    ```

   Көшіру қажет маршрутизаторлардың ID-лерін сақтаңыз.

1. Егер жобаңызда IPsec туннельдері болса, туннельді құру үшін пайдаланылатын маршрутизаторлардың ID-лерін сақтаңыз.

1. Қауіпсіздік топтарының тізімін алыңыз:

    ```console
    ```

   Әдепкі топтардан басқа барлық қауіпсіздік топтарының атауларын сақтаңыз: `all`, `default`, `ssh+www`.

1. `all`, `default`, `ssh+www` әдепкі топтарының атаулары SDN Sprut пен Neutron-да бірдей. Осы топтардың идентификаторларын біліп, сақтаңыз:

    1. VK Cloud-тың [жеке кабинетіне](https://cloud.vk.com/app/) өтіңіз.
    1. Жобаны таңдаңыз.
    1. **Виртуалды желілер** → **Firewall баптаулары** бөліміне өтіңіз.
    1. SDN Sprut және Neutron ішіндегі `all`, `default`, `ssh+www` топтарының ID-лерін тауып, сақтаңыз.

## 2. Маршрутизаторларды, желілерді және ішкі желілерді SDN Sprut-қа көшіріңіз

1. Әкімші ВМ-ге SSH арқылы қосылып, `neutron-2-sprut` директориясына өтіңіз.
1. Маршрутизаторлар тізімі бар CSV форматындағы конфигурациялық файл жасаңыз. Бұл файл скрипт арқылы базалық желілік мәндерді көшіру үшін қажет болады.

   Файл пішімі:

    ```csv
    <ID маршрутизатора 1>,<std|adv>
    <ID маршрутизатора 2>,<std|adv>
    ```

   Мұндағы `<std|adv>` — SDN Sprut ішіндегі маршрутизатор көшірмесінің түрі:

    - `std` — көшірме [стандартты](/kz/networks/vnet/concepts/router#standard) маршрутизатор түрінде жасалады.
    - `adv` — көшірме [кеңейтілген](/kz/networks/vnet/concepts/router#advanced) маршрутизатор түрінде жасалады. Егер жобаңыздағы маршрутизатор IPsec туннелін құру үшін пайдаланылса, осы параметрді көрсетіңіз.

   {cut(Маршрутизаторларды көшіруге арналған конфигурациялық файл мысалы)}

    ```csv
    35fbfb7c-aaaa-aaaa-aaaa-c3fb4d833dee,adv
    a2dd20ca-bbbb-bbbb-bbbb-7ea8b00a81a6,std
    ```
   {/cut}

1. Маршрутизаторларды көшіру скриптін іске қосыңыз:

    ```console
    ./copy-router-and-networks.sh config.csv
    ```

   Мұнда `config.csv` — маршрутизаторлар тізімі бар конфигурациялық файлдың атауы.

   Скрипт орындалғаннан кейін көрсетілген маршрутизаторлар SDN Sprut-қа олармен байланысқан статикалық маршруттармен, желілермен және ішкі желілермен бірге көшіріледі. Әрбір көшірілген мәннің атауына `-sprut` постфиксі қосылады.

## 3. Скрипттің орындалуын тексеріңіз

Көшірілген мәндерді қарап шығып, барлық маршрутизаторлар, желілер және ішкі желілер көшірілгеніне көз жеткізіңіз:

1. Желілер тізімін тексеріңіз:

    ```console
    openstack network list
    ```

1. Ішкі желілер тізімін тексеріңіз:

    ```console
    openstack subnet list
    ```

1. Стандартты маршрутизаторлар тізімін тексеріңіз:

    ```console
    openstack router list
    ```

1. Қосылған кеңейтілген маршрутизаторлар тізімін тексеріңіз. Ол үшін жеке кабинетке өтіңіз немесе REST API сұрауын орындаңыз:

    ```html
    sprut_api_base="https://infra.mail.ru:9696/v2.0"

    token=$(openstack token issue -c id -f value)

    sprut_advanced_routers=$(curl -s -X GET "${sprut_api_base}/direct_connect/dc_routers" \
        -H "Content-Type: application/json" \
        -H "X-Auth-Token: $token" \
        -H "X-SDN:SPRUT")
    echo "Advanced routers collected:"
    echo "$sprut_advanced_routers" | jq .
    echo ""
    ```

   {note:info}
   OpenStack CLI кеңейтілген маршрутизаторларды тізімде көрсетпейді.
   {/note}

## 4. (Опционалды) Кеңейтілген маршрутизаторды баптаңыз

Егер жобада IPsec туннелі пайдаланылса, скрипт жасаған кеңейтілген маршрутизаторды [баптаңыз](/kz/networks/vnet/how-to-guides/onpremise-connect/advanced-router) және оны IPsec туннелін қосуға дайындаңыз.

## 5. (Опционалды) IPsec туннелін көшіріңіз

Бірдей селекторлары бар туннельдер (ішкі желілердің бастапқы және мақсатты ауқымдары) әртүрлі SDN-де орналасса да, бір уақытта қатар өмір сүре алмайды. Сондықтан SDN Sprut-та SDN Neutron ішіндегі туннельге ұқсас туннельді алдын ала жасауға болмайды, себебі бұл бастапқы туннель жұмысына қиындықтар туғызады.

1. SDN Neutron ішіндегі бастапқы туннельді [жойыңыз](/kz/networks/vnet/instructions/vpn#vpn_tunnelin_zhoyu).
1. SDN Sprut-та бапталған кеңейтілген маршрутизатор үшін жаңа VPN туннелін [қосыңыз](/kz/networks/vnet/instructions/vpn#vpn_tunnelin_kosu) және IPsec саясатының параметрлерін орнатыңыз.

## 6. Қауіпсіздік топтарын SDN Sprut-қа көшіріңіз

1. Әкімші ВМ-ге SSH арқылы қосылыңыз.
1. `neutron-2-sprut` директориясына өтіңіз.
1. Қауіпсіздік топтарын SDN Sprut-қа көшіріңіз:

    ```console
     ./copy-security-group.sh \
        --group-mapping=<id группы 1 в SDN Neutron>=<id группы 1 в SDN Sprut>,<id группы 2 в SDN Neutron>=<id группы 2 в SDN Sprut>,... \
        --groups=<название группы 3>,<название группы 4>,...
    ```

   Мұнда:

    - `--group-mapping`: параметр әртүрлі SDN-дегі бұрыннан бар қауіпсіздік топтарын ID бойынша өзара сәйкестендіруге мүмкіндік береді. Бұл көшірілетін топтардағы ережелерде дереккөз CIDR емес, басқа қауіпсіздік тобы болып көрсетілген жағдайда қажет. Әдетте бұлар `all`, `default`, `ssh+www` топтары.
    - `--groups`: SDN Neutron ішіндегі SDN Sprut-қа көшірілуі керек қауіпсіздік тобының атауы.

   Пәрмен мысалы:

    ```console
    ./copy-security-group.sh \
        --group-mapping=4a361c99-cccc-cccc-cccc-b59fb469d354=121a7574-dddd-dddd-dddd-66baa01400d6,14f303dc-aaaa-aaaa-aaaa-9637c9c342df=3371584a-bbbb-bbbb-bbbb-1cc9f3e89601 \
        --groups=web-server-security,spark-k8s-6681-master,ml_sec_group
    ```

   Қауіпсіздік топтары `-sprut` постфиксімен көшіріледі.

## 7. Қауіпсіздік топтарын тексеруді орындаңыз

SDN Sprut ішіндегі қауіпсіздік топтары SDN Neutron ішіндегі қауіпсіздік топтарына ұқсас екеніне көз жеткізіңіз. SDN Neutron-нан көшірілген SDN Sprut топтарының `-sprut` постфиксі болады.

1. Әкімші ВМ-ге SSH арқылы қосылып, `neutron-2-sprut` директориясына өтіңіз.
1. Скриптті іске қосыңыз:

    ```console
    ./check-if-all-sprut-sg-present.sh
    ```

   Егер барлық қажетті топтар көшірілген болса, скрипт орындалғаннан кейін мына хабарлама қайтарылады:

    ```console
    All security groups have corresponding '-sprut' groups.
    ```

## 8. (Опционалды) SDN Sprut-та Floating IP-адрес жасаңыз

Floating IP-адрестерді басқа SDN-ге көшіру мүмкін емес. Егер жобаңыздың инфрақұрылымында Floating IP-адрестер болса, SDN Sprut-та [жаңа Floating IP-адрестерді жасаңыз](/kz/networks/vnet/instructions/ip/floating-ip#add) және оларды жазып алыңыз.

## 9. ВМ желілік интерфейстерін SDN Sprut-қа ауыстырыңыз

{note:warn}
Бұл операция желілік байланысты үзу арқылы орындалады. Егер жұмыс істеп тұрған желілік инфрақұрылымды көшіріп жатсаңыз, сервис пайдаланушыларын техникалық жұмыстар туралы ескертіңіз және бұл операцияны ең аз жүктеме сағаттарында орындаңыз.
{/note}

ВМ-ді SDN Sprut-қа көшіру үшін оның порттарын жаңа ішкі желілерге қосу қажет. Бұл ВМ-дегі порттар санына және қосылым түріне қарай әртүрлі тәсілдермен орындалады:

- Бірнеше ВМ-ді бір скриптті орындау арқылы ауыстыруға болады, бірақ бұл ВМ-дердің әрқайсысында тек бір желілік интерфейс болуы және `ext-net` сыртқы желісіне тікелей қосылымдары болмауы тиіс.
- Бір ВМ-ді жеке скриптті орындау арқылы ауыстыруға болады. Бұл тәсіл жобаңызда бірнеше желілік интерфейсі бар, яғни маршрутизатор, прокси немесе шекаралық фаервол рөлін атқаратын ВМ болса, қолайлы.
- Егер конфигурацияңызда ВМ `ext-net` сыртқы желісіне тікелей қосылған болса, оны SDN Sprut-қа өзіңізге ыңғайлы кез келген тәсілмен көшіріңіз:

    - SDN Sprut-та [Floating IP-адрес қосыңыз](/kz/networks/vnet/instructions/ip/floating-ip#add) және оны ВМ-ге қосыңыз.
    - SDN Sprut-та ВМ-ді [қайта жасаңыз](/kz/computing/iaas/instructions/vm/vm-create) және оны `internet` сыртқы желісіне қосыңыз.

    - SDN Sprut ішіндегі мақсатты ішкі желі атауы.

1. Миграция жүргізілетін техникалық терезені бөліңіз. Ауыстыру кезінде ВМ желілік байланысын жоғалтады. Бір ВМ-ді ауыстыру шамамен 45 секундта орындалады.
1. Әкімші ВМ-ге SSH арқылы қосылып, `neutron-2-sprut` директориясына өтіңіз.
1. ВМ-нің белсенді екеніне көз жеткізіңіз:

    ```console
    openstack server show <имя ВМ>
    ```

   Жауапта ВМ күйі **ACTIVE** болуы тиіс.
1. `neutron-2-sprut` директориясына өтіңіз.
1. Көшіру скриптін іске қосыңыз:

    ```console
    ./migrator.sh --opt <clone/noclone>
    ```

   Мұндағы `<clone/noclone>` MAC және IP-адрестер туралы ақпаратты сақтауға мүмкіндік береді:

    - `clone` — MAC және IP-адрестер туралы ақпаратты сақтай отырып көшіру.
    - `noclone` — мақсатты ішкі желіден кездейсоқ MAC және IP-адрестерді тағайындай отырып көшіру.

1. SDN Sprut-қа көшіру қажет ВМ атауын енгізіңіз.
1. ВМ-ді қосу қажет SDN Sprut ішіндегі желі атауын енгізіңіз.
1. ВМ-ді қосу қажет SDN Sprut ішіндегі ішкі желі атауын енгізіңіз.
1. Скрипт орындалғаннан кейін көшірілген ВМ консольін ашыңыз немесе SSH арқылы қосылып, қосылған желілік интерфейстерге IP-адрес алу үшін DHCP серверіне сұрау жіберіңіз:

   {tabs}

   {tab(Linux)}

    ```console
    dhclient
    ```
   {/tab}

   {tab(Windows)}

    ```console
    ipconfig /release
    ipconfig /renew
    ```
   {/tab}

   {/tabs}

{/tab}
