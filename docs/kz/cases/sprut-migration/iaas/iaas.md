## Дайындық қадамдары

1. Миграция үшін жобаны таңдаңыз.
1. Жобаңызда қандай SDN қосылғанын [қараңыз](/kz/tools-for-using-services/account/instructions/project-settings/manage#sdn_view). Егер SDN Sprut қосылмаған болса, [техникалық қолдауға](/kz/contacts) жүгініңіз.
1. Миграцияны орындау үшін әкімшінің жұмыс орнын дайындаңыз:

    1. Интернетке қолжетімділігі бар ішкі желіде [ВМ жасаңыз](/kz/computing/iaas/instructions/vm/vm-create). Келесі параметрлерді көрсетіңіз:

        - **Операциялық жүйе**: Ubuntu таңдаңыз.
        - **Firewall баптаулары**: CLI арқылы ВМ-ге қол жеткізу үшін `ssh` көрсетіңіз.
        - **Сыртқы IP тағайындау**: опцияны қосыңыз.
        - Қалған параметрлерді өз қалауыңыз бойынша таңдаңыз.

    1. ВМ жасалғанын күтіңіз және [оған SSH арқылы қосылыңыз](/kz/computing/iaas/instructions/vm/vm-connect/vm-connect-nix).

    1. Команданы орындап, пакеттер индекстерін жаңартыңыз:

        ```console
        sudo apt-get update
        ```

    1. [jq](https://jqlang.github.io/jq/) утилитасын орнатыңыз:

        ```console
        sudo apt-get install -y jq
        ```

    1. [OpenStack клиентін](/kz/tools-for-using-services/cli/openstack-cli#1_openstack_klientin_ornatynyz) орнатыңыз және жоба ішінде [аутентификациядан өтіңіз](/kz/tools-for-using-services/cli/openstack-cli#openstack-authorize).

    1. Миграцияны орындауға арналған скрипттері бар репозиторийді көшіріңіз:

        ```console
        git clone https://github.com/vk-cs/neutron-2-sprut.git
        ```
    1. `neutron-2-sprut` директориясына өтіңіз.
    1. Осы директориядағы `.sh` кеңейтімі бар барлық файлдарға орындау құқықтарын қосатын команданы орындаңыз:

        ```console
        chmod +x .sh
        ```

## 1. Миграция үшін жоба инфрақұрылымы туралы деректерді жинаңыз

Жобаның желілік инфрақұрылымын бағалап, көшіру қажет мәндерді анықтаңыз:

1. Әкімші ВМ-іне SSH арқылы қосылыңыз.
1. ВМ тізімін алыңыз:

    ```console
    openstack server list
    ```
    Жауаптан келесі мәліметтерді тауып, сақтаңыз:

     - Бірнеше желілік интерфейсі бар виртуалды машиналардың ID-лері.
     - Тағайындалған Floating IP-адрестері бар виртуалды машиналардың ID-лері және IP-адрестері.
     - Көшіру қажет қалған барлық ВМ-дердің ID-лері.
1. Маршрутизаторлар тізімін алыңыз:

    ```console
    openstack router list
    ```

    Көшіру қажет маршрутизаторлардың ID-лерін сақтаңыз.

1. Егер жобаңызда IPsec-туннельдер болса, туннельді құру үшін пайдаланылатын маршрутизаторлардың ID-лерін сақтаңыз.

1. Қауіпсіздік топтарының тізімін алыңыз:

    ```console
    openstack security group list
    ```

    Әдепкі топтардан басқа барлық қауіпсіздік топтарының атауларын сақтаңыз: `all`, `default`, `ssh+www`.

1. Әдепкі `all`, `default`, `ssh+www` топтарының атаулары SDN Sprut және Neutron ішінде бірдей. Осы топтардың идентификаторларын біліп, сақтаңыз:

    1. VK Cloud жеке кабинетіне [өтіңіз](https://cloud.vk.com/app/).
    1. Жобаны таңдаңыз.
    1. **Виртуалды желілер** → **Firewall баптаулары** бөліміне өтіңіз.
    1. SDN Sprut және Neutron ішіндегі `all`, `default`, `ssh+www` топтарының ID-лерін тауып, сақтаңыз.

## 2. Маршрутизаторларды, желілерді және ішкі желілерді SDN Sprut-қа көшіріңіз

1. Әкімші ВМ-іне SSH арқылы қосылып, `neutron-2-sprut` директориясына өтіңіз.
1. Маршрутизаторлар тізімі бар CSV форматындағы конфигурациялық файл жасаңыз. Бұл файл скрипт көмегімен базалық желілік мәндерді көшіру үшін қажет болады.

    Файл пішімі:

    ```csv
    <МАРШРУТИЗАТОР_1>,<КӨШІРМЕ_ТҮРІ>
    <МАРШРУТИЗАТОР_2>,<КӨШІРМЕ_ТҮРІ>
    ```

    Мұнда:

   - `<МАРШРУТИЗАТОР_1>`, `<МАРШРУТИЗАТОР_2>` — маршрутизаторлардың идентификаторлары.
   - `<КӨШІРМЕ_ТҮРІ>` — SDN Sprut-тағы маршрутизатор көшірмесінің түрі:

      - `std` — көшірме [стандартты](/kz/networks/vnet/concepts/router#vnet-router-standard) маршрутизатор түрінде жасалады.
      - `adv` — көшірме [кеңейтілген](/kz/networks/vnet/concepts/router#vnet-router-advanced) маршрутизатор түрінде жасалады. Егер жобаңыздағы маршрутизатор IPsec-туннель құру үшін пайдаланылса, осы параметрді көрсетіңіз.

    Маршрутизаторларды көшіруге арналған конфигурациялық файл мысалы:

    ```csv
    35fbfb7c-aaaa-aaaa-aaaa-c3fb4d833dee,adv
    a2dd20ca-bbbb-bbbb-bbbb-7ea8b00a81a6,std
    ```

1. Маршрутизаторларды миграциялау скриптін іске қосыңыз:

    ```console
    ./copy-router-and-networks.sh <ФАЙЛ_АТАУЫ>
    ```

    Мұнда `<ФАЙЛ_АТАУЫ>` — маршрутизаторлар тізімі бар конфигурациялық файлдың атауы.

    Мысал:

    ```console
    ./copy-router-and-networks.sh config.csv
    ```

    Скрипт орындалғаннан кейін көрсетілген маршрутизаторлар SDN Sprut-қа статикалық маршруттармен, желілермен және олар қосылған ішкі желілермен бірге көшіріледі. Әрбір көшірілген мәннің атауына `-sprut` постфиксі қосылады.

## 3. Скрипттің орындалуын тексеріңіз

Көшірілген мәндерді қарап шығып, барлық маршрутизаторлар, желілер және ішкі желілер көшірілгеніне көз жеткізіңіз:

1. Желілер тізімін тексеріңіз:

    ```console
    openstack network list
    ```

1. Ішкі желілер тізімін тексеріңіз:

    ```console
    openstack subnet list
    ```

1. Стандартты маршрутизаторлар тізімін тексеріңіз:

    ```console
    openstack router list
    ```

1. Қосылған кеңейтілген маршрутизаторлар тізімін тексеріңіз. Ол үшін жеке кабинетке өтіңіз немесе REST API сұрауын орындаңыз:

    ```console
    sprut_api_base="https://infra.mail.ru:9696/v2.0"

    token=$(openstack token issue -c id -f value)

    sprut_advanced_routers=$(curl -s -X GET "${sprut_api_base}/direct_connect/dc_routers" \
        -H "Content-Type: application/json" \
        -H "X-Auth-Token: $token" \
        -H "X-SDN:SPRUT")
    echo "Advanced routers collected:"
    echo "$sprut_advanced_routers" | jq .
    echo ""
    ```

    {note:info}
    OpenStack CLI тізімде кеңейтілген маршрутизаторларды көрсетпейді.
    {/note}

## 4. (Қосымша) Кеңейтілген маршрутизаторды баптаңыз

Егер жобада IPsec-туннель пайдаланылса, скрипт жасаған кеңейтілген маршрутизаторды [баптаңыз](/kz/networks/vnet/how-to-guides/onpremise-connect/advanced-router) және оны IPsec-туннелін қосуға дайындаңыз.

## 5. (Қосымша) IPsec-туннелін көшіріңіз

Бірдей селекторлары (ішкі желілердің бастапқы және мақсатты диапазондары) бар туннельдер, тіпті әртүрлі SDN ішінде болса да, бір уақытта қатар өмір сүре алмайды. Сондықтан SDN Sprut-та SDN Neutron-тағы туннельге ұқсас туннельді алдын ала жасауға болмайды, өйткені бұл бастапқы туннельдің жұмысына қатысты мәселелерге әкеледі.

1. SDN Neutron-тағы бастапқы туннельді [жойыңыз](/kz/networks/vnet/instructions/vpn#vnet-vpn-delete).
1. SDN Sprut-та бапталған кеңейтілген маршрутизатор үшін жаңа VPN-туннелін [қосыңыз](/kz/networks/vnet/instructions/vpn#vnet-vpn-add) және IPsec-саясатының параметрлерін орнатыңыз.

## 6. Қауіпсіздік топтарын SDN Sprut-қа көшіріңіз

1. Әкімші ВМ-іне SSH арқылы қосылыңыз.
1. `neutron-2-sprut` директориясына өтіңіз.
1. Қауіпсіздік топтарын SDN Sprut-қа көшіріңіз:

    ```console
     ./copy-security-group.sh \
        --group-mapping=<NEUTRON_ТОП_1>=<SPRUT_ТОП_1>,<NEUTRON_ТОП_2>=<SPRUT_ТОП_2>,... \
        --groups=<ТОП_3>,<ТОП_4>,...
    ```

    Мұнда:

    - `--group-mapping` — параметр әртүрлі SDN-дердің бұрыннан бар қауіпсіздік топтарын ID бойынша өзара сәйкестендіруге мүмкіндік береді. Бұл көшірілетін топтарда дереккөз CIDR емес, басқа қауіпсіздік тобы болатын ережелер бар болса қажет. Әдетте, бұл `all`, `default`, `ssh+www` топтары. Параметр аргументтері:
      - `<NEUTRON_ТОП_1>`, `<NEUTRON_ТОП_2>` — SDN Neutron ішіндегі қауіпсіздік топтарының идентификаторлары;
      - `<SPRUT_ТОП_1>`, `<SPRUT_ТОП_2>`  — SDN Sprut ішіндегі қауіпсіздік топтарының идентификаторлары;

    - `--groups` — параметр SDN Neutron ішіндегі SDN Sprut-қа көшірілуі тиіс қауіпсіздік топтарының атауларын көрсетеді. Параметр аргументтері: `<ТОП_3>`, `<ТОП_4>` — көшірілетін топтардың атаулары.

    Команда мысалы:

    ```console
    ./copy-security-group.sh \
        --group-mapping=4a361c99-cccc-cccc-cccc-b59fb469d354=121a7574-dddd-dddd-dddd-66baa01400d6,14f303dc-aaaa-aaaa-aaaa-9637c9c342df=3371584a-bbbb-bbbb-bbbb-1cc9f3e89601 \
        --groups=web-server-security,spark-k8s-6681-master,ml_sec_group
    ```

    Қауіпсіздік топтары `-sprut` постфиксімен көшіріледі.

## 7. Қауіпсіздік топтарын тексеруді орындаңыз

SDN Sprut-тағы қауіпсіздік топтары SDN Neutron-тағы қауіпсіздік топтарына ұқсас екеніне көз жеткізіңіз. SDN Neutron-тан көшірілген SDN Sprut топтарында `-sprut` постфиксі бар.

1. Әкімші ВМ-іне SSH арқылы қосылып, `neutron-2-sprut` директориясына өтіңіз.
1. Скриптті іске қосыңыз:

    ```console
    ./check-if-all-sprut-sg-present.sh
    ```

    Егер барлық қажетті топтар көшірілсе, скрипт орындалуы нәтижесінде келесі хабарлама қайтарылады:

    ```console
    All security groups have corresponding '-sprut' groups.
    ```

## 8. (Қосымша) SDN Sprut-та Floating IP-адрес жасаңыз

Floating IP-адрестерін басқа SDN-ге көшіру мүмкін емес. Егер жобаңыздың инфрақұрылымында Floating IP-адрестер болса, SDN Sprut-та [жаңаларын жасаңыз](/kz/networks/vnet/instructions/ip/floating-ip#vnet-floating-ip-add) және оларды жазып алыңыз.

## 9. ВМ желілік интерфейстерін SDN Sprut-қа ауыстырыңыз

{note:warn}
Бұл операция желілік байланыс үзілуімен орындалады. Егер сіз жұмыс істеп тұрған желілік инфрақұрылымды миграциялап жатсаңыз, сервис пайдаланушыларын техникалық жұмыстар туралы алдын ала ескертіңіз және бұл операцияны жүктеме ең аз болатын уақытта орындаңыз.
{/note}

ВМ-ді SDN Sprut-қа көшіру үшін оның порттарын жаңа ішкі желілерге қосу қажет. Бұл ВМ порттарының санына және қосылу түріне байланысты әртүрлі тәсілдермен орындалады:

- Бірнеше ВМ-ді бір скриптті орындау арқылы ауыстыруға болады, бірақ бұл ВМ-дерде тек бір желілік интерфейс болуы және сыртқы `ext-net` желісіне тікелей қосылымдары болмауы керек.
- Бір ВМ-ді жеке скриптті орындау арқылы ауыстыруға болады. Бұл тәсіл жобаңызда бірнеше желілік интерфейсі бар, яғни маршрутизатор, прокси немесе шекаралық файервол рөлін атқаратын ВМ болса қолайлы.
- Егер конфигурацияңызда ВМ сыртқы `ext-net` желісіне тікелей қосылған болса, оны SDN Sprut-қа өзіңізге ыңғайлы кез келген тәсілмен көшіріңіз:

  - SDN Sprut-та [Floating IP-адрес қосыңыз](/kz/networks/vnet/instructions/ip/floating-ip#vnet-floating-ip-add) және оны ВМ-ге байланыстырыңыз.
  - SDN Sprut-та [ВМ-ді қайта жасаңыз](/kz/computing/iaas/instructions/vm/vm-create) және оны сыртқы `internet` желісіне қосыңыз.

1. Көшіру қажет ВМ тізімі бар CSV форматындағы файл жасаңыз.

    Файл пішімі:

    ```csv
    <ВМ_1>,<SPRUT_ЖЕЛІ_1>,<SPRUT_ІШКІ_ЖЕЛІ_1>,<SPRUT_FLOATING-IP>

    ```

    Мұнда:

    - `<ВМ_1>`, `<ВМ_2>` — SDN Sprut-қа көшіру қажет ВМ атаулары.
    - `<SPRUT_ЖЕЛІ_1>`, `<SPRUT_ЖЕЛІ_2>` — ВМ-ді қосу қажет SDN Sprut-тағы желілердің атаулары.
    - `<SPRUT_ІШКІ_ЖЕЛІ_1>`, `<SPRUT_ІШКІ_ЖЕЛІ_2>` — ВМ-ді қосу қажет SDN Sprut-тағы ішкі желілердің атаулары.
    - (Қосымша) `<SPRUT_FLOATING_IP>` — ВМ-ге байланыстыру қажет SDN Sprut-тағы Floating IP-адрес.

    ВМ тізімі бар файл мысалы:

    ```csv
    web-server-1,web-app-network-sprut,app-subnet-sprut,83.166.236.59
    web-server-2,web-app-network-sprut,app-subnet-sprut
    ```

1. Миграция жүргізілетін техникалық терезені бөліңіз. ВМ ауысу кезінде желілік байланысын жоғалтады. Бір ВМ-ді ауыстыру шамамен 45 секунд алады.

1. ВМ-дердің белсенді екеніне көз жеткізіңіз:

    ```console
    openstack server list
    ```

    SDN Sprut-қа көшіру қажет барлық ВМ үшін күйі `ACTIVE` болуы тиіс.

1. Миграция скриптін іске қосыңыз:

    ```console
    ./migrator-multiple.sh <ФАЙЛ_АТАУЫ> \
     --all-secgroup-sprut-id=<ALL_ТОБЫНЫҢ_ID> \
     --ssh-www-secgroup-sprut-id=<SSH+WWW_ТОБЫНЫҢ_ID>
    ```

    Мұнда:

    - `<ФАЙЛ_АТАУЫ>` — ВМ тізімі бар файл атауы.
    - (Қосымша ) `--all-secgroup-sprut-id` — `all` қауіпсіздік тобы жобаға әдепкі бойынша қосылады және SDN Sprut пен Neutron ішінде бір атауға ие. Егер топ ВМ-де пайдаланылса, OpenStack CLI қауіпсіздік тобының SDN-ге тиесілігін ажырата алуы үшін Sprut-тағы топтың ID-сін көрсетіңіз. Параметр аргументі: `<ALL_ТОБЫНЫҢ_ID>` — SDN Sprut-тағы `all` қауіпсіздік тобының идентификаторы.
    - (Қосымша ) `--ssh-www-secgroup-sprut-id` — `ssh+www` қауіпсіздік тобы жобаға әдепкі бойынша қосылады және SDN Sprut пен Neutron ішінде бір атауға ие. Егер топ ВМ-де пайдаланылса, OpenStack CLI қауіпсіздік тобының SDN-ге тиесілігін ажырата алуы үшін Sprut-тағы топтың ID-сін көрсетіңіз.
