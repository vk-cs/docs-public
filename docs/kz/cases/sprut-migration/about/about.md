## Миграция не үшін қажет

Бұлттың желілік инфрақұрылымының сенімділігі мен қолжетімділігін арттыру үшін жобаларды [Sprut](/kz/networks/vnet/concepts/sdn#vnet-sdn-sprut) — VK Cloud компаниясының меншікті әзірлемесі болып табылатын SDN-ге көшіру ұсынылады. Ол ауқымды көлемде желілер мен осы желілердің үстіндегі желілік функциялардың тұрақты жұмысын қамтамасыз етеді.

SDN Sprut бұрын қолданылған [SDN Neutron](/kz/networks/vnet/concepts/sdn#vnet-sdn-neutron) мүмкіндіктерін кеңейту үшін әзірленді. SDN Neutron шағын желілермен жұмыс істеуде тиімді, бірақ ірі желілермен жұмыс істегенде (мыңдаған және ондаған мың порт) шектеулері бар. SDN барлық виртуалды желілер мен сервистердің негізі болғандықтан, бұл шектеулер пайдаланушы жобаларының да, бүкіл платформаның да тұрақтылығына, ауқымдалуына және дамуына елеулі әсер етті.

SDN Sprut Openstack Neutron API-мен толық үйлесімді және SDN Neutron-пен салыстырғанда бірқатар артықшылықтарға ие:

- оңайырақ ауқымдалады және кез келген көлемдегі желілерге жарамды;
- VK Cloud сервистерімен және үшінші тарап сервистерімен еркін интеграциялануға мүмкіндік береді;
- өнімділігі жоғары;
- SDN Neutron-да қолжетімсіз сервистері бар;
- OpenStack-тен тәуелсіз.

Қазір Sprut — барлық жаңа жобалар үшін негізгі SDN, бірақ ескі жобалар SDN Neutron пайдаланады. Жобаларыңызды SDN Sprut-қа өз бетіңізше көшіре аласыз. Жеке кабинеттен [жобаңыз қай SDN-ге қосылғанын білуге болады](/kz/tools-for-using-services/account/instructions/project-settings/manage#project-sdn-view).

{note:warn}
SDN Neutron қолдауы 2025 жылдың соңында тоқтатылады. Осыдан кейін SDN Neutron пайдаланушылары қауіпсіздік мәселелеріне және жаңартулардың болмауына тап болуы мүмкін.
{/note}

Жобаларыңызды SDN Sprut-қа өз бетіңізше көшіре аласыз.

Миграцияның функционалдық артықшылықтары:

- [Кеңейтілген маршрутизатор](/kz/networks/vnet/how-to-guides/onpremise-connect/advanced-router) BGP протоколы бойынша динамикалық маршрутизацияны қолдайды, бұл істен шығуға төзімді маршрутизация сұлбаларын құруға мүмкіндік береді.
- [Cloud Direct Connect](https://cloud.vk.com/direct-connect) — компанияның жергілікті желісі мен VK Cloud арасындағы бөлінген қосылымдар.
- [Ортақ желілер](/kz/networks/vnet/concepts/net-types#vnet-net-types-shared-net) бір аймақтағы жобаларды біріктіру үшін.
- Істен шығуға төзімділікті арттыруға арналған орталықтандырылмаған DHCP сервисі.
- PaaS сервистері аясындағы әртүрлі жобалар арасындағы оқшауланған трафик.
- Бұлт ішіндегі орталықтандырылмаған және істен шығуға төзімді [жеке DNS сервисі](/kz/networks/dns/instructions/private-dns).

Миграцияның функционалдық емес артықшылықтары:
- REST API және платформа сервистерімен немесе үшінші тарап сервистерімен интеграциялаудың басқа тетіктері.
- SDN Neutron-пен салыстырғанда өнімділіктің 34%-ға дейін артуы.
- API және UI жауап беру уақытының қысқаруы.
- SDN компоненттерін көлденең ауқымдауға дайындық.
- Желілік мәндердің жұмыс қабілетін автоматты түрде қалпына келтіру.
- Жүйенің оқиғаларға нақты уақыт режимінде әрекет етуі есебінен істен шығуға төзімділікті жақсарту.
- Желілік инфрақұрылымды автоматты конфигурациялау және мониторингтеу мүмкіндігі.

Барлық әрекеттер төрт кезеңге бөлінген:

4. Миграциядан кейінгі тестілеу және оңтайландыру.

## {heading(Миграция құралдары)[id=migration_instruments]}

Жобаңызды SDN Sprut-қа келесі құралдар арқылы өз бетіңізше көшіре аласыз:

- Terraform: көшірудің ең қарапайым тәсілі, бірақ көбірек техникалық терезені қажет етеді (сервис жұмыс істемейтін уақыт).
- Жеке кабинет және Openstack CLI: көшіруге дайындалуға көбірек уақытты қажет етеді, бірақ техникалық терезе аз болады.

Жобаларды көшіру кезінде келесі скрипттер қолданылады:

|===
|Скрипт
|Сипаттама

|[copy-router-and-networks.sh](https://github.com/vk-cs/neutron-2-sprut/blob/guide_v3/copy-router-and-networks.sh)
|Скрипт SDN Neutron ішіндегі маршрутизаторлардың және оларға қосылған желілер мен ішкі желілердің конфигурацияларын көшіреді және SDN Sprut-та соларға ұқсастарын жасайды
