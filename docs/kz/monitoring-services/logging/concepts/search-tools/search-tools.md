Іздеу тілі мыналарға мүмкіндік береді:

- бір немесе бірнеше сүзгілеуші өрнек бойынша іздеуге;
- логикалық операциялар мен салыстыру операцияларын қолдануға.

Логтарды сүзу үшін іздеу жолында сүзгі параметрлерін пайдаланыңыз:

| Параметр | Формат | Сипаттамасы | Мысал |
| ---------------------- | ----------------------------------------- | ------ | ------ |
| `message` | `message: "мән"` | Хабарламаларында берілген мәндер бар жазбаларды іздеу. Әдепкі параметр болып табылады, сұрауда оны көрсетпеуге болады | `message: "Hello world!"` |
| `timestamp` | `timestamp <салыстыру операциясы> "мән"` | Берілген уақыт аралығында жіберілген жазбаларды іздеу | `timestamp >= "2022-04-10T00:00:00Z"` |
| `level` | `level <салыстыру операциясы> мән` | Берілген логтау деңгейлері бар жазбаларды іздеу. Қолжетімді логтау деңгейлері — `DEBUG`, `INFO`, `WARN`, `ERROR`, `CRITICAL` | `level = INFO` |
| `payload` | `payload: "мән"` | `payload` ішінде берілген мәндер бар жазбалар мәтіні бойынша іздеу | `payload: warning` |
| `payload.field.search` | `payload.<field>: "мән"` | `payload` элементтері бойынша жазбаларды лексикографиялық іздеу. Егер ағаштың түбір элементі параметрлердің ешқайсысымен сәйкес келмесе, `payload` префиксін көрсетпеуге болады. `payload` ішінде элементтің бар-жоғын `EXISTS` операторы арқылы тексеруге болады: `payload.result EXISTS`. Мұндай сүзгі `payload` ішінде `result` элементі бар жазбаларды шығарады | `payload.status: created` |

Бір параметрдің бірнеше мәні бойынша жазбаларды сүзу үшін мәндерді бірінен кейін бірін тізіп шығыңыз.

{note:warn}

Егер іздеуде өрістермен өрнек көрсетілмесе, әдепкі бойынша іздеу `message` өрісіндегі ішкі жол бойынша орындалады.

{/note}

## Талаптар

1. Өрнектегі `"`, `'` және `＼` таңбалары `＼` көмегімен экрандалады.
1. Мәнді тырнақшаға алмауға болады, егер ол:

    - латын әліпбиінің әрпінен басталса және тек латын әліпбиінің әріптерін, цифрларды және астыңғы сызу таңбаларын қамтыса;
    - таңбасыз бүтін сан болса.

1. `timestamp` үшін қолжетімді мәндер: RFC3339.

## Салыстыру операторлары

| Оператор | Түсіндірмесі |
| -------- | ----------- |
| `=` | Тең |
| `<>` | Тең емес |
| `>` | Үлкен |
| `<` | Кіші |
| `<=` | Кіші немесе тең |
| `>=` | Үлкен немесе тең |
| `:` | Қамтиды |

## Бірнеше шарты бар сүзгілер

Бір сүзгіде бірнеше шартты логикалық операторлардың көмегімен біріктіруге болады:

- `AND`
- `OR`
- `NOT`

## Есептеу тәртібі

Белгілі бір есептеу тәртібін беру үшін жақшаларды пайдаланыңыз:

```sql
параметр1: "мән1" AND (параметр2 = "мән2" OR параметр3 < "мән3")
```
