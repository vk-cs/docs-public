Qdrant — жоғары өлшемді векторлар түрінде ұсынылған деректерді іздеуге және талдауға арналған әмбебап векторлық СУБД және API-сервис. Qdrant жасанды интеллектке, машиналық оқытуға, ұсыным жүйелеріне, табиғи тілді өңдеуге және мультимодальды деректерге байланысты қолданбаларға жарайды.

Qdrant мүмкіндіктері:

- жоғары өнімділікпен үлкен көлемдегі векторлық деректерді сақтау және басқару;  
- жоғары өлшемді кеңістіктерде HNSW алгоритмін пайдалана отырып, ең жақын көршілерді (ANN) іздеу;  
- TensorFlow, PyTorch, Hugging Face және OpenAI сияқты танымал ML-фреймворктарымен интеграция;  
- атрибуттар бойынша сүзгілеу және сұрыптаумен бірге векторлық іздеуді біріктірілген түрде пайдалану;  
- үлкен деректер жиындарымен жұмыс істеуге арналған икемді көлденең масштабталу.  

Qdrant RESTful API және танымал бағдарламалау тілдеріндегі клиенттік кітапханаларды ұсынады, бұл қолданбалармен интеграцияны жеңілдетеді.

Qdrant сервисін пайдалану арқылы сіз [Marketplace](/kz/start/legal/vk/marketplace) және [Qdrant](https://github.com/qdrant/qdrant/blob/master/LICENSE) лицензиялық келісімдерімен келісесіз.

Жобада Qdrant сервисін өрістету үшін:

1. VK Cloud-та [тіркеліңіз](/kz/intro/onboarding/account) және [жеке кабинетке өтіңіз](https://kz.cloud.vk.com/app).
1. Өрістетілген сервисі бар ВМ-ге және интернеттен Qdrant веб-интерфейсіне қол жеткізу үшін:

    1. Егер ол бұрын жасалмаған болса, интернетке қолжетімді [желі жасаңыз](/kz/networks/vnet/instructions/net#zhelini_zhasau).
    1. Өрістетілген сервисі бар ВМ орналастырылатын [ішкі желі баптауларында](/kz/networks/vnet/instructions/net#zhelini_redakciyalau) **Жеке DNS** опциясын өшіріңіз.

1. **Магазин приложений** бөліміне өтіңіз.
1. **Qdrant** сервисінің карточкасында **Подробнее** батырмасын басыңыз.
1. Сервис сипаттамасы бар бетте **Подключить** батырмасын басыңыз.
1. **Параметры сервера** бетінде деректерді енгізіңіз.
    1. Qdrant параметрлерін көрсетіңіз:

        - **Как будет размещен Qdrant**: сервисті орналастыру түрін таңдаңыз.

            - `external` — VK Cloud ұсынған сыртқы IP-мекенжаймен және домендік атпен. Интернеттен өрістетілген Qdrant сервисі бар ВМ-ге қол жеткізу үшін осы орналастыру түрін таңдаңыз.
            - `internal` — тек VK Cloud ішінде қолжетімділікпен.

        - **S3-ке резервтік көшіруді қосу**: Qdrant деректерінің резервтік көшірмелерін [объектілік қоймада](/kz/storage/s3/concepts/about) күн сайын жасау үшін опцияны қосыңыз. Қойма автоматты түрде жасалады. Соңғы 7 көшірме сақталады.

        - **Мониторингті қосу**: өрістетілген Qdrant орналасқан виртуалды машинаның метрикаларын [Cloud Monitoring](/kz/monitoring-services/monitoring) сервисіне автоматты түрде жіберу үшін опцияны қосыңыз. Бұл метрикалар жеке кабинеттің **Мониторинг** бөлімінде қолжетімді болады.

    1. Сервис инстансы өрістетілетін виртуалды машинаның параметрлерін көрсетіңіз:

        - **Желі**: өрістетілген сервисі бар ВМ орналастырылатын желіні таңдаңыз. Егер сіз `external` орналастыру түрін көрсетсеңіз, интернетке қолжетімді желіні және **Жеке DNS** опциясы өшірілген ішкі желіні таңдаңыз.
        - **Зона доступности**: ВМ іске қосылатын [ЦОД-ты](/kz/start/concepts/architecture#az) таңдаңыз.
        - **Тип виртуальной машины**: [алдын ала орнатылған ВМ конфигурациясын](/kz/computing/iaas/concepts/vm/flavor) таңдаңыз.

    1. Жүйелік диск пен деректер дискісінің параметрлерін көрсетіңіз.

        - **Размер диска**: ВМ дискісінің қажетті көлемін гигабайтпен көрсетіңіз.
        - **Тип диска**: [диск түрлерінің бірін](/kz/computing/iaas/concepts/data-storage/disk-types#disk_types) таңдаңыз — `HDD`, `SSD` немесе `High-IOPS SSD`.

1. **Келесі қадам** батырмасын басыңыз.
1. Сервисті өрістетуге арналған инфрақұрылымның құнымен танысып, **Подключить тариф** батырмасын басыңыз.

    Бұлттық инфрақұрылымды өрістету басталады және сервис қасиеттері беті ашылады. Процесс күйі **Магазин приложений → Мои сервисы** бөлімінде де көрсетіледі.

    Сәтті өрістетуден кейін VK Cloud-тағы аккаунтыңызға байланыстырылған поштаға келесі хабарламалар келеді:

    - VK Cloud жеке кабинетіндегі сервис инстансына сілтемесі бар хабарлама;
    - сервиске кіру деректері бар бір реттік сілтемесі бар хат.

1. Хаттағы бір реттік сілтеме бойынша өтіп, сервиске кіру деректерін сақтаңыз:

    - `qdrant_url` — Qdrant API-іне сілтеме;
    - `api_key` — Qdrant API-іне қосылу кілті;
    - `keypair` — сервис өрістетілген виртуалды машинаға SSH арқылы қосылуға арналған жеке кілт.

   {note:info}

   Егер кіру деректері жоғалса, жаңаларын [генерациялаңыз](../../instructions/pr-instance-manage#update_access).

   {/note}

1. Qdrant сервисінің веб-интерфейсіне өтіңіз:

    1. `<АДРЕС_СЕРВИСА>/dashboard` сілтемесі бойынша өтіңіз, мұндағы `<АДРЕС_СЕРВИСА>` — `qdrant_url` мәніндегі сілтеме.
    1. API кілті ретінде `api_key` параметрінің мәнін көрсетіңіз.

2. (Опционалды) Сервиспен жұмысты бастау үшін [нұсқаулықты](https://qdrant.tech/documentation/interfaces) пайдаланыңыз. Қажет болса, [Qdrant ресми құжаттамасын](https://qdrant.tech/documentation) оқып шығыңыз.
