Бұл нұсқаулық WordPress-сайтыңыз үшін доменді ауыстырып, SSL сертификаттарын баптауға көмектеседі. Келесі мысал қарастырылады:

- WordPress сервисі HTTPD (Apache) процесін және SELinux мәжбүрлі қолжетімділікті бақылау жүйесін пайдалана отырып, AlmaLinux басқаруындағы серверде өрістетілген.
- Бұл сервер `old-domain.com` доменінде орналасқан және өзіңіз дербес тіркеген `new-domain.com` жаңа доменіне көшіріледі.

## 1. Сервердегі домендік атауды өзгертіңіз

1. Apache виртуалды хостының конфигурациясын өңдеңіз:

    1. Сайтыңызға арналған Apache конфигурация файлын ашыңыз. Әдетте ол `/etc/httpd/conf.d/your-site.conf` жолында орналасады.
    1. `VirtualHost` блогын тауып, ондағы ағымдағы домендік атауды жаңасына ауыстырыңыз.

        Жаңа домендік атауы бар блоктың мысалы:

        ```apache
        <VirtualHost *:80>
            ServerName new-domain.com
            ServerAlias www.new-domain.com
            DocumentRoot /var/www/html
            <Directory "/var/www/html">
                AllowOverride All
                Require all granted
            </Directory>
        </VirtualHost>
        ```

1. Егер қосымша конфигурация файлдары пайдаланылса (мысалы, ішкі домендер үшін), олардағы домендік атауды жаңасына ауыстырыңыз.

1. Келесі команда арқылы Apache-ті қайта іске қосыңыз:

    ```console
    sudo systemctl restart httpd
    ```

1. Егер SELinux қосылған болса, оны жаңа доменмен жұмыс істеуге баптаңыз:

    1. Apache жаңа конфигурацияларға қол жеткізе алуы үшін `/var/www/html/` бумасы үшін SELinux қауіпсіздік контекстін қалпына келтіріңіз. Ол үшін келесі команданы орындаңыз:

        ```console
        sudo restorecon -Rv /var/www/html/
        ```

    1. HTTPD демонына сыртқы желілік қосылымдарды орнатуға және пайдаланушылардың home-директорияларына жүгінуге рұқсат беріңіз. Ол үшін келесі командаларды орындаңыз:

        ```console
        sudo setsebool -P httpd_can_network_connect 1
        sudo setsebool -P httpd_enable_homedirs 1
        ```

1. SSL сертификатын баптаңыз.

    1. Let’s Encrypt-тен сертификаттарды автоматты түрде алу үшін `Certbot` бумасын келесі командамен орнатыңыз:

        ```console
        sudo dnf install certbot python3-certbot-apache
        ```

    1. `Certbot` іске қосып, жаңа доменге SSL сертификатын автоматты түрде орнату үшін нұсқауларды орындаңыз:

        ```console
        sudo certbot --apache -d new-domain.com -d www.new-domain.com
        ```

    1. `Certbot` HTTP-ті HTTPS-ке қайта бағыттауды ұсынғанда, осы опцияны қосыңыз. Бұл пайдаланушыларды қауіпсіз қосылымға автоматты түрде бағыттауға мүмкіндік береді.

    1. SSL сертификаты орнатылғаннан кейін, келесі команданы орындап, оны автоматты түрде тексеру мен жаңартуды қосыңыз:
  
        ```console
        sudo systemctl enable certbot-renew.timer
        ```

    1. Келесі командамен Apache-ті қайта іске қосыңыз:

        ```console
        sudo systemctl restart httpd
        ```

        {note:info}

        Осы сәттен бастап WordPress дашбордына өту үшін `http://new-domain.com/wp-admin` мекенжайын пайдаланыңыз.

        {/note}

## 2. WordPress баптауларындағы домендік атауды өзгертіңіз

1. WordPress баптауларындағы URL мекенжайларын өзгертіңіз:

    1. Браузерде `http://new-domain.com/wp-admin` мекенжайына өтіңіз.
    1. Әкімші логині мен құпиясөзін пайдаланып WordPress жүйесіне кіріңіз.
    1. **Settings → General** бөліміне өтіңіз.
    1. **WordPress Address (URL)** және **Site Address (URL)** өрістеріндегі домендік атауды жаңасына ауыстырыңыз:

        ```txt
        https://new-domain.com
        ```

    1. **Save Changes** түймесін басыңыз.

1. Контент пен дерекқор баптауларындағы ескі доменге барлық сілтемелерді келесі командаларды орындап, жаңаларына ауыстырыңыз:

    ```console
    cd /var/www/html
    wp search-replace 'http://old-domain.com' 'https://new-domain.com' --allow-root
    ```

1. Егер **WP Super Cache** немесе **W3 Total Cache** сияқты кэштеу плагинін пайдалансаңыз, плагин баптаулары арқылы бүкіл кэшті тазалаңыз.
1. Сайттағы тұрақты URL мекенжайларының құрылымын қайта генерациялаңыз. Ол үшін **Settings → Permalinks** бөліміне өтіп, **Save Changes** түймесін басыңыз.

    {note:info}

    Осы сәттен бастап WordPress-сайтыңызға өту үшін `http://new-domain.com` мекенжайын пайдаланыңыз.

    {/note}

## 3. Сайттың дұрыс жұмысын тексеріңіз

1. SSL сертификатын тексеріңіз:

    1. Сайтқа өтіп, оның HTTPS протоколы арқылы қатесіз ашылатынына көз жеткізіңіз.
    1. SSL сертификатының дұрыстығын тексеру үшін [SSL Labs](https://www.ssllabs.com/ssltest) сияқты онлайн құралдарды пайдаланыңыз.

1. Ескі доменге барлық сұраулардың жаңасына дұрыс қайта бағытталатынына көз жеткізіңіз. Мұны [Redirect Checker](https://www.redirect-checker.org) құралы арқылы жасауға болады.

1. Сайттың функционалдығын тексеріңіз:

    1. Сайт беттері бойынша өтіп, сілтемелердің дұрыс жұмыс істейтініне көз жеткізіңіз.
    1. Жаңа URL сілтемелері бойынша барлық медиафайлдардың жүктелетініне көз жеткізіңіз.

## 4. Сайт қауіпсіздігін күшейтіңіз

Қауіпсіздікті жақсарту үшін HSTS баптаңыз — бұл HTTPS протоколы арқылы қорғалған қосылымды мәжбүрлі түрде белсендіретін механизм. Ол үшін Apache конфигурациясына келесі жолдарды қосыңыз:

```apache
<VirtualHost *:443>
    Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains"
</VirtualHost>
```

## 5. Сайтыңызға сілтемелерді жаңартыңыз

1. Егер сайтыңыз іздеу жүйелерімен әлдеқашан индекстелген болса, Google Search Console және Яндекс Вебмастерге жаңа XML сайт картасын жіберіп, онда жаңа домендік атауды көрсетіңіз.
1. Егер әлеуметтік желілерді немесе басқа сыртқы ресурстарды белсенді түрде жүргізсеңіз, олардағы доменіңізге сілтемелерді жаңартыңыз.
