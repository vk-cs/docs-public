Сіз өз инфрақұрылымыңызды ықтимал веб-қауіптерден Check Point CloudGuard Network сервисінің көмегімен қорғай аласыз. Бұл сервис кез келген типтегі бұлтта қауіптердің алдын алуға және желіні қорғауға арналған кешенді шешім болып табылады. Check Point CloudGuard Network сервисі мыналарды қамтиды:

* Security Gateway BYOL — виртуалды шлюз (келесі буындағы файервол);
* Security Management BYOL — сервисті басқару сервері.

{cut(Сервис жұмысының принципі)}

1. Бір немесе әртүрлі желілерде алдын ала орнатылған GAiA ОЖ-сы бар екі ВМ құрылады: виртуалды шлюз және басқару сервері.
1. Осы ВМ-дердің әрқайсысында алғашқы қосылу кезінде бастапқы баптау шебері іске қосылады, ол осы ВМ-дермен кейін веб-интерфейс арқылы жұмыс істеу үшін бастапқы параметрлерді баптауға мүмкіндік береді.
1. Шлюзге қажетті желілер қосылады.
1. Сервисті басқару басқару серверіне қосылатын SmartConsole қолданбасының көмегімен жүзеге асырылады. Бұл қолданба мыналарға мүмкіндік береді:

   * виртуалды шлюздің басқару серверімен байланысын баптауға;
   * қажетті лицензияларды қосып, қауіпсіздік саясатын баптауға;
   * резервтік көшіруді баптауға.

{/cut}

Нұсқаулықтың барлық қадамдарынан өткеннен кейін, сіз:

1. Виртуалды шлюзді өрістетіп, баптайсыз.
1. Басқару серверін өрістетіп, баптайсыз.
1. Шлюз бен басқару серверіне желілік қолжетімділікті баптайсыз.
1. Шлюздің желілік интерфейстерін баптайсыз.
1. Қауіпсіздік саясатын баптайсыз.
1. Резервтік көшіру нұсқаларымен танысасыз.

Check Point CloudGuard Network сервисін пайдалана отырып, сіз [қолданбалар дүкенінің](/kz/start/legal/vk/marketplace) лицензиялық келісімімен, сондай-ақ [Check Point лицензиялық келісімімен](https://www.checkpoint.com/support-services/software-license-agreement-limited-hardware-warranty) келісесіз.

{note:warn}

Check Point CloudGuard Network BYOL (Bring Your Own Licence) моделі бойынша ұсынылады. Сервисті пайдалануға лицензия сатып алу үшін Check Point серіктесіне [cpcloud@proway.tech](mailto:cpcloud@proway.tech) поштасына жазыңыз.

{/note}

## 1. Check Point CloudGuard Network сервисін өрістетіңіз

1. **Security Gateway BYOL** шлюзін өрістетіңіз:

   1. VK Cloud жеке кабинетіне [өтіңіз](https://kz.cloud.vk.com/app).
   1. **Қолданбалар дүкені** бөліміне өтіңіз.
   1. **Check Point CloudGuard Network — Security Gateway BYOL** сервисінің карточкасында **Толығырақ** батырмасын басыңыз.
   1. Сервис бетінде **Қосу** батырмасын басыңыз.
   1. Сервисті баптау бетінде шлюз параметрлерін орнатыңыз:

       - **Желі**: интернетке қолжетімділігі бар бұрын жасалған желіні таңдаңыз немесе оның [типін](/kz/networks/vnet/concepts/sdn) және ішкі желі мекенжайын көрсетіп, жаңасын жасаңыз.
       - Қалған параметрлерді өзгертусіз қалдырыңыз.

   1. **Келесі қадам** батырмасын басыңыз.
   1. Растау бетінде қосылудың қорытынды параметрлерін тексеріп, **Тарифті қосу** батырмасын басыңыз.

   Өрістету аяқталғаннан кейін поштаңызға шлюзді баптау нұсқаулығына арналған бір реттік сілтеме келеді. Осы нұсқаулықты сақтап қойыңыз.

1. **Security Management BYOL** басқару серверін өрістетіңіз:

   1. VK Cloud жеке кабинетіне [өтіңіз](https://kz.cloud.vk.com/app).
   1. **Қолданбалар дүкені** бөліміне өтіңіз.
   1. **Check Point CloudGuard Network — Security Management BYOL** сервисінің карточкасында **Толығырақ** батырмасын басыңыз.
   1. Сервис бетінде **Қосу** батырмасын басыңыз.
   1. Сервисті баптау бетінде басқару серверінің параметрлерін орнатыңыз:

       - **Желі**: өрістетілген сервисі бар ВМ орналастырылатын, интернетке қолжетімділігі бар бұрын жасалған желіні таңдаңыз немесе оның типін (Neutron немесе Sprut) және ішкі желі мекенжайын көрсетіп, жаңа желі жасаңыз.
       - Қалған параметрлерді [ұсынымдарға](#preparatory_steps) сәйкес таңдаңыз немесе әдепкі мәндерді қалдырыңыз.

    1. **Келесі қадам** батырмасын басыңыз.
    1. Растау бетінде қосылудың қорытынды параметрлерін тексеріп, **Тарифті қосу** батырмасын басыңыз.

   Өрістету аяқталғаннан кейін поштаңызға басқару серверін баптау нұсқаулығына арналған бір реттік сілтеме келеді. Осы нұсқаулықты сақтап қойыңыз.

## {heading(2. Виртуалды шлюзді баптаңыз)[id=setup_gateway]}

1. VK Cloud жеке кабинетіне [өтіңіз](https://kz.cloud.vk.com/app).
1. **Бұлттық есептеулер** → **Виртуалды машиналар** бөліміне өтіңіз.
1. Виртуалды машиналар тізімінде виртуалды шлюз ретінде жасалған ВМ атауын басыңыз.
1. ВМ бетінде **Консоль** қойындысына өтіңіз.
1. GAiA операциялық жүйесінің CLISH пәрмен жолына кіру үшін шлюзді баптау нұсқаулығында берілген логин мен құпиясөзді енгізіңіз.
1. Шлюз ВМ-де жаңа құпиясөзді келесі пәрменнің көмегімен орнатыңыз:

   ```console
   set user admin password
   ```
1. Пәрмендерді орындаңыз:

   ```console
   set hostname <ИМЯ_ХОСТА>
   set interface eth0 ipv4-address <ВНУТРЕННИЙ_IP> subnet-mask <МАСКА_ПОДСЕТИ>
   set static-route default nexthop gateway address <IP_ШЛЮЗА_ПО_УМОЛЧАНИЮ> on
   save config
   ```

   Мұнда:

   * `<ИМЯ_ХОСТА>` — шлюздің виртуалды машинасының атауы. ВМ атауы **Security Gateway BYOL** сервисін өрістету кезінде автоматты түрде жасалады. Осы параметрдің көмегімен ВМ атауын өзгерте аласыз.
   * `<ВНУТРЕННИЙ_IP>` — шлюзді баптау нұсқаулығында көрсетілген шлюздің ішкі IP мекенжайы.
   * `<МАСКА_ПОДСЕТИ>` — шлюзді баптау нұсқаулығында көрсетілген ішкі желі маскасы.
   * `<IP_ШЛЮЗА_ПО_УМОЛЧАНИЮ>` — шлюзді баптау нұсқаулығында көрсетілген әдепкі шлюзге дейінгі статикалық маршрут.

## 4. Шлюз ВМ-і мен басқару серверінің логикалық томдарының өлшемдерін үлкейтіңіз

Егер шлюз бен басқару сервері дискісі 100 ГБ-тан үлкен виртуалды машиналарда өрістетілсе, осы ВМ-дер үшін `lv_current` немесе `lv_log` логикалық томдарының өлшемін үлкейтіңіз:

1. VK Cloud жеке кабинетіне [өтіңіз](https://kz.cloud.vk.com/app).
1. **Бұлттық есептеулер** → **Виртуалды машиналар** бөліміне өтіңіз.
1. Виртуалды машиналар тізімінде шлюз немесе Check Point CloudGuard Network басқару сервері ретінде жасалған ВМ атауын басыңыз.
1. ВМ бетінде **Консоль** қойындысына өтіңіз.
1. Қажетті ВМ-ді баптау нұсқаулығында берілген логинді және ВМ-ді баптау кезінде таңдалған құпиясөзді енгізіңіз.
1. GRUB 2 жүктеушісі үшін құпиясөз орнатыңыз:

   ```console
   set grub2-password
   ```

1. Сараптамалық режим үшін құпиясөз орнатыңыз (Linux жүйесіндегі бір пайдаланушылық режимнің баламасы):

   ```console
   set expert-password
   ```

1. Баптаулардағы өзгерістерді сақтаңыз:

   ```console
   save config
   ```

1. ВМ-ді қайта жүктеңіз:

   ```console
   reboot
   ```

1. Жүктеу кезінде `Press any key to see the boot menu` шақыруы пайда болғанда, кез келген пернені басыңыз.
1. **Start in maintenance mode** режимін таңдаңыз.
1. Шлюзді (немесе басқару серверін) баптау кезінде орнатылған логинді және GRUB2 жүктеушісі үшін орнатылған құпиясөзді енгізіңіз.
1. LVM логикалық томдар менеджерін іске қосыңыз:

    ```console
    lvm_manager
    ```

1. `2` опциясын таңдау үшін `Resize lv_current/lv_log Logical Volume` енгізіңіз.
1. Пернетақтадан тиісті санды енгізіп, қажетті логикалық томды таңдаңыз.
1. Ұсынылған ауқымнан кез келген мәнді орнатыңыз.

Қосымша ақпарат [sk95566 мақаласында](https://support.checkpoint.com/results/sk/sk95566) берілген (мақалаға қолжетімділік Check Point CloudGuard Network өнімін сатып алғаннан кейін ашылады).

## {heading(5. Шлюзге желілік қолжетімділікті баптаңыз)[id=configure_network_access_to_gateway]}

1. VK Cloud жеке кабинетіне [өтіңіз](https://kz.cloud.vk.com/app).
1. **Виртуалды желілер** → **Firewall баптаулары** бөліміне өтіңіз.
1. Шлюзді баптау нұсқаулығында көрсетілген қауіпсіздік тобы үшін желіңіздегі IP мекенжайларынан келетін кіріс трафикке рұқсат беретін ережелерді қосыңыз:
   1. Қажетті қауіпсіздік тобының атауын басыңыз.
   1. **Кіріс трафик** блогында **+ Ереже қосу** түймесін басыңыз.
   1. Трафик түрін таңдап, порттарды көрсетіңіз:

      {tabs}

      {tab(Барлық протоколдар және барлық порттар)}

      Желіңіздегі IP мекенжайларынан кез келген кіріс трафикке рұқсат беру үшін:

      1. **Тип** өрісінде **Барлық протоколдар және барлық порттар** таңдаңыз.
      1. **Қашықтағы мекенжай** бөлімінде **IP мекенжайлары ауқымы** таңдаңыз және желіңіздегі IP мекенжайлары ауқымын автоматты түрде қою үшін **Менің IP-ді пайдалану** түймесін басыңыз.

      {/tab}

      {tab(HTTPS)}

      Кез келген IP мекенжайларынан TCP `443` портына кіріс трафикке рұқсат беру үшін **HTTPS** таңдап, `443` портын көрсетіңіз.

      {/tab}

      {tab(SSH)}

      Кез келген IP мекенжайларынан TCP `22` портына кіріс трафикке рұқсат беру үшін **SSH** таңдап, `22` портын көрсетіңіз.

      {/tab}

      {/tabs}

   1. **Ережені сақтау** түймесін басыңыз.

1. GAiA Portal веб-интерфейсі арқылы виртуалды шлюзге қосылып, First Time Configuration Wizard баптау шеберінің көмегімен шлюзді баптауды аяқтаңыз:

   1. ```https://<ВНЕШНИЙ_IP_АДРЕС>:443``` адресіне өтіңіз, мұнда `<ВНЕШНИЙ_IP_АДРЕС>` — шлюзді баптау нұсқаулығында көрсетілген шлюздің сыртқы IP мекенжайы.
   1. Шлюзді баптау нұсқаулығында берілген логинді және [баптау](#setup_gateway) кезінде орнатылған құпиясөзді енгізіңіз.
   1. Пайда болған терезеде **SIC** бөлімінде **Activation Key** өрісіне құпиясөз орнатыңыз. Бұл құпиясөзді жазып қойыңыз. Ол шлюзді басқару серверіне қосу кезінде қажет болады.

Енді сіз GAiA Portal веб-интерфейсі арқылы `https://<ВНЕШНИЙ_IP_АДРЕС>` адресі бойынша (немесе SSH арқылы `<ВНЕШНИЙ_IP_АДРЕС>:22`) шлюз ВМ-іне қосыла аласыз.

## {heading(6. Настройте сетевой доступ к серверу управления)[id=configure_network_access_to_management]}

1. VK Cloud жеке кабинетіне [өтіңіз](https://kz.cloud.vk.com/app).
1. **Виртуалды желілер** → **Firewall баптаулары** бөліміне өтіңіз.
1. Басқару серверін баптау нұсқаулығында көрсетілген қауіпсіздік тобы үшін желіңіздегі IP мекенжайларынан келетін кіріс трафикке рұқсат беретін ережелерді қосыңыз:
   1. Қажетті қауіпсіздік тобының атауын басыңыз.
   1. **Кіріс трафигі** блогында **+ Ережені қосу** түймесін басыңыз.
   1. Трафик түрін таңдап, порттарды көрсетіңіз:

      {tabs}

      {tab(Барлық протоколдар және барлық порттар)}

      Желіңіздегі IP мекенжайларынан кез келген кіріс трафикке рұқсат беру үшін:

      1. **Тип** өрісінде **Барлық протоколдар және барлық порттар** таңдаңыз.
      1. **Қашықтағы мекенжай** бөлімінде **IP мекенжайлары ауқымы** таңдаңыз және желіңіздегі IP мекенжайлары ауқымын автоматты түрде қою үшін **Менің IP-ді пайдалану** түймесін басыңыз.

      {/tab}

      {tab(HTTPS)}

      Кез келген IP мекенжайларынан TCP `443` портына кіріс трафикке рұқсат беру үшін **HTTPS** таңдап, `443` портын көрсетіңіз.

      {/tab}

      {tab(SSH)}

      Кез келген IP мекенжайларынан TCP `22` портына кіріс трафикке рұқсат беру үшін **SSH** таңдап, `22` портын көрсетіңіз.

      {/tab}

      {tab(Басқа)}

      SmartConsole қолданбасы арқылы басқару серверіне қосылу үшін басқа порттарды орната аласыз:

      1. **Тип** өрісінде **Басқа** таңдаңыз.
      1. Кез келген IP мекенжайларынан осы порттарға кіріс трафикке рұқсат беру үшін TCP `19009`, `18210`, `18190`, `443` порттарын көрсетіңіз.

      {/tab}

      {/tabs}

   1. **Ережені сақтау** түймесін басыңыз.

1. GAiA Portal веб-интерфейсі арқылы басқару серверіне қосылып, First Time Configuration Wizard баптау шеберінің көмегімен баптауды аяқтаңыз:

   1. `https://<ВНЕШНИЙ_IP_АДРЕС>:443` адресіне өтіңіз, мұнда `<ВНЕШНИЙ_IP_АДРЕС>` — басқару серверін баптау нұсқаулығында көрсетілген басқару серверінің сыртқы IP мекенжайы.
   1. Басқару серверін баптау нұсқаулығында берілген логинді және [баптау](#setup_management) кезінде орнатылған құпиясөзді енгізіңіз.
   1. First Time Configuration Wizard баптау шебері терезесінде **Next** түймесін басыңыз.
   1. **Deployment Options** қадамында баптауларды өзгертпей, **Next** түймесін басыңыз.
   1. **Management Connection** қадамында баптауларды өзгертпей, **Next** түймесін басыңыз.
   1. **Device Information** қадамында баптауларды өзгертпей, **Next** түймесін басыңыз.
   1. **Date and Time Settings** қадамында баптауларды өзгертпей, **Next** түймесін басыңыз.
   1. **Installation Type** қадамында **Security Gateway and/or Security Management** таңдаңыз.
   1. **Products** қадамында **Check Point CloudGuard Network — Security Management BYOL** сервисін басқару сервері ретінде баптау үшін жалауша арқылы тек **Security Management** таңдаңыз. Қажет болған жағдайда сервисті шлюз және басқару сервері ретінде бір уақытта баптай аласыз. Бұл туралы толығырақ [сервистің ресми құжаттамасынан](https://sc1.checkpoint.com/documents/R81.20/WebAdminGuides/EN/CP_R81.20_Installation_and_Upgrade_Guide/Content/Topics-IUG/Installing-Standalone.htm) оқыңыз.
   1. **Security Management Administrator** қадамында баптауларды өзгертпей, **Next** түймесін басыңыз.
   1. **Security Management GUI Clients** қадамында баптауларды өзгертпей, **Next** түймесін басыңыз.
   1. **First Time Configuration Wizard Summary** қадамында **Finish** түймесін басып, жүйені қайта жүктеуді растаңыз.

Енді сіз GAiA Portal веб-интерфейсі арқылы `https://<ВНЕШНИЙ_IP_АДРЕС>` адресі бойынша (немесе SSH арқылы `<ВНЕШНИЙ_IP_АДРЕС>:22`) басқару серверінің ВМ-іне қосыла аласыз, мұнда `<ВНЕШНИЙ_IP_АДРЕС>` — басқару серверін баптау нұсқаулығында көрсетілген басқару серверінің сыртқы IP мекенжайы.

## 7. Желілік интерфейстерді қосыңыз

Әдепкі бойынша ВМ үшін бір желілік интерфейс жасалады. Шлюзге желілік интерфейс қосу үшін:

1. VK Cloud жеке кабинетіне [өтіңіз](https://kz.cloud.vk.com/app).
1. **Бұлттық есептеулер** → **Виртуалды машиналар** бөліміне өтіңіз.
1. Виртуалды машиналар тізімінде шлюз ретінде жасалған ВМ атауын басыңыз.
1. **Сети** қойындысына өтіңіз.
1. Қажетті желіні шлюз ВМ-іне [қосыңыз](/kz/computing/iaas/instructions/vm/vm-add-net#vm_ge_zhelini_kosu) және осы желі үшін қауіпсіздік тобына ережелерді [қосыңыз](/kz/networks/vnet/instructions/secgroups#add_rule).

   {note:info}

   Желілік интерфейс қайта жүктеуді қажет етпей, GAiA операциялық жүйесінде қосылады және жойылады.

   {/note}

1. GAiA Portal веб-интерфейсін немесе SSH пайдаланып, GAiA ОЖ-сында желілік интерфейсті [баптаңыз](https://sc1.checkpoint.com/documents/R81.20/WebAdminGuides/EN/CP_R81.20_Gaia_AdminGuide/Content/Topics-GAG/Physical-Interfaces.htm?tocpath=Network%20Management%7CNetwork%20Interfaces%7C_____1).

## 8. Қауіпсіздік саясатын баптаңыз

1. Check Point CloudGuard Network сервисін басқаруға арналған SmartConsole қолданбасын [сілтеме](https://sc1.checkpoint.com/documents/Jumbo_HFA/R81.20_SC/R81.20/R81.20_Downloads.htm) арқылы орнатыңыз немесе оны шлюздің не басқару серверінің GAiA Portal веб-интерфейсі арқылы жүктеп алыңыз:

   1. ```https://<ВНЕШНИЙ_IP_АДРЕС>``` адресіне өтіңіз, мұнда `<ВНЕШНИЙ_IP_АДРЕС>` — баптау нұсқаулығында көрсетілген шлюздің немесе басқару серверінің сыртқы IP мекенжайы.
   1. Сол жақ мәзірден **Maintenance** → **Download SmartConsole** таңдаңыз.
   1. SmartConsole дистрибутивін жүктеп алу үшін **Download** түймесін басыңыз.
   1. SmartConsole орнату файлын іске қосыңыз.
   1. Орнатушының барлық ұсыныстарын қабылдап, оның жұмысы аяқталғанша күтіңіз.

1. SmartConsole көмегімен басқару серверіне [қосылыңыз](https://sc1.checkpoint.com/documents/R81.20/WebAdminGuides/EN/CP_R81.20_SecurityManagement_AdminGuide/Content/Topics-SECMG/Connecting-to-Security-Management-Server-through-SmartConsole.htm?tocpath=_____7#Connecting_to_the_Security_Management_Server_through_SmartConsole), ол үшін басқару сервері үшін орнатылған логин мен құпиясөзді, сондай-ақ баптау нұсқаулығында көрсетілген басқару серверінің сыртқы IP мекенжайын көрсетіңіз.
1. Шлюзді басқару серверіне [қосыңыз](https://sc1.checkpoint.com/documents/R81.20/WebAdminGuides/EN/CP_R81.20_SecurityManagement_AdminGuide/Content/Topics-SECMG/Creating-New-Security-Gateway.htm?tocpath=Managing%20Gateways%7C_____1), ол үшін мыналарды көрсетіңіз:

   * **Name**: шлюзді [баптау](#setup_gateway) кезінде `hostname` параметрінде берілген шлюз ВМ-інің атауы.
   * **IPv4 Address**: шлюзді [баптау](#setup_gateway) кезінде `<ВНУТРЕННИЙ_IP>` параметрінде берілген шлюздің ішкі IP мекенжайы.

   Қосу кезеңінде шлюзге [желілік қолжетімділікті баптау](#configure_network_access_to_gateway) кезінде көрсеткен SIC-құпиясөзді енгізіңіз.

1. Шлюз үшін қажетті бағдарламалық blade-терді қосыңыз:

   1. Тізімдегі қажетті шлюз атауын екі рет басыңыз.
   1. Шлюз қасиеттерінде **Network Security** қойындысында қажетті баптауларды белгілеңіз.

1. Access Control Policy саясатын [жасаңыз](https://sc1.checkpoint.com/documents/R81.20/WebAdminGuides/EN/CP_R81.20_SecurityManagement_AdminGuide/Content/Topics-SECMG/Introducing-the-unified-access-control-policy.htm?tocpath=Creating%20an%20Access%20Control%20Policy%7C_____1).
1. Қажет болған жағдайда **Custom Threat Prevention** немесе **Autonomous Threat Prevention** режимінде [Threat Prevention Policy](https://sc1.checkpoint.com/documents/R81.20/WebAdminGuides/EN/CP_R81.20_ThreatPrevention_AdminGuide/Content/Topics-TPG/The_Check_Point_Threat_Prevention_Solution.htm?TocPath=The%20Check%20Point%20Threat%20Prevention%20Solution%7C_____0#The_Check_Point_Threat_Prevention_Solution) саясатын, сондай-ақ QoS Policy және HTTPS Inspection саясаттарын баптаңыз.
1. Шлюз бен басқару серверіне лицензияларды [орнатыңыз](https://sc1.checkpoint.com/documents/R81.20/WebAdminGuides/EN/CP_R81.20_SecurityManagement_AdminGuide/Content/Topics-SECMG/Managing-Server-and-Gateway-Licenses.htm?tocpath=Managing%20Gateways%7CManaging%20%20Licenses%7C_____1).

[Access Control Policy](https://sc1.checkpoint.com/documents/R81.20/WebAdminGuides/EN/CP_R81.20_SecurityManagement_AdminGuide/Content/Topics-SECMG/Installing-the-Access-Control-Policy.htm?tocpath=Creating%20an%20Access%20Control%20Policy%7C_____10) және [Threat Prevention Policy](https://sc1.checkpoint.com/documents/R81.20/WebAdminGuides/EN/CP_R81.20_SecurityManagement_AdminGuide/Content/Topics-SECMG/Installing-Threat-Prevention-Policy.htm) саясаттарын шлюзге орнатыңыз.

## 9. Резервтік көшіруді қарастырыңыз

{note:warn}

Check Point CloudGuard Network сервисі инстанстарын VK Cloud құралдарымен резервтік көшіру және қалпына келтіру қолдау көрсетілмейді.

{/note}

Резервтік көшіру нұсқалары [sk108902 мақаласында](https://support.checkpoint.com/results/sk/sk108902) сипатталған және GAiA-ның келесі кіріктірілген резервтік көшіру рәсімдерін қамтиды:

* **System Backup** — GAiA ОЖ конфигурациясы мен басқару сервері базасының резервтік көшірмесін орындайды және мерзімдік резервтік көшіру үшін пайдаланылады. Резервтік көшірменің өлшемі:

  * шлюздер үшін ондаған немесе жүздеген МБ;
  * басқару сервері үшін жүздеген немесе мыңдаған МБ.

* **Snapshot** — жүйелік баптаулар мен келесі өнімдердің суретін жасайды:

  * файлдық жүйе (root);
  * жүйе конфигурациясы (интерфейстер, маршруттау, хост атауы және т.б.);
  * бағдарламалық blade-тер конфигурациясы;
  * басқару дерекқоры (басқару серверінде).

  Сурет өлшемі — бірнеше гигабайт. Негізінен GAiA операциялық жүйесін жаңа нұсқаға жаңарту алдында немесе Jumbo Hotfix бағдарламалық пакетін орнату алдында пайдаланылады.
