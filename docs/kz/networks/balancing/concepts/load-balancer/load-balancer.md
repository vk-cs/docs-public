## Жалпы мәліметтер

Жүктеме теңгергіші кіріс трафикті бірнеше виртуалды бэкенд-серверлер арасында бөлуге мүмкіндік береді және сол арқылы осы серверлер ұсынатын сервистер үшін жоғары қолжетімділікті қамтамасыз етеді. Бір немесе бірнеше сервер істен шыққан жағдайда, трафик қалған серверлерге қайта бағытталады. Сондай-ақ [жеке түрі](../load-balancer#zhukteme_tengergishterinin_turleri) жүктеме теңгергіші VK Cloud басқа сервистерінің жұмысын қамтамасыз ету үшін пайдаланылады.

{note:warn}

 VK Cloud жүктеме теңгергіштерін жобаларда пайдалану ұсынылады. Үшінші тарап шешімдерінің жұмыс қабілеттілігіне кепілдік берілмейді. Атап айтқанда, Windows Server Network Load Balancer (NLB) және Software Load Balancer (SLB) жұмысына қолдау көрсетілмейді).

{/note}

VK Cloud жүктеме теңгергіші [OpenStack Octavia](https://docs.openstack.org/octavia/latest/), негізінде құрылған, оның негізінде [HAProxy](http://www.haproxy.org/). жатыр. OpenStack Octavia жүктеме теңгергішіне келесі мүмкіндіктерді береді:

- HTTP-, HTTPS-, UDP- және TCP-қосылымдарын проксилеу және теңгеру (соңғылары — соның ішінде proxy protocol қолдауымен).

  [Proxy protocol](https://www.haproxy.org/download/1.8/doc/proxy-protocol.txt) серверге қосылымдар туралы ақпаратты (мысалы, клиенттің IP-мекенжайын) беруге мүмкіндік береді. Сервер мұндай ақпаратты өңдей алуы үшін, оның өзі де осы протоколды қолдауы керек. Proxy protocol NGINX, Apache, Envoy және [басқа өнімдерде](https://www.haproxy.com/blog/use-the-proxy-protocol-to-preserve-a-clients-ip-address/#proxy-protocol-support).

- HTTP/1.1-ге қосымша HTTP/2-қосылымдарын проксилеу және теңгеру/1.1.

- SSL-қосылымдарын жүктеме теңгергішінде терминациялау.

- Тайм-ауттарды (time-outs) икемді баптау).

- Әртүрлі теңгеру әдістері.

- Бэкендтер үшін салмақтарды баптау.

Істен шығуға төзімділік үшін жүктеме теңгергішінің екі данасы қолданылады, олардың бірі `active`, режимінде, ал екіншісі `standby`. режимінде болады. Күйлерді синхрондау және осы теңгергіштер арасында трафикті ауыстыру VRRP протоколының көмегімен орындалады. Пайдаланушы тұрғысынан мұндай істен шығуға төзімді конфигурация бір жүктеме теңгергіші сияқты көрінеді.

Жүктеме теңгергішінің ең жоғары өнімділігін қамтамасыз ету үшін:

- Бэкендтерге HTTP/1.1 протоколы бойынша [keep-alive](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Keep-Alive).
- Жүктеме теңгергішіне кіретін трафик үшін стандартты Ethernet MTU 1500 байт мәнін пайдаланыңыз.

Осы шарттар орындалған жағдайда, жүктеме теңгергіші секундына 1-ден 1,5 гигабитке дейін трафикті және секундына шамамен 10000 сұрауды өңдеп, шекті өнімділікті көрсете алады.

## Жүктеме теңгергіштерінің түрлері

VK Cloud жүйесінде теңгергіштің екі түрі бар. Теңгергіш түрін жеке кабинетте [көруге](../../instructions/manage-lb#zhukteme_tengergishterinin_tizimin_zhne_olar_turaly_akparatty_karau) болады.

Теңгергіштердің әртүрлі түрлері әртүрлі тарифтеледі.

{tabs}

{tab(Стандартты теңгергіш)}

Мұндай теңгергішті пайдаланушы жасайды және қолданбалардың істен шығуға төзімділігін және икемді масштабталуын қамтамасыз ету үшін пайдаланылуы мүмкін:

- [Cloud Servers сервисімен](/kz/computing/iaas) жұмыс істегенде кіріс трафикті виртуалды серверлер арасында бөлу үшін.
- [Cloud Containers сервисімен](/kz/kubernetes/k8s), жұмыс істегенде, мысалы [Ingress controller](/kz/kubernetes/k8s/how-to-guides/ingress) (соның ішінде [алдын ала орнатылған](/kz/kubernetes/k8s/concepts/addons-and-settings/addons#ingress_controller_nginx_65f458d8)) түріндегі сервистерді пайдаланғанда [Load Balancer](/kz/kubernetes/k8s/how-to-guides/load-balancer).

{/tab}

- [Cloud Containers сервисінде](/kz/kubernetes/k8s) Kubernetes API-ге қолжетімділікті қамтамасыз ету үшін әрбір Kubernetes кластері үшін.
- [Cloud Databases сервисінде](/kz/dbs/dbaas) кластерлік конфигурациядағы ДҚ инстанстары үшін инстанс түйіндері арасында жүктемені теңгеру мақсатында. Бұл мына инстанстарға қолданылады:
  - MySQL;
  - PostgreSQL;
  - Postgres Pro Enterprise;
  - Postgres Pro Enterprise 1С.

{/tab}
