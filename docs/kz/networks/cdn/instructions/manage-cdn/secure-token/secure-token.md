Опция [Secure token](/kz/networks/cdn/concepts/secure-token) опциясы CDN-ресурсының файлдарын қалаусыз жүктеп алудан қорғауға мүмкіндік береді.

## Qauipsiz token қосуды орнату

{tabs}
{tab(Жеке кабинет)}

{include(/kz/_includes/_open-cdn.md)}

1. қойындысына өтіңіз **Безопасность**.
1. опциясын қосыңыз **Активировать Secure Token**.
1. Құпия кілтті енгізіңіз — ұзындығы 6-дан 32 таңбаға дейінгі кездейсоқ таңбалар жиынтығы. Енгізілген құпия кілтті қауіпсіз жерде сақтаңыз.
1. (Опционалды) Белгілі бір IP-мекенжайдан ғана қол жеткізуге рұқсат беру үшін **Добавить IP-адрес к токену**, опциясын қосыңыз. IP-мекенжай CDN-ресурсынан тыс, контент көзі орналасқан сайтта [скриптті](/kz/networks/cdn/concepts/secure-token#ipscripts) орналастыру арқылы беріледі.
1. батырмасын басыңыз **Сохранить изменения**.

{/tab}
{tab(API)}

Сұрау денесіндегі `options` параметрлерін орнатыңыз `secure_key`:

- Опцияны қосу үшін `"enabled": true`, мәнін көрсетіңіз  — `"enabled": false`.
- Құпия кілтті орнату үшін `key`. параметріне ұзындығы 6-дан 32 таңбаға дейінгі таңбалар жиынтығын енгізіңіз. Мысал: `"key": "examplesecurekey"`.
- Токенге IP-мекенжай қосуды қосу үшін `2` параметріне `type`. мәнін енгізіңіз. IP-мекенжай қосуды өшіру үшін `0`. мәнін енгізіңіз. Мысалдар: `"type": 2`, `"type": 0`.

Сұрау мысалы:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281' \
--header 'X-Auth-Token: example6UjMOd' \
--header 'Content-Type: application/json' \
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options": {
        "secure_key": {
            "enabled": true, 
            "key": "examplesecurekey",
            "type": 2
        }
    }
}'
```

{note:info}
Secure token қосуды орнатқаннан кейін CDN-ресурсының контентіне қол қойылмаған сілтемелер арқылы қол жеткізу мүмкін болмайды.
Контенттің бір бөлігіне кәдімгі сілтемелер арқылы қол жеткізуді ұйымдастыру үшін, ол үшін бөлек, [источнигі](/kz/networks/cdn/instructions/create-resource) бар тағы бір ресурс [жасаңыз](/kz/networks/cdn/concepts/origin-groups).
{/note}

## Қол қойылған сілтемелер арқылы қол жеткізуді баптау

Баптау пайдаланушылар контентке қол жеткізетін ресурста орындалады:

1. Қол қойылған сілтемені генерациялау үшін ресурсыңызға қажетті [скриптті](/kz/networks/cdn/concepts/secure-token#scripts) қосыңыз. Скриптті таңдау **Добавить IP-адрес к токену**.
1. Пайдаланушылар қажетті файлдарға тек қол қойылған сілтемелер арқылы қол жеткізетіндей етіп ресурсыңызды ұйымдастырыңыз. Бұл ретте CDN VK Cloud сервисі бастапқы серверге жүгінген кезде, құпия кілттің бар-жоғына қарамастан, контентке қол жеткізуі тиіс.
