## {heading(Сұрау тақырыптарын қосу)[id=cdn-http-headers-add-request]}

**Сұрау тақырыптарын қосу** опциясы CDN сервері бастапқы серверге сұрауға қосатын жеке HTTP тақырыптарын орнатуға мүмкіндік береді.

HTTP тақырыбын сұрауға қосу үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **HTTP тақырыптары** қойындысына өтіңіз.
1. **Сұрау тақырыптарын қосу** опциясын қосыңыз.
1. Тақырып атауын және мәнін көрсетіңіз. Өрістер үшін рұқсат етілген таңбалар:

    - **Тақырып атауы**: латын әріптері, сандар, `_` және `-` арнайы таңбалары;
    - **Мәні**: латын әріптері, сандар, `_`, `.`, `/`, `:`, `-`, `=` арнайы таңбалары және бос орындар.
1. (Қосымша) Тағы бір сұрау тақырыбын қосу үшін **Тақырып қосу** түймесін басыңыз. Тақырып атауын және мәнін көрсетіңіз. Сіз 50-ге дейін сұрау тақырыбын орната аласыз.
1. (Қосымша) Сұрау тақырыбын жою үшін ![trash-icon](../../../../../assets/trash-icon.svg "inline") түймесін басыңыз.
1. **Өзгерістерді сақтау** түймесін басыңыз.

{/tab}

{tab(API)}

{include(../../../../../_includes/_api_cdn_create_change.md)}

Сұрау денесінің `options` блогында `staticRequestHeaders` параметрлерін көрсетіңіз:

- Опцияны қосу үшін `"enabled": true`, өшіру үшін `"enabled": false` көрсетіңіз.
- `value` параметрінде тақырыптар мен мәндерді `name: value` форматында көрсетіңіз. Сіз 50-ге дейін сұрау тақырыбын көрсете аласыз. Параметрлер үшін рұқсат етілген таңбалар:

  - `name`: латын әріптері, сандар, `_` және `-` арнайы таңбалары.
  - `value`: латын әріптері, сандар, `_`, `.`, `/`, `:`, `-`, `=` арнайы таңбалары және бос орындар. Мәні арнайы таңбадан басталмауы және тек арнайы таңбалардан ғана тұрмауы керек.

Сұрау үлгісі:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options":
        {
        "staticRequestHeaders": {
            "enabled": true,
            "value": {
                "Header-One": "Value 1",
                "Header-Two": "Value 2"
            }
        },
    }
}'
```

{/tab}

## {heading(CORS тақырыптарын қолдау)[id=cdn-http-headers-cors]}

{linkto(../../../../../storage/s3/concepts/access/s3-cors#s3-s3-cors)[text=Cross-Origin Resource Sharing (CORS)]} механизмі пайдаланушының браузері бастапқы доменнен өзгеше домендерде орналасқан ресурстарға қол жеткізу рұқсаттарын алуы үшін Access-Control-Allow-Origin жауап тақырыбын пайдаланады.

Access-Control-Allow-Origin HTTP тақырыбын баптау үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **HTTP тақырыптары** қойындысына өтіңіз.
1. **HTTP-тақырып Access-Control-Allow-Origin** опциясын қосыңыз.
1. Тақырыпты баптау нұсқасын таңдаңыз:

    - `*, барлық домендер үшін`: барлық домендер үшін контентті көрсетуге рұқсат береді.
    - `"$http_origin", барлық домендер үшін`: барлық домендер үшін контентті көрсетуге рұқсат береді, бірақ Access-Control-Allow-Origin жауап тақырыбында сұрау жіберілген домен беріледі.
    - `"$http_origin", егер дереккөз төменде көрсетілсе`: көрсетілген домендер үшін контентті көрсетуге рұқсат береді:

        1. Контентті көрсету қажет доменді көрсетіңіз.
        1. (Қосымша) Тағы бір домен қосу үшін ![plus-icon](../../../../../assets/plus-icon.svg "inline") **Домен қосу** түймесін басыңыз.
        1. (Қосымша) Доменді тізімнен жою үшін ![trash-icon](../../../../../assets/trash-icon.svg "inline") түймесін басыңыз.

        Жауап тақырыбында сұрау жіберілген домен беріледі.
1. **Өзгерістерді сақтау** түймесін басыңыз.

{/tab}

{tab(API)}

{include(../../../../../_includes/_api_cdn_create_change.md)}

Сұрау денесінің `options` блогында `cors` параметрлерін көрсетіңіз:

- Опцияны өшіру және ешқандай мәндерді көрсетпеу үшін `cors: null` жазыңыз.
- Опцияны қосу үшін `"enabled": true`, өшіру үшін `"enabled": false` көрсетіңіз.
- `value` параметрінде контентті көрсетуге рұқсат берілетін домендерді көрсетіңіз.

Сұрау үлгісі:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options":
        {
        "cors": {
            "enabled": true,
            "value": [
                "first.example.com",
                "second.example.com"
            ]
        }
    }
}'
```

{/tab}

## {heading(Рұқсат етілген HTTP әдістерін баптау)[id=cdn-security-allowed-http]}

**Рұқсат етілген HTTP-әдістер** опциясы CDN-ресурсына жіберілетін сұраулардың рұқсат етілген HTTP әдістерін басқаруға мүмкіндік береді. Әдепкі бойынша мына әдістерге рұқсат етілген: `GET`, `HEAD`, `OPTIONS`, `DELETE`. `POST`, `PUT`, `PATCH` әдістеріне рұқсат беру үшін [техникалық қолдау қызметіне](/kz/contacts) хабарласыңыз.

Рұқсат етілген әдістерді баптау үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **HTTP-тақырыптар** қойындысына өтіңіз.
1. **Рұқсат етілген HTTP-әдістер** опциясын қосыңыз.
1. CDN-ресурсы өңдеуі тиіс әдістерді таңдаңыз.
1. **Өзгерістерді сақтау** түймесін басыңыз.

{/tab}

{tab(API)}

{include(../../../../../_includes/_api_cdn_create_change.md)}

Сұрау денесіндегі `options` блогында `allowedHttpMethods` параметрлерін көрсетіңіз:

- опцияны қосу үшін `"enabled": true`, өшіру үшін `"enabled": false` мәнін көрсетіңіз;
- `value` параметрінде қолжетімді болуы тиіс әдістерді көрсетіңіз.

Сұрау мысалы:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options":
        {
        "allowedHttpMethods": {
            "enabled": true,
            "value": [
                "GET",
                "HEAD"
            ]
        },
    }
}'
```

{/tab}

{/tabs}

Егер сервер көрсетілген әдісті танымаған болса, 501 (Not Implemented) кодымен жауап қайтарылады. Егер серверге әдіс белгілі болса, бірақ ол CDN-ресурсында рұқсат етілмесе, 405 (Method Not Allowed) кодымен жауап қайтарылады.
