## HTTP тақырыптарын баптау

Опция **Сұрау тақырыптарын қосу** опциясы CDN сервері бастапқы серверге сұрауға қосатын жеке HTTP тақырыптарын орнатуға мүмкіндік береді.

HTTP тақырыбын сұрауға қосу үшін:

{tabs}

{tab(Жеке кабинет)}

{include(/kz/_includes/_open-cdn.md)}

1. қойындысына өтіңіз **HTTP-тақырыптар**.
1. опциясын қосыңыз **Добавление заголовков запросов**.
1. Тақырып атауын және мәнін көрсетіңіз. Өрістер үшін рұқсат етілген таңбалар:

    - **Тақырып атауы**: латын әріптері, сандар `_` және `-`;
    - **Мәні**: латын әріптері, сандар `_`, `.`, `/`, `:`, `-`, `=` арнайы таңбалары және бос орындар.
1. (Қосымша) Тағы бір сұрау тақырыбын қосу үшін **Тақырып қосу**, түймесін басыңыз. Тақырып атауын және мәнін көрсетіңіз. Сіз 50-ге дейін сұрау тақырыбын орната аласыз.
1. (Қосымша) Сұрау тақырыбын жою үшін ![trash-icon](/kz/assets/trash-icon.svg "inline"), түймесін басыңыз.
1. батырмасын басыңыз **Өзгерістерді сақтау**.

{/tab}

{tab(API)}

{include(/kz/_includes/_api_cdn_create_change.md)}

Сұрау денесінің `options` параметрлерін көрсетіңіз `staticRequestHeaders`:

- Опцияны қосу үшін `"enabled": true`, көрсетіңіз  — `"enabled": false`.
- Параметрде `value` параметрінде тақырыптар мен мәндерді `name: value`. форматында көрсетіңіз. Сіз 50-ге дейін сұрау тақырыбын көрсете аласыз. Параметрлер үшін рұқсат етілген таңбалар:

  - `name`: латын әріптері, сандар `_` және `-`.
  - `value`: латын әріптері, сандар `_`, `.`, `/`, `:`, `-`, `=` арнайы таңбалары және бос орындар. Мәні арнайы таңбадан басталмауы және тек арнайы таңбалардан ғана тұрмауы керек.

Сұрау үлгісі:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options":
        {
        "staticRequestHeaders": {
            "enabled": true,
            "value": {
                "Header-One": "Value 1",
                "Header-Two": "Value 2"
            }
        }
}
}'
```

{/tab}

{/tabs}

HTTP тақырыбын жауапқа қосу үшін:

1. (Қосымша) Тағы бір сұрау тақырыбын қосу үшін ![plus-icon](/kz/assets/plus-icon.svg "inline") **Тақырып қосу**, түймесін басыңыз. Тақырып атауын және мәнін көрсетіңіз. Сіз 50-ге дейін жауап тақырыбын орната аласыз.

## CORS тақырыптарын қолдау

Механизм [Cross-Origin Resource Sharing (CORS)](/kz/storage/s3/concepts/access/s3-cors) механизмі пайдаланушының браузері бастапқы доменнен өзгеше домендерде орналасқан ресурстарға қол жеткізу рұқсаттарын алуы үшін Access-Control-Allow-Origin жауап тақырыбын пайдаланады.

Access-Control-Allow-Origin HTTP тақырыбын баптау үшін:

{tabs}

{tab(Жеке кабинет)}

{include(/kz/_includes/_open-cdn.md)}

1. қойындысына өтіңіз **HTTP-тақырыптар**.
1. опциясын қосыңыз **HTTP-тақырып Access-Control-Allow-Origin**.
1. Тақырыпты баптау нұсқасын таңдаңыз:

    - `*, для всех доменов`: барлық домендер үшін контентті көрсетуге рұқсат береді.
    - `"$http_origin", для всех доменов`: барлық домендер үшін контентті көрсетуге рұқсат береді, бірақ Access-Control-Allow-Origin жауап тақырыбында сұрау жіберілген домен беріледі.
    - `"$http_origin", если источник указан ниже`: көрсетілген домендер үшін контентті көрсетуге рұқсат береді:

        1. Контентті көрсету қажет доменді көрсетіңіз.
        1. (Қосымша) Тағы бір домен қосу үшін ![plus-icon](/kz/assets/plus-icon.svg "inline") **Домен қосу**, түймесін басыңыз.
        1. (Қосымша) Доменді тізімнен жою үшін ![trash-icon](/kz/assets/trash-icon.svg "inline"), түймесін басыңыз.

        Жауап тақырыбында сұрау жіберілген домен беріледі.
1. батырмасын басыңыз **Өзгерістерді сақтау**.

{/tab}

{tab(API)}

{include(/kz/_includes/_api_cdn_create_change.md)}

Сұрау денесінің `options` параметрлерін көрсетіңіз `cors`:

- Опцияны өшіру және ешқандай мәндерді көрсетпеу үшін `cors: null`.
- Опцияны қосу үшін `"enabled": true`, көрсетіңіз  — `"enabled": false`.
- Параметрде `value` параметрінде контентті көрсетуге рұқсат берілетін домендерді көрсетіңіз.

Сұрау үлгісі:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options":
        {
        "cors": {
            "enabled": true,
            "value": [
                "first.example.com",
                "second.example.com"
            ]
        }
    }
}'
```

{/tab}

## Рұқсат етілген HTTP әдістерін баптау

Опция **Рұқсат етілген HTTP-әдістер** опциясы CDN-ресурсына жіберілетін сұраулардың рұқсат етілген HTTP әдістерін басқаруға мүмкіндік береді. Әдепкі бойынша мына әдістерге рұқсат етілген.

Рұқсат етілген әдістерді баптау үшін:

{tabs}

{tab(Жеке кабинет)}

{include(/kz/_includes/_open-cdn.md)}

1. қойындысына өтіңіз **HTTP-тақырыптар**.
1. опциясын қосыңыз **Рұқсат етілген HTTP-әдістер**.
1. CDN-ресурсы өңдеуі тиіс әдістерді таңдаңыз.
1. батырмасын басыңыз **Өзгерістерді сақтау**.

{/tab}

{tab(API)}

{include(/kz/_includes/_api_cdn_create_change.md)}

Сұрау денесіндегі `options` параметрлерін көрсетіңіз `allowedHttpMethods`:

- опцияны қосу үшін `"enabled": true`, мәнін көрсетіңіз  — `"enabled": false`;
- в параметре `value` параметрінде қолжетімді болуы тиіс әдістерді көрсетіңіз.

Сұрау мысалы:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options":
        {
        "allowedHttpMethods": {
            "enabled": true,
            "value": [
                "GET",
                "HEAD"
            ]
        }
}
}'
```

{/tab}

{/tabs}

Егер сервер көрсетілген әдісті танымаған болса, 501 (Not Implemented) кодымен жауап қайтарылады. Егер серверге әдіс белгілі болса, бірақ ол CDN-ресурсында рұқсат етілмесе, 405 (Method Not Allowed) кодымен жауап қайтарылады).
