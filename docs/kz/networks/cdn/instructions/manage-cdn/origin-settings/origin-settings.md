## {heading(Дереккөзбен өзара әрекеттесу протоколын өзгерту)[id=cdn-origin-settings-protocol]}

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **Негізгі баптаулар** қойындысына өтіңіз.
1. Дереккөзбен өзара әрекеттесу протоколын таңдаңыз:

    - `HTTP` — дереккөздегі контент тек HTTP (80 порт) арқылы қолжетімді немесе дереккөз HTTPS-тен HTTP-ке қайта бағыттауды қолдайды.
    - `HTTPS` — дереккөздегі контент тек HTTPS (443 порт) арқылы қолжетімді немесе дереккөз HTTP-тен HTTPS-ке қайта бағыттауды қолдайды.
    - `HTTP және HTTPS` — дереккөздегі контент HTTP және HTTPS арқылы қолжетімді. Бұл жағдайда CDN-ресурстан дереккөзге жасалатын сұрауда соңғы пайдаланушы сұрауының протоколы сақталады. Бұл ретте CDN-ресурстың кэшінде әрбір протокол үшін файлдың екі нұсқасы сақталады.
1. **Өзгерістерді сақтау** түймесін басыңыз.

{/tab}

{tab(API)}

{include(../../../../../_includes/_api_cdn_create_change.md)}

Сұрау денесіндегі `originProtocol` параметрінде мына мәнді көрсетіңіз:

- `HTTP` — дереккөздегі контент тек HTTP (80 порт) арқылы қолжетімді немесе дереккөз HTTPS-тен HTTP-ке қайта бағыттауды қолдайды.
- `HTTPS` — дереккөздегі контент тек HTTPS (443 порт) арқылы қолжетімді немесе дереккөз HTTP-тен HTTPS-ке қайта бағыттауды қолдайды.
- `MATCH` — дереккөздегі контент HTTP және HTTPS арқылы қолжетімді. Бұл жағдайда CDN-ресурстан дереккөзге жасалатын сұрауда соңғы пайдаланушы сұрауының протоколы сақталады. Бұл ретте CDN-ресурстың кэшінде әрбір протокол үшін файлдың екі нұсқасы сақталады.

Сұрау үлгісі:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "originProtocol": "HTTPS"
}'
```

## {heading(Деректер көзін өзгерту)[id=cdn-origin-settings-data-source]}

Баптау CDN-ресурс контентті алатын дереккөзді өзгертуге мүмкіндік береді.

Егер CDN-ресурсты жасау кезінде бір дереккөз көрсетілсе, автоматты түрде көрсетілген параметрлері бар бір дереккөзден тұратын дереккөздер тобы жасалады. Сондықтан CDN-ресурсты өңдеу кезінде тек дереккөздер тобын өзгерту қолжетімді, бір дереккөзді таңдау қолжетімді емес.

Дереккөзді өзгерту үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **Негізгі баптаулар** қойындысына өтіңіз.
1. **Дереккөздер тобы** параметрінде ашылмалы тізімнен топты таңдаңыз.

    Егер тізімде қажетті топ болмаса:

     1. **Дереккөздер тобын қосу** тармағын таңдаңыз.
     1. Дереккөздер тобының атауын беріңіз.
     1. Бірінші дереккөзді қосыңыз:

        {include(../../../../../_includes/_cdn_add_origin.md)}

     1. (Қосымша) Басқа дереккөздерді қосыңыз.

     1. (Қосымша) Жеке дереккөздерді қосыңыз немесе өшіріңіз. Егер бұл топтағы осындай түрдегі жалғыз дереккөз болса, `Белсенді` дереккөзді өшіруге болмайды. Өшірілген дереккөз контентті CDN-ресурстар сұрағанда пайдаланылмайды.

     1. (Қосымша) Дереккөз түрінің оң жағындағы ![pencil-icon](../../../../../assets/pencil-icon.svg "inline") белгішесін басып, жеке дереккөз параметрлерін өзгертіңіз.

     1. (Қосымша) Дереккөз түрінің оң жағындағы ![trash-icon](../../../../../assets/trash-icon.svg "inline") белгішесін басып, жеке дереккөзді жойыңыз.

        Егер бұл топтағы осындай түрдегі жалғыз дереккөз болса, `Белсенді` дереккөзді жоюға болмайды.

     1. Қосылған барлық дереккөздердің домендері үшін (соның ішінде өшірілгендері үшін де) DNS жазбалары бапталғанына көз жеткізіңіз.

        {note:warn}
        Егер {var(cloud)} DNS-ке сұрау арқылы доменнің қолжетімділігін тексере алмаса, дереккөздер тобын жасау мүмкін болмайды.
        {/note}

     1. Контентті CDN серверлері сұраған кезде дереккөзді таңдау тәртібіне әсер ету үшін **Дереккөздегі 4XX және 5XX кодтары кезінде тізімдегі келесі дереккөзді пайдалану** опциясын таңдаңыз.

        Бұл опция таңдалса, CDN серверлері өздері жүгініп жатқан дереккөз қолжетімсіз болса, тізімдегі келесі дереккөздерге жүгінеді. Егер барлық дереккөздер қолжетімсіз болса, CDN серверлері контент сұралған кезде тізімдегі соңғы дереккөздің жауабын қайтарады. Дереккөзді таңдау туралы толығырақ {linkto(../../../../../networks/cdn/concepts/origin-groups#cdn-origin-groups)[text=Дереккөздер топтары]} бөлімінен оқыңыз.

        Бұл опция дереккөздер тобында бірден көп дереккөз бапталған болса қолжетімді.

     1. **Топ құру** түймесін басыңыз.
1. **Өзгерістерді сақтау** түймесін басыңыз.

{/tab}

{tab(API)}

{include(../../../../../_includes/_api_cdn_create_change.md)}

Сұрау денесінде мына параметрлерді көрсетіңіз:

- `origin` — IP мекенжайы немесе дереккөздің домендік атауы және, егер бапталған болса, порты.
- `originGroup` — дереккөздер тобының идентификаторы немесе атауы.

Сұрауда `origin` параметрін де, `originGroup` параметрін де пайдалана аласыз.

Жеке дереккөзді өзгертуге арналған сұрау үлгісі:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "origin": "yoursite.com"
}'
```

Дереккөздер тобын өзгертуге арналған сұрау үлгісі:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
}'
```

{note:info}
Дереккөздер тобының өзін өңдеу немесе қосу үшін дереккөздер топтарымен жұмыс істеуге арналған әдістерді пайдаланыңыз.
{/note}

{/tab}

## {heading(Host тақырыбын өзгерту)[id=cdn-origin-settings-host-header]}

CDN серверлері дереккөздерден контент сұраған кезде HTTP сұрауларында міндетті [Host](https://developer.mozilla.org/en-US/docs/Web/HTTP/Headers/Host) тақырыбын көрсетеді. Бұл тақырыпты пайдалану дереккөздердегі қажетті виртуалды хостқа жүгінуге мүмкіндік береді.

Әдепкі бойынша Host тақырыбының мәні бірінші CNAME-ге сәйкес келеді.

Тақырыпты өзгерту үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **Негізгі баптаулар** қойындысына өтіңіз.
1. **Дереккөздер мен домендер конфигурациясы** блогын ашыңыз.
1. **Host тақырыбын өзгерту** параметрінде дереккөздерге жүгінген кезде `Host` HTTP тақырыбының мәнін өзгерту керек пе, соны таңдаңыз:

    - `Өзгертпеу` — тақырып мәні ретінде дереккөздер тобындағы бірінші дереккөздің домен аты немесе IP мекенжайы пайдаланылады.

        **Мысал:**

        Екі дереккөзден тұратын топ бапталған болсын:

        - `203.0.113.222:8080`,
        - `images.example.com`.

        Онда CDN серверлері осы дереккөздердің кез келгеніне жүгінген кезде `Host: 203.0.113.222:8080` тақырыбы пайдаланылады.

    - `Кастом` — тақырып мәні ретінде көрсетілген доменнің немесе IP мекенжайының мәні пайдаланылады.

        **Мысал:**

        Екі дереккөзден тұратын топ бапталған болсын:

        - `203.0.113.222:8080`,
        - `images.example.com`.

        Сондай-ақ опция үшін `content-source.example.org` мәні көрсетілген болсын.

        Онда CDN серверлері осы дереккөздердің кез келгеніне жүгінген кезде `Host: content-source.example.org` тақырыбы пайдаланылады.

    - `Қайта жіберу` — тақырып мәні ретінде бірінші бапталған жеке доменнің атауы пайдаланылады.

        **Мысал:**

        Екі дереккөзден тұратын топ бапталған болсын:

        - `203.0.113.222:8080`,
        - `images.example.com`.

        Сондай-ақ CDN үшін екі жеке домен бапталған болсын:

        - `cdn.contoso.com`,
        - `cdn.example.org`.

        Онда CDN серверлері осы дереккөздердің кез келгеніне жүгінген кезде `Host: cdn.contoso.com` тақырыбы пайдаланылады.
1. **Өзгерістерді сақтау** түймесін басыңыз.

{/tab}

{tab(API)}

{include(../../../../../_includes/_api_cdn_create_change.md)}

Сұрау денесінде мына параметрлерді көрсетіңіз:

- Host тақырыбы ретінде ресурс үшін бірінші CNAME берілуі үшін (әдепкі бойынша), `forward_host_header` және `hostHeader` параметрлері үшін `"enabled": false` көрсетіңіз.
- Host тақырыбы ретінде бірінші бапталған жеке доменнің атауы берілуі үшін мына параметрлерді көрсетіңіз:

    ```json
    "forward_host_header": {
        "enabled": true,
        "value": true
    },
    "hostHeader": {
        "enabled": false
    }
    ```

- Host тақырыбы ретінде кастом домен немесе IP мекенжайы берілуі үшін мына параметрлерді көрсетіңіз:

    ```json
    "forward_host_header": {
        "enabled": false,
        "value": false
    },
    "hostHeader": {
        "enabled": true,
        "value": "<заголовок>"
    }
    ```

Хостқа кастом тақырып орнатуға арналған сұрау үлгісі:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "options": {
        "forward_host_header": {
            "enabled": false,
            "value": false
        },
        "hostHeader": {
            "enabled": true,
            "value": "content-source.example.org"
        }
    }
}'
```

{/tab}

{/tabs}

{note:warn}

Опцияға өзгерістер енгізгеннен кейін {linkto(../../../../../networks/cdn/instructions/manage-cdn/content-settings#cdn-content-settings-cache-clearing)[text=CDN-ресурс кэшін тазалау]} қажет.

{/note}

## {heading(Шилдингті қосу)[id=cdn-origin-settings-shielding]}

Дереккөзді шилдингтеу дереккөзді жоғары жүктемелерден қорғауға мүмкіндік береді. Ол үшін прекэш-сервер пайдаланылады, ол пайдаланушылардың барлық сұрауларын жинап, қажетті контенттің өзі прекэш-сервер кэшінде болмаса, оларды дереккөзге бір-бірден жібереді.

{note:info}
Дереккөз шилдингін қосу CDN жиынтық трафигін тұтынуды арттырады.
{/note}

Дереккөз шилдингін қосу үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **Негізгі баптаулар** қойындысына өтіңіз.
1. **Дереккөз шилдингін белсендіру** опциясын қосыңыз.
1. Ескертуді оқып, қосуды растаңыз.
1. Дереккөзіңізге жақынырақ орналасқан деректер орталығын таңдаңыз.
1. **Өзгерістерді сақтау** түймесін басыңыз.

{/tab}
