## {heading(Рұқсат етілген HTTP-әдістерін баптау)[id=cdn-security-allowed-http]}

**Разрешенные HTTP-методы** опциясы CDN-ресурсына сұраулардың рұқсат етілген HTTP-әдістерін басқаруға мүмкіндік береді. Әдепкі бойынша мына әдістерге рұқсат етілген: `GET`, `HEAD`, `OPTIONS`, `DELETE`. `POST`, `PUT`, `PATCH` әдістеріне рұқсат беру үшін [техникалық қолдауға](/kz/contacts) жүгініңіз.

Рұқсат етілген әдістерді баптау үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **HTTP-заголовки** қойындысына өтіңіз.
1. **Разрешенные HTTP-методы** опциясын қосыңыз.
1. CDN-ресурсы өңдеуі тиіс әдістерді таңдаңыз.
1. **Сохранить изменения** батырмасын басыңыз.

{/tab}

{tab(API)}

{include(../../../../../_includes/_api_cdn_create_change.md)}

Сұрау денесіндегі `options` блогында `allowedHttpMethods` параметрлерін жазыңыз:

- опцияны қосу үшін `"enabled": true`, өшіру үшін — `"enabled": false` мәнін көрсетіңіз;
- `value` параметрінде қолжетімді болуы тиіс әдістерді жазыңыз.

Сұрау мысалы:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options":
        {
        "allowedHttpMethods": {
            "enabled": true,
            "value": [
                "GET",
                "HEAD"
            ]
        },
    }
}'
```

{/tab}

{/tabs}

Егер сервер көрсетілген әдісті танымаған болса, 501 (Not Implemented) коды бар жауап қайтарылады. Егер әдіс серверге белгілі болса, бірақ CDN-ресурсында оған рұқсат етілмесе, 405 (Method Not Allowed) коды бар жауап қайтарылады.

## {heading(Елдер бойынша қол жеткізу саясатын баптау)[id=cdn-security-access-policy-country]}

Баптау контентті белгілі бір елдерден рұқсатсыз қол жеткізуден қорғауға мүмкіндік береді.

Елдер бойынша қауіпсіздік саясатын баптау үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **Безопасность** қойындысына өтіңіз.
1. **Настроить политику доступа по странам** опциясын қосыңыз.
1. Саясат түрін таңдаңыз:

   - **Разрешающая** — таңдалғандардан басқа барлық елдерге контентке қол жеткізуге рұқсат береді;
   - **Блокирующая** — таңдалғандардан басқа барлық елдер үшін контентке қол жеткізуді бұғаттайды.

1. Контентке қол жеткізуге тыйым салу немесе рұқсат беру қажет елдерді таңдаңыз. Елдер тізімде [ISO 3166-1 alpha-2](https://www.iso.org/kz/iso-3166-country-codes.html) пішімінде көрсетілген. Көптік таңдау қолжетімді.
1. **Сохранить изменения** батырмасын басыңыз.

{/tab}

{tab(API)}

{include(../../../../../_includes/_api_cdn_create_change.md)}

Сұрау денесіндегі `options` блогында `country_acl` параметрлерін жазыңыз.

Ресей Федерациясы мен Қазақстаннан ғана контентке қол жеткізуге рұқсат беру үшін сұрау мысалы:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options":
        {
        "country_acl": {
            "enabled": true,
            "excepted_values": [
                "RU",
                "KZ"
            ],
            "policy_type": "deny"
        }
    }
}'
```

{/tab}

{/tabs}

## {heading(Домен бойынша қол жеткізу саясатын баптау)[id=cdn-security-access-policy-domain]}

Баптау контентті басқа сайттарда жариялаудан қорғауға мүмкіндік береді.

Домендер бойынша қауіпсіздік саясатын баптау үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **Безопасность** қойындысына өтіңіз.
1. **Настроить политику доступа по домену** опциясын қосыңыз.
1. Саясат түрін таңдаңыз:

   - **Разрешающая** — көрсетілгендерден басқа барлық домендерге контентке қол жеткізуге рұқсат береді;
   - **Блокирующая** — көрсетілгендерден басқа барлық домендер үшін контентке қол жеткізуді бұғаттайды.

1. `http://` немесе `https://` көрсетпей, доменді немесе домен маскасын енгізіңіз. Мысалы, `example.com`, `*.example.com`. Бірнеше доменді енгізуге болады.
1. (Опционалды) Көрсетілген сайттарға CDN-ресурсына тікелей сілтеме арқылы жүгінуге рұқсат беру немесе тыйым салу үшін **Обращение по прямой ссылке** опциясын қосыңыз.
1. **Сохранить изменения** батырмасын басыңыз.

{/tab}

{tab(API)}

{include(../../../../../_includes/_api_cdn_create_change.md)}

Сұрау денесіндегі `options` блогында `referrer_acl` параметрлерін жазыңыз.

`vk.com` доменінің тек ішкі домендері үшін контентке қол жеткізуге рұқсат беру үшін сұрау мысалы:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options":
        {
        "referrer_acl": {
            "enabled": true,
            "excepted_values": [
                "*.vk.com"
            ],
            "policy_type": "deny"
        }
      }
}'
```

{/tab}

{/tabs}

## {heading(IP-мекенжайлар бойынша қол жеткізу саясатын баптау)[id=cdn-security-access-policy-ip]}

Баптау белгілі бір IP-мекенжайлар үшін контентке қол жеткізуге тыйым салуға мүмкіндік береді.

IP-мекенжайлар бойынша қауіпсіздік саясатын баптау үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **Безопасность** қойындысына өтіңіз.
1. **Настроить политику доступа по IP-адресам** опциясын қосыңыз.
1. Саясат түрін таңдаңыз:

   - **Разрешающая** — көрсетілгендерден басқа барлық IP-мекенжайларға контентке қол жеткізуге рұқсат береді;
   - **Блокирующая** — көрсетілгендерден басқа барлық IP-мекенжайлар үшін контентке қол жеткізуді бұғаттайды.

1. Ішкі желі маскасымен бірге IP-мекенжайларды енгізіңіз. Мысалы, `192.168.3.2/32` немесе `2a03:d000:2980:7::8/128`. Қол жеткізу саясаты көрсетілген IP-мекенжайлар негізінде есептелген желі мекенжайлары бойынша жұмыс істейді. Егер екі немесе одан да көп IP-мекенжай бір желіге тиесілі болса, сол IP-мекенжайлардың біреуін ғана көрсету жеткілікті.
1. **Сохранить изменения** батырмасын басыңыз.

{/tab}

{tab(API)}

{include(../../../../../_includes/_api_cdn_create_change.md)}

Сұрау денесіндегі `options` блогында `ip_address_acl` параметрлерін жазыңыз.

`192.168.1.100/32` және `10.10.10.10/24` IP-мекенжайларынан контентке қол жеткізуге тыйым салу үшін сұрау мысалы:

```json
curl --location --request PUT 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources/175281'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
    "originGroup": 267760,
    "originGroup_name": "exampleorigin",
    "secondaryHostnames": [],
    "options":
        {
        "ip_address_acl": {
            "enabled": true,
            "excepted_values": [
                "192.168.1.100/32",
                "10.10.10.10/24"
            ],
            "policy_type": "allow"
        }
    }
}'   

```

{/tab}

{/tabs}

## {heading(TLS протоколы нұсқаларын таңдау)[id=cdn-security-tls-versions]}

Баптау CDN арқылы соңғы пайдаланушы алатын HTTPS-трафикті шифрлау үшін пайдаланылатын TLS протоколының белгілі бір нұсқаларын таңдауға мүмкіндік береді. Егер опция өшірілген болса, барлық қолжетімді нұсқалар пайдаланылады:

- SSL 3.0,
- TLS 1.0,
- TLS 1.1,
- TLS 1.2,
- TLS 1.3.

TLS протоколы нұсқаларын таңдау үшін:

{tabs}

{tab(Жеке кабинет)}

{include(../../../../../_includes/_open-cdn.md)}

1. **Безопасность** қойындысына өтіңіз.
1. **Настроить разрешенные версии TLS** опциясын қосыңыз.
1. Тізімнен қажетті нұсқаларды таңдаңыз немесе қажетсіздерін алып тастаңыз. Тізімде кемінде бір нұсқа болуы тиіс.
1. **Сохранить изменения** батырмасын басыңыз.

{/tab}

{/tabs}
