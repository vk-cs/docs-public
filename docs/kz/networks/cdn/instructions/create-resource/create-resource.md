Сіз CDN-ресурсын екі тәсілмен жасай аласыз:

- [CDN сервисінің интерфейсі арқылы](#cdn_servisinin_interfeysi_arkyly_zhasau). Бұл нұсқаны, егер бөгде [контент көздерін](../../concepts/origin-groups) көрсету немесе SSL-сертификаттарды қолмен баптау қажет болса, пайдаланыңыз.

- [text=Бакет интерфейсі арқылы]} [VK Object Storage](#baket_interfeysi_arkyly_zhasau) сервисіндегі [Бакет интерфейсі арқылы](/kz/storage/s3). сервисінде. Бұл нұсқаны бакет контент көзі ретінде әрекет етуі үшін пайдаланыңыз. Бакетке сәйкес CDN-ресурс автоматты түрде жасалады, SSL-сертификаттар да автоматты түрде бапталады.

## CDN сервисінің интерфейсі арқылы жасау

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. CDN-ресурсын жасау қажет жобаны таңдаңыз.
1. Бөлімге өтіңіз **CDN → CDN-ресурстар**.
1. түймесін басыңыз **Создать ресурс**.
1. блогында **Қолжетімділік пен хаттаманы баптау** блогында қажетті баптауларды орнатыңыз:

    - **Соңғы пайдаланушыларға контентке қолжетімділік**: егер ресурс жасалғаннан кейін контент тұтынушыларға жеткізіле бастауын қаласаңыз, осы опцияны қосыңыз.

        Егер опция өшірілсе, онда CDN-ресурс жасалғаннан кейін `Приостановлен` күйіне өтеді және контент жеткізілмейді. Контентке. [қолжетімділікті қосуға немесе өшіруге](../manage-cdn/enable-cdn) CDN-ресурсын жасағаннан кейін де болады.

    - **Көзбен өзара әрекеттесу хаттамасы**: выберите протокол, по которому [CDN-серверлер](../../concepts/about) контентті көздерден қай хаттама бойынша сұрататынын таңдаңыз:

       - `HTTP` — көздегі контент тек HTTP (80-порт) арқылы қолжетімді немесе көз HTTPS-тен HTTP-ке қайта бағыттауды қолдайды.
       - `HTTPS` — көздегі контент тек HTTPS (443-порт) арқылы қолжетімді немесе көз HTTP-тен HTTPS-ке қайта бағыттауды қолдайды.
       - (Әдепкі бойынша) `HTTP и HTTPS` — көздегі контент HTTP және HTTPS арқылы қолжетімді. Бұл жағдайда CDN-нен көзге жіберілетін сұрауда соңғы пайдаланушы сұрауының хаттамасы сақталады. Бұл ретте CDN кэшінде файлдың екі нұсқасы — әр хаттама үшін бір-бірден — сақталады.

1. блогында **Көздер мен домендер конфигурациясы** блогында контент сұралатын көздерді баптаңыз:

    - **Контентті сұрату**: CDN-серверлер контентті қай көздерден сұрататынын анықтаңыз:

        - `С одного источника` — в поле **Контент көзі** өрісінде көзге жүгіну қажет URL-ді көрсетіңіз.

            {cut(Требования к URL и примеры)}

            Талаптар:

            - URL тек латын әріптерін, сандарды `.` және `-`.
            - URL схема (`http://` немесе `https://`): қамтымауы тиіс: ол `http(s)://`. түрінде автоматты түрде қосылған. Нақты хаттама бұрын бапталған сәйкес параметр арқылы анықталады.
            - URL-де домендік атауды да, IP-мекенжайды да беруге болады. Сондай-ақ портты көрсетуге де болады.
            - Бүкіл URL-дің ең үлкен ұзындығы — 255 таңба. Ішкі доменнің ең үлкен ұзындығы — 63 таңба.

            Мысалдар:

            - `203.0.113.222:8080`,
            - `images.example.com`.

            {/cut}

            CDN-ресурсын жасау барысында көрсетілген параметрлері бар бір көзден тұратын көздер тобы автоматты түрде жасалады.

        - `С группы источников` — тізімнен топты таңдаңыз немесе ![plus-icon](./assets/plus-icon.svg "inline") **Көздер тобын қосу**.

           Жаңа топты қосқанда оның атауын беріңіз және [оған көздерді қосыңыз](/kz/networks/cdn/instructions/manage-origin-groups). Әрбір көз үшін URL мен түрін (`Активный` немесе `Резервный`), көрсетіңіз, сондай-ақ CDN-серверлер контент сұратқанда пайдаланылатын көздерді қосулы күйде белгілеңіз.

           {cut(Требования к URL и примеры)}

           Талаптар:

           - URL тек латын әріптерін, сандарды және `.`, `-`.
           - URL схема (`http://` немесе `https://`): қамтымауы тиіс: ол `http(s)://`. түрінде автоматты түрде қосылған. Нақты хаттама CDN-ресурсын **Көзбен өзара әрекеттесу хаттамасы** при [создании](../../../../networks/cdn/instructions/create-resource) опциясымен анықталады.
           - URL-де тек домендік атауды (FQDN) ғана беруге болады).
           - Бүкіл URL-дің ең үлкен ұзындығы — 255 таңба. Ішкі доменнің ең үлкен ұзындығы — 63 таңба.

           Мысалдар:

           - `example.org`,
           - `images.example.com`.

           {/cut}
        
           Қосылған барлық көздердің домендері үшін (соның ішінде өшірулілері үшін де) DNS-жазбалар бапталғанына көз жеткізіңіз.

           {note:warn}
           Егер VK Cloud DNS-ке сұрау арқылы доменнің қолжетімділігін тексере алмаса, онда көздер тобын жасау мүмкін болмайды.
           {/note}

    - **Дербес домен**: CDN үшін пайдаланылатын дербес доменді көрсетіңіз. Осы доменге жүгінгенде контент CDN арқылы жеткізіледі. Доменнің толық білікті атауын (FQDN) пайдаланыңыз. Оған түбірлік домен атауын қоспаңыз `cdn.example.com`, емес `cdn.example.com.`.

        Вы можете добавить несколько доменов с помощью кнопки ![plus-icon](./assets/plus-icon.svg "inline") **Домен қосу** түймесінің көмегімен бірнеше домен қоса аласыз немесе олардың жанындағы ![Удалить](./assets/trash-icon.svg "inline") белгішесін басып, қажетсіз домендерді жоя аласыз.

        {note:warn}
        CDN-ресурсы жасалғаннан кейін дербес домендерді өзгерту мүмкін емес.
        {/note}

        Оны CNAME-жазбада көрсету үшін түпнұсқа доменнің атауын сақтау қажет.

    - **Host тақырыбын өзгерту**: бұрын бапталған көздерге жүгінгенде, [оның мәнін өзгерту керек пе](/kz/networks/cdn/instructions/manage-cdn/origin-settings#host_takyrybyn_ozgertu) соны таңдаңыз.
    
        - `Не менять` — тақырып мәні бірінші CNAME-пен, яғни бірінші бапталған дербес доменнің атауымен сәйкес келеді.

            {cut(Пример)}

            Екі көзден тұратын топ бапталған болсын:

            - `203.0.113.222:8080`,
            - `images.example.com`.

            Сондай-ақ CDN-ресурсында CNAME жасау үшін екі дербес домен бапталған болсын:

            - `cdn.contoso.com`,
            - `cdn.example.org`.

            Онда CDN-серверлер осы көздердің кез келгеніне жүгінгенде `Host: cdn.contoso.com`.

            {/cut}

        - `Кастомный` — тақырып ретінде пайдаланылатын доменді немесе IP-мекенжайды көрсетіңіз. Көз көрсетілген тақырып үшін сұрауларды өңдей алуы тиіс.

            {cut(Пример)}

            Екі көзден тұратын топ бапталған болсын:

            - `203.0.113.222:8080`,
            - `images.example.com`.

            мәні көрсетілген болсын `content-source.example.org`.

            Онда CDN-серверлер осы көздердің кез келгеніне жүгінгенде `Host: content-source.example.org`.

            {/cut}

        - `Пересылать` — көзден контент сұратылғанда CDN-ге сұрау жіберілгендегідей Host тақырыбы жіберіледі.

            {cut(Пример)}

            Екі көзден тұратын топ бапталған болсын:

            - `203.0.113.222:8080`,
            - `images.example.com`.

            Сондай-ақ CDN-ресурсында CNAME жасау үшін екі дербес домен бапталған болсын:

            - `cdn.contoso.com`,
            - `cdn.example.org`.
        
            Онда Host тақырыбының мәні мынадай болады:
            
            - мекенжайына жүгінгенде `cdn.contoso.com` — `Host: cdn.contoso.com`;
            - мекенжайына жүгінгенде  `cdn.example.org` — `Host: cdn.example.org`.

            {/cut}

1. блогында **Шифрлау баптаулары** блогында бұрын бапталған дербес домендерге HTTPS арқылы қол жеткізу кезінде пайдаланылатын SSL-сертификатқа жауап беретін опцияны таңдаңыз:

    - `Не использовать` — сертификат пайдаланылмайды: дербес домендерге тек HTTP арқылы ғана жүгінуге болады.

    - (Әдепкі бойынша) `Let's Encrypt` — тегін [Let's Encrypt](https://letsencrypt.org/ru). сертификаты пайдаланылады. Сертификат CDN-ресурсы жасалғаннан кейін, көз-серверлер қолжетімді болғанда және дербес домендер үшін CNAME-жазбаларға қатысты DNS-тегі өзгерістер таралған кезде жасалады. Әдетте бұған 30 минутқа дейін уақыт кетеді.

      {note:warn}
      Бұл опцияны таңдағанда **Соңғы пайдаланушыларға контентке қолжетімділік**.
      {/note}

    -  `Свой сертификат` — шифрлау үшін пайдаланылатын сертификатты тізімнен таңдаңыз. Сертификат таңдау үшін қолжетімді болуы үшін, оны, [сертификаттар қоймасына қосыңыз](../../../../networks/cdn/instructions/manage-certificates).

1. түймесін басыңыз **Создать ресурс**.

{/tab}

Сұрау мысалы:

```json
curl --location --request POST 'https://kz.cloud.vk.com/api/cdn/api/v1/projects/examplef8f67/resources'\
--header 'X-Auth-Token: example6UjMOd'\
--header 'Content-Type: application/json'\
--data '{
   "cname": "cdn.example.com",
   "originGroup": 132,
   "secondaryHostnames": [
      "1.example.com",
      "2.example.com"
   ]
}'
```

## CDN-ресурсын жұмысқа дайындау

1. CDN-ресурсы үшін CNAME-жазбасын жасаңыз. Бұл URL-мекенжайды өзгертуге мүмкіндік береді. Мысалы, егер сіз CDN-ресурсын веб-сайтыңыздың кескіндерін ұсыну үшін пайдалансаңыз `images.example.com`, түріндегі CDN-ресурсына нұсқайды `cl-541e19d9.service.cdn.msk.vkcs.cloud`.

   Егер сіз VK Cloud DNS сервисін пайдалансаңыз [нұсқаулықты](/kz/networks/dns/instructions/publicdns/records#add).

   Егер жасау кезінде түпнұсқа доменді сақтамаған болсаңыз, оны CDN-ресурсы туралы ақпараттан таба аласыз:

   {include(/kz/_includes/_open-cdn.md)}

1. Статикалық файлдарға дейінгі жолда түпнұсқа доменді дербес доменге ауыстырыңыз.

1. [Контентті жүктеңіз](../upload-content) VK Cloud интерфейсіне (егер CDN-ресурсы бакеттен жасалмаған болса).
