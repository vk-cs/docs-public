DNS VK Cloud сервисі жария және жеке желілердегі домендік аймақтар мен жазбаларды басқаруға арналған құралдарды ұсынады. Ол VK Cloud бұлтты сервистер экожүйесімен біріктірілген және трафикті бағдарлауды баптауға, ресурстардың қолжетімділігін қамтамасыз етуге және инфрақұрылым компоненттері арасындағы қауіпсіз өзара әрекеттесуді ұйымдастыруға мүмкіндік береді.

## {heading(Қызмет туралы)[id=private-dns]}

*Жеке DNS* — платформаның жобалық желілерінде жұмыс істейтін DNS-сервердің функционалдығы. Инстанстарға DNS атаулары арқылы жүгінуге мүмкіндік береді.

Сервис жеке аймақты және виртуалды машиналар порттарының атауларын баптауды қолдайды. DNS-сервер желідегі DHCP порттарымен бірдей мекенжайлар бойынша жауап береді. Желіде жеке DNS жұмыс істеуі үшін DHCP-сервер қосулы болуы керек.

<warn>

Жеке DNS сұрауларын қайта жіберу серверлері — `8.8.8.8`, `8.8.4.4`. Бұл мекенжайларды өзгертуге қолдау көрсетілмейді.

</warn>

## {heading(Жария DNS)[id=public-dns]}

*Жария DNS* интернетте көрінетін DNS аймақтарын басқарады.

Жария DNS-тің негізгі функциялары:

* A, AAAA, CNAME, MX, TXT, SRV және басқа жазбаларды баптау.
* Географиялық үлестірілген, істен шығуға төзімді қосымша инфрақұрылымында жүктемені теңгерімдеу.
* Ресурстардың IP мекенжайлары өзгерген кезде жазбаларды автоматты түрде жаңарту (мысалы, жүктеме теңгергіштері үшін).

Жария DNS-пен жұмыс істеу үшін VK Cloud [жеке кабинетінің](/kz/tools-for-using-services/account/concepts/rolesandpermissions) пайдаланушыларына арналған [рөлдерді](https://kz.cloud.vk.com/app/) баптаңыз:

- DNS аймақтары мен ресурстық жазбаларды көру үшін — Бақылаушы.

- DNS аймақтары мен ресурстық жазбаларды өңдеу үшін:

    - Жоба иесі,
    - Желі әкімшісі,
    - Суперәкімші.

Жария DNS үшін келесі квоталар мен лимиттер қолданылады:

- Жоба ішіндегі DNS аймақтарының ең көп саны — 100.
- Бір DNS аймағы үшін A түріндегі ресурстық жазбалардың ең көп саны — 750.
- Бір DNS аймағы үшін әрбір түрдегі ресурстық жазбалардың ең көп саны — 500.
