VK Cloud қолданба инфрақұрылымы орналасқан деректер орталығы істен шыққан жағдайда да жұмысын жалғастыратын ақауға төзімді қолданбаларды жасауға мүмкіндік береді.

Ақауға төзімділікті қамтамасыз ету үшін қолданба инфрақұрылымы бір аймақтың бірнеше [қолжетімділік аймақтарында](/kz/start/concepts/architecture#az) қайталануы керек, ал аймақтардың бірі қолжетімсіз болған жағдайда балансировка сұрауларды көрші аймақтардағы серверлерге қайта бағыттауы тиіс. Мұндай архитектура үшін қолжетімділік аймақтарының ең аз саны — екі, оңтайлысы — үш.

DNSBalancer сервисі қолжетімділік аймақтары арасында сұрауларды теңестіруді орындайды.

{cut(Сервис жұмысының принципі)}

DNSBalancer сервисі қолданба өрістетілген ВМ-ларға, барлық қолжетімділік аймақтарында орнатылады.

Сервис өзі орнатылған ВМ-ның жария IP-мекенжайының (A-жазбасының) және басқа қолжетімділік аймақтарындағы ВМ-лардың қолжетімділігін тексереді.

Егер сервис өзінің ВМ IP-мекенжайы қолжетімсіз екенін анықтаса, ол жария DNS-тегі сәйкес A-жазбаны жояды.

Егер сервис көрші ВМ-ның IP-мекенжайы қолжетімсіз екенін анықтаса:

1. Сервис өзінің ВМ жария мекенжайымен A-жазбасын жасайды (егер A-жазбасы әлі жасалмаса).
2. Сервис қолжетімсіз жария мекенжайлары бар A-жазбаларды жояды.

Мақсатты сервиске сұраулар тізімдегі тек қолжетімді IP-мекенжайлар арқылы ғана жіберіледі.

Тексеру берілген уақыт аралығы арқылы циклдік түрде орындалады. Егер кейінгі тексеру кезінде ВМ қолжетімді болса, оның IP-мекенжайына сәйкес A-жазба тізімге қайта қосылады.

{/cut}

Мақалада DNSBalancer сервисін әртүрлі қолжетімділік аймақтарындағы үш ВМ-ға қалай орнатып, баптау керектігі көрсетілген. Сервис бапталғаннан кейін қолданбаға сұраулар жұмыс істеп тұрған ВМ-лардың біріне жіберіледі. Егер ВМ қолжетімсіз болса, оған сұраулар жіберілмейді.

## Дайындық қадамдары

1. [Өтіңіз](https://kz.cloud.vk.com/app) VK Cloud жеке кабинетіне.

## 2. Сервис конфигурациясы файлын жасаңыз

1. ВМ-лардың бірінде терминал сессиясын ашыңыз.
1. конфигурация файлын жасаңыз `/etc/dnsbalancer/config.yaml`.
1. Файлда сервисті баптауға арналған негізгі параметрлерді көрсетіңіз:

    ```yaml
    config:
        url: "https://mcs.mail.ru/public-dns/" 
        auth_url: "https://infra.mail.ru:3535/v3/" 
        login: "example@test" 
        password: "123456example" 
        project_id: "777e19938fe947666d5d3be5c00053d7" 
        zone: "example.com" 
        own_ip: "213.213.213.213" 
        is_self_maintenance: true 
        driver: "vkcloudns" 
        protocol: "tcp" 
        port: 80
        timeout: 180 
    ```

    Мұнда:

    - `url`: Publicdns API эндпоинты.
    - `auth_url`: авторизацияға арналған URL **API арқылы қолжеткізу** қойындысындағы **Auth URL**.
    - `login`: пайдаланушы логині **API арқылы қолжеткізу** қойындысындағы **Username**.
    - `password`: пайдаланушы құпиясөзі.
    - `project_id`: жоба идентификаторы **API арқылы қолжеткізу** қойындысындағы **Project ID**.
    - `zone`: бақылануы тиіс және ол үшін A-жазбалар жасалатын жария DNS-аймақ.
    - `own_ip`: сервис данасы орналасқан ВМ-ның IPv4-мекенжайы.
    - `is_self_maintenance`:
      - `true` — мақсатты IP-мекенжай мен DNSBalancer сервисінің IP-мекенжайы бірдей.
      - `false`— мақсатты IP-мекенжай мен DNSBalancer сервисінің IP-мекенжайы әртүрлі.
    - `driver`: DNS-серверге арналған бэкенд түрі. Тек `vkcloudriver`.
    - `protocol`: IP-мекенжайды тексеру хаттамасы:
      - `icmp` — арқылы тексеру `ICMP Echo Request`, утилитасы сияқты жұмыс істейді `ping`.
      - `tcp` — мақсатты IP-мекенжаймен TCP-қосылым орнату арқылы тексеру.
    - `port`: тексерілуі қажет порт.
    - `timeout`: тексерулер арасындағы уақыт, секундпен, ұсынылатын мәні 180-тен 360-қа дейін 360.

1. Қадамдарды әр ВМ үшін қайталаңыз.

## 3. Демонды жасап, іске қосыңыз

1. ВМ-лардың бірінде терминал сессиясын ашыңыз.
1. Келесі пәрменнің көмегімен `dnsbalancer.service` файлын жасаңыз:

   ```console
   sudo vim /etc/systemd/system/dnsbalancer.service
   ```  

1. Жасалған файлға келесі жолдарды қосыңыз:

    ```console
    [Unit]
    Description=Service for checking A-Records into Zone
    After=network.target

    [Service]
    Type=simple
    WorkingDirectory=/usr/local/bin
    ExecStart=/usr/local/bin/dnsbalancer
    Restart=always

    [Install]
    WantedBy=multi-user.target

1. Перезагрузите процесс `systemd` с помощью команды:

   ```console
   sudo systemctl daemon-reload
   ```

1. Запустите демон при помощи команды:

   ```console
   sudo systemctl start dnsbalancer.service
   ```

1. Выполните команду, чтобы демон запускался автоматически во время загрузки системы:

   ```console
   sudo systemctl enable dnsbalancer.service
   ```

1. Повторите действия для каждой ВМ.

В результате сервис будет передавать в DNS-зону A-записи только доступных ВМ, поэтому все запросы к приложению будут отправляться только к активному серверу. Приложение продолжит работу даже в случае отключения одной или двух ВМ.
