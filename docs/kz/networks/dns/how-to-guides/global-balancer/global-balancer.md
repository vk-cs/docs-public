VK Cloud әртүрлі деректер орталықтарында ([қолжетімділік аймақтарында](/kz/start/concepts/architecture#az)). орналастырылған бірнеше бэкенд-серверді пайдаланатын географиялық үлестірілген архитектураның есебінен ақауға төзімді қолданбаларды жасауға мүмкіндік береді. Қолжетімділік аймақтарының бірінде апат болған жағдайда, инфрақұрылым трафикті басқа деректер орталықтарындағы жұмысқа қабілетті серверлерге автоматты түрде қайта бағыттап, бос тұрып қалуды азайтады.

Бұл үлгіні іске асыру үшін DNS деңгейіндегі жүктемені теңестіру механизмі қолданылады, ол серверлердің қолжетімділігін тұрақты түрде тексеріп, сұрауларды резервтік серверлерге қайта бағыттайды.

Бұл мысалда әртүрлі қолжетімділік аймақтарындағы үш ВМ DNS сервисінің теңгерімдеу топтары арқылы біріктіріледі.

## Дайындық қадамдары

1. Егер бұл бұрын жасалмаса [DNS-теңгерімдеуге](/kz/networks/dns/concepts/global-balancer) [техникалық қолдауға](/kz/contacts) жүгіну арқылы қолжеткізіңіз.
1. [Өтіңіз](https://kz.cloud.vk.com/app) VK Cloud жеке кабинетіне.
1. Интернетке қолжетімді, әртүрлі [қолжетімділік аймақтарында](/kz/computing/iaas/instructions/vm/vm-create) орналасқан үш ВМ таңдаңыз немесе [жасаңыз](/kz/start/concepts/architecture#az).
1. Осы ВМ-лардың сыртқы IP-мекенжайларын жазып алыңыз. Бұл мысалда:

    - `83.166.234.101`;
    - `109.120.188.102`;
    - `217.16.23.103`.

1. Жасалған ВМ-лар үшін көрсетілген портқа `169.254.169.100/32` IP-мекенжайынан келетін кіріс трафикті [рұқсат етіңіз](/kz/networks/vnet/instructions/secgroups). Бұл — сервистің қолжетімділігін тексеруге арналған VK Cloud техникалық мекенжайы. Егер қауіпсіздік тобының ережесі осы мекенжайдан ВМ портына қолжеткізуді шектесе, DNS-теңгерімдеу ВМ-ды қолжетімсіз деп белгілеп, оған трафик жібермейді.
1. [жасаңыз](/kz/networks/dns/instructions/publicdns/dns-zone#add) Бұл мысалда `example.dns.com`.
1. Егер бұл әлі жасалмаса, доменіңізді домен тіркеушісінде тіркеп, оны VK Cloud NS-серверлеріне делегирлеңіз. DNS-жазбаларының жаңартылуын күтіңіз: әдетте бұл 10–30 минутты алады.
1. DNS-теңгерімдеудің жұмысын тексеру үшін компьютеріңізге `dig` утилитасын орнатыңыз:
...
   DNS-теңгерімдеуді тексеру үшін `dig` утилитасының баламаларын пайдалана аласыз. Бұл жағдайда тексеру пәрмендері өзгеше болады.

   {/note}

## 1. DNS-аймаққа A-жазбаларды қосыңыз

DNS-аймақта қосылған әр ВМ-ның сыртқы IP-мекенжайына сәйкес келетін A-жазбаларды [жасаңыз](/kz/networks/dns/instructions/publicdns/records#add).

## 2. Теңгерімдеу тобын жасаңыз

1. DNS-аймақ үшін `Round Robin` теңгерімдеу түрімен теңгерімдеу тобын [жасаңыз](/kz/networks/dns/instructions/publicdns/global-balancer#add).
1. Теңгерімдеу тобына DNS-аймақта қолжетімді барлық A-жазбаларды қосыңыз:

   - мониторинг түрін таңдаңыз `ICMP`;
   - қалған параметрлерді әдепкі мәндермен қалдырыңыз.

## 3. DNS-теңгерімдеудің жұмысын тексеріңіз

1. Компьютерде пәрменді орындаңыз:

   {tabs}

   {tab(dig)}

   ```console
   for i in {1..100}; do dig www.example.dns.com  A @ns4.msc.mail.ru +short; done | sort | uniq -c
   ```

   {/tab}

   {tab(nslookup)}

   ```console
   for i in {1..100}; do 
       nslookup -type=A www.example.dns.com ns4.msc.mail.ru | grep -A1 "Name:" | tail -1
   done | sort | uniq -c
   ```

   {/tab}

   {/tabs}

   Күтілетін жауап:

   ```bash
   33 83.166.234.101
   34 109.120.188.102
   33 217.16.23.103
   ```

   Мұнда жолдың басында DNS-теңгерімдеу әр серверге жіберген сұраулар саны көрсетіледі. Жауап теңгерімдеудің жұмыс істеп тұрғанын және сұрауларды барлық қолжетімді серверлер (қолжетімділік аймақтары) арасында шамамен тең бөлетінін көрсетеді).

1. VK Cloud жеке кабинетінде үш ВМ-ның екеуін [тоқтатыңыз](/kz/computing/iaas/instructions/vm/vm-manage#start_stop_restart_vm) Бұл сервердің істен шығуын немесе деректер орталығының қолжетімсіздігін модельдейді.
1. Компьютерде пәрменді қайталаңыз:

   {tabs}

   {tab(dig)}

   ```console
   for i in {1..100}; do dig www.example.dns.com  A @ns4.msc.mail.ru +short; done | sort | uniq -c
   ```

   {/tab}

   {tab(nslookup)}

   ```console
   for i in {1..100}; do 
       nslookup -type=A www.example.dns.com ns4.msc.mail.ru | grep -A1 "Name:" | tail -1
   done | sort | uniq -c
   ```

   {/tab}

   {/tabs}

   Күтілетін жауап:

   ```bash
   100 217.16.23.103
   ```

   Мұнда жауап DNS-теңгерімдеу барлық сұрауларды бір серверге (қолжетімділік аймағына) жіберетінін көрсетеді, өйткені қалған екі сервер қолжетімсіз.
