# {heading(VPN-туннелі жұмыс істемейді)[id=vnet-vpn-tunnel-psk]}

{include(/kz/_includes/_translated_by_ai.md)}

Егер жасалған VPN-туннелі `DOWN` күйінде болып, қайта жүктегеннен кейін де жұмыс істей бастамаса, бұған ортақ пайдалану кілтінде (PSK) арнайы таңбалардың қолданылуы себеп болуы мүмкін.

Ортақ пайдалану кілтін жасау кезінде `-`, `+`, `&`, `!`, `@`, `#`, `$`, `%`, `^`, `*`, `(`, `)`, `,`, `.`, `:`, `;`, `_`, `=`, `<`, `>`, `{`, `}`, `/` арнайы таңбаларын пайдалануға болады, бірақ олардың кейбірі, мысалы `#`, `&` немесе `{`, операциялық жүйе арқылы интерпретациялануы мүмкін. Бұл жағдайда кілт қате болады және VPN-туннелі жұмыс істемейді.

### {heading(Шешім)[id=vnet-vpn-tunnel-psk-resolve]}

1. Клиент желісі жағында `/etc/ipsec.secrets` файлындағы кілтті қос тырнақша `"` арқылы экранирлеңіз.

    Мысал:

    ```plaintext

    192.0.2.200 192.0.2.100 : PSK "1#$@5{840...9RTS86&"

    ```

1. Егер бұлттық желі жағында кілтті OpenStack CLI арқылы жасап, экранирлеу қолданылған-қолданылмағанын есіңізде сақтамасаңыз, `/etc/ipsec.secrets` файлындағы кілтті алып, оған тырнақшалар қосып ауыстырыңыз:

    ```console

    openstack vpn ipsec site connection set <ID_VPN-ТУННЕЛЯ> --psk "<КЛЮЧ_СОВМЕСТНОГО_ИСПОЛЬЗОВАНИЯ>"

    ```
    Егер кілтті жеке кабинет арқылы жасаған болсаңыз, ол әлдеқашан экранирленген.

1. Егер мәселе сақталса, [техникалық қолдау қызметіне хабарласыңыз](/kz/contacts).
