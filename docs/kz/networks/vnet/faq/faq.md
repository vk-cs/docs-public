{cut(Менің бұлтымда ішкі желілер не үшін пайдаланылады және олар қандай қолжетімділік аймақтарына жатады?)}

Ішкі желілерді талдау үшін [техникалық қолдауға](/kz/contacts) хабарласыңыз.
Өтініште мыналарды көрсетіңіз:

- [жоба PID](/kz/tools-for-using-services/account/instructions/project-settings/manage#zhoba_identifikatoryn_alu);
- [ішкі желілер ID](/kz/networks/vnet/instructions/net#zheliler_men_ishki_zheliler_tizimin_sonday_ak_olar_turaly_akparatty_karau).

{/cut}

{cut(IP мекенжайы үшін PTR-жазбаларын қалай басқаруға болады?)}

PTR-жазбасы арқылы кері DNS-сұрауды баптау үшін [техникалық қолдауға](/kz/contacts) хабарласыңыз.
Өтініште IP-адресті және оған сәйкес домен атауын көрсетіңіз.

{/cut}

{cut(Шлюздің IP мекенжайы басқа жобадан қолжетімді бола ма?)}

Әртүрлі жобалар арасындағы желілік байланыстылықты ұйымдастыру үшін [ортақ желілерді пайдаланыңыз](/kz/networks/vnet/instructions/shared-nets). Желіні бір жобадан екіншісіне бере аласыз, содан кейін екі жобада да маршруттауды баптай аласыз.

{/cut}

{cut(Сыртқы SMTP серверлеріне дейінгі трафик шектеле ме?)}

Спамнан қорғау үшін жаңа жобаларда SMTP порттары (25, 465, 587) арқылы шығатын трафик әдепкі бойынша шектелген.

Егер поштаны жібере алмасаңыз, шектеуді алып тастау үшін [техникалық қолдауға](/kz/contacts) жүгініңіз.

{/cut}

{cut(VK Cloud IP мекенжайларынан жария ресурстарға қолжетімділікке шектеулер бар ма?)}

VK Cloud жария ресурстарға бұғаттаусыз интернетке стандартты қолжеткізуді ұсынады.

Мұны интернетке қолжеткізуі бар ВМ-ді [құру арқылы](/kz/networks/vnet/quick-start) тексере аласыз.

{/cut}

{cut(Windows Server-де VPN баптаудың қандай ерекшеліктері бар?)}

Windows Server жүйесінде VPN (IPsec/IKEv2) жұмысы үшін рұқсат беретін ережелерді баптаңыз:

1. VK Cloud қауіпсіздік топтарында.

   UDP протоколы бойынша 500 және 4500 порттарына кіріс трафигін ашыңыз. Көзі ретінде VPN-клиенттердің IP-адресін немесе адрестер ауқымын көрсетіңіз.

1. Виртуалды машинадағы Windows Брандмауэрінде.

   UDP бойынша 500 және 4500 порттарына кіріс қосылымдарға рұқсат беретін ережелерді жасаңыз.

{/cut}

{cut(Floating IP-адресін қалай ауыстыруға болады?)}

ВМ-ге жаңа Floating IP-адресті тағайындау үшін:

1. [Ағымдағы Floating IP-адресті](/kz/networks/vnet/instructions/ip/floating-ip#disassociate) порттан ажыратыңыз.
1. [Жаңа Floating IP-адрес алыңыз](/kz/networks/vnet/instructions/ip/floating-ip#add).

{note:warn}

Қолданыстағы Floating IP-адрестер ешбір портқа тағайындалмаса да тарифтелмейді. Егер ажыратылған Floating IP-адрес бұдан былай қажет болмаса, оны [жойыңыз](/kz/networks/vnet/instructions/ip/floating-ip#delete).

{/note}

{/cut}

2. [Портты](/kz/networks/vnet/instructions/ports#portty_kosu_dc4476c4) екінші виртуалды машинаға қосыңыз.

{/cut}
