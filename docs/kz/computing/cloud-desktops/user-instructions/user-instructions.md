# {heading(Пайдаланушыға арналған қадамдық нұсқаулықтар)[id=desktops-user-instructions]}

{include(/kz/_includes/_translated_by_ai.md)}

Нұсқаулықтар виртуалды жұмыс үстеліне қосылуға және онымен жұмысты бастауға көмектеседі. Жұмысты бастамас бұрын компьютеріңізге қосылу үшін қажетті қолданбаны — Cloud Desktop Client орнатыңыз.

## {heading(Cloud Desktop Client қолданбасын орнату)[id=desktops-install-client]}

{tabs}
{tab(Windows)}

1. {ifdef(public)}[cloud-desktop-setup.exe](https://hub.mcs.mail.ru/repository/client-agent-remoteclick/client/release/win/cloud-desktop-setup.exe) орнату файлын жүктеп алыңыз{/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf)}`cloud-desktop-setup.exe` орнату файлын Cloud Desktop әкімшісінен сұраңыз{/ifdef}.
1. Файлды іске қосып, Cloud Desktop Client қолданбасының орнатылуын күтіңіз.

   {note:warn}
   Cloud Desktop Client орнату үшін әкімші құқықтары қажет.
   {/note}

1. Компьютерді қайта жүктеңіз.

{/tab}
{tab(macOS)}

1. {ifdef(public)}[cloud-desktop-setup.dmg](https://hub.mcs.mail.ru/repository/client-agent-remoteclick/client/release/mac/cloud-desktop-setup.dmg) орнату файлын жүктеп алыңыз{/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf)}`cloud-desktop-setup.dmg` орнату файлын Cloud Desktop әкімшісінен сұраңыз{/ifdef}.
1. Файлды іске қосыңыз.
1. Қолданбаны **Программалар** қалтасына жылжытыңыз.

{note:warn}
Қолданбаны алғаш іске қосқанда оның интернеттен жүктелгені туралы ескерту көрсетіледі. Ескертуді елемей, қолданбаны ашуға келісіңіз.
{/note}

{/tab}
{tab(РЕД ОС)}

1. {ifdef(public)}[cloud-desktop-setup.rpm](https://hub.mcs.mail.ru/repository/client-agent-remoteclick/client/release/redos/cloud-desktop-setup.rpm) орнату файлын жүктеп алыңыз{/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf)}`cloud-desktop-setup.rpm` орнату файлын Cloud Desktop әкімшісінен сұраңыз{/ifdef}.
1. Файлды іске қосып, Cloud Desktop Client қолданбасының орнатылуын күтіңіз.

   {note:warn}
   Cloud Desktop Client орнату үшін әкімші құқықтары қажет.
   {/note}

1. Компьютерді қайта жүктеңіз.

{/tab}
{/tabs}

## {heading(Cloud Desktop серверіне және жұмыс үстеліне қосылу)[id=desktops-client-connect]}

1. Жұмыс үстеліне қосылу үшін қажет ақпаратты Cloud Desktop әкімшісінен біліп алыңыз:

   - Cloud Desktop серверінің атауы және IP-мекенжайы.
   - Серверге қосылу порты. Әдетте `5031` порты пайдаланылады.
   - Серверге қосылу үшін екі факторлы аутентификация қажет пе және қажет болса, авторизацияның екінші факторын қалай баптау керек.
   - Жұмыс үстелдері пулының және сізге бөлінген жұмыс үстелінің атаулары.
   - Жұмыс үстеліне арналған логин мен құпиясөз. Әдетте ұйымыңыздағы аутентификация үшін қолданылатын логин мен құпиясөзбен сәйкес келеді.
   - {linkto(../../../computing/cloud-desktops/concepts/about#desktops-about-pool)[text=Жұмыс үстелдері пулының түрі]}: сессиялық немесе жекелендірілген.

   {note:info}
   Сессиялық жұмыс үстелінің баптауларындағы және мазмұнындағы кез келген өзгерістер онымен {linkto(#desktops-end-of-work)[text=жұмысты аяқтағаннан]} кейін жоғалады.

   Жекелендірілген жұмыс үстелін баптауға, онда деректерді сақтауға және қолданбаларды орнатуға болады. Барлық өзгерістер кәдімгі компьютердегідей бір қосылудан келесісіне дейін сақталады.
   {/note}

1. {linkto(#desktops-install-client)[text=Орнатыңыз]} және Cloud Desktop Client қолданбасын іске қосыңыз.
1. ![](../../../assets/settings-icon.svg "inline") белгішесін басыңыз.
1. Ашылған терезеде **HTTPS пайдалану** және **Сертификаттарды елемеу** опцияларын таңдап, **Қолдану** түймесін басыңыз.
1. **Сервер қосу** түймесін басып, **Серверге қосылуды баптау** терезесінде параметрлерді көрсетіңіз:

   - Cloud Desktop серверінің атауы.
   - Cloud Desktop серверінің IP-мекенжайы.
   - Серверге қосылу порты.
   - Серверге қосылуға арналған логин мен құпиясөз. Ұйымыңыздағы аутентификация үшін қолданылатын логин мен құпиясөзбен сәйкес келеді.

1. **Қолдану** түймесін басыңыз.

   {note:info}
   Cloud Desktop Client келесі іске қосылғанда қосылған сервер бірден қосылуға қолжетімді болады.
   {/note}

1. Серверді таңдап, **Қосылу** түймесін басыңыз.

   Егер Cloud Desktop сервисі үшін екі факторлы аутентификация бапталған болса, келесі әрекеттердің бірін орындаңыз:

   {ifdef(public)}
   - Серверге алғаш қосылғанда авторизацияның екінші факторын баптаңыз. Нақты сценарий екі факторлы авторизация провайдеріне байланысты. Мысалы, QR-кодты Google Authenticator қолданбасы арқылы сканерлеп, кейін қолданбадағы кодты веб-беттегі өріске енгізу қажет болуы мүмкін.
   - Серверге қайта қосылғанда авторизацияның екінші факторын көрсетіңіз.
     {/ifdef}

   Қолжетімді жұмыс үстелдері пулдарының тізімі ашылады.

1. Қажетті пулды және қажетті жұмыс үстелін таңдап, **Қосылу** түймесін басыңыз.

   Виртуалды жұмыс үстелі бөлек терезеде ашылады.

1. Егер ОС баптауларында **Қауіпсіздік экраны** қарастырылған болса, ![](../../../assets/command-icon.svg "inline") белгішесін басып, **Қауіпсіздік экраны** тармағын таңдаңыз. Ашылған қауіпсіздік экранында CTRL+ALT+DEL пернелер тіркесімін басыңыз.
1. Жұмыс үстеліне арналған логин мен құпиясөзді енгізіңіз.

## {heading(Жұмыс үстеліне жылдам қол жеткізу үшін жарлық жасау)[id=desktops-shortcut]}

{tabs}
{tab(Windows)}

1. {linkto(#desktops-install-client)[text=Орнатыңыз]} және Cloud Desktop Client қолданбасын іске қосыңыз.
1. Қажетті серверді таңдап, **Қосылу** түймесін басыңыз.

   Қолжетімді жұмыс үстелдері пулдарының тізімі ашылады.

1. Қажетті виртуалды жұмыс үстелі үшін ![](../../../assets/more-icon.svg "inline") басып, **Жұмыс үстеліне қосу** тармағын таңдаңыз.

Жұмыс үстелінде Cloud Desktop Client қолданбасын іске қоспай-ақ таңдалған виртуалды жұмыс үстеліне жылдам қосылуға арналған жарлық пайда болады.

{/tab}
{/tabs}

## {heading(Толық экран және терезелік режимдер арасында ауысу)[id=desktops-change-display-mode]}

Әдепкі бойынша жұмыс үстелі оған қосылған кезде толық экран режимінде ашылады. Қажет болса, терезелік режимге және кері ауыса аласыз:

- Толық экран режимінен шығу үшін экранның жоғарғы шекарасындағы мәзірдегі **Терезеге кішірейту** белгішесін басыңыз немесе ALT+ENTER пернелерін басыңыз.
- Толық экран режиміне қайта оралу үшін терезенің жоғарғы оң жақ бұрышындағы стандартты **Кеңейту** белгішесін басыңыз немесе ALT+ENTER пернелерін басыңыз.
- Меңзерді жұмыс үстелі бар терезенің сыртына шығару үшін CTRL+ALT пернелерін басыңыз.

## {heading(Windows жылдам пернелерін пайдалануды баптау)[id=desktops-change-hotkey-mode]}

1. Экранның немесе жұмыс үстелі терезесінің жоғарғы шекарасындағы мәзірдегі ![Команда](../../../assets/command-icon.svg "inline") белгішесін басыңыз.
1. **Windows пернелер тіркесімдерін пайдалану** тізімінен қажетті режимді таңдаңыз:

   - `Осы компьютерде`: жылдам пернелер тек жергілікті компьютерде жұмыс істейді.
   - `Қашықтағы компьютерде`: жылдам пернелер тек виртуалды жұмыс үстелінде жұмыс істейді.
   - `Тек толық экран режимінде`: терезелік режимде жылдам пернелер тек жергілікті компьютерде, ал толық экран режимінде тек виртуалды жұмыс үстелінде жұмыс істейді.

## {heading(Бейнетрансляцияның кеңсе режимін қосу)[id=desktops-enable-office-mode]}

Кеңсе режимі {linkto(../../../computing/cloud-desktops/concepts/work-modes#desktops-office-mode)[text=кескін сапасын арттырады]}, бұл экранды жаңарту жиілігі жоғары болу қажет емес жағдайларда, мысалы кеңсе қолданбаларымен жұмыс істегенде.

Кеңсе режимін қосу үшін:

1. Экранның немесе жұмыс үстелі терезесінің жоғарғы шекарасындағы мәзірдегі ![](../../../assets/settings-icon.svg "inline") белгішесін басыңыз.
1. **Бейнетрансляция режимі** тізімінен `Кеңсе` тармағын таңдаңыз.

## {heading(Перифериялық құрылғыларды қосу)[id=desktops-client-connect-peripherals]}

Жұмыс үстеліне {linkto(../../../computing/cloud-desktops/concepts/work-modes#desktops-peripherals)[text=қолжетімді]} перифериялық құрылғыларды қосу үшін:

1. Экранның немесе жұмыс үстелі терезесінің жоғарғы шекарасындағы мәзірден қажетті белгішені басыңыз, мысалы принтер белгішесін.

   Таңдалған түрдегі перифериялық құрылғылар тізімі ашылады, мысалы қолжетімді принтерлер тізімі.

1. Тізімнен жалаушалар арқылы қажетті құрылғыларды таңдаңыз.
1. Дәл осылай басқа түрдегі перифериялық құрылғыларды таңдаңыз, мысалы жергілікті компьютер қалталары немесе USB құрылғылары.

   Барлық таңдалған құрылғылар виртуалды жұмыс үстелінде қолжетімді болады.

## {heading(Жұмыс үстелінен ажыратылу)[id=desktops-end-of-work]}

Жұмыс үстелінен келесі тәсілдердің бірімен ажыратыңыз:

- Жұмыс үстелінің қонақ ОС жүйесінен стандартты тәсілмен шығыңыз.
- Cloud Desktop Client қолданбасын жабыңыз.

  Сессиялық жұмыс үстелі жойылады, ал жекелендірілген жұмыс үстелі келесі қосылуға дейін бұғатталады.

## {heading(Cloud Desktop серверінен ажыратылу)[id=desktops-client-disconnect]}

1. Экранның немесе жұмыс үстелі терезесінің жоғарғы шекарасындағы мәзірдегі ![](../../../assets/exit-icon.svg "inline") белгішесін басыңыз.
1. **Сеансты аяқтау** терезесінде **Аяқтау** түймесін басыңыз.

{note:warn}
{linkto(#desktops-end-of-work)[text=ажыратылмаған]} жұмыс үстелдері қосулы күйінде қалып, есептеу ресурстарын тұтынуды жалғастырады.
{/note}
