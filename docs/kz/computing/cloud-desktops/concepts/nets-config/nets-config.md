Cloud Desktop сервисінің желілік конфигурациясына мыналар кіреді:

- сервистің инфрақұрылымының қызметтік ВМ-лерін орналастыруға арналған желі;
- жұмыс үстелдері пулдарын қосуға арналған бір немесе бірнеше желі.

Осы желілердің барлығы интернетке шығатын бір маршрутизаторға қосылады. Қызметтік ВМ-лерді және жұмыс үстелдері пулдарының кез келген санын бір желіде орналастыруға рұқсат етіледі.

Желілік конфигурацияны баптаудың автоматты және қолмен орындалатын режимдері қолданылады. Режим сервис инфрақұрылымына арналған [желіні баптау](../../instructions/config/setup-net) кезеңінде таңдалады және жұмыс үстелдері пулын алғаш рет жасау іске қосылғанға дейін өзгертілуі мүмкін. Осыдан кейін сервис желілерін баптау режимін өзгерту мүмкін болмайды.

### {heading(Желілерді автоматты түрде баптау)[id=automatic_net_setup]}

Желі автоматты режимде бапталған кезде Cloud Desktop сервисінің желілік баптауларында мыналар көрсетіледі:

- ешбір желіге байланыстырылмаған, интернетке шығатын маршрутизатор;
- қажетті көлемдегі IP мекенжайлар диапазоны.

Cloud Desktop инстансымен бірге инстанстың барлық ВМ-лері: қызметтік ВМ-лер де, жұмыс үстелдері пулдары да орналастырылатын желі автоматты түрде жасалады.

### {heading(Желілерді қолмен баптау)[id=manual_net_setup]}

Қолмен баптау кезінде әртүрлі желілік конфигурацияларды жасауға болады:

- қызметтік ВМ-лерді және жұмыс үстелдері пулдарының кез келген санын қосу үшін автоматты режимдегідей бір желіні пайдалану;
- қызметтік ВМ-лер үшін бөлек желіні және әрбір пул үшін бөлек желілерді жасау;
- кез келген басқа нұсқаларды баптау.

Қызметтік ВМ-лерді орналастыруға арналған желі сервис инфрақұрылымын [баптау](../../instructions/config/setup-net) кезінде, ал жұмыс үстелдері пулдарына арналған желілер [пулды құру](../../instructions/desktops-pool/add) кезінде көрсетіледі.

### {heading(Порттар санына қойылатын талаптар)[id=ports_number]}

Қолмен баптау кезінде желілерде бос порттар санына қойылатын талаптардың орындалуын өз бетінше бақылау қажет:

- Қызметтік ВМ-лерге арналған желіде кемінде 16 бос порт болуы тиіс.
- Пул желісіндегі бос порттар саны мынадан кем болмауы керек:

  - егер пул дербестендірілген болса, жұмыс үстелдерінің ең аз саны;
  - егер пул сессиялық болса, жұмыс үстелдерінің ең аз саны мен ыстық резервтегі жұмыс үстелдерінің санының қосындысы.

  Бос порттардың ұсынылатын саны пулдағы жұмыс үстелдерінің ең көп санына тең, әр жұмыс үстеліне бір порттан.

- Егер желі сервистің бірнеше компоненті үшін пайдаланылса, мысалы қызметтік ВМ-лер мен пул үшін, қажетті порттар саны осы компоненттерге қойылатын талаптардың қосындысы ретінде есептелуі тиіс.

Бос порттардың *P* саны келесі формула бойынша есептеледі:

{formula}
P = 2^{32-M}-C-2
{/formula}

Мұндағы *M* — ішкі желі маскасы, *C* — әлдеқашан жасалған порттардың саны.

{note:warn}

Егер желілер тек Cloud Desktop инстансы үшін ғана емес, басқа қажеттіліктер үшін де пайдаланылса, уақыт өте келе олардағы бос порттар саны өзгеруі мүмкін. Порттар жетіспеген жағдайда жұмыс үстелдері қателікпен жасалады.

{/note}
