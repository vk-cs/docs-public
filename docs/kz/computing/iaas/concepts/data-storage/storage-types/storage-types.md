# {heading(Деректерді сақтау түрлері)[id=iaas-concepts-storage-types]}

{include(/kz/_includes/_translated_by_ai.md)}

{var(cloud)} келесі сақтау түрлерін қолдайды:

- _Блоктық (block-based)_. Деректерді қатты дискілердегі сияқты логикалық блоктар түрінде сақтайды. Блоктық сақтау ВМ және контейнерлердің деректерін, сондай-ақ диск снимоктарын сақтау үшін қолайлы. {var(cloud)} ішінде бұл түр ВМ-ге қосылатын дискілер және [Kubernetes кластерлеріндегі дискілер](/kz/kubernetes/k8s/concepts/storage) арқылы ұсынылған.

  {var(cloud)} ішіндегі блоктық дискілер желілік және жергілікті болуы мүмкін, сақтау сенімділігімен, репликацияланатын деректер көшірмелерінің санымен және [өнімділік](../volume-sla) параметрлерімен ерекшеленеді. Толығырақ — [блоктық сақтау дискілері](../disk-types) бөлімінде.

- _Объектілік (object-based)_. Деректерді метадеректері бар тәуелсіз объектілер түрінде сақтайды. Сирек қолжеткізілетін үлкен көлемдегі деректерді (резервтік көшірмелер, BigData, үлкен медиафайлдар және т.б.) сақтау үшін қолайлы. {var(cloud)} ішінде бұл түр S3 API қолдауы бар [VK Object Storage](/kz/storage/s3) сақтауымен ұсынылған.
- _Файлдық (file-based)_. Бірнеше виртуалды машинаға қосуға болатын виртуалды файлдық жүйе. Бұлттың жеке желісі ішінде орналасқан клиенттер арасында файлдармен орталықтандырылған алмасуды іске асырады. Деректерге жүгінгенде ең аз кідірісті талап ететін тұрақты тапсырмаларға қолайлы. Толығырақ — [файлдық сақтау орны](../file-share) бөлімінде.

Сақтау түрлерінің әрқайсысының өз артықшылықтары мен кемшіліктері бар. Түрді таңдау қолданбаның қажеттіліктеріне, сондай-ақ өнімділікке, ауқымдылыққа және деректерді басқаруға қойылатын талаптарға байланысты.
