# {heading(Файлдық сақтау орнын жасау)[id=iaas-fs-create]}

{include(/kz/_includes/_translated_by_ai.md)}

Cloud Servers ішінде файлдық сақтау орындарын жеке кабинет, OpenStack CLI немесе API арқылы жасауға болады. API көмегімен файлдық сақтау орнын жасау үшін [файлдық сақтау орындарын басқаруға арналған OpenAPI спецификациясын](/kz/tools-for-using-services/api/api-spec/iaas-api/manila-api) пайдаланыңыз.

{tabs}

{tab(Жеке кабинет)}

1. {ifdef(public)}[Өтіңіз](https://kz.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cer)}{linkto(../../../../../intro/authorization/lk_entry#prerequisites_vkc_ui)[text=Өтіңіз]}{/ifdef} {var(cloud)} жеке кабинетіне.
1. Файлдық сақтау орнын жасау қажет [жобаны](/kz/tools-for-using-services/account/concepts/projects) таңдаңыз.
1. **Бұлтты есептеулер** → **Файлдық сақтау орны** бөліміне өтіңіз.
1. **Сақтау орнын жасау** немесе **Жасау** батырмасын басыңыз.
1. Файлдық сақтау орнының параметрлерін орнатыңыз:

   - **Файлдық сақтау орнының атауы**: тек латын әріптерінен, цифрлардан және `-`, `_` және `.` арнайы таңбаларынан тұруы мүмкін.
   - **Сақтау орнының өлшемі**: ГБ-тағы сақтау орнының өлшемі. Квота шегінде болуы, 10 ГБ-тан кем емес және 10000 ГБ-тан артық емес болуы тиіс.
   - **Протокол**: сақтау орнына Windows ОЖ-нен қатынау үшін CIFS протоколын, Linux-тен — NFS протоколын таңдаңыз.
   - **Желі**: ұсынылғандардың ішінен таңдаңыз немесе жаңасын жасаңыз. Жаңа желі үшін сондай-ақ **Ішкі желі мекенжайын** көрсетіңіз.
   - **Файлдық сақтау орнының желісі**: ұсынылғандардың ішінен таңдаңыз немесе жаңасын жасаңыз.
   
1. **Келесі қадам** батырмасын басыңыз.
1. (Опционалды түрде) Сақтау орнына қатынау ережелерін қосыңыз:

   1. **Жаңа ережені қосу** түймесін басыңыз.
   1. Бастапқы көздің IP мекенжайын немесе ішкі желі мекенжайын енгізіңіз.
   1. Қатынау режимін таңдаңыз.
   1. **Ережені қосу** батырмасын басыңыз.

   {note:info}
   Қатынау [ережелерін](/kz/computing/iaas/instructions/fs-manage/fs-access-rules#iaas-fs-access-rules-adding) файлдық сақтау орнын жасағаннан кейін де қосуға болады.
   {/note}

1. **Файлдық серверді қосу** түймесін басыңыз.

{/tab}

{tab(OpenStack CLI)}

1. OpenStack клиенті [орнатылғанына](/kz/tools-for-using-services/cli/openstack-cli#1_ustanovite_klient_openstack) көз жеткізіңіз және жобада [аутентификациядан өтіңіз](/kz/tools-for-using-services/cli/openstack-cli#openstack-authorize).
1. Manila клиенті [орнатылғанына](/kz/tools-for-using-services/cli/openstack-cli#openstack-install-package) көз жеткізіңіз.
1. Файлдық сақтау орнының бар желісін таңдаңыз немесе жаңасын жасаңыз.

   {tabs}
    
   {tab(Бар желіні таңдау)}
        
   1. Келесі команданы орындап, файлдық сақтау орнының бар желілерінің тізімін алыңыз:

      ```console
      openstack share network list
      ```

      Қажетті желінің атауын немесе идентификаторын жазып алыңыз.

   1. (Опционалды түрде) Келесі команданы орындап, таңдалған желінің қасиеттерін қараңыз:

      ```console
      openstack share network show <СЕТЬ>
      ```

      Мұнда `<СЕТЬ>` — жобада бар файлдық сақтау орны желісінің атауы немесе идентификаторы.

   {/tab}
    
   {tab(Жаңа желіні жасау)}
    
   Команданы орындаңыз:

   ```console
   openstack share network create --neutron-net-id <ID_СЕТИ> --neutron-subnet-id <ID_ПОДСЕТИ> --name <ИМЯ_СЕТИ>
   ```

   Мұнда:

   - `<ID_СЕТИ>` — жобада бар приватті желінің идентификаторы.
   - `<ID_ПОДСЕТИ>` — оның ішкі желісінің идентификаторы.
   - `<ИМЯ_СЕТИ>` — жасалатын файлдық сақтау орны желісіне арналған атау.

   Жасалған желінің атауын немесе идентификаторын жазып алыңыз.

   {/tab}
    
   {/tabs}

1. Келесі команданың көмегімен файлдық сақтау орнын жасаңыз:

   ```console
   openstack share create --name <ИМЯ_ХРАНИЛИЩА> --share-network <СЕТЬ> <ПРОТОКОЛ> <РАЗМЕР> 
   ```

   Мұнда:

   - `<ИМЯ_ХРАНИЛИЩА>` — жасалатын файлдық сақтау орнының атауы.
   - `<СЕТЬ>` — алдыңғы қадамда таңдалған немесе жасалған желінің атауы немесе идентификаторы.
   - `<ПРОТОКОЛ>` — қатынау протоколы. Сақтау орнына Windows ОЖ-нен қатынау үшін CIFS протоколын, Linux-тен — NFS протоколын көрсетіңіз.
   - `<РАЗМЕР>` — ГБ-тағы файлдық сақтау орнының қажетті өлшемі. Квота шегінде болуы, 10 ГБ-тан кем емес және 10000 ГБ-тан артық емес болуы тиіс.

{/tab}
