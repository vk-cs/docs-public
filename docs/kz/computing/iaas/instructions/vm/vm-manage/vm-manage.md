VK Cloud платформасында виртуалды машинаның параметрлері мен күйін [жеке кабинетте](https://kz.cloud.vk.com/app/) немесе OpenStack CLI командалық жол интерфейсі арқылы өзгертуге болады.

## Виртуалды машинаның ID алу

OpenStack CLI көмегімен ВМ басқару үшін виртуалды машинаның идентификаторы қажет.

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. ВМ тізімінде идентификаторын білу қажет виртуалды машинаның атауын басыңыз.
1. ВМ бетінде **Жалпы ақпарат** қойындысына өтіңіз.
1. Кестеден **Виртуалды машинаның ID** жолын табыңыз.

{/tab}

{tab(OpenStack CLI)}

1. OpenStack CLI клиентін [орнатыңыз](/kz/tools-for-using-services/cli/openstack-cli) және егер әлі жасалмаса, жобада аутентификациядан өтіңіз.

1. Келесі әрекеттердің бірін орындаңыз.

    - Егер ВМ атауы белгілі болса, виртуалды машина туралы ақпаратты алып, `id` жолын табыңыз:

      ```console
      openstack server show <Имя ВМ>
      ```

    - Егер ВМ нақты атауы белгісіз болса, виртуалды машиналар тізімін шығарып, қажеттісін табыңыз:

      ```console
      openstack server list
      ```

{/tab}

{/tabs}

## {heading(ВМ іске қосу, тоқтату, қайта жүктеу)[id=start_stop_restart_vm]}

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. ВМ күйін келесі тәсілдердің бірі арқылы өзгертіңіз.

    - Топтық операциялар арқылы — бірнеше ВМ үшін:

        1. Виртуалды машиналар тізімінде күйін өзгерту қажет ВМ-дерді жалаушалар арқылы таңдаңыз.
        1. ВМ тізімінің үстінде қажетті әрекеті бар батырманы басыңыз.
        1. Әрекетті растаңыз.

    - Контекстік мәзір арқылы — бір ВМ үшін:

        1. Виртуалды машиналар тізімінде күйін өзгерту қажет ВМ үшін ![ ](/kz/assets/more-icon.svg "inline") белгішесін басыңыз.
        1. Әрекетті таңдап, растаңыз.

    - Виртуалды машина бетінде:

        1. Виртуалды машиналар тізімінде күйін өзгерту қажет ВМ атауын басыңыз.
        1. ВМ бетінде **Жалпы ақпарат** қойындысына өтіңіз.
        1. ВМ параметрлері бар кестенің оң жағында қажетті әрекет белгішесін басыңыз.
        1. Әрекетті растаңыз.

{/tab}

{tab(OpenStack CLI)}

1. OpenStack клиентін [орнатыңыз](/kz/tools-for-using-services/cli/openstack-cli) және егер әлі жасалмаса, жобада аутентификациядан өтіңіз.

1. Қажетті команданы орындаңыз.

    - ВМ іске қосу:

      ```console
      openstack server start <ID виртуальной машины>
      ```

    - ВМ тоқтату:

      ```console
      openstack server stop <ID виртуальной машины>
      ```

    - ВМ қайта жүктеу:

      ```console
      openstack server reboot <ID виртуальной машины>
      ```

{/tab}

## ВМ мәжбүрлі қайта жүктеу

Егер виртуалды машина жауап бермесе, мәжбүрлі қайта жүктеуді қолданыңыз.

{note:info}

ВМ мәжбүрлі қайта жүктеу қоректендіруді өшіру және қайта қосуға (power cycling) сәйкес келеді.<br/>Қайта жүктеу ВМ операциялық жүйесінің дұрыс аяқталуын (graceful shutdown) білдіреді.

{/note}

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. ВМ мәжбүрлі қайта жүктеуді келесі тәсілдердің бірі арқылы орындаңыз.

    - Контекстік мәзір арқылы:

        1. Виртуалды машиналар тізімінде қажетті ВМ үшін ![ ](/kz/assets/more-icon.svg "inline") белгішесін басыңыз.
        1. **Мәжбүрлі қайта жүктеу** тармағын таңдап, әрекетті растаңыз.

    - Виртуалды машина бетінде:

        1. Виртуалды машиналар тізімінде күйін өзгерту қажет ВМ атауын басыңыз.
        1. ВМ бетінде **Жалпы ақпарат** қойындысына өтіңіз.
        1. ВМ параметрлері бар кестенің үстінде **Тағы** батырмасын басыңыз.
        1. **Мәжбүрлі қайта жүктеу** батырмасын басып, әрекетті растаңыз.

{/tab}

{tab(OpenStack CLI)}

1. OpenStack клиентін [орнатыңыз](/kz/tools-for-using-services/cli/openstack-cli) және егер әлі жасалмаса, жобада аутентификациядан өтіңіз.

1. ВМ мәжбүрлі қайта жүктеуді орындаңыз:

   ```console
   openstack server reboot --hard <ID виртуальной машины>
   ```

{/tab}

{/tabs}

    - Контекстік мәзір арқылы:

        1. Виртуалды машиналар тізімінде қажетті ВМ үшін ![ ](/kz/assets/more-icon.svg "inline") белгішесін басыңыз.
        1. Әрекетті таңдап, растаңыз.

    - Виртуалды машина бетінде:

        1. Виртуалды машиналар тізімінде күйін өзгерту қажет ВМ атауын басыңыз.
        1. ВМ бетінде **Жалпы ақпарат** қойындысына өтіңіз.

## {heading(ВМ жою)[id=delete_vm]}

VK Cloud платформасында виртуалды машина әдепкі бойынша root-дискімен бірге жойылады. Қосымша дискілерді жою сәйкес саясат параметрлеріне байланысты.

{note:err}

Жою операциясы қайтарылмайды! ВМ жоймас бұрын қажетті деректерді сақтаңыз.

{/note}

1. Виртуалды машинаны жоюға дайындаңыз.

    - Егер root-дискіні сақтау қажет болса, оны клондаңыз немесе [қажетсізге ауыстырыңыз](../vm-root-replace).

    - Сақтау қажет қосымша дискілерді [ВМ-ден ажыратыңыз](../../volumes/volumes-connect#dismount_disk).

      Жою саясаты параметрінде `delete_on_termination`: `False` қасиеті орнатылған қосымша дискілерді ВМ-нен ажыратпауға болады.

      {cut(Дискілердің жою саясатын тексеру — OpenStack CLI)}

        1. OpenStack клиенті [орнатылғанына](/kz/tools-for-using-services/cli/openstack-cli#1_openstack_klientin_ornatynyz) көз жеткізіңіз және жобада [аутентификациядан өтіңіз](/kz/tools-for-using-services/cli/openstack-cli#3_autentifikaciyadan_otiniz).

        1. Команданы орындаңыз:

         ```console
         openstack server show <ID виртуальной машины> --os-compute-api-version 2.42
         ```

           `volumes_attached` жолын табыңыз. Онда дискілердің ID-лері және олардың жою саясаты — `delete_on_termination` қасиеті (`True` немесе `False`) көрсетіледі.

      {/cut}

1. Қажетті деректерді сақтағаннан кейін виртуалды машинаны жойыңыз.

   {tabs}

   {tab(Жеке кабинет)}

    1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
    1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
    1. ВМ-ді келесі тәсілдердің бірі арқылы жойыңыз.

        - Топтық операциялар арқылы — бірнеше ВМ үшін:

            1. Виртуалды машиналар тізімінде жою қажет ВМ-дерді таңдаңыз.
            1. ВМ тізімінің үстіндегі мәзірде **Жою** батырмасын басыңыз.
            1. Әрекетті растаңыз.

        - Контекстік мәзір арқылы — бір ВМ үшін:

            1. Виртуалды машиналар тізімінде жою қажет ВМ үшін ![ ](/kz/assets/more-icon.svg "inline") белгішесін басыңыз.
            1. **Жою** тармағын таңдап, әрекетті растаңыз.

        - Виртуалды машина бетінде:

            1. Виртуалды машиналар тізімінде жою қажет ВМ атауын басыңыз.
            1. ВМ бетінде **Жалпы ақпарат** қойындысына өтіңіз.
            1. ВМ параметрлері бар кестенің оң жағында себет белгішесін басыңыз.
            1. Әрекетті растаңыз.

   {/tab}

   {tab(OpenStack CLI)}

    1. OpenStack клиентін [орнатыңыз](/kz/tools-for-using-services/cli/openstack-cli) және егер әлі жасалмаса, жобада аутентификациядан өтіңіз.
    1. ВМ жойыңыз:

      ```console
      openstack server delete <ID виртуальной машины>
      ```

   {/tab}

## ВМ атауын өзгерту және типін өзгерту

VK Cloud ішінде виртуалды машинаның атауын өзгертуге, сондай-ақ ВМ типін — процессорлар санын (vCPU) және жедел жад көлемін (RAM) өзгертуге болады.

{note:info}

ВМ типі өзгерген кезде виртуалды машина қайта жүктеледі.

{/note}

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. ВМ атауын немесе типін келесі тәсілдердің бірі арқылы өзгертіңіз.

    - Контекстік мәзір арқылы:

        1. Виртуалды машиналар тізімінде қажетті ВМ үшін ![ ](/kz/assets/more-icon.svg "inline") белгішесін басыңыз.
        1. Әрекетті таңдаңыз.
        1. Енгізу өрісін толтырып, **Сақтау** батырмасын басыңыз.

    - Виртуалды машина бетінде:

        1. Виртуалды машиналар тізімінде күйін өзгерту қажет ВМ атауын басыңыз.
        1. ВМ бетінде **Жалпы ақпарат** қойындысына өтіңіз.
        1. ВМ параметрлері бар кестенің үстінде **Тағы** батырмасын басыңыз.
        1. Әрекетті таңдаңыз.
        1. Енгізу өрісін толтырып, **Сақтау** батырмасын басыңыз.

{/tab}

{tab(OpenStack CLI)}

1. OpenStack клиентін [орнатыңыз](/kz/tools-for-using-services/cli/openstack-cli) және егер әлі жасалмаса, жобада аутентификациядан өтіңіз.

1. Қажетті әрекетті орындаңыз.

    - ВМ атауын өзгерту:

      ```console
      openstack server set --name <новое имя> <ID виртуальной машины>
      ```

    - ВМ типін өзгерту:

        1. Қолжетімді конфигурация үлгілерінің тізімін алып, үлгі ID-сін көшіріңіз:

         ```console
         openstack flavor list --all
         ```

        1. ВМ типін өзгерту процесін іске қосып, оның аяқталуын күтіңіз:

         ```console
         openstack server resize --flavor <ID шаблона> <ID виртуальной машины>
         ```

{/tab}

{/tabs}

## Тегтерді тағайындау

Тегтер виртуалды машиналар тізімін сүзуге және қажетті ВМ-ді жылдамырақ табуға мүмкіндік береді. Бар тегтерді пайдаланыңыз немесе өз тегіңізді жасап, оған түс таңдаңыз.

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. Қажетті ВМ үшін ![ ](/kz/assets/more-icon.svg "inline") батырмасын басыңыз.
1. Қажетті тегтерді жалаушалармен белгілеңіз.
1. Жаңа тег қосу үшін **Жаңа тег құру** батырмасын басыңыз, оның атауын енгізіңіз, түсін таңдаңыз және **Тег қосу** батырмасын басыңыз.

{/tab}

{tab(OpenStack CLI)}

1. OpenStack клиентін [орнатыңыз](/kz/tools-for-using-services/cli/openstack-cli) және егер әлі жасалмаса, жобаға аутентификациядан өтіңіз.

1. Қажетті команданы орындаңыз.

    - ВМ тегтерінің тізімін шығару:

      ```console
      nova server-tag-list <ID виртуальной машины>
      ```

    - ВМ-ге тег тағайындау:

      ```console
      nova server-tag-add <ID виртуальной машины> <тег>
      ```

    - ВМ-дегі тегті жою:

      ```console
      nova server-tag-delete <ID виртуальной машины> <тег>
      ```

    - ВМ-дегі барлық тегтерді жою:

      ```console
      nova server-tag-delete-all <ID виртуальной машины>
      ```

{/tab}

{/tabs}

## {heading(Құпиясөзді орнату және өзгерту)[id=password]}

[Әдепкі пайдаланушы тіркелгісінің](../../../concepts/oper-system#default_account) құпиясөзін ОС-қа алғашқы авторизация кезінде өзіңіз орнатуыңыз қажет. Орнатылған құпиясөзді кейін өзгертуге болады.

Құпиясөзді орнату шарттары:

- виртуалды машина іске қосылған және ОС жүктеу процесі аяқталған;
- QEMU қонақ агенті орнатылған және іске қосылған (күйі — `running`).

{note:info}

VK Cloud бейнесінен жасалған виртуалды машиналарда QEMU қонақ агенті әдепкі бойынша бар.

{/note}

QEMU қонақ агентінің күйін тексеру командасы:

{tabs}

{tab(Windows)}

```console
sc query qemu-ga
```

{/tab}

{tab(Linux)}

```console
systemctl status qemu-guest-agent
```

{/tab}

{/tabs}

Құпиясөзді орнату үшін:

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. Келесі тәсілдердің бірін пайдаланыңыз:

    - Контекстік мәзір арқылы:

        1. Виртуалды машиналар тізімінде қажетті ВМ үшін ![ ](/kz/assets/more-icon.svg "inline") батырмасын басыңыз.
        1. **Құпиясөз орнату** тармағын таңдаңыз.
        1. Жаңа құпиясөзді енгізіңіз немесе **Генерациялау** батырмасын басыңыз.
        1. **Құпиясөзді орнату** батырмасын басыңыз.

    - Виртуалды машина бетінде:

        1. Виртуалды машиналар тізімінде күйін өзгерту қажет ВМ атауын басыңыз.
        1. ВМ бетінде **Жалпы ақпарат** немесе **Консоль** қойындысына өтіңіз.
        1. ВМ параметрлері бар кестенің үстінде немесе консоль терезесінің үстінде **Құпиясөз орнату** батырмасын басыңыз.
        1. Жаңа құпиясөзді енгізіңіз немесе **Генерациялау** батырмасын басыңыз.
        1. **Құпиясөзді орнату** батырмасын басыңыз.

{/tab}

{tab(OpenStack CLI)}

1. OpenStack клиентін [орнатыңыз](/kz/tools-for-using-services/cli/openstack-cli) және егер әлі жасалмаса, жобаға аутентификациядан өтіңіз.

1. Команданы орындаңыз:

   ```console
   openstack server set --root-password <ID виртуальной машины>
   ```

## Құпиясөзді қалпына келтіру

Әдепкі пайдаланушы үшін генерацияланған құпиясөзді қалпына келтіру тек приватты желіде жасалған Windows виртуалды машиналары үшін қолжетімді. Құпиясөзді алу үшін ВМ жасау кезінде таңдалған кілт жұбының приватты кілті қажет.

{note:warn}

Егер ВМ жасалғаннан кейін құпиясөз өзгертілген болса немесе ВМ сыртқы желіде жасалған болса, [жаңа құпиясөз орнатыңыз](#password).

{/note}

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. Құпиясөзді алу терезесін келесі тәсілдердің бірімен ашыңыз.

    - Контекстік мәзір арқылы:

        1. Виртуалды машиналар тізімінде қажетті ВМ үшін ![ ](/kz/assets/more-icon.svg "inline") батырмасын басыңыз.
        1. **Құпиясөзді алу** тармағын таңдаңыз.

    - Виртуалды машина бетінде:

        1. Виртуалды машиналар тізімінде күйін өзгерту қажет ВМ атауын басыңыз.
        1. ВМ бетінде **Жалпы ақпарат** қойындысына өтіңіз.
        1. ВМ параметрлері бар кестенің үстінде **Тағы** батырмасын басыңыз.
        1. **Құпиясөзді алу** таңдаңыз.

1. Ашылған терезеде **Приватты кілт файлы** батырмасын басыңыз немесе кілт мазмұнын **Приватты кілт** өрісіне енгізіңіз.
1. **Құпиясөзді шифрдан шығару** батырмасын басыңыз.
1. Терезеде **Құпиясөз** өрісі пайда болады. Өрістегі белгішені басып, құпиясөзді көшіріңіз.

{/tab}

## Кілт арқылы ВМ-ге қолжетімділікті қалпына келтіру

SSH арқылы кілт жұбын пайдаланып Linux виртуалды машинасына қолжетімділікті қалпына келтіру үшін ОС пайдаланушысының құпиясөзін білу қажет.

1. Жария кілтті алыңыз.

    - Егер жария кілт файлы жоғалған болса, приватты кілт файлы бар директорияға өтіп, команданы орындаңыз:

      ```console
      ssh-keygen -y -f <имя файла с приватным ключом> > <имя файла с публичным ключом>
      ```

    - Егер приватты кілт файлы жоғалған болса, [жаңа кілт жұбын жасаңыз](/kz/tools-for-using-services/vk-cloud-account/instructions/account-manage/keypairs#kilttik_zhupty_zhasau).

1. Жария кілт файлына сілтеме жасаңыз.

    1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
    1. **Объектілік сақтау → Бакеттер** бөліміне өтіңіз.
    1. Бар бакетті пайдаланыңыз немесе [жаңасын жасаңыз](/kz/storage/s3/instructions/buckets/create-bucket).
    1. Жария кілті бар файлды бакетке [қосыңыз](/kz/storage/s3/instructions/objects/upload-object).
    1. Осы файлға сілтеме арқылы [қолжетімділікті қосыңыз](/kz/storage/s3/instructions/access-management/signed-url).
    1. Жария кілт файлына сілтемені көшіріңіз.

1. Жария кілтті виртуалды машинаға жүктеңіз.

    1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
    1. Виртуалды машиналар тізімінде қажетті ВМ атауын басыңыз.
    1. ВМ бетінде **Консоль** қойындысына өтіңіз.
    1. ОС ВМ жүйесіне кіру үшін әдепкі пайдаланушы атын және құпиясөзді пайдаланыңыз.
    1. Жария кілт файлын жүктеңіз:

      ```console
      wget <ссылка на файл с публичным ключом>
      ```

    1. Жария кілтті `authorized_keys` файлына көшіріңіз:

      ```console
      cat <файл с публичным ключом> >> ~/.ssh/authorized_keys
      ```

1. Кілт арқылы қолжетімділікті тексеріңіз.

    1. Приватты кілт сақталған компьютердің терминалына өтіңіз.
    1. SSH арқылы ВМ-ге қосылыңыз:

      ```console
      ssh -i <путь к файлу с приватным ключом> <логин>@<внешний IP-адрес ВМ>
      ```

## Оқиғалар журналын қарау

Оқиғалар журналы ВМ күйінің өзгеруі және пайдаланушы әрекеттері туралы ақпаратты қамтиды.

{tabs}

{tab(OpenStack CLI)}

1. OpenStack клиентін [орнатыңыз](/kz/tools-for-using-services/cli/openstack-cli) және егер әлі жасалмаса, жобаға аутентификациядан өтіңіз.

1. Қажетті команданы орындаңыз.

    - Оқиғалар журналын қарау (кеңейтілген формат):

      ```console
      openstack server event list --long <ID виртуальной машины>
      ```

    - Оқиғалар журналын қарау (қысқа формат):

      ```console
      openstack server event list <ID виртуальной машины>
      ```

    - Оқиға туралы толық ақпаратты шығару үшін **Request ID** бағанындағы оқиға идентификаторын пайдаланыңыз:

      ```console
      openstack server event show <ID виртуальной машины> <ID события>
      ```

Оқиғалар мысалдары:

- `start` / `stop` — ВМ іске қосу / тоқтату;
- `resize` / `confirmResize` — ВМ типін өзгерту процесін іске қосу / растау;
- `extend_volume` — диск көлемін ұлғайту;
- `create` — ВМ жасау;
- `live-migration` — ВМ миграциясы.

{/tab}
