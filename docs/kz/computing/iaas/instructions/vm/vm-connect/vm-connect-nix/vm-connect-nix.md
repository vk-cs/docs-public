# {heading(Linux ВМ-іне қосылу)[id=iaas-vm-connect-nix]}

{include(/kz/_includes/_translated_by_ai.md)}

Linux виртуалды машинасына қосылудың ұсынылатын тәсілі — SSH протоколы арқылы кілттік жұпты пайдалану.

## {heading(1. SSH арқылы қосылу мүмкіндігін тексеріңіз)[id=iaas-vm-connect-nix-ssh]}

SSH арқылы қосылу үшін виртуалды машинада SSH сервері іске қосылған болуы қажет. {var(cloud)} платформасында жасалған Linux виртуалды машиналарында әдепкі бойынша OpenSSH-сервер орнатылады және іске қосылады.

Қосылмас бұрын төмендегі шарттардың орындалатынына көз жеткізіңіз:

- виртуалды машина іске қосылған және ОЖ жүктелу процесі аяқталған;
- ВМ желі баптауларында SSH протоколы (TCP-порт `22`) бойынша қашықтан қол жеткізуге рұқсат берілген — `ssh` қауіпсіздік тобын [қосыңыз](/kz/networks/vnet/instructions/secgroups#vnet-secgroups-instance-add-sg);
- ВМ-де қосылуға қолжетімді сыртқы IP-мекенжай ([Floating IP](/kz/networks/vnet/concepts/ips-and-inet#vnet-ips-and-inet-floating-ip)) бар.

  {note:info}
  ВМ-нің сыртқы IP-мекенжайын {var(cloud)} жеке кабинетінде (https://kz.cloud.vk.com/app/) **Бұлттық есептеулер → Виртуалды машиналар** бөлімінен көруге болады.
  {/note}

Сыртқы IP-мекенжайдың орнына виртуалды машинаның толық домендік атын (FQDN) пайдалануға болады. Толық домендік ат арқылы қосылу үшін DNS-жазбалары дұрыс бапталған болуы қажет.

{note:info}
{var(cloud)} доменіңізді делегирлей алатын [DNS-серверлерін](/kz/networks/dns/instructions/publicdns) ұсынады.
{/note}

{var(cloud)} платформасы ұсынатын FreeBSD және Bitrix бейнелерінен жасалған виртуалды машиналарда `ext-net` сыртқы желісімен жұмыс істеу қолдау көрсетілмейді. Мұндай ВМ-ге SSH арқылы қосылу үшін ол DHCP қосылған приват желіде болуы қажет.

Егер виртуалды машинаға SSH арқылы қосылу мүмкін болмаса немесе қолжетімсіз болса, [VNC-консолін](../../vm-console#vm-console-vnc) пайдаланыңыз.

## {heading(2. Есептік деректерді дайындаңыз)[id=iaas-vm-connect-nix-credentials]}

ВМ-ге SSH арқылы қосылу үшін ОЖ әдепкі бойынша жасалған пайдаланушының атын [анықтаңыз](../../../../concepts/oper-system#iaas-oper-system-default-account). Оны жазып алыңыз.

## {heading(3. Кілттік жұптың бар-жоғын тексеріңіз)[id=iaas-vm-connect-nix-key-pared]}

ВМ-ге SSH арқылы кілттік жұпты пайдаланып қосылу үшін ашық кілт виртуалды машинада `~/.ssh/authorized_keys` файлында сақталуы, ал жеке кілті бар файл қосылу орындалатын компьютерде орналасуы қажет.

Егер виртуалды машина {var(cloud)} платформасында жасалған болса:

- ашық кілт виртуалды машинада автоматты түрде сақталды;
- **Жаңа кілт жасау** опциясы таңдалса, `.pem` кеңейтімі бар файлдағы жеке кілт ВМ жасалған компьютерге жүктелді.

Егер жұптағы кілттердің бірі немесе екеуі де жоғалған болса, [қалпына келтіру нұсқаулығын](../../vm-manage#iaas-vm-manage-recovery-access) орындаңыз.

## {heading(4. ВМ-ге қосылыңыз)[id=iaas-vm-connect-nix-os]}

{tabs}

{tab(Linux/macOS)}

1. (Қосымша) Кілт файлына қол жеткізу құқықтарын баптаңыз:

   ```console
   chmod 400 <ПУТЬ_К_КЛЮЧУ>
   ```

1. Терминалда команданы орындаңыз.

    - IP-мекенжай арқылы қосылу үшін:

      ```console
      ssh -i <ПУТЬ_К_КЛЮЧУ> <ИМЯ_ПОЛЬЗОВАТЕЛЯ>@<ВНЕШНИЙ_IP-АДРЕС_ВМ>
      ```

    - Толық домендік ат арқылы қосылу үшін:

      ```console
      ssh -i <ПУТЬ_К_КЛЮЧУ> <ИМЯ_ПОЛЬЗОВАТЕЛЯ>@<ПОЛНОЕ_ДОМЕННОЕ_ИМЯ_ВМ>
      ```

   {include(../../../../../../_includes/_ssh-warning.md)[tags=new-host]}

   {note:info}
   Bitrix {var(cloud)} бейнесінен жасалған виртуалды машинаға алғаш қосылғанда, консольде `root` пайдаланушысы үшін құпиясөзді өзгерту туралы сұрау пайда болады. Ағымдағы `bitrix` құпиясөзін көрсетіп, жаңа құпиясөзді екі рет енгізіңіз.
   {/note}

{/tab}

{tab(Windows 10/11)}

1. (Қосымша) Жеке кілт файлына қол жеткізу құқықтарын баптаңыз:

    1. `.pem` жеке кілт файлының қасиеттерін ашыңыз.
    1. **Қауіпсіздік** қойындысына өтіп, **Қосымша** түймесін басыңыз. **Қосымша қауіпсіздік параметрлері** терезесі ашылады.
    1. **Мұрагерлікті өшіру** түймесін басып, ашылған терезеде **Осы нысаннан мұраланған барлық рұқсаттарды жою** тармағын таңдаңыз.
    1. **Қосымша қауіпсіздік параметрлері** терезесінде **Қосу** түймесін басыңыз.
    1. Ашылған терезеде **Субъектіні таңдаңыз** түймесін басып, ВМ-ге қосылу үшін пәрмен жолы іске қосылатын Windows ОЖ пайдаланушысын көрсетіңіз.
    1. Ашық тұрған баптау терезелерінде **ОК** түймесін кезекпен басып, өзгерістерді растаңыз.

1. Пәрмен жолында команданы орындаңыз.

    - IP-мекенжай арқылы қосылу үшін:

      ```console
      ssh -i <ПУТЬ_К_КЛЮЧУ> <ИМЯ_ПОЛЬЗОВАТЕЛЯ>@<ВНЕШНИЙ_IP-АДРЕС_ВМ>
      ```

    - Толық домендік ат арқылы қосылу үшін:

      ```console
      ssh -i <ПУТЬ_К_КЛЮЧУ> <ИМЯ_ПОЛЬЗОВАТЕЛЯ>@<ПОЛНОЕ_ДОМЕННОЕ_ИМЯ_ВМ>
      ```

   {include(../../../../../../_includes/_ssh-warning.md)[tags=new-host]}

   {note:info}
   Bitrix {var(cloud)} бейнесінен жасалған виртуалды машинаға алғаш қосылғанда, консольде `root` пайдаланушысы үшін құпиясөзді өзгерту туралы сұрау пайда болады. Ағымдағы `bitrix` құпиясөзін көрсетіп, жаңа құпиясөзді екі рет енгізіңіз.
   {/note}

{/tab}

{tab(Windows 7/8)}

Windows 7/8 жүйесінде SSH арқылы қосылу үшін PuTTY қолданбасын пайдаланыңыз.

1. PuTTY-ді [орнатыңыз](https://www.putty.org/).
1. PuTTY Private Key Files форматында жеке кілт файлын жасаңыз:

    1. PuTTY жиынтығына кіретін PuTTYgen қолданбасын іске қосыңыз.
    1. **Conversions** мәзірінде **Import Key** түймесін басып, ашылған терезеде `.pem` жеке кілт файлын таңдаңыз.
    1. **Save private key** түймесін басыңыз.
    1. Ашылған терезеде директорияны таңдаңыз, файл атауын енгізіңіз, **Файл түрі** өрісінде `.ppk` кеңейтімі көрсетілгенін тексеріп, **Сақтау** түймесін басыңыз.

1. `.ppk` жеке кілт файлын Pageant ішіне қосыңыз:

    1. PuTTY жиынтығына кіретін Pageant қолданбасын іске қосыңыз.
    1. Тапсырмалар жолағындағы Pageant белгішесін тінтуірдің оң жақ түймесімен басыңыз.
    1. Контекстік мәзірден **Add key** тармағын таңдаңыз.
    1. Алдыңғы қадамда жасалған `.ppk` жеке кілт файлын таңдаңыз.

1. PuTTY-ді іске қосыңыз:

    1. **Host Name (or IP address)** өрісіне виртуалды машинаның сыртқы IP-мекенжайын немесе оның толық домендік атын енгізіңіз. `22` портын және `SSH` байланыс түрін көрсетіңіз.
    1. Сол жақтағы ағашта **Connection → SSH → Auth** тармағын ашыңыз.
    1. **Allow agent forwarding** жалаушасын орнатыңыз.
    1. **Private key file for authentication** өрісінде `.ppk` жеке кілт файлын таңдаңыз.
    1. **Open** түймесін басыңыз.
    1. Егер бұл ВМ-ге алғашқы қосылу болса, пайда болған терезеде оны растаңыз.
    1. Ашылған терминал терезесінде ОЖ пайдаланушысының атын енгізуге шақыру пайда болады.
    1. ВМ операциялық жүйесінің пайдаланушы атын енгізіңіз. Нәтижесінде сервермен қосылым орнатылуы тиіс.

       {note:info}
       Bitrix {var(cloud)} бейнесінен жасалған виртуалды машинаға алғаш қосылғанда, консольде `root` пайдаланушысы үшін құпиясөзді өзгерту туралы сұрау пайда болады. Ағымдағы `bitrix` құпиясөзін көрсетіп, жаңа құпиясөзді екі рет енгізіңіз.
       {/note}

{/tab}

{/tabs}
