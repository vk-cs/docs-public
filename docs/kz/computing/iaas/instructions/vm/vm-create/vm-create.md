Cloud Servers ішінде виртуалды машиналарды жеке кабинет, OpenStack CLI немесе Terraform арқылы жасауға болады. Terraform көмегімен виртуалды машина жасау үшін [Terraform бөліміндегі нұсқаулықты](/kz/tools-for-using-services/terraform/how-to-guides/iaas/create) пайдаланыңыз.

VK Cloud командасы кейбір ОЖ образдарын дайындады. Дайын образдардың тізімі жеке кабинеттегі [жаңа виртуалды машина жасау терезесінде](https://kz.cloud.vk.com/app/services/infra/servers/add) қолжетімді. Сондай-ақ ОЖ образын өзіңіз [импорттай](../../images/images-manage#obrazdy_importtau) аласыз.

{note:info}

ВМ баптауға қолжетімді параметрлер әртүрлі операциялық жүйелер үшін өзгеше болуы мүмкін.

{/note}

ВМ жасау үшін:

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. Шот балансының оң екенін және виртуалды машинаның қажетті конфигурациясын жасау үшін [квоталардың](/kz/tools-for-using-services/account/concepts/quotasandlimits) жеткілікті екенін тексеріңіз.
1. **Бұлтты есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. **Инстанс құру** немесе **Қосу** түймесін басыңыз.
1. **Конфигурация** блогында ВМ параметрлерін орнатыңыз:

    - **Операциялық жүйе**: [операциялық жүйе нұсқасын](../../../concepts/oper-system) немесе бұрын VK Cloud ішінде [жасаған](../../images/images-manage#obrazdy_zhasau) не [импорттаған](../../images/images-manage#obrazdy_importtau) образды таңдаңыз.
    - **Виртуалды машина санаты**: ВМ-нің алдын ала орнатылған конфигурацияларының [санатын](../../../concepts/vm/flavor#vm_categories) таңдаңыз.
    - **Виртуалды машина түрі**: ВМ-нің алдын ала орнатылған конфигурациясын таңдаңыз.

      Графикалық үдеткіші бар ВМ жасау үшін [Cloud GPU конфигурация шаблоны](/kz/computing/gpu/concepts/about) қажет. Егер тізімде қажетті шаблон болмаса, оны [сұратыңыз](/kz/computing/gpu/connect).

    - **Қолжетімділік аймағы**: ВМ іске қосылатын [қолжетімділік аймағын](/kz/computing/iaas/concepts/avail-zone) таңдаңыз.
    - **Конфигурациядағы машиналар саны**: қажет ВМ санын көрсетіңіз.

1. **Жалпы ақпарат** блогын толтырыңыз:

    - **Виртуалды машина атауы**: тек латын әріптерін, сандарды немесе `-`, `_` және `.` арнайы таңбаларын пайдаланыңыз.
    - **Тегтер**: қажет болса, ВМ үшін [тегті көрсетіңіз](../vm-manage#tegterdi_tagayyndau) немесе жаңасын жасаңыз.
    - **ВМ іске қосылғанда скрипттерді баптау**: бірінші іске қосу кезінде орындалатын bash-скриптті немесе [cloud-config](https://cloudinit.readthedocs.io/en/latest/reference/examples.html) сценарийін қосу үшін опцияны қосыңыз.

1. **Диск өлшемі және жылдамдығы** блогында диск параметрлерін көрсетіңіз.

    - **Диск өлшемі**: ВМ дискінің қажетті өлшемін гигабайтпен көрсетіңіз.

      Дискінің максималды өлшемі [шектелген](/kz/tools-for-using-services/account/concepts/quotasandlimits#no_quotas_limits). Көлемі үлкенірек дискісі бар ВМ жасау үшін [OpenStack CLI](/kz/tools-for-using-services/cli/openstack-cli) пайдаланыңыз.

    - **Диск түрі**: [диск түрін](../../../concepts/data-storage/disk-types#disk_types) таңдаңыз.

1. **Желі және firewall баптаулары** блогында желі баптауларын орнатыңыз.

    - **Желі**: бар желіні таңдаңыз немесе жаңасын жасаңыз. Жаңа желі жасалған кезде қосымша өрістер пайда болады:

        - **SDN**: [виртуалды желілерді басқару жүйесін](/kz/networks/vnet/concepts/sdn) таңдаңыз. Жаңа аккаунттар үшін опция қолжетімсіз және әдепкі бойынша [Sprut](/kz/networks/vnet/concepts/sdn#sprut) пайдаланылады.

         {note:info}

          Әртүрлі SDN-де жасалған желілер бір-біріне қолжетімсіз, бірақ оларды [Floating IP-адрестер](/kz/networks/vnet/concepts/ips-and-inet#floating-ip) арқылы байланыстыра аласыз.

         {/note}

        - **Ішкі желі адресі**: `XXX.XXX.XXX.XXX/XX` форматындағы CIDR-ды көрсетіңіз, мысалы, `10.0.0.0/24`.

    - **Конфигурациялық дискіні пайдалану**: егер желіде DHCP сервері болмаған кезде виртуалды машинада желіні баптау қажет болса, опцияны қосыңыз. Егер [сыртқы желі](/kz/networks/vnet/concepts/net-types#external_net) немесе [DHCP](/kz/networks/vnet/concepts/ips-and-inet#zhelilik_adresteu) өшірілген желі таңдалса, опция автоматты түрде қосылады.
    - **DNS атауы**: [жеке DNS](/kz/networks/dns/instructions/private-dns) үшін домендік атауды көрсетіңіз. Ол арқылы ішкі желіде инстанстарға жүгінуге болады.
    - **Виртуалды машина кілті**: SSH арқылы қосылуға арналған кілтті таңдаңыз немесе жаңасын жасаңыз.

      `Жаңа кілт жасау` таңдалса, ВМ жасалғаннан кейін `*.pem` жеке кілт файлы автоматты түрде жүктеледі. Оны сақтаңыз, кейін SSH арқылы серверге қосылу үшін қажет болады.

      Кілттің ашық бөлігі серверге автоматты түрде қосылады.

    - **Firewall баптаулары**: қажетті қауіпсіздік топтарын таңдаңыз.

      SSH арқылы қосылу үшін `ssh` немесе `ssh+www` тобын қосыңыз. Желілік қолжетімділік ережелерін баптау туралы толығырақ [Қауіпсіздік топтары](/kz/networks/vnet/instructions/secgroups) бөлімінде.

    - (Опционалды) **Сыртқы IP тағайындау**: ВМ үшін жаңа немесе бар [Floating IP-адресті](/kz/networks/vnet/concepts/ips-and-inet#floating-ip) тағайындау үшін опцияны қосыңыз. Егер [желі](/kz/networks/vnet/concepts/net-types#standartty_zheli) үшін интернетке қолжетімділік қосылған болса, опция қолжетімді болады.

      {note:info}

      Егер бұрын ВМ үшін [сыртқы желі](/kz/networks/vnet/concepts/net-types#external_net) таңдалған болса, опция әрқашан қосулы болады.

      {/note}

    - (Опционалды) **Доменді сыртқы IP-ге байланыстыру**: опцияны қосып, FQDN-ді сыртқы IP-адреске байланыстыру үшін доменді көрсетіңіз. DNS аймағына таңдалған IP-адрес үшін [A-жазба](/kz/networks/dns/instructions/publicdns/records#add) қосылады. Егер **Сыртқы IP тағайындау** опциясы қосылған болса, опция қолжетімді болады.

      {note:warn}

      DNS аймағының CNAME-жазбасымен сәйкес келетін доменге сыртқы IP-адресті байланыстыруға болмайды. Бұл жағдайда ВМ жасалады, бірақ A-жазба қосылмайды.

      {/note}

    - **Мониторингті қосу**: [мониторинг сервисіне](/kz/monitoring-services/monitoring/concepts) метрикаларды жіберу үшін агент орнату мақсатында опцияны қосыңыз.

1. **Резервтік көшіру** блогында қажетті параметрлерді көрсетіңіз.

   Әдепкі бойынша резервтік көшіру қосулы және оған ұсынылатын баптаулар көрсетілген, бірақ оларды өзгерте аласыз:

    - **Жоспар атауы**: резервтік көшіру жоспары үшін еркін атау көрсетіңіз.
    - **Толық бэкаптарды сақтау стратегиясын (GFS) қосу**: сақтау мерзімі аяқталғаннан кейін резервтік көшірмелер автоматты түрде жойылуы үшін опцияны қосыңыз.

        - **Апталық толық бэкаптарды сақтау**: резервтік көшірмені қанша апта сақтау керегін көрсетіңіз.
        - **Айлық толық бэкаптарды сақтау**: резервтік көшірмені қанша ай сақтау керегін көрсетіңіз.
        - **Жылдық толық бэкаптарды сақтау**: резервтік көшірмені қанша жыл сақтау керегін көрсетіңіз.

      {note:info}

      Егер **Толық бэкаптарды сақтау стратегиясын (GFS) қосу** опциясы өшірілген болса, тек ең соңғы резервтік көшірмелер сақталады, олардың саны **Макс. толық бэкаптар саны** өрісінде көрсетіледі. Көрсетілген шекке жеткенде, ескі резервтік көшірмелер автоматты түрде жойылады.

      {/note}

    - **Инкременттік бэкаптарды қосу**: толық резервтік көшірме аптасына бір рет жасалуы үшін опцияны қосыңыз. Аптаның қалған барлық күндерінде инкременттік резервтік көшірмелер жасалады.
    - **Резервтік көшіру кестесі**: аптаның күндері мен резервтік көшіруді іске қосу уақытын таңдаңыз. Уақыт GMT+03:00 уақыт аймағында көрсетіледі.
    - **Жойылмайтын бэкаптарды қосу**: резервтік көшірмелерді [объектілерді бұғаттау](../../../../../storage/s3/concepts/objects-lock) арқылы жойылудан қорғау үшін опцияны қосыңыз. Бұғаттаудың екі түрі қолжетімді:

        - `Уақытша қорғау`: көшірмелерді белгіленген күн саны бойы жоюдан және қайта жазудан қорғайды. Екі режимі бар:
            - `Compliance`: қатаң бұғаттау. Көрсетілген сақтау кезеңі ішінде алынбайды және өзгертілмейді.
            - `Governance`: жоба әкімшісі алып тастай алатын икемді бұғаттау.
        - `Мерзімсіз қорғау`: бұғаттау алынғанға дейін көшірмені жоюдан қорғайды.

1. **Құру** түймесін басыңыз.

1. ВМ жасалғанша күтіңіз. Бұл процесс біраз уақыт алуы мүмкін. Құру аяқталған кезде ВМ сипаттамалары мен қосылу нұсқаулығы бар бет ашылады.

{/tab}

{tab(OpenStack CLI)}

1. OpenStack CLI-пен жұмыс істеуге дайындалыңыз:

    1. Екі факторлы аутентификацияны [қосыңыз](/kz/tools-for-using-services/vk-cloud-account/instructions/account-manage/manage-2fa).
    1. Ағымдағы пайдаланушы үшін API арқылы қолжетімділікті [белсендіріңіз](/kz/tools-for-using-services/api/rest-api/enable-api).
    1. OpenStack клиентінің [орнатылғанына](/kz/tools-for-using-services/cli/openstack-cli#1_openstack_klientin_ornatynyz) көз жеткізіп, жобада [аутентификациядан өтіңіз](/kz/tools-for-using-services/cli/openstack-cli#3_autentifikaciyadan_otiniz).

1. Деректерді жинаңыз:

    1. Қолжетімді ВМ түрлерінің тізімін алып, қажетті идентификаторды сақтаңыз:

      ```console
      openstack flavor list
      ```

    1. Қолжетімді ВМ образдарының тізімін алып, қажетті идентификаторды сақтаңыз:

      ```console
      openstack image list
      ```

    1. Қауіпсіздік топтарының тізімін алыңыз:

      ```console
      openstack security group list
      ```

        - Linux ВМ жасап, оған SSH арқылы қосылу үшін `ssh` немесе `ssh+www` тобының идентификаторын сақтаңыз.
        - Windows ВМ жасап, оған RDP арқылы қосылу үшін `rdp` немесе `rdp+www` тобының идентификаторын сақтаңыз.

       Желілік қолжетімділік ережелерін баптау туралы толығырақ [Қауіпсіздік топтары](/kz/networks/vnet/instructions/secgroups) бөлімінде.

    1. Қолжетімді желілердің тізімін алып, қажетті идентификаторды сақтаңыз:

      ```console
      openstack network list
      ```

        - Егер `ext-net` желісі таңдалса, виртуалды машинаға сыртқы IP-адрес автоматты түрде тағайындалады.
        - Егер жеке желі таңдалса, виртуалды машинаға жасалғаннан кейін [Floating IP-адресті](/kz/networks/vnet/instructions/ip/floating-ip) тағайындауға болады.

    1. Қолжетімді кілттер жұбының тізімін алып, қажетті кілттер жұбының атауын сақтаңыз:

      ```console
      openstack keypair list
      ```

       Жаңа кілттер жұбын жасау үшін:
        1. Кілтті жасаңыз:

            ```console
            ssh-keygen -t rsa -b 2048 -f ~/.ssh/my_key -N ""
            ```

        1. Кілтті жүктеңіз:

            ```console
            openstack keypair create --public-key ~/.ssh/my_key.pub --type ssh <ИМЯ_КЛЮЧЕВОЙ_ПАРЫ>
            ```

1. Жүктеу дискін жасаңыз:

   ```console
   openstack volume create root-volume --size <ОБЪЕМ_ДИСКА> --image <ID_ОБРАЗА> --availability-zone MS1 --bootable
   ```

   Мұнда:

    - `<ОБЪЕМ_ДИСКА>` — диск өлшемі ГБ.

      Максималды өлшем шектеулі. Толығырақ — [Квоталар мен лимиттер](/kz/tools-for-using-services/account/concepts/quotasandlimits#no_quotas_limits) бөлімінде.

    - `<ID_ОБРАЗА>` — бұрын алынған образ идентификаторы.


1. ВМ жасаңыз:

   ```console
   openstack server create <ИМЯ_ВМ> \
                           --volume <ID_ДИСКА> \
                           --network <ID_СЕТИ> \
                           --flavor <ID_ШАБЛОНА_КОНФИГУРАЦИИ> \
                           --key-name <ИМЯ_КЛЮЧЕВОЙ_ПАРЫ> \
                           --availability-zone MS1
   ```

   {note:warn}

   Егер `--network` опциясында сыртқы желі (`ext-net`) көрсетілсе, ВМ жасау командасына `--use-config-drive` опциясын қосыңыз.

   {/note}

   Виртуалды машина жасалғаннан кейін ол туралы ақпарат көрсетіледі. `adminPass` өрісін тауып, оның мәнін сақтаңыз. Ол серверге VNC консолі арқылы кіру үшін қажет болады.

1. Жасалған ВМ күйін тексеріңіз:

   ```console
   openstack server list
   ```

   Жасалған машина қолжетімді ВМ тізімінде пайда болып, `ACTIVE` мәртебесіне ие болуы тиіс.

{note:info}

CLI арқылы ВМ жасаудың кейбір ерекшеліктері:

- Егер қолжетімділік аймағы көрсетілмесе, кездейсоқ аймақ таңдалады.
- Егер диск көрсетілмесе, «эфемерлік диск» жасалады (шектеулерімен).

{/note}
