Виртуалды машинаны желіге порт арқылы қосуға болады. Порт — ВМ-нің виртуалды желілік картасы, ол үшін келесі мәндер бапталады:

- қауіпсіздік топтары;
- қосылған желі;
- DNS атауы.

Желілер мен порттар туралы толығырақ [Желілер мен ішкі желілерді басқару](/kz/networks/vnet/instructions/net) және [Порттарды басқару](/kz/networks/vnet/instructions/ports) бөлімдерінен қараңыз.

## ВМ-ге желіні қосу

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. Қажетті ВМ-ні таңдап, **Желілер** қойындысына өтіңіз.
1. **Қосылым қосу** түймесін басыңыз.
1. Ашылған терезеде:

    1. **Атауы**: желі атауын көрсетіңіз.
    1. **Қосу үшін желі**: тізімнен мәнді таңдаңыз:

        - **Жаңа желі құру**: жаңа желіні құру туралы толығырақ [Желілер мен ішкі желілерді басқару](/kz/networks/vnet/instructions/net) бөлімінен қараңыз.
        - **Сыртқы желі (ext-net)**: виртуалды машинаға IP-мекенжай автоматты түрде тағайындалады.
        - Бар желі.

       Бар желіні таңдаған кезде келесі параметрлерді орнатыңыз:

        - **Сыртқы IP тағайындау**: ВМ-ге интернет арқылы қол жеткізу қажет болса, қосыңыз.
        - **DNS атауы**: [приват DNS](/kz/networks/dns/instructions/private-dns) арқылы ВМ-ге жүгінуге болатын атау.
        - **IP-мекенжайды көрсету**: ішкі желі адрестер пулынан нақты IP-мекенжайды көрсету үшін қосыңыз.

    1. **Firewall баптаулары**: қажетті қауіпсіздік топтарын көрсетіңіз. Толығырақ [Қауіпсіздік топтары](/kz/networks/vnet/instructions/secgroups) бөлімінен қараңыз.

1. **Сақтау** түймесін басыңыз.

{/tab}

{tab(OpenStack CLI)}

OpenStack клиенті [орнатылғанына](/kz/tools-for-using-services/cli/openstack-cli#1_ustanovite_klient_openstack) көз жеткізіңіз және жобаға [аутентификациядан өтіңіз](/kz/tools-for-using-services/cli/openstack-cli#3_proydite_autentifikaciyu).

### Бар портқа қосылу

1. Желіге қосу жоспарланған виртуалды машинаның идентификаторын алыңыз:

   ```console
   openstack server list
   ```

1. Желілердің және олардың құрамындағы ішкі желілердің тізімін алыңыз:

   ```console
   openstack network list
   ```

1. Қажетті желідегі қолжетімді порттардың тізімін алыңыз:

   ```console
   openstack port list --network <ИМЯ_ИЛИ_ID_СЕТИ>
   ```

1. Таңдалған портты ВМ-ге қосыңыз:

   ```console
   openstack server add port <ID_ВМ> <ID_ПОРТА>
   ```

1. Порттың ВМ-ге сәтті қосылғанына көз жеткізіңіз:

   ```console
   openstack port list --server <ID_ВМ>
   ```

### Жаңа порт құру

1. Желілердің және олардың құрамындағы ішкі желілердің тізімін алыңыз:

   ```console
   openstack network list
   ```

1. Қосылу жоспарланған ішкі желі үшін IP-мекенжайлар тізімін алыңыз:

   ```console
   openstack subnet list --network <ИМЯ_ИЛИ_ID_СЕТИ>
   ```

1. Қауіпсіздік топтарының тізімін алыңыз:

   ```console
   openstack security group list
   ```

1. Портты төмендегі тәсілдердің бірімен құрыңыз:

    - Қажетті желіде және `default` қауіпсіздік тобымен:

      ```console
      openstack port create <ИМЯ_ПОРТА> --network <ИМЯ_ИЛИ_ID_СЕТИ>
      ```

    - Параметрлерді көрсете отырып:

      ```console
      openstack port create <ИМЯ_ПОРТА> \
                            --network <ИМЯ_ИЛИ_ID_СЕТИ> \
                            --mac-address <MAC-АДРЕС> \
                            --fixed-ip subnet=<ИМЯ_ИЛИ_ID_ПОДСЕТИ>,ip-address=<IP-АДРЕС_ПОРТА> \
                            --security-group <ID_ГРУППЫ_БЕЗОПАСНОСТИ>
      ```

      Қолдау көрсетілетін параметрлердің толық тізімін көру үшін команданы орындаңыз:

      ```console
      openstack port create --help
      ```

      Сондай-ақ портты [DHCP опцияларымен](https://github.com/Juniper/contrail-controller/wiki/Extra-DHCP-Options) құруға болады.

      Қажет болған жағдайда портқа [Floating IP-мекенжайын тағайындаңыз](/kz/networks/vnet/instructions/ip/floating-ip).

   Нәтижесінде құрылған порттың идентификаторы және ол туралы басқа ақпарат шығарылады.

1. Желіге қосу жоспарланған виртуалды машинаның идентификаторын алыңыз:

   ```console
   openstack server list
   ```

1. Құрылған портты ВМ-ге қосыңыз:

   ```console
   openstack server add port <ID_ВМ> <ID_ПОРТА>
   ```

1. Порттың ВМ-ге сәтті қосылғанына көз жеткізіңіз:

   ```console
   openstack port list --server <ID_ВМ>
   ```

## ВМ желісін жою

{tabs}

{tab(Жеке кабинет)}

1. [Өтіңіз](https://kz.cloud.vk.com/app/) VK Cloud жеке кабинетіне.
1. **Бұлттық есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. Қажетті ВМ-ні таңдап, **Желілер** қойындысына өтіңіз.
1. Қажетті желі үшін төмендегілердің бірін орындаңыз:

    - Желі жалаушасын белгілеңіз, содан кейін **Жою** түймесін басыңыз.
    - Қосылған желі үшін ![ ](/kz/assets/more-icon.svg "inline") белгішесін басып, **Қосылымды жою** тармағын таңдаңыз.

1. Ашылған терезеде **Растау** түймесін басыңыз.

ВМ-ге желі қосылымы жойылады, бірақ бұрын жасалған порт **Қосылмаған** күйінде қалады. Порттарды жою туралы толығырақ [Порттарды басқару](/kz/networks/vnet/instructions/ports) бөлімінен қараңыз.

{/tab}

{tab(OpenStack CLI)}

1. OpenStack клиенті [орнатылғанына](/kz/tools-for-using-services/cli/openstack-cli#1_ustanovite_klient_openstack) көз жеткізіңіз және жобаға [аутентификациядан өтіңіз](/kz/tools-for-using-services/cli/openstack-cli#3_proydite_autentifikaciyu).
1. Виртуалды машинаның ID мәнін алыңыз:

   ```console
   openstack server list
   ```

1. ВМ-дегі порттар тізімін алыңыз:

   ```console
   openstack port list --server <ID_ВМ>
   ```

1. Қажет емес портты жойыңыз:

   ```console
   openstack port delete <ИМЯ_ИЛИ_ID_ПОРТА>
   ```

{/tab}
