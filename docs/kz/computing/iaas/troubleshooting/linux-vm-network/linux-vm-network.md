# {heading(Виртуалды машинаға қосылу мүмкін емес)[id=iaas-linux-vm-network]}

{include(/kz/_includes/_translated_by_ai.md)}

Бұл нұсқаулықта виртуалды машиналардағы желі конфигурациясы мәселелерін диагностикалау және жою бойынша ұсынымдар берілген.

Мысал ретінде мыналар болжанады:

- виртуалды машинада тек бір желілік интерфейс бар;
- виртуалды машинаға SSH арқылы қол жеткізудің жоқтығы мәселесі шешілуде.

Сипатталған қадамдарды басқа қолданбаға, мысалы NGINX веб-серверіне қолжетімділік жоғалғанда немесе виртуалды машинада бірнеше желілік интерфейс болғанда да пайдалануға болады.

## {heading(Жұмысты бастамас бұрын)[id=iaas-linux-vm-network]}

1. Мәжбүрлі қайта жүктеуді орындаңыз.

   Бұл операция виртуалды машина жауап бермесе немесе оның желілік интерфейсі қате инициализацияланса көмектесуі мүмкін:

   1. [Тоқтатыңыз](../../instructions/vm/vm-manage#iaas-vm-manage-start-stop-restart) виртуалды машинаны.
   1. Тоқтатылған виртуалды машина үшін [орындаңыз](../../instructions/vm/vm-manage#iaas-vm-manage-restart) мәжбүрлі қайта жүктеуді.

   Егер бұл мәселені шешпесе, қалған қадамдарды орындап, диагностикаға көшіңіз.

1. [Көз жеткізіңіз](../../instructions/vm/vm-manage#iaas-vm-manage-start-stop-restart), виртуалды машина іске қосылғанына.
1. [Көз жеткізіңіз](../../instructions/vm/vm-console#vm-console-vnc), виртуалды машинаның VNC-консоліне қол жеткізіп, онда авторизациялана алатыныңызға.

   Қажет болса, логин үшін [құпиясөзді қалпына келтіріңіз](../../instructions/vm/vm-manage#iaas-vm-manage-password-recovery).

## {heading(1. Желілік интерфейс баптауларын тексеріңіз)[id=iaas-linux-vm-network]}

Кейде қосылуға виртуалды машинаның желілік интерфейсінің қате инициализациясы немесе оның дұрыс емес баптаулары кедергі келтіреді. [Желілік интерфейс баптауларын](/kz/computing/iaas/how-to-guides/interface-settings-check) тексеріңіз.

## {heading(2. Қажетті қолданбалардың виртуалды машинада іске қосылғанын тексеріңіз)[id=iaas-linux-vm-network]}

Егер желілік интерфейс дұрыс бапталған болса, бірақ қолжетімділік болмаса, виртуалды машинада қажетті қолданбалар мен сервистердің, мысалы SSH-тың, іске қосылғанын тексеріңіз. Олар іске қосылмаған болуы немесе стандартты емес портта жұмыс істеуі мүмкін. 

SSH жұмысын тексеріңіз:

1. Виртуалды машинаның консоліне қосылып, авторизацияланыңыз.
1. SSH сервисінің іске қосылғанын тексеріңіз:

   ```console
   sudo systemctl status ssh
   ```

   Команда шығысы бойынша сервистің іске қосылған-қосылмағанын анықтаңыз:

   - `Active: active (running)`: SSH сервисі іске қосылған.
   - `Active: inactive (dead)`: SSH сервисі іске қосылмаған.

1. SSH сервисінің күйіне байланысты келесі әрекеттерді орындаңыз:

   {tabs}

   {tab(Егер SSH сервисі іске қосылған болса)}

   1. SSH сервисі қай портта жұмыс істейтінін анықтаңыз:

      ```console
      sudo cat /etc/ssh/sshd_config | grep -w Port
      ```

      Шығыста порт нөмірі болады, мысалы `Port 22`.

      Егер сервис стандартты `22` портында жұмыс істесе — [ВМ файерволы баптауларын тексеруге өтіңіз](). Әйтпесе келесі қадамға өтіңіз.

   1. Порт нөмірін көрсетіп, ВМ-ге қосылыңыз:

      ```console
      ssh -i <ПУТЬ_К_КЛЮЧУ> <ИМЯ_ПОЛЬЗОВАТЕЛЯ>@<ВНЕШНИЙ_IP-АДРЕС_ВМ> -p <ПОРТ>
      ```
   1. Виртуалды машинаға SSH арқылы қолжетімділіктің бар-жоғын тексеріңіз. Қолжетімділік болмаса, [ВМ файерволы баптауларын тексеруге өтіңіз]().

   {/tab}

   {tab(Егер SSH сервисі іске қосылмаған болса)}

   1. SSH сервисі қай портта жұмыс істейтінін анықтаңыз:

      ```console
      sudo cat /etc/ssh/sshd_config | grep -w Port
      ```

      Шығыста порт нөмірі болады, мысалы `Port 22`.

   1. Басқа процестердің бұл портты пайдаланбайтынына көз жеткізіңіз. SSH сервисінің логтарын қараңыз:

      ```console
      sudo journalctl -xeu ssh
      ```

      Егер логтарда осындай жол болса, онда SSH сервисінің портын басқа процесс пайдаланып тұр:

      ```text
      error: Bind to port 22 on 0.0.0.0 failed: Address already in use.
      ```

   1. Қай процесс портты алып қойғанын анықтаңыз:

      1. `netstat` утилитасын орнатыңыз:

         {tabs}
         {tab(Debian, Ubuntu)}

         ```console
         sudo apt install net-tools -y
         ```

         {/tab}
         {tab(AlmaLinux, CentOS)}

         ```console
         sudo yum install net-tools -y
         ```

         {/tab}
         {/tabs}

      1. Команданы орындаңыз:

         ```console
         sudo netstat -plntu | grep :22
         ```

         Бұл шығыс мысалында `22` портын PID `1234` бар `some-other-service` процесі пайдаланып тұр:

         ```text
         tcp   0   0 0.0.0.0:22   0.0.0.0:*  LISTEN   1234/some-other-service
         tcp6  0   0 :::22        :::*       LISTEN   1234/some-other-service
         ```

   1. SSH портын пайдаланып тұрған процесті тоқтатыңыз:

      - не `systemctl` арқылы:

          ```console
          sudo systemctl stop some-other-service
          ```

      - не процестің жұмысын мәжбүрлі түрде аяқтап:

         ```console
         sudo kill 1234
         ```

   1. Тоқтатылған процеске сәйкес сервистің баптауларын өзгертіңіз, ол `22` портынан өзгеше портты пайдалануы үшін.
   1. Сервисті қайта іске қосыңыз:

      ```console
      sudo systemctl restart some-other-service
      ```

   1. SSH сервисін қайта іске қосыңыз:

      ```console
      sudo systemctl restart sshd
      ```

   1. SSH сервисінің сәтті іске қосылғанына көз жеткізіңіз:

      ```console
      sudo systemctl status sshd
      ```

      {caption(Шығыс фрагментінің мысалы)[align=left;position=above]}
      ```text
      Active: active (running)
      ```
      {/caption}

      Егер SSH сервисі іске қосылмаса, `/etc/ssh/sshd_config` конфигурациялық файлындағы сервис баптауларын тексеріңіз.

      Сервистің жұмысындағы мәселелер туралы қосымша ақпарат алу үшін логтарды қараңыз:

      ```console
      sudo journalctl -xeu ssh
      ```

   1. Виртуалды машинаға SSH арқылы қолжетімділіктің бар-жоғын тексеріңіз. Қолжетімділік болмаса, [ВМ файерволы баптауларын тексеруге өтіңіз]().

   {/tab}

   {/tabs}

## {heading(3. Виртуалды машинаның файервол баптауларын тексеріңіз)[id=iaas-linux-vm-network]}

Егер виртуалды машинада файервол (`iptables`, `ufw`, `firewalld` сияқты) бапталған болса, ол қосылуға кедергі келтіруі мүмкін.

Төменде барлық трафикке рұқсат бере отырып, файерволдың барлық ережелерін уақытша өшіру жолы сипатталған. Бұл мәселенің дәл файерволда екенін анықтауға көмектеседі.

{note:warn}
Қосылу мәселесі локализацияланғаннан кейін, файервол баптауларын (қажетті түзетулермен) қайтарыңыз. Егер бүкіл трафикке рұқсат етілсе, виртуалды машинаның қауіпсіздігі төмендейді.
{/note}

Файервол баптауларын тексеру үшін:

1. Виртуалды машинаның консоліне қосылып, авторизацияланыңыз.
1. Файерволды өшіріңіз:

   {tabs}
   {tab(ufw)}

   ```console
   sudo ufw disable
   ```

   {/tab}
   {tab(firewalld)}

   ```console
   sudo systemctl stop firewalld
   ```

   {/tab}
   {tab(iptables)}

   1. Қолданыстағы `iptables` ережелерін сақтаңыз:

      ```console
      sudo iptables-save | sudo tee ~/iptables.rules
      ```

   1. Командаларды орындаңыз:

      ```console
      sudo iptables -P INPUT ACCEPT
      sudo iptables -P FORWARD ACCEPT
      sudo iptables -P OUTPUT ACCEPT
      sudo iptables -t nat -F
      sudo iptables -t mangle -F
      sudo iptables -F
      sudo iptables -X
      ```

   {/tab}
   {/tabs}

1. Виртуалды машинаға SSH арқылы қолжетімділіктің бар-жоғын тексеріңіз.

   Егер қолжетімділік пайда болса, файервол ережелерін түзетіп, оны қайта қосыңыз.

   Егер қолжетімділік пайда болмаса, файерволды қайта қосыңыз.

Файерволды қайта қосу үшін:

1. Виртуалды машинаның консоліне қосылып, авторизацияланыңыз.
1. Команданы орындаңыз:

   {tabs}
   {tab(ufw)}

   ```console
   sudo ufw enable
   ```

   {/tab}
   {tab(firewalld)}

   ```console
   sudo systemctl start firewalld
   ```

   {/tab}
   {tab(iptables)}

   ```console
   sudo iptables-restore < ~/iptables.rules
   ```

   {/tab}
   {/tabs}

## {heading(4. Қауіпсіздік топтарының баптауларын тексеріңіз)[id=iaas-linux-vm-network]}

Қате бапталған қауіпсіздік топтары, тіпті виртуалды машина деңгейінде кедергілер болмаса да, SSH арқылы қосылуға кедергі келтіруі мүмкін.

Төменде барлық трафикке рұқсат берілетіндей етіп, қауіпсіздік топтарының ережелерін уақытша баптау жолы сипатталған. Бұл мәселенің дәл қауіпсіздік топтарында екенін анықтауға көмектеседі.

{note:warn}
SSH арқылы қосылу мәселесі локализацияланғаннан кейін, қауіпсіздік топтарының ережелерін (қажетті түзетулермен) қайта баптаңыз. Егер бүкіл трафикке рұқсат етілсе, виртуалды машинаның қауіпсіздігі төмендейді.
{/note}

Қауіпсіздік топтарының баптауларын тексеру үшін:

1. {ifdef(public)}[Өтіңіз](https://kz.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cer)}{linkto(../../../../intro/authorization/lk_entry#prerequisites_vkc_ui)[text=Өтіңіз]}{/ifdef} {var(cloud)} жеке кабинетіне.
1. Қажетті виртуалды машина орналасқан жобаны таңдаңыз.
1. **Бұлтты есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. Қажетті виртуалды машинаның атауын басыңыз.
1. **Желілер** қойындысына өтіңіз.
1. Қажетті желілік қосылым үшін ![ ](/kz/assets/more-icon.svg "inline") батырмасын басып, **Қосылымды өңдеу** тармағын таңдаңыз.
1. **Firewall баптаулары** параметрінде:

   1. Барлық таңдалған қауіпсіздік топтарын жойыңыз.
   1. Ашылмалы тізімнен `default` және `all` («бәріне рұқсат етілген») қауіпсіздік топтарын таңдаңыз.

      `default` қауіпсіздік тобы кез келген шығыс трафикке рұқсат береді. `all` қауіпсіздік тобы кез келген кіріс трафикке рұқсат береді.

1. **Сақтау** батырмасын басыңыз.
1. Виртуалды машинаға SSH арқылы қолжетімділіктің бар-жоғын тексеріңіз.

   Егер қолжетімділік пайда болса, қауіпсіздік топтарын түзетіп, оларды `all` тобының орнына қайта қосыңыз.

   Егер қолжетімділік пайда болмаса, бастапқы баптауларға оралыңыз.

Қауіпсіздік топтарын қайта қосу үшін:

1. {ifdef(public)}[Өтіңіз](https://kz.cloud.vk.com/app/){/ifdef}{ifdef(private,private-pg,private-pdf,private-pg-pdf,private-cer)}{linkto(../../../../intro/authorization/lk_entry#prerequisites_vkc_ui)[text=Өтіңіз]}{/ifdef} {var(cloud)} жеке кабинетіне.
1. Қажетті виртуалды машина орналасқан жобаны таңдаңыз.
1. **Бұлтты есептеулер → Виртуалды машиналар** бөліміне өтіңіз.
1. Қажетті виртуалды машинаның атауын басыңыз.
1. **Желілер** қойындысына өтіңіз.
1. Қажетті желілік қосылым үшін ![ ](/kz/assets/more-icon.svg "inline") батырмасын басып, **Қосылымды өңдеу** тармағын таңдаңыз.
1. **Firewall баптаулары** параметрінде:

   1. `all` қауіпсіздік тобын жойыңыз.
   1. Ашылмалы тізімнен қажетті қауіпсіздік топтарын таңдаңыз.

      Егер таңдалған қауіпсіздік топтарында шығыс трафикке рұқсат беретін ереже болмаса, `default` қауіпсіздік тобын да таңдаңыз. Бұл топ шығыс трафикке рұқсат береді. Әйтпесе виртуалды машинаның желіге қолжетімділігі болмайды.

   {note:info}
   Стандартты `22` портында жұмыс істейтін SSH сервисінің трафигін өткізуі үшін `default` және `ssh` («тек ssh рұқсат етілген») топтарын таңдау жеткілікті.
   {/note}
