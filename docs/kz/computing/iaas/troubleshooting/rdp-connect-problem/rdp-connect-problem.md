# {heading(RDP арқылы ВМ-ге қосылу мәселесі)[id=iaas-rdp-connect-problem]}

{include(/kz/_includes/_translated_by_ai.md)}

[Виртуалды машинаға RDP арқылы қосылғанда](/kz/computing/iaas/instructions/vm/vm-connect/vm-connect-win) `Қашықтағы сеанс ажыратылды, себебі қашықтағы жұмыс үстелдерін лицензиялаудың қолжетімді серверлері жоқ` қатесі пайда болады. Осыдан кейін жұмыс сеансы аяқталады.

Windows Server жүйесінде әдепкі бойынша бір уақытта тек екі қашықтан қосылу сеансы қолжетімді. Бұл ретте бір есептік жазбамен [VNC-консольді](../../instructions/vm/vm-console) бір уақытта пайдаланып, ВМ-ге RDP арқылы қосылуға болмайды — кейінірек іске қосылған сеанс белсенді болып қалады.

Белсенді сеанстар санының лимитін арттыру үшін келесілер бапталып, белсендірілген болуы керек:

- қашықтағы жұмыс үстелі қызметтерінің рөлдері (Remote Desktop Services);
- қашықтағы жұмыс үстелі қызметтерін лицензиялау рөлдері (Remote Desktop Licensing);
- қосылуларға арналған лицензиялар пакеті.

### {heading(Шешім)[id=iaas-rdp-connect-problem-decision]}

Уақытша шешім ретінде ВМ-ге RDP арқылы әкімшілендіру режимінде қосылуға болады. Бұл белсенді сеанстар санының лимиті таусылған болса да мүмкін.

{note:warn}
Әкімшілендіру режиміндегі сеанс тек әкімші құқықтары бар есептік жазба үшін қолжетімді.
{/note}

Әкімшілендіру режиміндегі сеансқа қосылу үшін:

1. Келесі пәрменді орындаңыз:

   ```console
   mstsc /v:<IP-АДРЕС_ВМ> /admin
   ```
   Мұндағы `<IP-АДРЕС_ВМ>` — виртуалды машинаның IP мекенжайы немесе толық домендік атауы.

1. Ашылған терезеде есептік деректерді енгізіңіз.
1. Қосылуды растаңыз.
