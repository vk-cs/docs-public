## Виртуалды машиналар

{cut(Серверлер қайда орналасқан?)}

Серверлер РФ аумағында да, одан тыс жерлерде де орналасқан [VK Cloud бірнеше дата-орталықтарында](../concepts/avail-zone) орналасқан.

{/cut}

{cut(Қосылу жылдамдығы қандай?)}

ВМ үшін трафик бойынша шектеусіз 1 Гбит/с кіріс және шығыс байланыс арнасы беріледі, бірақ қосылудың нақты жылдамдығы бірнеше себепке байланысты төмен болуы мүмкін:

- Сыртқы факторлар. Мысалы, желілік маршруттағы кідірістер, пакеттердің жоғалуы, қосылу сапасы, сондай-ақ клиент жағындағы арнаның шектеулі өткізу қабілеті немесе жүктемесі.
- Қолданба немесе протокол деңгейіндегі шектеулер. Мысалы, ВМ-де жоғары жылдамдықтарға оңтайландырылмаған БҚ немесе деректерді беру протоколдары қолданылады.
- Сервердің немесе клиенттің есептеу ресурстарының жеткіліксіздігі (CPU, жад, диск). Мысалы, диск жылдамдығы қолжетімді жүктеу жылдамдығынан төмен.

{/cut}

{cut(ВМ үшін жаңа құпиясөзді қалай орнатуға болады?)}

[Нұсқаулықты](../instructions/vm/vm-manage#password) пайдаланыңыз.

{/cut}

{cut(Жоғары өнімді CPU қолжетімсіз)}

Жоғары өнімді CPU-ға қол жеткізу үшін [техникалық қолдауға](/kz/contacts) жүгініңіз.

{/cut}

{cut(Сыртқы IP мекенжайы қанша тұрады?)}

Floating IP мекенжайлары мен виртуалды машиналар порттарындағы IP мекенжайларының (`ext-net`) өзекті бағалары [прайс-парақта](https://cloud.vk.com/pricelist) орналастырылған.

{/cut}

{cut(Инстанс жасағым келеді, бірақ лайықты конфигурация жоқ)}

Егер ВМ жасау кезінде лайықты ВМ конфигурациясын таппасаңыз, [техникалық қолдауға](/kz/contacts) жүгініңіз.

CPU және RAM арақатынасы коэффициенті `1:1` немесе одан төмен болатын конфигурацияларды пайдалану ұсынылмайды. Мұндай конфигурацияларда өнімділіктің тар орындары болады және олар нақты міндеттерді орындау үшін, мысалы, машиналық оқыту немесе объектілерді тану үшін қолданылуы мүмкін.

{/cut}

{cut(ВМ жасау кезінде квоталар жетіспейді)}

Жобадағы ресурстарды босатыңыз немесе квоталарды [көбейтіңіз](/kz/tools-for-using-services/account/instructions/project-settings/manage#increase-quota).

{/cut}

{cut(Windows 7/8/10 жүйесімен ВМ жасай алмаймын)}

Windows 7/8/10 сияқты Windows тұқымдасына жататын клиенттік операциялық жүйелерді VK Cloud бұлтында пайдалану мүмкін емес. Бұл шектеу барлық жобалар үшін орнатылған және оны алып тастау мүмкін емес.

{/cut}

{cut(Инстанста графикалық элементтер нашар өңделеді)}

Виртуализация жүйесінде графиканы өңдеу үшін CPU ресурстары пайдаланылады, олар видеодрайвердің болуын қажет ететін графикалық элементтерді өңдеуге арналмаған, сондықтан сапасы ұқсас жергілікті құрылғылардан өзгеше болуы мүмкін.

{/cut}

{cut(Openstack CLI қосылмайды)}

Openstack CLI-ға конфигурация файлы арқылы қосылуға болады. Орнату, баптау және қосылу параметрлері туралы ақпарат [басқару утилиталары (CLI) бөлімінде](/kz/tools-for-using-services/cli/openstack-cli) келтірілген.

{/cut}

{cut(Виртуалды сервер консоліне қалай өтуге болады?)}

VNC консолі виртуалды машина бетінде **Бұлтты есептеулер → Виртуалды машиналар** бөлімінде қолжетімді. Толығырақ — [ВМ диагностикасы](../instructions/vm/vm-console#vnc_konsoli) мақаласында.

{/cut}

{cut(VK Cloud панелінде инстанс консолі көрсетілмейді)}

Браузердің соңғы нұсқасын пайдаланып жатқаныңызға көз жеткізіңіз. Қажет болса, кэшті тазалаңыз.

{note:info}

Консольде әдеттегі перне тіркесімдері, дыбысты беру және алмасу буфері қолжетімсіз.

{/note}

{/cut}

{cut(CPU немесе RAM көлемін ұлғайтуға бола ма?)}

Иә. Егер машина әлдеқашан жасалған болса, [оның түрін өзгертіңіз](../instructions/vm/vm-manage#vm_atauyn_ozgertu_zhne_tipin_ozgertu).

{/cut}

{cut(ВМ өнімділігін қалай бақылауға болады?)}

Жасалған ВМ бетінде **Мониторинг** қойындысында.

{/cut}

{cut(ВМ масштабтау қалай жүреді?)}

VK Cloud виртуалды машинасын масштабтау келесі кезеңдер арқылы өтеді:

1. ВМ тоқтатылады.
1. Виртуалды машинаға CPU, RAM немесе HDD қосылады.
1. ВМ қайта іске қосылады.

Биллинг итерациясы сағатына бір рет жүреді — осы уақыт ішінде ресурстар құнының есебі өзгереді.

{/cut}

{cut(ВМ үшін Floating IP мекенжайын тағайындай аламын ба?)}

Сіз ВМ үшін [қолданыстағы](/kz/networks/vnet/instructions/ip/floating-ip#associate) Floating IP мекенжайын тағайындай аласыз немесе [жаңасын](/kz/networks/vnet/instructions/ip/floating-ip#add) қоса аласыз.

{note:warn}

Жаңа Floating IP мекенжайы кездейсоқ түрде тағайындалады.

{/note}

{/cut}

{cut(VPS пен VDS арасындағы айырмашылық қандай?)}

Көзге көрінетін айырмашылық жоқ.

Провайдерлер бір кешенді қызметті ұсынады — VPS/VDS виртуалды серверін жалға алу. Веб-қызметтердің жұмысы үшін қанша және қандай ресурстар қажет екенін таңдаңыз: процессорлар саны, сақтау көлемі, жинақтауыш түрі, операциялық жүйелер және басқа параметрлер. Провайдер оларды сұраныс бойынша толық көлемде алуыңызды қамтамасыз етеді.

{/cut}

{cut(Виртуализация қалай іске асырылған?)}

Провайдер физикалық серверлерде клиенттердің көптеген ВМ-дері орналасқан виртуализация ортасын жаяды. ВМ бір-бірінен оқшауланған, клиенттер оларға қашықтан, шифрланған қосылымдар арқылы қол жеткізеді. VK Cloud виртуализацияны KVM + OpenStack негізінде өзінің жетілдірулерімен іске асырады.

{/cut}

{cut(Қосылғаннан кейін бұлтты сервердің конфигурациясын өзгертуге бола ма?)}

Иә, болады. Бұл процесс виртуалды машинаны қайта жүктеумен бірге жүреді.

{/cut}

{cut(Неліктен ВМ тоқтатылған болса да есептен шығару жалғасады?)}

Егер ВМ жұмысы тоқтатылса, онда келесі қызметтер үшін есептен шығару жалғасады:

- лицензияларды пайдалану (Windows және RDS, егер белсендірілген болса);
- дискілік кеңістікті жалға алу;
- бар резервтік көшірмелерді сақтау.

{/cut}

{cut(Бұлтты VPS/VDS серверлері бөлінген серверден (dedicated) несімен ерекшеленеді?)}

Пайдаланушы тұрғысынан бұлттағы серверлер бөлінген физикалық серверден еш айырмашылығы жоқ: сіз де root құқықтарын, желі баптауларына қолжетімділікті аласыз, файлдармен кез келген әрекеттерді орындай аласыз, кез келген қажетті бағдарламалық қамтамасыз етуді орнатып, баптай аласыз.

Қызмет құнын толық бақылауға алу үшін VPS/VDS жалға алған дұрыс. Ағымдағы қажеттіліктерге байланысты ВМ-дерді минуттар ішінде жасай немесе жоя аласыз, олардың қуатын тоқтатпай-ақ арттыра немесе азайта аласыз (даунтаймсыз). Бөлінген серверлерде бұлтты VPS/VDS сияқты икемділік жоқ, бұл ресурстардың толық пайдаланылмауына және шекті жүктемелер кезінде қолданбалардың істен шығу қаупіне әкеледі.

{/cut}

{cut(Виртуалды машиналарды қанша рет жасауға және жоюға болады?)}

Ресурстарды жасау және жою операцияларын шектеусіз рет орындауға болады.

{/cut}

{cut(Неліктен жоғары өнімді CPU үшін жиілік дұрыс көрсетілмейді?)}

Операциялық жүйедегі ақпарат виртуалды ядролардың нақты жиілігіне сәйкес келмейді. Бұл виртуализация ерекшелігіне байланысты. [Тестілеу кезінде](/kz/cases/performance-testing/cpu-testing) жоғары өнімді CPU-ға тән өнімділіктің артқанын көре аласыз.

{/cut}

{cut(ВМ желілік интерфейсіне нақты MAC мекенжайын қалай беру керек?)}

1. MAC мекенжайын көрсете отырып, [жаңа порт жасаңыз](/kz/networks/vnet/instructions/ports#portty_kosu):

   ```console
   openstack port create --network <ID_СЕТИ> <ИМЯ_ПОРТА> --mac-address <MAC-АДРЕС>
   ```

1. Жасалған портты ВМ-ге қосыңыз:

   ```console
   openstack server add port <ID_ВМ> <ID_ПОРТА>
   ```
   
1. ВМ операциялық жүйесінде [жаңа интерфейсті баптаңыз](/kz/computing/iaas/how-to-guides/interface-settings-check).

{/cut}

{cut(VK Cloud-тан жергілікті гипервизорға ВМ-ді қалай көшіруге болады?)}

1. ВМ [образын жасаңыз](/kz/computing/iaas/instructions/images/images-manage#obrazdy_zhasau).
1. Образды [экспорттаңыз](/kz/computing/iaas/instructions/images/images-manage#obrazdy_eksporttau).
1. Образды жергілікті гипервизорда виртуалды машина жасау үшін пайдаланыңыз.

{/cut}

{cut(Windows ОС бар ВМ-де аудио құрылғыларын қалай пайдалануға болады?)}

Виртуалды машиналарда дыбыстық карталар жоқ. Аудионы ойнату үшін дыбысты RDP протоколы арқылы компьютеріңізге қайта бағыттаңыз:

1. RDP клиентінің параметрлерін ашыңыз.
1. **Жергілікті ресурстар** қойындысына өтіңіз.
1. **Параметрлер** түймесін басыңыз.
1. **Осы компьютерде ойнату** опциясын таңдаңыз.

{/cut}

{cut(3D-стриминг және интерактивті қолданбалар үшін қандай сервистер қолайлы?)}

3D-графиканы рендерлеу және стриминг үшін [GPU бар ВМ](/kz/computing/gpu/concepts/about) қолайлы. Оңтайлы конфигурацияны таңдау үшін [сату бөліміне](https://cloud.vk.com/contacts/) жүгініңіз.

{/cut}

{cut(ВМ-ге ISO образын қосуға бола ма?)}

Иә, ВМ-ге ISO образын қоса аласыз.

ISO образдары туралы толығырақ — [Windows ОС бар ВМ образын жасау](/kz/computing/iaas/how-to-guides/win-image) бөлімінде.
{/cut}

{cut(Жаңа IP мекенжайымен ВМ көшірмесін қалай жасауға болады?)}

1. ВМ дискінің [снимогын жасаңыз](/kz/computing/iaas/instructions/volumes/volumes-snapshots).
1. Снимоктан [жаңа виртуалды машина жасаңыз](/kz/computing/iaas/instructions/volumes/volumes-snapshots#use_snapshots).

ВМ жасау кезінде оған жаңа IP мекенжайы тағайындалады.

{/cut}

{cut(ВМ жасау үшін жаңа операциялық жүйені қосуға бола ма?)}

Иә, қажетті ОС бар образды [импорттаңыз](/kz/computing/iaas/instructions/images/images-manage#obrazdy_importtau).

{/cut}

{cut(Неліктен маған квоталарды көбейтуден бас тартты?)}

Платформада пайдаланудың қатаң ережелері бар. Егер сіздің қызметіңіз трафикті бүркемелеумен, проксилеумен немесе заңнаманы бұзумен байланысты деген күдік туындаса, бұл квоталардың бұғатталуына әкелуі мүмкін.

{/cut}

{cut(ВМ-ді жойдым, неге қаражат есептен шығарылуы жалғасуда?)}

Виртуалды машинаны жою барлық байланысты ресурстардың жойылғанын білдірмейді. Дискілер, Floating IP мекенжайлары және резервтік көшірмелер бөлек тарифтеледі және ВМ жойылғаннан кейін жобада қала беруі мүмкін.

{/cut}

{cut(ВМ-ді қалпына келтіре аламын ба?)}

ВМ-ді, мысалы, файлдық жүйе істен шыққанда, қолжетімділік жоғалғанда, ОС жүктеушісі бұзылғанда қалпына келтіруге тырысуға болады.

ВМ-ді қалпына келтіру үшін [техникалық қолдауға](/kz/contacts) жүгініңіз.

{/cut}

## Дискілер мен образдар

{cut(Дискіні қалай үлкейтуге болады?)}

Дискіні VK Cloud панелі арқылы **Виртуалды машиналар** бөлімінде немесе **Бұлтты есептеулер** бөліміндегі **Дискілер** бөлімінде үлкейтуге болады.

Толық ақпарат [Дискілерді басқару](../instructions/volumes/volumes-manage#change_disk_size) мақаласында қолжетімді.

{/cut}

{cut(Дискіні кішірейту мүмкін емес)}

VK Cloud платформасында тек диск өлшемін ұлғайту қолжетімді.

{/cut}

{cut(Миграция процесі дискілердің өнімділігіне әсер ете ме?)}

Миграция сәтінде тек оқу өнімділігінің төмендеуі мүмкін, бірақ әдетте оқу үшін өнімділік қоры жеткілікті болады және төмендеу байқалмайды.

{/cut}

{cut(ВМ дискін басқа жобаға қалай беруге болады?)}

[Нұсқаулықты](../instructions/volumes/volumes-transfer) пайдаланыңыз.

{/cut}

{cut(Өз ОС образымды қалай жүктеуге болады?)}

VK Cloud платформасы алдын ала дайындалған және жүктелген образдардан виртуалды машиналар жасауға мүмкіндік береді. Образды дайындау бұлтты провайдерлер сервистерінде жұмыс істеуге қажетті бағдарламалық компоненттер мен драйверлер жиынтығын орнатудан тұрады, толығырақ [VK Cloud ішіне Hyper-V ВМ миграциясы](/kz/intro/migration/migrate-hyperv) және [VK Cloud ішіне VMware ВМ миграциясы](/kz/intro/migration/migrate-vmware) мақалаларында.

{/cut}

{cut(ВМ root-дискін қалай ауыстыруға болады?)}

Root-дискіні ауыстыру VK Cloud жобасында басқа диск бұрыннан жасалған жағдайда ғана мүмкін. Бұл бос диск те, операциялық жүйесі бар жүктелетін диск те болуы мүмкін. Нұсқаулық [root-дискіні ауыстыру](../instructions/vm/vm-root-replace) мақаласында берілген.

{/cut}

{cut(Диск снимогын жасауға бола ма?)}

Диск снимогын жасау [жеке кабинетте немесе OpenStack CLI арқылы](../instructions/volumes/volumes-snapshots) қолжетімді. Жасалған снимок дискінің өзі жойылғанға дейін сақталады.

{/cut}

{cut(Снимоктан дискіні қалай қалпына келтіруге болады?)}

[Нұсқаулықты](../instructions/volumes/volumes-snapshots) пайдаланыңыз.

{/cut}

{cut(Диск түрі қандай жылдамдықпен өзгереді?)}

Диск түрін өзгерту келесі жылдамдықпен жүреді:

- 70MB/s: егер диск ВМ-ге қосылған болса.
- 250MB/s: егер диск ВМ-ден ажыратылған болса.

Өшірілген виртуалды машинаға қосылған дискінің түрін өзгертуге болмайды.

{/cut}

{cut(Басқа ЦОД-қа түрлендірілгеннен кейін диск көшірмелері сақтала ма?)}

Дискіні басқа ЦОД-қа түрлендірген кезде бастапқы дискінің деректері жүктелетін айна жасалады. Түрлендіруден кейін мұндай айналар сақталмайды.

{/cut}

{cut(Барлық дискілерді бірден кеңейтуге бола ма, әлде әрқайсысын бөлек кеңейту керек пе?)}

Барлық дискілерді бір уақытта кеңейтуге болады.

{/cut}

{cut(Диск өлшемі өзгерген кезде виртуалды машина қайта жүктеле ме?)}

ВМ дискісінің өлшемін өзгерту ВМ-ді қайта жүктемей, жұмыс барысында жүреді.

{/cut}

{cut(Неліктен Дискіні жою түймесі көрсетілмейді?)}

Егер диск ВМ-ге қосылған болса, **Дискіні жою** түймесі болмайды.
[Дискіні ВМ-ден ажыратыңыз](../instructions/volumes/volumes-connect#dismount_disk), содан кейін түйме дискінің контекстік мәзірінде және диск бетінің өзінде қолжетімді болады.

{/cut}

{cut(Дискіні бұлттан қалай жүктеуге болады?)}

Образды [жасаңыз](/kz/computing/iaas/instructions/images/images-manage#obrazdy_zhasau) және [экспорттаңыз](/kz/computing/iaas/instructions/images/images-manage#obrazdy_eksporttau).

## Файлдық сақтау

{cut(Файлдық сақтау не үшін арналған?)}

Бұл сервис қашықтағы файлдық жүйені жасауға, файлдық жүйені виртуалды машиналарға монтаждауға, содан кейін инстанстардан файлдық жүйеге және кері деректерді оқуға және жазуға мүмкіндік береді.

{/cut}

Жоқ, мұндай мүмкіндік қарастырылмаған.

{/cut}

{cut(Файлдық сақтауды жобалар арасында пайдалануға бола ма?)}

Жоқ, мұндай функционал қарастырылмаған.

{/cut}

{cut(Файлдық сақтауды снимоктан қалпына келтіруге бола ма?)}

Снимоктар жасау туралы толығырақ [Снимоктармен жұмыс](../instructions/fs-manage/fs-snapshots#creating_a_snapshot) мақаласында.

{/cut}

{cut(Файлдық сақтау снимоктарын кесте бойынша жасау мүмкіндігі бар ма?)}

Жоқ, мұндай мүмкіндік қарастырылмаған.

Файлдық сақтау снимогын VK Cloud [жеке кабинеті](https://kz.cloud.vk.com/app/) арқылы немесе API пайдаланып қолмен жасауға болады.

{/cut}

{cut(Файлдық сақтауларда бос орынды бақылауға арналған агенттер бар ма?)}

{/cut}

{cut(Файлдық сақтауды басқа бұлттағы VPN арқылы қосуға бола ма?)}

Мұндай мүмкіндік жоқ.

Сақтаудың [қолжетімділік аймағы](/kz/start/concepts/architecture#az) жоба [аймағына](/kz/tools-for-using-services/account/concepts/regions) байланысты:

Иә, болады. Файлдық сақтауды ВМ-ге қосу туралы [Файлдық сақтауды қосу және ажырату](../instructions/fs-manage/fs-connect) мақаласында.

## Лицензиялау

{cut(Windows Server лицензиясын қалай алуға болады?)}

Windows ОС негізінде жасалатын ВМ-ге Windows Server операциялық жүйесінің лицензиялық көшірмесі алдын ала орнатылған. Егер [ВМ-ге қойылатын талаптар](../../vm-licenses/ms-lic#requirements) сақталса, лицензиялық көшірмені белсендіру ВМ қосылған кезде автоматты түрде жүреді. ОС белсендіру қатесі туындаса, виртуалды машинаның ID-н көрсете отырып, [техникалық қолдауға](/kz/contacts) жүгініңіз.

{/cut}

{cut(Қандай лицензияларды ала аламын?)}

Ұсынылатын стандартты лицензиялар тізімі шектеулі, қолжетімді тізімді [прайс-парақтан](https://cloud.vk.com/pricelist) көре аласыз.

{/cut}

{cut(Лицензиялаудың қандай түрлері бар?)}

Microsoft бюджетті барынша оңтайлы пайдалануға мүмкіндік беретін бірнеше лицензиялау моделін ұсынады:

- Ядроға. Бұл лицензиялау моделі пайдаланушылардың немесе құрылғылардың шектеусіз санына қолжетімділік береді.
- Пайдаланушыға. Бір пайдаланушыға шектеусіз құрылғылар санында лицензияға қолжетімділік беру үшін арналған.
- Құрылғыға. Бұл модельде лицензия сіздің серверіңізге жүгінетін әр құрылғы үшін сатып алынады. Құрылғымен жұмыс істейтін пайдаланушылар саны шектелмейді. «Құрылғыға» лицензиялары бірнеше қызметкер бір құрылғыны пайдалана алатын, мысалы бірнеше ауысымда жұмыс істейтін компанияларда шығындарды азайтуға және әкімшілендіруді жеңілдетуге мүмкіндік береді.

[Microsoft](/kz/computing/vm-licenses/ms-lic) лицензиялау ережелеріне сәйкес «ядроға» нұсқасы ВМ-нің әрбір виртуалды ядросын лицензиямен жабуды талап етеді. ВМ ядроларының санына қарамастан, әрбір 2 виртуалды CPU лицензиялануға жатады, толығырақ Microsoft мақаласында.

{/cut}

{cut(Лицензия түрін өзгертуге бола ма?)}

Иә, болады. Ол үшін [нұсқаулықты пайдаланыңыз](/kz/computing/iaas/instructions/vm/vm-lic-manage).
