Кластердегі бір сервиске немесе қолданбаға тиесілі серверлік топтардың (server groups) үлестірілуі шешілетін міндеттерге байланысты:

- Егер сервисте бір рөлге ие бірнеше серверлік топтар (ВМ, ДҚ инстанстары) болса, мысалы бірнеше қолданба сервері болса, істен шығуға төзімділікті арттыру үшін оларды әртүрлі физикалық серверлерге үлестіріңіз.
- Егер сервисте өзара әрекеттесу кідірісі маңызды болатын әртүрлі рөлдері бар серверлік топтар болса (мысалы, қолданба сервері мен деректер базасы сервері), трафик бір физикалық сервер аясында өтуі үшін оларды бір есептеу түйініне орналастырыңыз.

VK Cloud [серверлік топтарды үлестірудің төрт саясатын](../../concepts/server-group) қолдайды, `soft` саясаттары үшін `compute-api` 2.15-тен жоғары нұсқасын қолдау қажет.

{note:warn}

Серверлік топтар үшін рұқсат етілген операциялар:

- топта жаңа серверлер жасау;
- топтағы бар серверлерді жою.

Серверлік топтар үшін рұқсат етілмейтін операциялар:

- бар серверді топқа қосу;
- топтағы барлық серверлерді жоймай тұрып серверлік топты жою.

{/note}

Серверлік топтарды үлестіру ережелерін анықтау үшін инстанс қасиеттерінде топты көрсетіңіз:

1. Команда арқылы серверлік топтар тізімін шығарыңыз:

    ```console
    openstack server group list
    ```

1. `affinity` немесе `anti-affinity` саясатындағы серверлік топты табыңыз. Егер серверлік топтар табылмаса, оны команда арқылы жасаңыз:

    ```console
    ​openstack server group create <название группы серверов> --policy <политика>
    ```

1. Болашақ ВМ үшін жүктелетін дискілерді жасаңыз. Үш диск үшін мысал:

   ```console
   for i in 1 2 3;do openstack volume create --size 10 --image 98af6254-XXXX-XXXX-XXXX-81858ce9302a --availability-zone MS1 --bootable root-volume-$i;done
   ```

1. Серверлік топты көрсете отырып, қажетті виртуалды машиналарды жасаңыз. Үш ВМ үшін мысал:

    ```console
    for i in 1 2 3;do openstack --insecure server create --flavor STD3-1-1 --volume root-volume-$i --hint group=<ID со свойством политики> --nic net-id=<ID сети> vm-affinity-$i;done
    ```

1. ВМ жасалу процесінің аяқталуын күтіңіз. Орналастырылатын ВМ сәтті жасалғанына және `ACTIVE` күйінде тұрғанына көз жеткізіңіз.

{note:info}

Егер шектеулерге сәйкес келетін есептеу түйіндері табылмаса, ресурстарды жасауға арналған сұраулар қателікпен аяқталады.

{/note}
