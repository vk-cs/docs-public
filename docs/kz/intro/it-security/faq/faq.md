## Талаптарға сәйкестік

{cut(VK Cloud қандай өңірлік және халықаралық қауіпсіздік стандарттарына сәйкес келеді?)}

Толық ақпарат [Сертификаттар, лицензиялар және аттестаттар](../../certificates) бөлімінде және [Сертификаттар мен лицензиялар](https://cloud.vk.com/cloud-platform/certificates/) бетінде орналастырылған.

Қосымша құжаттар құпияны жария етпеу туралы келісім (NDA) жасалғаннан кейін беріледі.

{/cut}

{cut(VK Cloud PCI DSS талаптарына сәйкес келе ме?)}

Иә, сәйкес келеді. Толық ақпарат [Сертификаттар, лицензиялар және аттестаттар](../certificates) бөлімінде және [Сертификаттар мен лицензиялар](https://cloud.vk.com/cloud-platform/certificates/) бетінде орналастырылған.

{/cut}

{cut(Халықаралық деңгейде танылған компаниялар жүргізетін VK Cloud қауіпсіздік шараларын тәуелсіз бағалау (аудит) процесі бар ма?)}

Иә, бар. Толығырақ [Платформаның қауіпсіздігі](../platform-security#kauipsizdik_tekserulerin_zhurgizu) бөлімінде.

{/cut}

{cut(VK Cloud қызметтерді ұсыну үшін қосалқы мердігерлерді тарта ма?)}

VK Cloud қызметтерді ұсыну үшін қосалқы мердігерлерді тарта алады. Бұл туралы ақпарат оған әсер етуі мүмкін клиенттерге хабарланады.

VK Cloud қосалқы мердігерлермен жасалған шарттарда VK Cloud-тың өз клиенттері алдындағы ақпараттық қауіпсіздік бойынша міндеттемелеріне сәйкес келетін қажетті ақпараттық қауіпсіздік шаралары көрсетілуін бақылайды.

{/cut}

## Жауапкершілікті бөлу

{cut(VK Cloud пен клиент арасындағы жауапкершілік қалай бөлінеді?)}

VK Cloud бұлттық инфрақұрылымның қауіпсіздігіне жауап береді, клиент бұлт ішінде орналастырылған өз жүйелері мен деректерінің қауіпсіздігіне жауап береді.

[Облако 152-ФЗ](https://cloud.vk.com/solutions/152-fz/) арнайы сервисін пайдаланған кезде дербес деректерді сақтау үшін жауапкершілік те бөлінеді.

Толығырақ [Бұлт қауіпсіздігі](https://cloud.vk.com/cloud-security/) бетінде.

{/cut}

{cut(Клиент қолданбасының соңғы нүктелерін (endpoints) шифрлау қамтамасыз етіле ме?)}

Бұл үшін жауапкершілік клиенттің өзіне жүктеледі.

Жауапкершілікті бөлу туралы толығырақ жоғарыдағы сұрақтан оқыңыз.

{/cut}

{cut(152-ФЗ аясында дербес деректердің ақпараттық жүйелері (ИСПДн) үшін өз ақпаратты криптографиялық қорғау құралдарын (СКЗИ) пайдалануға бола ма?)}

Иә, болады.

{/cut}

{cut(VK Cloud ішінде сақталатын және өңделетін ақпаратты таңбалауға бола ма?)}

Иә, клиент өз ақпараты үшін ықтимал таңбалау тәсілдерін өзі таңдайды.

{/cut}

## Есептік жазбалар

{cut(Есептік жазбаларды кім жасайды?)}

Есептік жазбалар келесі жолдармен жасалуы мүмкін:

- пайдаланушы [тіркелгеннен](/kz/intro/onboarding/account) кейін өздігінен;
- LDAP-жүйелерден экспорттау арқылы.

Тіркелгеннен кейін клиент қол жеткізу құқықтарын, соның ішінде пайдаланушыларды шақыруды, қол жеткізу деңгейін, екі факторлы аутентификацияны және т.б. баптауды өзі басқарады.

{/cut}

{cut(Жеке кабинет пайдаланушылары үшін қауіпсіз аутентификацияның қандай әдістері қолданылады?)}

Аутентификация ашық түрде берілмейтін логин мен парольдің көмегімен орындалады. Қосымша ретінде TOTP пайдаланып [екі факторлы аутентификацияны (2FA) қосуға](/kz/tools-for-using-services/api/rest-api/enable-api#api_arkyly_kolzhetimdilikti_belsendiru) болады.

{/cut}

{cut(Жеке кабинет аккаунтын қорғау үшін не ұсынылады?)}

Бірнеше санаттағы таңбалардан тұратын ұзын және күрделі парольдерді пайдаланыңыз, сондай-ақ аккаунт үшін [екі факторлы аутентификацияны (2FA) қосыңыз](/kz/tools-for-using-services/api/rest-api/enable-api#api_arkyly_kolzhetimdilikti_belsendiru).

{/cut}

{cut(VK Cloud пайдаланушылары үшін бірыңғай кіру технологиясына (Single Sign-On, SSO) қолдау көрсетіле ме?)}

Иә, қолдау көрсетіледі. SAML немесе ADFS протоколдарын пайдаланып, VK Cloud [жеке кабинетіне](https://kz.cloud.vk.com/app/) кіру үшін клиенттің SSO жүйесін қосуға болады.

{/cut}

{cut(Клиент талаптарына сәйкес келетін пароль саясатын қолдануға бола ма?)}

Иә, болады. Ол үшін SAML немесе ADFS протоколдарын пайдаланып клиенттің LDAP-серверін платформаға қосып, осы сервердегі пароль саясатын бақылау қажет.

{/cut}

{cut(Қол жеткізуді рөлдік үлгі бойынша бөлу (Role Based Access Control, RBAC) қолданыла ма?)}

Иә, қолданылады. Толығырақ [Құқықтар мен рұқсаттар](/kz/tools-for-using-services/account/concepts/rolesandpermissions) бөлімінде.

{/cut}

{cut(Пайдаланушыны жасамай-ақ ресурстарға уақытша қол жеткізуді (Just in Time Access, JIT) баптауға бола ма?)}

Иә, мұны клиент жағында SAML пайдаланып іске асыруға болады.

{/cut}

{cut(Аккаунт компрометацияланған жағдайда не істеу керек?)}

Бірден [парольді өзгертуге](/kz/tools-for-using-services/vk-cloud-account/instructions/account-manage/editinfo#kupiyasozdi_ozgertu) тырысыңыз. Егер нәтиже болмаса, [техникалық қолдауға](/kz/contacts) жүгініңіз, өтінімде қызмет түрін **VK Cloud** → **Мәселе туралы хабарлау** → **Жеке кабинет және есептік жазба** → **Есептік жазба бұзылды** деп таңдаңыз.

{/cut}

## Желілік қауіпсіздік

{cut(Желілік қауіпсіздік топтарын қалай баптауға болады?)}

Толық ақпарат [Қауіпсіздік топтары](/kz/networks/vnet/instructions/secgroups) бөлімінде орналастырылған.

{/cut}

{cut(DDoS-тен қорғаныс (Anti-DDoS) және веб-файервол (Web Application Firewall, WAF) қолданыла ма?)}

Бұл сервистерді [жеке кабинетте](https://kz.cloud.vk.com/app/) **Арнайы сервистер** бөлімінде немесе [сайтта](https://cloud.vk.com/waf/) өтінім қалдыру арқылы қоса аласыз. Осы арнайы сервистер туралы толығырақ [WAF және AntiDDoS](/kz/security/firewall-and-protection-ddos) бөлімінде.

{/cut}

{cut(VK Cloud қандай DDoS қорғаныс құралдары мен веб-файерволдарды ұсынады?)}

- **AntiDDoS** арнайы сервисін пайдаланған кезде клиент StormWall, ServicePipe және Variti шешімдерінің арасынан таңдай алады.
- **WAF** арнайы сервисін пайдаланған кезде клиент Positive Technologies Cloud WAF, SolidWAF және WebMonitorex шешімдерінің арасынан таңдай алады.

{/cut}

{cut(Клиенттер қалай оқшауланған?)}

Клиенттер арасындағы оқшаулау мына деңгейлерде қамтамасыз етіледі:

- желі,
- гипервизор,
- қолданба.

VK Cloud клиентінің [Жобалары](/kz/tools-for-using-services/account/concepts/projects) басқа клиенттерден оқшауланған. Ақпараттық ағындар клиенттік ВМ деңгейінде басқарылады. VK Cloud инфрақұрылымдық ресурстары виртуализация ортасында жұмыс істеу үшін пайдаланылмайтын арнайы бөлінген физикалық серверлерде іске қосылған. Клиенттік желілерден сервистік желілерге қол жеткізу мүмкін емес. ВМ жасау кезінде пайдаланушы файервол қағидаларын өзі баптайды немесе ұсынылатын қауіпсіздік тобын таңдайды.

{/cut}

{cut(Басқа біреудің деректеріне қол жеткізу мүмкін бе?)}

Жоқ. VK Cloud клиентінің жобалары әр клиент үшін жеке жасалады және басқа клиенттерден оқшауланады. Клиент жобаларын басқа клиенттердің қайта пайдалануына жол берілмейді.

{/cut}

## Басқа

{cut(Ақпараттық қауіпсіздік оқиғаларының мониторингі қалай жүзеге асырылады?)}

VK және VK Cloud инфрақұрылымдарының ақпараттық қауіпсіздік оқиғаларын SOC VK кезекші ауысымы 24/7 режимінде бақылайды. Ақпараттық қауіпсіздік оқиғаларын жинау және сақтау үшін SIEM пайдаланылады. Ақпараттық қауіпсіздік оқиғалары журналдарына қол жеткізу SIEM құралдарымен шектелген.

VK Cloud пайдаланушылары үшін оқиғалар мониторингінің келесі сервистері қолжетімді:

- [Cloud Audit](/kz/tools-for-using-services/account/instructions/actionslogs),
- [Cloud Logging](/kz/monitoring-services/logging),
- [Cloud Monitoring](/kz/monitoring-services/monitoring).

{/cut}

{cut(Уақыт қалай синхрондалады?)}

VK инфрақұрылымының NTP-серверлері пайдаланылады, олар өз кезегінде [NTP Pool](https://www.ntppool.org/zone/ru) пулымен синхрондалады.

{/cut}

{cut(VK Cloud әрекеттер журналының сыртқы SIEM-жүйелерге берілуін баптауға бола ма?)}

Иә, болады. Толығырақ [API анықтамасында](/kz/tools-for-using-services/api/api-spec/logging).

{/cut}

{cut(VK Cloud әрекеттер журналы қанша уақыт сақталады?)}

Шамамен бір жыл. Журналдан клиенттің SIEM-жүйелеріне ақпарат жеткізуді баптау ұсынылады.

{/cut}

{cut(VK Cloud сервистерін пайдалану кезінде алынатын логтар мен басқа жазбалар қалай қорғалады?)}

Жазбалар осы ақпаратқа қол жеткізуді шектеу және бақылау арқылы қорғалады.

{/cut}

{cut(Ақпараттық қауіпсіздік инциденттері туындаған кезде не істеу керек?)}

Ақпараттық қауіпсіздік инциденттеріне күдік туралы [техникалық қолдау](/kz/contacts) арқылы хабарлаңыз. Өтініштен кейін жауапкершілік аймағы анықталады:

- Егер ақпараттық қауіпсіздік инциденті клиенттің жауапкершілік аймағында болса, ол инцидентті өздігінен шешеді.
- Егер ақпараттық қауіпсіздік инциденті VK Cloud жауапкершілік аймағында болса, онда инцидентпен VK Cloud айналысады.

VK Cloud клиенттерді оларға әсер етуі мүмкін ақаулар мен инциденттер туралы [Сервистер жұмысының мәртебесі](https://status.kz.cloud.vk.com) бөлімінде және [Telegram-арнада](https://t.me/vk_cloud_alerts) хабардар етеді.

{/cut}

{cut(Клиент мамандарының немесе сыртқы ұйымдарды тарта отырып, VK Cloud сервистерін осалдықтардың бар-жоғына сканерлеуді жүргізуге бола ма?)}

Иә, келісім бойынша болады.

{/cut}

{cut(VK Cloud резервтік көшіру сервисін ұсына ма?)}

Иә, толығырақ [Cloud Backup](/kz/storage/backups) бөлімінде.

Резервтік көшіру және қалпына келтіру процестерін ұйымдастыру үшін жауапкершілік пайдаланушылардың өздеріне жүктеледі.

{/cut}

{cut(VK Cloud сервистерінің резервтік көшірмелерін бөлек сақтау ұйымдастырылған ба?)}

Иә, ұйымдастырылған. Резервтік көшірмелерді сақтау үшін географиялық бөлінген S3-қоймасы пайдаланылады (деректердің сақталуына SLA 99.9999%).

{/cut}

{cut(Деректердің кепілді жойылуы қамтамасыз етіле ме?)}

Иә, қамтамасыз етіледі.

{/cut}

{cut(VK Cloud әкімшілерінің клиент деректеріне қол жеткізуінен қорғаныс қалай қамтамасыз етіледі?)}

VK Cloud Security Operations Center (SOC) және SIEM-жүйелердің (Security Information and Event Management) бақылауында болады.

Әкімшілер міндетті аутентификациядан өтеді және арнайы конфигурацияланған аралық хосттарды (jump hosts) пайдаланады.

Толығырақ [Платформаның қауіпсіздігі](../platform-security) бөлімінде.

{/cut}

{cut(Жалпыға қолжетімді деректерді беру желілері арқылы беру кезінде деректерді қорғау қалай қамтамасыз етіледі?)}

Қауіпсіз TLS және HTTPS хаттамалары пайдаланылады. Қажет болған жағдайда клиенттер деректерді қосымша шифрлауды баптай алады.

{/cut}

{cut(Қалыпты жұмыс рәсімдерін немесе қауіпсіздік рәсімдерін айналып өте алатын утилиталарды пайдалануға бола ма?)}

VK Cloud пайдалану аясында VK Cloud-пен келісілмей, қалыпты жұмыс рәсімдерін немесе қауіпсіздік рәсімдерін айналып өте алатын қызметтік бағдарламаларды (утилиталарды) пайдалануға тыйым салынады. VK Cloud сервистерін пайдалана отырып, сіз платформамен жұмыс істеу қағидаларын реттейтін [келісімдердің](/kz/start/legal) шарттарын қабылдайсыз.

Өз кезегінде VK Cloud мұндай бағдарламаларды VK Cloud персоналының пайдалануы қатаң шектелетініне және бақыланатынына кепілдік береді.

{/cut}

{cut(Пайдаланушыларға әсер етуі мүмкін VK Cloud өзгерістері туралы қалай білуге болады?)}

VK Cloud пайдаланушыларды оларға әсер етуі мүмкін сервистер жұмысының мәртебесі, ақаулар мен өзгерістер туралы [Сервистер жұмысының мәртебесі](https://status.kz.cloud.vk.com) бөлімінде және [Telegram-арнада](https://t.me/vk_cloud_alerts) уақтылы хабардар етеді.

{/cut}

{cut(Уақытша файлдар мен құжаттар қаншалықты жиі жойылады?)}

Уақытша файлдар мен құжаттар 1 ай ішінде жойылады.

{/cut}
