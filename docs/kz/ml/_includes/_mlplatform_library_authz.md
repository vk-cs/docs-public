{includetag(intro_part)}

Cloud ML Platform Python кітапханасы ұсынатын әдістердің көпшілігін пайдалану үшін _қолжеткізу токенінің_ көмегімен авторизациядан өту қажет.

Қолжеткізу токендерінің (refresh-токендердің) қолданылу аймағын анықтайтын келесі рөлдердің бірі болады:

- `Пользователь`. Бұл рөлдегі токен кітапханада қолжетімді әдістердің көпшілігін пайдалануға мүмкіндік береді.
- `Администратор`. Бұл рөлдегі токен кітапханада қолжетімді барлық әдістерді пайдалануға мүмкіндік береді. Мысалы, осы рөлдегі қолжеткізу токені арқылы басқа токендерді басқаруға болады.

{/includetag}

{includetag(main_part)}

Қолжеткізу құқықтарына байланысты токенді әртүрлі тәсілмен алуға болады:

- VK Cloud жеке кабинеті арқылы (әдетте `Администратор` рөлі бар қолжеткізу токені);
- Cloud ML Platform кітапханасы арқылы.

## {heading(Жеке кабинетте қолжеткізу токенін жасау)[id=create_token_management]}

Егер пайдаланушының VK Cloud жеке кабинетіне қолжеткізуі болса, ол қолжеткізу токенін өз бетінше жасап, екі рөлдің бірін таңдай алады. Мұндай токеннің әрекет ету мерзімі шектелмейді.

1. [VK Cloud жеке кабинетіне](https://kz.cloud.vk.com/app/) өтіңіз.
1. Токен жасау қажет жобаны таңдаңыз.
1. **ML Platform → Токены** бөліміне өтіңіз.
1. **Создать токен доступа** батырмасын басыңыз.
1. Ашылған терезеде токен параметрлерін көрсетіңіз:

   - **Название**: кез келген мән болуы мүмкін.
   - **Роль**: таңдалған рөл (`Пользователь` немесе `Администратор`).

1. **Создать** батырмасын басыңыз.
1. Ашылған терезеде токен мәнін көшіріп алып, құрылғыңызға сақтаңыз. Терезені жапқаннан кейін токен мәнін қалпына келтіру мүмкін болмайды. Егер ол жоғалса, жаңа токен жасаңыз.
1. **Готово** батырмасын басыңыз.

{note:err}

Қолжеткізу токеніңізді ешкімге бермеңіз.

{/note}

## {heading(Cloud ML Platform кітапханасы арқылы қолжеткізу токенін жасау)[id=create_token_console]}

Егер пайдаланушының VK Cloud жеке кабинетіне қолжеткізуі болмаса, қолжеткізу токенін кітапхана арқылы тіркеу токенінің көмегімен жасауға болады:

1. `Администратор` рөлі бар қолжеткізу токенінің иесі кітапхана арқылы пайдаланушы үшін [тіркеу токенін (register-токен)](#create_registration_token) жасайды. Ол бір реттік және шектеулі өмір сүру уақытына (TTL) ие. Тіркеу токенінде қолжеткізу токенінің барлық параметрлері: атауы, рөлі, әрекет ету уақыты беріледі.

2. Пайдаланушы тіркеу токені арқылы кітапханада әдістерді шақыру кезінде авторизациялау үшін [қолжеткізу токенін (refresh-токен)](#create_access_token) жасайды. Қолжеткізу токенінің барлық параметрлерін әкімші тіркеу токенінде алдын ала белгілейді, оларды өзгертуге болмайды.

### {heading(Тіркеу токенін жасау)[id=create_registration_token]}

Тіркеу токені тек Cloud ML Platform кітапханасы арқылы жасалады.

{note:err}

Қарапайымдылық үшін қолжеткізу токенінің мәні Python скриптінің мысалында көрсетілген, ал тіркеу токенінің мәні `print()` арқылы шығарылады.

Production-ортада токендерді ашық түрде пайдаланбаңыз. Сезімтал деректермен жұмыс істеу үшін орта айнымалыларын, құпиялар қоймаларын немесе басқа құралдарды пайдаланыңыз.

{/note}

1. Егер бұл әлі жасалмаса, [кітапхананы орнатыңыз](/kz/ml/mlplatform/mlplatform-lib/lib-install).
1. Егер бұл әлі жасалмаса, `Администратор` рөлі бар [қолжеткізу токенін жасаңыз](#create_token_management).
1. Python скриптін орындаңыз:

   ```python
   from mlplatform_client import MLPlatform
   from mlplatform_client.serializers.auth import MLPTokenType
    
   ADMIN_REFRESH_TOKEN = "<ТОКЕН_АДМИНИСТРАТОРА>"
    
   mlp = MLPlatform(ADMIN_REFRESH_TOKEN)

   register_token = mlp.create_register_token(
         client_name="<ИМЯ_ТОКЕНА_ДОСТУПА>",
         access_ttl="<ВРЕМЯ_ЖИЗНИ_РЕГИСТРАЦИОННОГО_ТОКЕНА>",
         refresh_ttl="<ВРЕМЯ_ЖИЗНИ_ТОКЕНА_ДОСТУПА>",
         token_type=<РОЛЬ_ТОКЕНА_ДОСТУПА>)

   print(register_token)
   ```

   Мұнда:
   - `<ТОКЕН_АДМИНИСТРАТОРА>`: `Администратор` рөлі бар қолжеткізу токенінің мәні.
   - `client_name`: осы тіркеу токені арқылы жасалатын қолжеткізу токенінің атауы.
   - `access_ttl`: тіркеу токенінің өмір сүру уақыты, мысалы, `2h45m30s`. Әрекет ету мерзімі өткен тіркеу токенін қолжеткізу токенін жасау үшін пайдалану мүмкін болмайды.
   - `refresh_ttl`: қолжеткізу токенінің өмір сүру уақыты, мысалы, `2h`. Егер бұл параметр көрсетілмесе, қолжеткізу токені мерзімсіз әрекет етеді.

   - `token_type`: қолжеткізу токенінің рөлі:

     - `MLPTokenType.USER` мәні `Пользователь` рөліне сәйкес келеді;
     - `MLPTokenType.ADMIN` мәні `Администратор` рөліне сәйкес келеді.

   Осы скрипт орындалғаннан кейін жасалған тіркеу қолжеткізу токенінің мәні шығарылады.

1. Басқа пайдаланушы берілген параметрлері бар қолжеткізу токенін жасау үшін пайдалана алуы үшін осы тіркеу токенін оған беріңіз.

### Тіркеу токені арқылы қолжеткізу токенін жасау

{note:err}

Қарапайымдылық үшін тіркеу токенінің мәні Python скриптінің мысалында көрсетілген, ал қолжеткізу токенінің мәні `print()` арқылы шығарылады.

Production-ортада токендерді ашық түрде пайдаланбаңыз. Сезімтал деректермен жұмыс істеу үшін орта айнымалыларын, құпиялар қоймаларын немесе басқа құралдарды пайдаланыңыз.

{/note}

1. Егер бұл әлі жасалмаса, [кітапхананы орнатыңыз](/kz/ml/mlplatform/mlplatform-lib/lib-install).
1. Сізде тіркеу токені бар екеніне көз жеткізіңіз. Егер ол жоқ болса, `Администратор` рөлі бар қолжеткізу токенінің иесінен [тіркеу токенін жасауды](#create_registration_token) сұраңыз.
1. Python скриптін орындаңыз:

   ```python
   from mlplatform_client import MLPlatform
 
   REGISTER_TOKEN = "<ЗНАЧЕНИЕ_РЕГИСТРАЦИОННОГО_ТОКЕНА>"

   mlp = MLPlatform()
   refresh_token = mlp.create_refresh_token(REGISTER_TOKEN)
 
   print(refresh_token)
   ```

   Осы скрипт орындалғаннан кейін жасалған қолжеткізу токенінің мәні шығарылады.

   Қолжеткізу токенінің параметрлері (атауы, рөлі және әрекет ету уақыты) тіркеу токенімен анықталады.

1. Токен мәнін көшіріп алып, құрылғыңызға сақтаңыз.

{note:err}

Қолжеткізу токеніңізді ешкімге бермеңіз.

{/note}

## {heading(Токен көмегімен авторизациялау)[id=authorisation_by_token]}

{note:err}

Қарапайымдылық үшін қолжеткізу токенінің мәні Python скриптінің мысалында көрсетілген. Production-ортада токендерді ашық түрде пайдаланбаңыз. Сезімтал деректермен жұмыс істеу үшін орта айнымалыларын, құпиялар қоймаларын немесе басқа құралдарды пайдаланыңыз.

{/note}

1. [Жеке кабинет арқылы](#create_token_management) немесе [Cloud ML Platform кітапханасы арқылы](#create_token_console) қолжеткізу токенін жасаңыз.

   Қандай әдістерді пайдалану жоспарланғанына байланысты белгілі бір рөлі бар қолжеткізу токені қажет болуы мүмкін. [Кітапхана әдістерінің анықтамалығында](/kz/ml/mlplatform/mlplatform-lib/lib-reference) нақты әдісті шақыру үшін токенде қандай рөл болуы керектігі көрсетіледі.

1. MLPlatform класының объектісін жасап, конструкторға алынған токеннің мәнін беріңіз.

   **Мысал:**

   ```python
   from mlplatform_client import MLPlatform

   REFRESH_TOKEN = "<ЗНАЧЕНИЕ_ТОКЕНА_ДОСТУПА>"
   mlp = MLPlatform(REFRESH_TOKEN)
   ```

Енді осы объектіні пайдаланып, өзіңізге қажет әдістерді шақыра аласыз. Объектіні жасау кезінде көрсетілген токен әдісті шақыру кезінде авторизациялау үшін пайдаланылады.

Егер авторизация сәтсіз болса, ұқсас қате туралы хабар көрсетіледі:

- Егер токен рөлі әдісті шақыру үшін жеткіліксіз болса:

  ```text
  mlplatform_client.core.exceptions.ForbiddenException: <Response [403]>. Info: {"status":403,"warnorCode":"ForbiddenException","warnorParams":{"field":"KtoTam token does not contain the following scopes: ['ml-platform:Manage']"}}
  ```

- Егер токеннің әрекет ету мерзімі өтіп кетсе:

  ```text
  mlplatform_client.core.exceptions.ForbiddenException: <Response [403]>. Info: {"status":403,"warnorCode":"ForbiddenException","warnorParams":{"field":"Forbidden"}}
  ```
