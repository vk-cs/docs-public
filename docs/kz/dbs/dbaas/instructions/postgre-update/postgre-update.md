# {heading(PostgreSQL нұсқасын жаңарту)[id=dbaas-postgre-update]}

{include(/kz/_includes/_translated_by_ai.md)}

ДБ инстанстары үшін PostgreSQL нұсқаларын мерзімді түрде жаңартып отыру ұсынылады. Жаңа нұсқаларда жаңа мүмкіндіктер қосылып, қателер түзетілген.

Жаңарту алдында:

1. Жаңартуды жоспарлап отырған ДҚ инстансының [сақтық көшірмесін](../../../../storage/backups/instructions/create-backup-copy)
   орындаңыз.
1. Сақтық көшірмеден жаңартылуы жоспарланған инстанспен бірдей инстансты өрістетіңіз. Ол үшін
   жаңартуды орындап, барлық деректердің қолжетімді болып қалатынын тексеріңіз.

{note:warn}

Жаңарту кезінде ДҚ инстансы қолжетімсіз болады. Жаңартуды алдын ала жоспарлаңыз.

{/note}

PostgreSQL ДБ инстансын жаңарту үшін:

{tabs}

{tab(Жеке кабинет)}

1. VK Cloud [жеке кабинетіне](https://kz.cloud.vk.com/app/) өтіңіз.
1. Қажетті ДҚ инстансы орналасқан жобаны таңдаңыз.
1. **Дерекқорлар** → **Дерекқор инстанстары** бөліміне өтіңіз.
   Егер ДҚ инстансы үшін жаңарту қолжетімді болса, тиісті хабарлама оның атауының оң жағында орналасады.
1. Жаңарту қолжетімді болатын ДҚ инстансына өтіңіз.
1. **Жаңартуды орнату** түймесін басыңыз.
1. Пайда болған терезеде тізімнен қажетті нұсқаны таңдаңыз.
1. **Нұсқаны өзгерту** түймесін басыңыз.

{/tab}

{tab(API)}

1. [Trove сервисінің API пайдалану үшін URL алыңыз](../../../../tools-for-using-services/api/rest-api/endpoints).

   URL көрінісі: `https://<эндпоинт API>/<идентификатор проекта>/`.

1. API-ге сұрауларды авторизациялау үшін [токен алыңыз](../../../../tools-for-using-services/api/rest-api/case-keystone-token).

1. Жаңартуды орындаңыз. Жаңарту рәсімі {linkto(
   ../../concepts/work-configs#dbaas-work-configs)[text=конфигурацияға]} байланысты:

   {tabs}

   {tab(Single и Master-Replica)}

    1. ДҚ инстанстарының тізімін алыңыз:

       {tabs}

       {tab(cURL (Linux bash \ macOS zsh))}

       ```console
       curl \
         --request GET \
         --url https://<эндпоинт API>/<идентификатор проекта>/instances \
         --header 'X-Auth-Token: <токен>'
       ```

       {/tab}

       {tab(cURL (Windows PowerShell))}

       ```console
       curl `
         --request GET `
         --url https://<эндпоинт API>/<идентификатор проекта>/instances `
         --header 'X-Auth-Token: <токен>'
       ```

       {/tab}

       {/tabs}

    1. API сұрауына берілген жауаптан `Master` рөлі бар инстанстың идентификаторын анықтаңыз. **Master-Replica** конфигурациясында
       барлық инстанстарды жаңарту `Master` рөлі бар ДҚ инстансын жаңарту арқылы орындалады. Жаңартуды тек
       `Replica` рөлі бар инстанстар үшін ғана орындау мүмкін емес.

       Мұндай инстанстың идентификаторын анықтау үшін:

        1. `instances[].name` өрісінен қажетті атауы бар инстансты табыңыз.
        1. Бұл PostgreSQL инстансы екеніне көз жеткізіңіз:

           ```json
           {
             "instances": [
               {
                 "id": "<идентификатор инстанса>",
                 "name": "<имя инстанса>",
                 ...
                 "datastore": {
                   "type": "<тип СУБД: postgresql>",
                   "version": "<версия СУБД>"
                 },
                 ...
               },
             ...]
           }
           ```

        1. `instances[].replica_of` өрісінің бар-жоғын тексеріңіз:

            - Егер мұндай өріс бар болса, онда бұл `Replica` рөлі бар инстанс, ал өрістегі мән — `Master` рөлі бар инстанстың қажет идентификаторы.
            - Егер мұндай өріс жоқ болса, онда бұл `Master` рөлі бар инстанс. Осы инстанстың идентификаторын `instances[].id` өрісінен
              жазыңыз.

    1. API сұрауына берілген жауаптан жаңартуға болатын нұсқаны анықтаңыз. Бұл үшін `Master` рөлі бар инстанс туралы
       мәліметтерді қараңыз. `Replica` рөлі бар инстанстар үшін жаңарту мүмкіндіктері туралы ақпарат
       берілмейді.

       ```json
       {
         "instances": [
           {
             "id": "<идентификатор инстанса с ролью Master>",
             ...
             "datastore": {
               ...,
               "version": "<версия СУБД>"
             },
             ...
             "upgrade_info": {
               "major_version": "<версия, на которую можно обновиться>",
               "minor_versions": []
             },
           },
         ...]
       }
       ```

       Жаңартуды `instances[].upgrade_info.major_version` өрісінде көрсетілген нұсқаға дейін орындауға болады. Егер өрісте —
       бос мән болса, онда ДҚ инстансы ең өзекті нұсқаны пайдаланып тұр және жаңарту қажет емес.

       ДҚ инстансын бірден бірнеше нұсқа алға жаңартқанда аралық нұсқаларды өткізіп жіберу мүмкін емес:

        - PostgreSQL 10 → 11 → 12 → 13 → 14 жаңартуын орындауға болады;
        - PostgreSQL 10 → 14 жаңартуын орындауға болмайды;

    1. API сұрауына берілген жауаптан `Master` рөлі бар инстанс үшін барлық репликаларды табыңыз және, егер олар бар болса,
       олардың идентификаторларын алыңыз. Идентификаторлар репликалар үшін жаңарту барысын бақылау үшін қажет.

       `Master` рөлі бар инстанстың идентификаторы бойынша іздеуді орындаңыз:

        - Егер осы идентификатор `instances[].replica_of` өрістерінде аталған болса, онда осындай мәндері бар өрістері бар инстанстар —
          репликалар. Олардың идентификаторларын жазыңыз.

          ```json
          {
             "instances": [
               {
                 "id": "<идентификатор инстанса c ролью Replica>",
                 ...,
                 "replica_of": "<идентификатор инстанса с ролью Master>",
                 ...
               },
             ...]
          }
          ```

        - Егер осы идентификатор `instances[].replica_of` өрістерінде табылмаса, онда репликалар жоқ, ал осындай идентификаторы бар инстанс — **Single** конфигурациясында. Қосымша әрекеттер қажет емес.

    1. Жаңарту процесін іске қосыңыз:

       {tabs}

       {tab(cURL (Linux bash \ macOS zsh))}

       ```console
       curl \
         --request PATCH \
         --url https://<эндпоинт API>/<идентификатор проекта>/instances/<идентификатор инстанса с ролью Master> \
         --header 'X-Auth-Token: <токен>' \
         --data '{
             "instance":
               {
                   "datastore_version": "<версия, на которую нужно обновиться>"
               }
          }'
       ```

       {/tab}

       {tab(cURL (Windows PowerShell))}

       ```console
       curl `
         --request PATCH `
         --url https://<эндпоинт API>/<идентификатор проекта>/instances/<идентификатор инстанса с ролью Master> `
         --header 'X-Auth-Token: <токен>' `
         --data '{
             \"instance\":
               {
                   \"datastore_version\": \"<версия, на которую нужно обновиться>\"
               }
          }'
       ```

       {/tab}

       {/tabs}

       Жаңарту басталады, процесс ұзақ уақыт алуы мүмкін.

    1. Жеке `Master` немесе `Replica` рөлі бар инстанстар үшін (бар болса) жаңарту барысын мерзімді түрде тексеріп отырыңыз:

       {tabs}

       {tab(cURL (Linux bash \ macOS zsh))}

       ```console
       curl \
         --request GET \
         --url https://<эндпоинт API>/<идентификатор проекта>/instances/<идентификатор инстанса> \
         --header 'X-Auth-Token: <токен>'
       ```

       {/tab}

       {tab(cURL (Windows PowerShell))}

       ```console
       curl `
         --request GET `
         --url https://<эндпоинт API>/<идентификатор проекта>/instances/<идентификатор инстанса> `
         --header 'X-Auth-Token: <токен>'
       ```

       {/tab}

       {/tabs}

       Жаңарту кезінде `instance.status` өрісінің мәні ретімен `BACKUP`, `UPGRADE` болып өзгереді.

       Жаңарту аяқталғаннан кейін:

        - `instance.status` өрісіндегі мән `ACTIVE` болып өзгереді.
        - `instance.last_task_result` өрісінде соңғы орындалған операцияның нәтижесі туралы ақпарат пайда болады. Сәтті жаңартылғандағы мәні — `MAJOR_UPGRADE_COMPLETED`.

   {/tab}

   {tab(Кластер)}

    1. ДҚ кластерлерінің тізімін алыңыз:

       {tabs}


    1. API сұрауына берілген жауаптан жаңартылуы керек кластердің идентификаторын және кластерді қандай нұсқаға дейін
       жаңартуға болатынын анықтаңыз:

       ```json
       {
         "clusters": [
           {
             "id": "<идентификатор кластера>",
             "name": "<имя кластера>",
             ...
             "datastore": {
               "type": "<тип СУБД: postgresql>",
               "version": "<версия СУБД>"
             },
             ...
             "upgrade_info": {
               "major_version": "<версия, на которую нужно обновиться>",
               "minor_versions": []
             },
             ...
           },
         ...]
       }
       ```

       Жаңартуды `clusters[].upgrade_info.major_version` өрісінде көрсетілген нұсқаға дейін орындауға болады. Егер өрісте —
       бос мән болса, онда ДҚ кластеры ең өзекті нұсқаны пайдаланып тұр және жаңарту қажет емес.

       ДҚ кластерін бірден бірнеше нұсқа алға жаңартқанда аралық нұсқаларды өткізіп жіберу мүмкін емес:

        - PostgreSQL 10 → 11 → 12 → 13 → 14 жаңартуын орындауға болады;
        - PostgreSQL 10 → 14 жаңартуын орындауға болмайды;

    1. ДҚ кластерін жаңарту процесін іске қосыңыз:

       {tabs}

       {tab(cURL (Linux bash \ macOS zsh))}

       ```console

         --url https://<эндпоинт API>/<идентификатор проекта>/clusters/<идентификатор кластера> \
         --header 'Content-Type: application/json' \
         --header 'X-Auth-Token: <токен>' \
         --data '{
               "datastore_upgrade":

                       "datastore_version": "<версия, на которую можно обновиться>"
                   }
           }'
       ```

       Жаңарту басталады, процесс ұзақ уақыт алуы мүмкін.

    1. Жаңарту барысын мерзімді түрде тексеріңіз: жаңартылып жатқан ДҚ кластеры туралы толық ақпаратты сұратыңыз:

       {tabs}

       {tab(cURL (Linux bash \ macOS zsh))}

       ```console
       curl \
         --request GET \
         --url https://<эндпоинт API>/<идентификатор проекта>/clusters/<идентификатор кластера> \
         --header 'X-Auth-Token: <токен>'
       ```

       {/tab}

       {tab(cURL (Windows PowerShell))}

       ```console
       curl `
         --request GET `
         --url https://<эндпоинт API>/<идентификатор проекта>/clusters/<идентификатор кластера> `
         --header 'X-Auth-Token: <токен>'
       ```

       {/tab}

       {/tabs}

       Жаңарту кезінде:

        - `cluster.task.name` өрісінде `UPGRADING_CLUSTER` мәні болады.
        - `cluster.instances[].status` өрісінде кластер инстанстары өз мәртебесін ретімен `BACKUP`,
          `UPGRADE` деп өзгертеді.

       Жаңарту аяқталғаннан кейін:

        - `cluster.task.name` өрісінде `NONE` мәні болады.
        - `cluster.last_task_result` өрісінде соңғы орындалған операцияның нәтижесі туралы ақпарат пайда болады. Сәтті жаңартылғандағы мәні — `MAJOR_UPGRADE_COMPLETED`.
