Zabbix — қолданбаларды, серверлерді және желілік құрылғыларды мониторингтеуге арналған БҚ.

Дерекқор кластеры инстанстарының мониторингі үшін Zabbix мониторинг жүйесін пайдалануға болады. Бұл бөлімде Zabbix мониторинг серверін өрістету туралы ақпарат берілмейді. Мұнда мониторинг сервері мен агенттерінің мүмкіндіктерін кеңейтетін қосымша орнатылатын компоненттер туралы ақпарат берілген. ДБ кластеры инстанстарында Zabbix агенті мониторинг жүйесінің клиенттік бөлігін өзіңіз орнатып, баптайсыз деп болжанады.

Zabbix үшін әзірленген көптеген алдын ала орнатылған шаблондар көптеген сервистердің күйі туралы ақпарат алуға мүмкіндік береді. Шешімдер Zabbix 3.4 және одан жоғары нұсқаларымен үйлесімді.

## Пайдаланушы параметрлері

- server (міндетті параметр) — Zabbix мониторинг серверінің IP-мекенжайы немесе атауы.
- listen_port(default: 10050) — пассивті тексерулерге арналған Zabbix агентінің порты.
- server_port(default: 10051) — белсенді тексерулерге арналған Zabbix серверінің порты.
- b64_pkcs12_container (default: none) — base4 форматына түрлендірілген, құпиясөзсіз PKS#12 форматындағы, құрамында куәландырушы орталықтың сертификаты, жабық кілт және Zabbix агентінің сертификаты бар файл.
- psk(default: none) — құпия кілт (Pre Shared Key).
- zabbix_agent_version (default: 3.4) — Zabbix агентінің нұсқасы (мүмкін мәндер: 3.4 немесе 5.0). Параметр PostgreSQL және PostgreSQL Pro үшін қолжетімді.
- hostname — Zabbix серверінде көрсетуге арналған хост атауы (бас және кіші әріптер ескеріледі). Рұқсат етілетін таңбалар: латынның кіші және бас әріптері, сандар, бос орындар және арнайы таңбалар: `.`, `_`, `-`. Әдепкі бойынша — ДҚБЖ инстансы ВМ атауы. ДҚБЖ кластерлері үшін пайдаланбаңыз.

## PostgreSQL мониторингі

- server (міндетті параметр) — Zabbix мониторинг серверінің IP-мекенжайы немесе атауы.
- listen_port(default: 10050) — пассивті тексерулерге арналған Zabbix агентінің порты.
- server_port(default: 10051) — белсенді тексерулерге арналған Zabbix серверінің порты.
- b64_pkcs12_container (default: none) — base4 форматына түрлендірілген, құпиясөзсіз PKS#12 форматындағы, құрамында куәландырушы орталықтың сертификаты, жабық кілт және Zabbix агентінің сертификаты бар файл.
- psk(default: none) — құпия кілт (Pre Shared Key).
- zabbix_agent_version (default: 3.4) — Zabbix агентінің нұсқасы. Мүмкін мәндер: 3.4 және 5.0. PostgreSQL және PostgreSQL Pro үшін Zabbix Agent 2 орнатуға арналған 5.0v2 нұсқасы да қолжетімді.
- hostname — Zabbix серверінде көрсетуге арналған хост атауы (бас және кіші әріптер ескеріледі). Рұқсат етілетін таңбалар: латынның кіші және бас әріптері, сандар, бос орындар және арнайы таңбалар: `.`, `_`, `-`. Әдепкі бойынша — ДҚБЖ инстансы ВМ атауы. ДҚБЖ кластерлері үшін пайдаланбаңыз.

Агенттің 3.4 нұсқасын орнатқан кезде Zabbix серверіне [pg_monz](https://www.google.com/url?q=https://github.com/pg-monz/pg_monz/blob/master/README-en.md&sa=D&source=docs&ust=1635767629305000&usg=AOvVaw3-3XfumQyub6xvlkGtZvjE) шаблонын орнату керек. Ол үшін [архивті тарқату](https://github.com/pg-monz/pg_monz/archive/refs/tags/2.2.tar.gz) және Zabbix серверінде `/pg_monz-2.2/pg_monz/template/` директориясынан шаблонды импорттау қажет. Бұл ретте шаблонды баптау қажет емес.

Агенттің 5.0 нұсқасы үшін Zabbix серверінің PostgreSQL Agent шаблонын пайдалану керек. Бұл ретте шаблонды баптау талап етілмейді.

Агенттің 5.0v2 нұсқасы үшін Zabbix серверінің PostgreSQL Agent 2 шаблонын пайдалану керек.

Шаблон баптауларында мыналарды орнату керек:

- PG.URI — tcp://127.0.0.1:5432;
- PG.DB — postgres.

## MySQL мониторингі

MySQL үшін Zabbix Agent кеңейтімін орнату төменде келтірілген метрикаларды жинауға мүмкіндік береді:

| Метрика                 | Сипаттамасы                                                                        |
|-------------------------|------------------------------------------------------------------------------------|
| mysql status            | Нұсқа, идентификатор, күй, үздіксіз жұмыс уақыты.                                  |
| connections status      | Байланыс қателері және үзілген қосылымдар.                                         |
| traffic                 | Қабылданған/жіберілген, секундтағы байт.                                           |
| temporary objects usage | Уақытша файлдарды, кестелерді және дискілердегі кестелерді пайдалану.              |
| keys usage              | Жазбалар саны, оқулар, MyISAM блоктары мен кэшін пайдалану.                        |
| operations count        | begin, commit, delete, insert, rollback, select, update үшін секундтағы амалдар.   |
| queries                 | Секундтағы сұраулар саны және баяу сұраулар саны.                                  |
| table locks             | Кестелердің дереу және күтілетін бұғаттауларының саны.                             |
| threads count           | Жұмыс істеп тұрған, жасалған, қосылған және кэштелген ағындар саны.                |

Zabbix агентінің 3.4 нұсқасын таңдағанда [mysbix](https://github.com/sergiotocalini/mysbix) шаблонының компоненттері орнатылады.

Пайдалануды бастау үшін [осы шаблонды](https://www.zabbix.com/documentation/current/ru/manual/xml_export_import/templates#%D0%B8%D0%BC%D0%BF%D0%BE%D1%80%D1%82) [импорттаңыз](https://raw.githubusercontent.com/sergiotocalini/mysbix/master/zbx3.4_template_db_mysql.xml).

{note:warn}

Zabbix серверінің кейбір нұсқаларында дәл осындай атаумен шаблон әлдеқашан орнатылған болуы мүмкін. Қайшылықтарды болдырмау үшін импорттаудан бұрын xml файлындағы шаблон атауын бірегей атауға өзгертуді ұсынамыз.

Агенттің v.5.0 нұсқасын таңдағанда Zabbix серверінің 5.0 және одан жоғары нұсқаларында алдын ала орнатылған [Template DB MySQL by Zabbix agent](https://git.zabbix.com/projects/ZBX/repos/zabbix/browse/templates/db/mysql_agent?at=refs%2Fheads%2Frelease%2F5.0) шаблонын пайдаланыңыз.

{/note}

| cluster member performance | басқару ағыны оқиғалары және сұраулар кезектерінің күйі                                                                             |

Пайдалануды бастау үшін [осы шаблонды](https://www.zabbix.com/documentation/current/ru/manual/xml_export_import/templates#%D0%B8%D0%BC%D0%BF%D0%BE%D1%80%D1%82) [импорттаңыз](https://raw.githubusercontent.com/MogiePete/zabbix-galera-template/master/App-Galera_Cluster.xml).

## ClickHouse мониторингі

ClickHouse үшін Zabbix Agent кеңейтімін орнату төменде келтірілген метрикаларды жинауға мүмкіндік береді:

| Метрика          | Сипаттамасы                                                                                                                                      |
|------------------|--------------------------------------------------------------------------------------------------------------------------------------------------|
| db status        | Дайындық, қосылымдардың күйі.                                                                                                                    |
| operations count | Секундтағы кірістірулер\жолдар, секундтағы сұраулар.                                                                                            |
| query            | Ағымдағы сұраулар саны, ағымдағы сұрауларды орындаудың ең көп уақыты, өңделген сұраулар мен кірістірулер саны, кірістіру кезіндегі кідірістер. |
| merge            | Секундтағы сығылмаған байттарды біріктірулер және жолдарды біріктірулер.                                                                         |
| replication      | Репликация үшін жадты пайдалану, лаг, кезектегі репликация тапсырмаларының саны.                                                                 |
| memory usage     | Фондық біріктірулер, мутациялар, жеткізулер үшін жадты пайдалану.                                                                                |

zabbix агентінің 3.4 нұсқасын таңдағанда [clickhouse-zabbix-template](https://www.google.com/url?q=https://github.com/Altinity/clickhouse-zabbix-template/&sa=D&source=docs&ust=1635766525505000&usg=AOvVaw1EbWUEyfblJ3WnlZpK9shE) шаблонының компоненттері орнатылады.

Пайдалануды бастау үшін [осы шаблонды](https://www.zabbix.com/documentation/current/ru/manual/xml_export_import/templates#%D0%B8%D0%BC%D0%BF%D0%BE%D1%80%D1%82) [импорттаңыз](https://raw.githubusercontent.com/Altinity/clickhouse-zabbix-template/master/zbx_clickhouse_template.xml).

Егер сіздің zabbix-server нұсқаңыз 5.0 немесе одан жоғары болса, кіріктірілген [Template DB ClickHouse by HTTP](https://git.zabbix.com/projects/ZBX/repos/zabbix/browse/templates/db/clickhouse_http?at=refs%2Fheads%2Frelease%2F5.0) шаблонын пайдаланыңыз.

Кеңейтімді орнату кезінде келесі параметрлерді көрсетіңіз:

- source_ip_addresses — clickhouse-server жүйесіне http арқылы қосылу үшін рұқсат етілген мекенжайларды [cllickhouse users](https://clickhouse.com/docs/ru/operations/settings/settings-users/#user-namenetworks) рұқсаттары баптауларына ұқсас форматта көрсету үшін (бөлгіш — үтір).
- zabbix_clickhouse_password — “{$CLICKHOUSE.PASSWORD}” макросындағы мәнді пайдалану үшін.
- mcs_user — {$CLICKHOUSE.USER} макросы үшін.

## Инстанстарды мониторинг серверіне қосу сценарийінің мысалы

1. Инстанс пен мониторинг сервері арасында белсенді және пассивті тексерулер порттары бойынша желілік байланыс болуын қамтамасыз етіңіз ([толығырақ осында](https://www.zabbix.com/documentation/current/ru/manual/appendix/items/activepassive)).
2. Берілетін деректерді шифрлау әдісін таңдаңыз (шифрлаусыз, TLS немесе PSK). Zabbix агентін орнату кезінде қорғалған қосылымның екі әдісін де таңдауға болады, мұндай жағдайда пайдалану барысында шифрлау тәсілін тек сервердегі қосылу конфигурациясын өзгерту арқылы ауыстыруға болады ([толығырақ осында](https://www.zabbix.com/documentation/current/ru/manual/encryption)).
3. Қажетті пайдаланушы параметрлерімен Zabbix агентін инстансқа немесе кластердің барлық даналарына орнатыңыз.
4. Инстанстарды Zabbix серверіне Configuration -> Hosts -> Create Hosts бөлімінде қосыңыз ([толығырақ осында](https://www.zabbix.com/documentation/5.4/ru/manual/config/hosts/host)).

{note:warn}

Host name, PSK identity өрістерінің мәні үшін инстанстың қысқа атауын (hostname -s) пайдаланыңыз (егер PSK қолданылса).

5. Мониторинг шаблон(дар)ын инстансқа немесе инстанс кіретін топқа тағайындаңыз ([толығырақ осында](https://www.zabbix.com/documentation/5.4/ru/manual/config/templates/linking)).
6. 10 минуттан кейін Monitoring -> Latest Data бөлімінде агенттен деректер серверге түсіп жатқанын тексеріңіз, ол үшін мазмұнды инстанс атауы бойынша сүзгіден өткізіңіз. Метрикалар бойынша тарихи мәліметтерді сол жерден — соңғы бағаннан қарай аласыз.

{/note}
