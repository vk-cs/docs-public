## Қызмет архитектурасы

Cloud Databases қызметінің (қызмет ретінде дерекқор) ықшамдалған түрдегі архитектурасы:

![Архитектура сервиса Cloud Databases](./assets/dbaas_architecture.png){params[width=30%; noBorder=true]}

VK Cloud басқару панелінде дерекқорды басқарудың алдын ала орнатылған құралдары бар виртуалды машина немесе кластер өрістетіледі. Виртуалды машина немесе кластер жасаған кезде реплика жасау керек пе ( **Master-Replica** конфигурациясы үшін) немесе түйіндер саны ( **Кластер** конфигурациясы үшін) таңдалады.

Одан кейін клиенттік қолданбалар желілік интерфейс арқылы (IP-мекенжай, коннектор, API) дерекқормен кәдімгі жергілікті дерекқор сияқты әрекеттеседі — желілік қосылу жылдамдығына байланысты түзетулермен.

## OpenSearch кластерінің архитектурасы

OpenSearch кластерінде деректер `data` рөлі бар түйіндерде сақталады және өңделеді. `master` рөлі бар кластер түйіндері кластерді басқаруға және оның күйін бақылауға жауап береді.

`master` және `data` рөлдері бір түйін аясында біріктірілуі мүмкін. Осылайша, VK Cloud-та келесілерден тұратын кластер жасауға болады:

- `data` рөлін қатар атқаратын тек `master` рөлі бар түйіндерден;
- `data` рөлі бар түйіндерден және жеке бөлінген `master` рөлі бар түйіндерден.

Түйін рөлдері туралы толығырақ [OpenSearch құжаттамасында](https://opensearch.org/docs/latest/tuning-your-cluster/cluster/).

VK Cloud платформасында OpenSearch кластері үшін [OpenSearch Dashboards](https://opensearch.org/docs/latest/dashboards/quickstart-dashboards/) сервисі орналастырылатын жеке түйін жасауға болады.
